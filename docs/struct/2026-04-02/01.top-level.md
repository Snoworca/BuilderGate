# 01. 최상위 디렉토리 구조

## 디렉토리 트리

```
ProjectMaster/
├── AGENTS.md                     # LLM 행동 규칙
├── CLAUDE.md                     # 프로젝트 퀵스타트 및 규칙
├── PRD.md                        # 제품 요구사항 문서
├── README.md                     # Vite 스캐폴드 README
├── dev.js                        # 개발 서버 동시 실행 스크립트
├── package-lock.json             # 루트 lockfile
├── .gitignore
├── .playwright-mcp/              # Playwright 스크린샷/콘솔 로그
├── .screenshot/                  # 스크린샷 아티팩트
├── docs/                         # 전체 문서 (SRS, 계획, 연구)
├── server/                       # Node.js/Express 백엔드
└── frontend/                     # React/Vite 프론트엔드
```

## 주요 루트 파일

### `dev.js` — 개발 서버 런처

- Node.js 스크립트로 `server/`와 `frontend/`를 동시 실행
- 서버: `npm run dev` (tsx watch, hot reload)
- 프론트엔드: `npx vite --host` (3초 딜레이 후 시작)
- 출력에 `[server]` / `[frontend]` 컬러 프리픽스 적용
- Windows에서 `taskkill /T /F`로 프로세스 트리 종료
- **규칙**: `kill {pid}` 또는 `taskkill /F /IM node.exe` 직접 사용 금지

### `PRD.md` — 제품 요구사항 문서

- BuilderGate의 전체 비전과 기능 요구사항 정의
- `docs/PRD.md`에도 동일 사본 존재

### `CLAUDE.md` — 프로젝트 가이드

- Claude Code용 퀵스타트, 규칙, API 레퍼런스
- 프로세스 종료 금지, 스크린샷 경로 규칙 등 명시
