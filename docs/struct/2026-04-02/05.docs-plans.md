## 계획 문서 구조 패턴

각 단계 디렉토리는 다음 패턴을 따름:

```
plan/stepN/
├── 00.index.md                     # 전체 계획 인덱스
├── 00-1.architecture.md            # 아키텍처 결정
├── 00-2.tech-decisions.md          # 기술 ADR
├── 01.phase-1-*.md                 # Phase 1 구현 계획
├── 02.phase-2-*.md                 # Phase 2 구현 계획
├── ...
├── final-validation.md             # 인수 기준
├── integration-test-guide.md       # 수동 테스트 체크리스트
├── verification/
│   └── phase-N-verification.md     # Phase별 검증 체크리스트
└── progress.json                   # 완료 추적 (Step 7, 8)
```

## 연구 문서 (`research/`)

실제 개발 과정에서 발생한 기술 이슈에 대한 심층 분석:

- **한글 IME 이중 입력**: xterm.js + 한글 입력 시 중복 문자 발생 원인 및 해결
- **리버스 프록시 아키텍처**: Vite dev 서버를 HTTPS 서버 뒤에 프록시하는 구조
- **Vite WS 프록시 ECONNABORTED**: WebSocket 프록시 연결 중단 문제 분석
- **cmux/tmux 연구**: 터미널 멀티플렉서 기능 벤치마크 및 영감
