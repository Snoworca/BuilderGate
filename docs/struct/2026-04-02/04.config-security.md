# 04. 설정 및 보안

## config.json5 구조

```json5
{
  server: { port: 4242 },
  pty: {
    termName: "xterm-256color",
    cols: 80, rows: 24,
    useConpty: true,        // Windows ConPTY vs winpty
    maxBufferSize: 65536,   // 64KB
    shell: "auto"           // auto/powershell/wsl/bash
  },
  session: { idleDelayMs: 200 },
  ssl: { /* 빈 경로 = 자동 생성 */ },
  security: { cors: { allowedOrigins: [] } },
  logging: { level, audit, directory, rotation },
  auth: {
    password: "enc(...)",   // AES-256-GCM 암호화
    jwtDuration: "30m"
  },
  fileManager: {
    maxFileSize: 1048576,   // 1MB
    blockedExtensions: [".exe", ".dll", ".so", ".bin"],
    blockedPaths: [".ssh", ".gnupg", ".aws"]
  },
  workspace: {
    dataPath: "./data/workspaces.json",
    maxWorkspaces: 10,
    maxTabsPerWorkspace: 8,
    maxTotalSessions: 32
  },
  twoFactor: { enabled: false, smtp: { /* enc(...) 비밀번호 */ } }
}
```

## 보안 계층

```
[브라우저] ──WSS──→ [HTTPS 서버]
                      │
                      ├── TLS 1.2-1.3 (자동 생성 self-signed 인증서)
                      ├── JWT HS256 (30분 만료, 블랙리스트)
                      ├── 선택적 2FA (이메일 OTP, 6자리)
                      ├── AES-256-GCM 설정 암호화
                      ├── 경로 순회 방지 (pathValidator)
                      ├── 차단 확장자/경로 (.exe, .ssh 등)
                      └── Helmet 보안 헤더 (HSTS, CSP 등)
```

### 인증 흐름

1. `POST /api/auth/login` — 비밀번호 검증
2. 2FA 활성 시: `tempToken` 반환 → OTP 이메일 발송
3. `POST /api/auth/verify` — OTP 검증 → 최종 JWT 발급
4. 2FA 비활성 시: 즉시 JWT 발급
5. WebSocket 연결 시 `?token=` 쿼리로 JWT 전달
6. 15분 주기 자동 갱신 (`useHeartbeat`)

### 암호화 시스템 (`CryptoService`)

- **알고리즘**: AES-256-GCM
- **키 파생**: PBKDF2 (10만 반복, SHA-256)
- **마스터 키 소스**: 머신 ID
- **저장 형식**: `enc(base64_iv:base64_encrypted:base64_tag)`
- **적용 대상**: config.json5 내 비밀번호, JWT 시크릿

### SSL (`SSLService`)

- 설정에 인증서 경로가 비어 있으면 `selfsigned` 라이브러리로 자동 생성
- `server/certs/` 디렉토리에 저장
- 만료 확인 기능 내장
- TLS 1.2-1.3, 강력한 암호 스위트 설정

### 설정 핫 리로드

`PATCH /api/settings` → `SettingsService`:

| 적용 범위 | 설명 |
|-----------|------|
| `immediate` | 즉시 반영 (CORS 등) |
| `new_logins` | 새 로그인부터 적용 (비밀번호 등) |
| `new_sessions` | 새 세션부터 적용 (PTY 기본값 등) |

서버 재시작 없이 런타임 설정 변경 가능.
