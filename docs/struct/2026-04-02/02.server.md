# 02. 서버 아키텍처

## 디렉토리 구조

```
server/
├── package.json          # buildergate-server, ESM, tsx watch
├── tsconfig.json         # ES2022, NodeNext, strict
├── config.json5          # JSON5 서버 설정 (Zod 검증)
├── config.json5.bak      # 암호화 전 백업
├── src/
│   ├── index.ts          # 메인 엔트리포인트
│   ├── test-runner.ts    # 커스텀 테스트 러너
│   ├── services/         # 비즈니스 로직
│   ├── routes/           # HTTP 라우트 핸들러
│   ├── middleware/       # Express 미들웨어
│   ├── ws/               # WebSocket 라우터
│   ├── types/            # TypeScript 타입 정의
│   ├── schemas/          # Zod 스키마
│   └── utils/            # 유틸리티
├── certs/                # 자동 생성 SSL 인증서
└── data/
    ├── workspaces.json   # 워크스페이스 영속 상태
    └── workspaces.json.bak
```

## 메인 엔트리포인트 (`src/index.ts`)

서버 시작 순서:
1. `CryptoService` 초기화 (머신 ID 기반 키 파생)
2. `RuntimeConfigStore` + `ConfigFileRepository` 초기화
3. `AuthService` 초기화 (JWT, 비밀번호 검증)
4. `TwoFactorService` 선택 초기화 (이메일 OTP)
5. `FileService`, `SettingsService` 초기화
6. `WorkspaceService` 초기화 (영속 상태 로드, 고아 탭 복구)
7. Express 라우트 등록
8. 개발 모드: Vite 리버스 프록시 생성 (`/` 경로)
9. SSL 인증서 로드/생성 → HTTPS 서버 시작
10. `WsRouter` 초기화 (WebSocket 메시지 버스)
11. HTTP→HTTPS 리다이렉트 서버 (포트 4241)
12. Graceful shutdown 설정

## 서비스 계층 (`src/services/`)

| 파일 | 역할 |
|------|------|
| `SessionManager.ts` | PTY 생명주기 관리. `node-pty` 프로세스 생성/추적, 터미널 I/O를 WS 구독자에게 라우팅. 셸 타입 해석(auto/powershell/wsl/bash). Windows CWD 추적(임시 파일 폴링). 싱글턴 |
| `AuthService.ts` | JWT HS256 발급/검증, AES 복호화된 비밀번호 검증, 토큰 블랙리스트(5분 주기 정리) |
| `CryptoService.ts` | AES-256-GCM + PBKDF2(10만 반복, SHA-256). 머신 ID 기반 마스터 키. `enc(...)` 형식 |
| `SSLService.ts` | TLS 인증서 로드/자동 생성(selfsigned), 만료 확인, TLS 1.2-1.3 설정 |
| `TwoFactorService.ts` | 6자리 OTP 생성, nodemailer 이메일 발송, 만료/최대 시도 제한 |
| `FileService.ts` | 파일 시스템 CRUD: listDirectory, readFile, copyFile, moveFile, deleteFile, mkdir. MIME 타입 매핑. pathValidator 보안 |
| `WorkspaceService.ts` | 워크스페이스/탭/그리드 CRUD + 영속화. `workspaces.json` 로드/저장. 디바운스 플러시(5초). 고아 탭 감지. 최대 10 워크스페이스, 8 탭/워크스페이스 |
| `SettingsService.ts` | 런타임 설정 패치/적용(Zod 검증). 서비스 핫 리로드(비밀번호, SMTP, CORS 변경) |
| `RuntimeConfigStore.ts` | 인메모리 편집 가능 설정 스냅샷. 필드별 적용 범위(immediate/new_logins/new_sessions) |
| `ConfigFileRepository.ts` | config.json5 읽기/쓰기(주석 보존). 쓰기 전 백업 생성 |

## 라우트 (`src/routes/`)

| 파일 | 마운트 경로 | 주요 엔드포인트 |
|------|-------------|----------------|
| `sessionRoutes.ts` | `/api/sessions` | CRUD, input, resize, shells 목록 |
| `fileRoutes.ts` | `/api/sessions/:id/` | cwd, files(목록/읽기), copy, move, delete, mkdir |
| `authRoutes.ts` | `/api/auth/` | login, verify(2FA), logout, refresh, status |
| `settingsRoutes.ts` | `/api/settings` | GET(스냅샷), PATCH(패치 적용) |
| `workspaceRoutes.ts` | `/api/workspaces` | 워크스페이스/탭/그리드 전체 CRUD, 순서 변경 |

## 미들웨어 (`src/middleware/`)

| 파일 | 역할 |
|------|------|
| `authMiddleware.ts` | JWT Bearer 토큰 추출 (Authorization 헤더 또는 `?token` 쿼리), `req.user` 어태치 |
| `securityHeaders.ts` | Helmet 기반 보안 헤더: HSTS, CSP, X-Frame-Options, X-Content-Type-Options |

## WebSocket (`src/ws/`)

| 파일 | 역할 |
|------|------|
| `WsRouter.ts` | `/ws` 경로 WebSocket 서버. JWT 인증(연결 시). 세션별 구독. 클라이언트→서버: subscribe, input, resize, ping. 서버→클라이언트: output, status, cwd, workspace/tab/grid 이벤트. 30초 heartbeat |

## 타입 (`src/types/`)

| 파일 | 정의 내용 |
|------|-----------|
| `index.ts` | Session, SessionDTO, SessionStatus, ShellType 등 중앙 re-export |
| `config.types.ts` | Config 전체 인터페이스(Server/PTY/Session/SSL/Security/Auth 등) |
| `auth.types.ts` | JWTPayload, LoginRequest/Response, OTPData, PendingAuth 등 |
| `file.types.ts` | DirectoryEntry, DirectoryListing, FileContent, CopyRequest 등 |
| `settings.types.ts` | EditableSettingsKey, SettingsSnapshot, SettingsPatchRequest 등 |
| `workspace.types.ts` | Workspace, WorkspaceTab, GridLayout, WorkspaceState 등 |
| `ws-protocol.ts` | ClientWsMessage, ServerWsMessage 유니온 타입 |

## 유틸리티 (`src/utils/`)

| 파일 | 역할 |
|------|------|
| `config.ts` | config.json5 로더, 평문 비밀번호 자동 암호화, Zod 검증, 싱글턴 export |
| `constants.ts` | SSL 기본값, TLS 설정, 암호 스위트, 보안 헤더 상수 |
| `errors.ts` | ErrorCode 열거형(60+), ErrorMessages/StatusCodes 매핑, AppError 클래스 |
| `pathValidator.ts` | 경로 순회 방지, 차단 경로/확장자 검사, 심볼릭 링크 해석 후 재검증 |
