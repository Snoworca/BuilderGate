# 요청 정리: React Mosaic Option A Slot Reorder Patch

작성일: 2026-04-22

## 배경

- 현재 grid mode는 `react-mosaic-component`의 기본 split DnD를 사용한다.
- 드래그 중 파란 반투명 guide는 셀 일부(`top/bottom/left/right`)만 덮는다.
- 사용자는 split insertion이 아니라 **셀 전체 target + slot reorder**를 원한다.

## 요구사항

1. 드래그 중 파란 guide는 대상 세션 셀 **전체**를 덮어야 한다.
2. drop 시 동작은 split 생성이 아니라 **move/insert 기반 reorder**여야 한다.
3. 예: `[1,2,3,4,5]`에서 `1`을 `5` 위치로 보내면 결과는 `[2,3,4,5,1]`이어야 한다.
4. 즉, swap이 아니라 source 제거 후 target slot에 삽입하고, 나머지 세션은 순서를 유지한 채 밀려야 한다.
5. 현재 `react-mosaic-component` 기반 구조를 유지하되, 필요하면 내부 patch/fork를 허용한다.
6. 사용자는 Option A, 즉 **library internal patch/fork 방식**을 선택했다.
7. 동작은 “모든 것이 완벽하게 동작”해야 하므로, drag target 의미와 저장/복원, 기존 mode semantics까지 함께 고려해야 한다.
8. 이 요구사항은 **`equal` 모드에서만** 동작해야 한다.
9. reorder는 셀 body 전체가 아니라 **창 이동 버튼/그립(handle)** 에서만 시작해야 한다.

## 작업 결정

- 구현 방향은 Option A로 고정한다.
- 즉, 앱 외부 overlay를 우회 계층으로 추가하는 것이 아니라, 우선 `react-mosaic-component` 내부 DnD 모델을 patch하는 계획을 세운다.

## 작업 가정

1. reorder 입력은 **창 이동 버튼/그립(handle)에서 시작하는 desktop 마우스 좌측 버튼 drag** 로 정의한다.
2. drop은 **마우스 좌측 버튼을 떼는 순간**으로 정의한다.
3. 우클릭은 기존 컨텍스트 메뉴와 충돌하므로 reorder 입력으로 사용하지 않는다.
4. touch long-press reorder는 현재 요구사항 범위에 포함하지 않는다.
5. “셀 전체에 drop”은 **그 셀의 현재 slot index를 목표 위치로 삼는 것**으로 해석한다.
6. 일반 규칙은 다음과 같다.
   - `target slot index`는 **source 제거 전, 현재 화면에 보이는 순서**에서 계산한다.
   - source를 먼저 제거한 뒤, 제거 전 계산한 `target slot index`에 source를 삽입한다.
   - source가 target보다 앞에 있으면, source는 target 뒤쪽으로 이동한 결과가 된다.
   - source가 target보다 뒤에 있으면, source는 target 앞쪽으로 이동한 결과가 된다.
7. source를 자기 자신이 있는 동일 slot에 drop하면 **no-op** 으로 처리한다.
8. 어떤 셀에도 해당하지 않는 바깥 영역에 drop하면 **no-op** 으로 처리한다.
9. 따라서 v1 계획에서는 별도 before/after split semantics를 두지 않는다.
