# 요구사항 정리: 세션 CWD 영속화 및 복원

## 사용자 원문

> 서버가 종료된 상태에서 다시 실행되면 세션이 종료되잖아.
> 만약에, 세션을 다시 시작할 때 마지막 디렉토리에서 재시작 하게 하려면 세션 임시 저장 파일을 만들고 이곳으로부터 복구 하도록 해야겠지?

## 요약

서버 재시작(정상/비정상 종료) 후 터미널 세션이 마지막 작업 디렉토리(CWD)에서 자동 복원되도록 하는 기능.

## 목표

1. 각 탭의 마지막 CWD를 workspaces.json에 영속화
2. 서버 재시작 시 고아 탭의 세션을 저장된 CWD에서 자동 재생성
3. 비정상 종료에도 최소한의 CWD 데이터 손실 (30초 이내)

## 변경 유형

feature (기능 추가)

## 제약사항

- 기존 workspaces.json 저장 패턴(원자적 쓰기: temp → backup → rename) 유지
- 기존 CWD 추적 메커니즘(임시파일 + watchFile) 활용
- 최대 세션 수 32개 제한 유지
- localhost 전용 보안 모델 유지

## 보안/동시성 분석 결과 (반영 필수)

### HIGH
- 프론트엔드 sessionId 동기화: 복구 후 새 sessionId가 생성되므로 프론트엔드는 REST로 최신 상태를 먼저 fetch해야 함

### MEDIUM
- CWD 인젝션: 제어문자/널바이트/개행 거부, 최대 길이 제한, PowerShell BOM 제거
- Symlink 공격: 임시파일 디렉토리 격리 또는 lstat 검증
- Shutdown 레이스: watchFile 중지 → CWD 스냅샷 → forceFlush 순서 보장
- restartTab 세션 누수 (기존 버그): deleteSession(oldId) 호출 필요
- workspaces.json 역직렬화 검증: lastCwd 필드 검증
- 비존재 CWD 디렉토리: fs.existsSync() → 홈 디렉토리 폴백
