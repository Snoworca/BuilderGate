# 프로젝트 개요

## 한 줄 정의

BuilderGate는 브라우저에서 여러 셸 세션을 워크스페이스와 탭 단위로 관리하고, 필요하면 그리드로 동시에 띄워 운영할 수 있게 만든 HTTPS 기반 개발 작업 환경입니다.

## 지금 이 저장소가 실제로 하는 일

- 로컬 HTTPS 앱으로 브라우저 터미널 제공
- 워크스페이스별 다중 탭 관리
- 탭 모드와 그리드 모드 전환
- 서버 권위 screen snapshot 기반 재연결/전환 복구
- 비밀번호 로그인과 선택적 TOTP
- 일부 런타임 설정 편집

## 현재 활성 제품 범위

### 서버가 실제 제공하는 핵심 계약

- `POST /api/auth/login`
- `POST /api/auth/verify`
- `POST /api/auth/refresh`
- `POST /api/auth/logout`
- `GET /api/settings`
- `PATCH /api/settings`
- `GET /api/workspaces`
- `POST/PATCH/DELETE /api/workspaces/*`
- `GET /api/sessions/shells`
- `GET /api/sessions/:id/cwd`
- `GET /api/sessions/:id/files*`
- `GET /api/sessions/telemetry`
- `GET/POST/DELETE /api/sessions/debug-capture/:id*`
- `/ws`

### 프런트에서 실제 마운트되는 핵심 화면

- 로그인 화면
- TOTP 검증 화면
- 워크스페이스 사이드바
- 탭 바
- 터미널 화면
- 그리드 모드
- 설정 화면

## 기술 스택

| 영역 | 현재 사용 기술 |
|---|---|
| 서버 | Node.js, Express 4, TypeScript, `node-pty`, `ws` |
| 프런트 | React 19, TypeScript, Vite 7, `react-mosaic-component`, xterm.js |
| 인증 | JWT + TOTP |
| 설정 | JSON5 + Zod + 런타임 editable subset |
| TLS | 자체 서명 인증서 자동 생성 또는 사용자 PEM 로드 |
| 저장 | `server/data/workspaces.json`, `server/data/totp.secret`, 브라우저 `localStorage` |

## 현재 구조에서 가장 중요한 해석

### 1. 이 프로젝트는 워크스페이스 중심 앱이다

구형 세션 목록형 API와 훅이 남아 있어도, 메인 앱 셸은 이미 워크스페이스/탭 중심으로 굳어졌습니다.

### 2. 실시간 경로는 단일 WebSocket이다

과거 SSE/HTTP 입력 구조가 아니라 현재는 `/ws` 하나로 입력, 출력, snapshot, 상태, 워크스페이스 브로드캐스트를 처리합니다.

### 3. 터미널 복구의 권위는 서버에 있다

최근 구조는 "클라이언트가 로컬 스냅샷만 믿는 방식"이 아니라 "서버가 headless terminal snapshot을 보내고, 클라이언트가 ack를 돌려주는 방식"입니다.

### 4. 그리드 레이아웃은 서버와 프런트에 중복 모델이 있지만 현재 활성 원본은 프런트 localStorage 쪽이다

서버 `WorkspaceService`는 `gridLayouts`를 저장하지만, 현재 메인 UI는 `useMosaicLayout()`을 통해 브라우저 `localStorage` 기반 트리를 사용합니다. 이 불일치는 작업 시 반드시 의식해야 합니다.
