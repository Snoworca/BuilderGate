# 테스트와 알려진 갭

## 이번 위키 갱신 시 직접 확인한 결과

2026-04-16에 직접 실행한 명령 결과:

- `server`: `npm test` 통과, `78 test(s) passed`
- `frontend`: `npm run build` 통과

프런트 빌드는 현재 깨져 있지 않습니다. 기존 위키의 "프런트 빌드 실패" 설명은 더 이상 맞지 않습니다.

## 서버 테스트 현실

서버는 별도 프레임워크가 아니라 자체 `test-runner.ts`로 주요 서비스/프로토콜을 검증합니다.

현재 커버되는 축:

- `RuntimeConfigStore`
- `SettingsService`
- `AuthService`
- `TOTPService`
- `SessionManager`
- `WsRouter`
- `WorkspaceService`
- `FileService`
- 일부 `authRoutes` 흐름
- headless snapshot serialization
- resize replay timing 회귀

즉 과거 위키처럼 "WsRouter와 WorkspaceService가 테스트 공백"이라고 적으면 현재는 부정확합니다.

## 프런트 테스트 현실

프런트는 unit test보다 Playwright 회귀 테스트 모음에 의존합니다.

현재 실행 방식:

- `frontend/package.json`에는 `test` 스크립트가 없음
- Playwright는 `npx playwright test`
- 기본 대상은 `https://localhost:2002`
- `frontend/playwright.config.ts`가 `node dev.js`를 webServer로 사용

## 상대적으로 현재 구조와 맞는 E2E 스위트

- `terminal-authority.spec.ts`
- `terminal-keyboard-regression.spec.ts`
- `header-context-menu-regression.spec.ts`
- `terminal-paste.spec.ts`

이들은 현재 `.workspace-screen`, `.terminal-view`, `.grid-cell`, helper textarea, debug capture 흐름을 기반으로 작성되어 있어 현재 구조와 더 가깝습니다.

## stale 가능성이 높은 E2E 스위트

- `pane-split.spec.ts`
- `pane-persistence.spec.ts`
- `pane-keyboard.spec.ts`
- `helpers.ts` 내부의 `.pane-*` 헬퍼

이유:

- `.pane-leaf`
- `.pane-resizer`
- `.pane-zoom-badge`
- `.status-prefix`

같은 선택자는 현재 메인 UI 구조의 중심 선택자가 아닙니다.

## 최신 검증 문서

최근 집중 검증 문서는 다음 경로가 중요합니다.

- `docs/report/2026-04-16.terminal-runtime-stabilization/01.summary.md`
- `docs/report/2026-04-16.terminal-runtime-stabilization/02.validation-and-artifacts.md`

이 문서들은 다음 현실을 반영합니다.

- workspace switch / reconnect 안정화
- `tab <-> grid` 전환 안정화
- PowerShell 경로 회귀 검증
- splitter drag 복구

이번 위키 갱신 중에는 전체 Playwright suite를 다시 끝까지 재실행하지는 않았고, 최신 dedicated validation 문서를 함께 참조했습니다.

## 아직 남아 있는 자동화 공백

### 서버 쪽

- `server/src/index.ts`의 전체 조립 경로
- 실제 HTTPS certificate generation/load 경로 전체
- Vite reverse proxy 경로
- production-like multi-process bootstrap

### 프런트 쪽

- unit test 부재
- terminal runtime layer에 대한 컴포넌트 수준 자동 테스트 부재
- localStorage/server `gridLayouts` 불일치 검증 부족

## 현재 코드 수준에서 기억해야 할 구조적 갭

### 1. 서버 `gridLayouts`와 프런트 active grid 원본이 다르다

자동 테스트나 문서가 "그리드 저장은 서버만 보면 된다"라고 가정하면 틀릴 수 있습니다.

### 2. 타입 정의와 실제 emit가 완전히 일치하지 않는다

- `workspace:deleting`는 emit되지만 프런트 handler 없음
- `session:error`, `tab:disconnected`는 타입은 있으나 현재 서버 emit 없음

### 3. 설정 스키마와 실제 editable subset은 다르다

현재 UI와 `PATCH /api/settings`는 전체 스키마를 노출하지 않습니다.

## 위키 해석 원칙

이 저장소의 테스트 관련 문서는 아래 원칙으로 읽어야 합니다.

1. "테스트 파일이 있다"와 "현재 구조를 충분히 검증한다"를 분리한다.
2. 현재 `server npm test`와 `frontend npm run build` 결과를 먼저 믿는다.
3. Playwright는 spec별로 최신성 차이가 크므로 selector와 시나리오를 함께 본다.
