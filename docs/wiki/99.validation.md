## 이번 갱신에서 바로잡은 핵심 오해

### 1. 포트 정보

오래된 문서는 `4242`, `4545`, `4001` 같은 값을 섞어 설명하지만 현재 기본 경로는 `2001/2002/2003`입니다.

### 2. 프런트 빌드 상태

기존 위키의 "프런트 빌드 실패" 설명은 현재 맞지 않습니다. 지금은 `npm run build`가 통과합니다.

### 3. 터미널 구조

현재 구조는 단순 tab별 xterm 재생성이 아니라:

- 서버 headless snapshot 권위
- `screen-snapshot` ack
- `TerminalRuntimeLayer` 기반 runtime 재사용

구조입니다.

### 4. 그리드 영속화 설명

서버 `gridLayouts`가 존재해도 현재 메인 그리드 UI는 브라우저 localStorage를 강하게 사용합니다.

### 5. 테스트 해석

Playwright spec는 모두 같은 최신성을 갖지 않습니다. `terminal-*`, `header-context-menu-regression` 계열은 현재 구조와 가깝고, `pane-*` 계열은 stale 가능성이 큽니다.
