# 백엔드 구조

## 최상위 계층

| 계층 | 역할 |
|---|---|
| `server/src/index.ts` | 부팅, 서비스 조립, HTTPS/redirect/WS 서버 생성 |
| `routes/` | HTTP 엔드포인트 노출 |
| `services/` | 비즈니스 로직, 런타임 상태, 영속화 |
| `middleware/` | 인증, 보안 헤더, localhost debug 가드 |
| `types/`, `schemas/`, `utils/` | 타입, 설정 검증, 유틸리티 |
| `ws/` | WebSocket 라우팅과 replay 제어 |

## 핵심 서비스

### `SessionManager`

현재 서버에서 가장 중요한 서비스입니다.

주요 책임:

- `node-pty`로 셸 프로세스 생성
- 세션별 상태 `running`/`idle` 추적
- 서버 쪽 headless terminal 상태 유지
- authoritative screen snapshot 생성
- resize 시 replay refresh 타이밍 제어
- CWD hook 주입 및 CWD 변화 추적
- PowerShell/WSL/bash 계열 셸 해석
- debug capture와 observability 제공

현재 구조의 핵심 포인트:

- 스크롤백 복구는 단순 문자열 버퍼가 아니라 headless terminal serialize 결과를 사용
- snapshot은 `seq`, `cols`, `rows`, `mode`, `replayToken`을 갖고 WS로 전송
- 클라이언트 ack 전에는 live output을 `WsRouter` 쪽 queue에 적재
- resize 직후에는 quiet window와 drain 상태를 본 뒤 snapshot refresh
- idle detection은 heuristic에서 시작하고 OSC 133을 감지하면 `osc133` 모드로 승격

### `WorkspaceService`

서버 원본 워크스페이스 상태를 가진 서비스입니다.

저장 상태:

- `workspaces`
- `tabs`
- `gridLayouts`

주요 책임:

- 워크스페이스 CRUD
- 탭 CRUD
- 탭의 `lastCwd` 영속화
- `workspaces.json` 로드와 원자적 저장
- `.bak` 백업 사용
- 서버 재시작 시 orphan tab 복구

주의할 점:

- 서버는 `gridLayouts`를 저장하지만 현재 메인 프런트는 이 값을 active UI 원본으로 쓰지 않습니다
- 즉 서버 상태와 실제 그리드 UI 상태가 분리된 부분입니다

### `AuthService`

- JWT 발급/검증/갱신/폐기
- 암호화된 비밀번호 복호화 후 timing-safe 비교
- blacklist cleanup timer 유지
- `localhostPasswordOnly` 옵션 지원

### `TOTPService`

- TOTP secret 생성/로드
- `server/data/totp.secret` 관리
- QR 코드 콘솔 출력과 data URL 생성
- pending auth 저장
- 최대 3회 시도 제한
- same-step replay 방지

### `SettingsService`

- editable subset validation
- patch merge
- 런타임 반영
- config 파일 저장
- 저장 실패 시 롤백

현재 반영 대상은 `AuthService`, `SessionManager`, `FileService`, `RuntimeConfigStore`입니다.

### `FileService`

- 세션의 현재 CWD 기준 파일 작업
- path traversal 방지
- blocked path / blocked extension 검사
- symlink realpath 재검증
- list/read/copy/move/delete/mkdir 지원

현재 주의점:

- 타입과 설정에는 `maxCodeFileSize`가 있지만 실제 읽기 제한 경로는 `maxFileSize` 중심입니다

### `SSLService`

- 사용자 인증서 로드
- 자체 서명 인증서 자동 생성
- 인증서 만료일 검사
- HTTPS 서버용 TLS 옵션 생성

## WebSocket 구조

`WsRouter`는 `/ws?token=<jwt>`만 받습니다.

주요 역할:

- 업그레이드 시 JWT 검증
- 클라이언트별 `clientId` 부여
- 세션별 subscribe/unsubscribe 관리
- `screen-snapshot` 전송
- ack 대기 중 output queueing
- resize 시 replay snapshot refresh
- `output`, `status`, `cwd`, `session:exited` 브로드캐스트
- 워크스페이스/탭/그리드 이벤트 브로드캐스트
- heartbeat와 replay telemetry 유지

## 보안과 운영

- Helmet 기반 보안 헤더
- HSTS
- `Permissions-Policy`
- 민감 응답 no-cache
- 동적 CORS
- Bearer 토큰 기반 보호 라우트
- debug capture는 localhost 요청만 허용

## 현재 구조에서 중요한 불일치와 함정

### 1. WS 타입 정의와 실제 emit가 완전히 같지 않다

- `workspace:deleting`는 서버가 emit하지만 프런트는 현재 handler가 없습니다
- `session:error`는 타입에 있으나 현재 서버가 emit하지 않습니다
- `tab:disconnected`도 타입과 handler는 있으나 현재 서버 emit가 없습니다

### 2. `gridLayouts`는 서버에 저장되지만 메인 프런트가 주로 localStorage를 사용한다

서버 API와 상태 모델은 살아 있지만, 현재 그리드 UI는 `useMosaicLayout()`의 브라우저 저장값을 우선 사용합니다.

### 3. `sessionRoutes`는 살아 있지만 메인 앱 설계의 중심이 아니다

현재 사용자는 워크스페이스/탭 UI를 통해 세션을 다루며, 직접 `GET /api/sessions` 기반 목록형 UX를 사용하지 않습니다.

### 4. `config.json5`의 일부 키는 현재 no-op 또는 제한적이다

- `pty.bufferFilter`: 현재 런타임 미사용
- `fileManager.maxCodeFileSize`: 타입/설정엔 있으나 read path 핵심 제한은 아님
