# 영속화와 데이터

## 서버 측 핵심 영속화 파일

### `server/data/workspaces.json`

현재 서버 공유 상태의 핵심 원본입니다.

저장 내용:

- 워크스페이스 목록
- 탭 목록
- 각 탭의 `sessionId`
- 각 탭의 `lastCwd`
- 서버 모델상의 `gridLayouts`

대략 구조:

```json
{
  "version": 1,
  "lastUpdated": "ISO timestamp",
  "state": {
    "workspaces": [],
    "tabs": [],
    "gridLayouts": []
  }
}
```

함께 쓰이는 보조 파일:

- `server/data/workspaces.json.bak`
- `server/data/workspaces.json.tmp` 잠시 생성 가능

### `server/data/totp.secret`

- 암호화된 TOTP secret
- 첫 초기화 시 자동 생성 가능
- 이후 재시작 시 재사용

### `server/certs/self-signed.crt`, `server/certs/self-signed.key`

- 사용자 인증서를 지정하지 않았을 때 자동 생성되는 개발용 인증서

## 서버 측 저장/복구 정책

### 워크스페이스 저장

- 일반 변경은 debounce flush
- 생성/삭제 계열은 즉시 flush
- temp 파일 작성 후 rename하는 원자적 저장
- 기존 파일은 `.bak` 백업 유지

### orphan tab 복구

서버 재시작 후 저장된 탭은 있는데 해당 `sessionId` PTY가 없으면 새 세션을 다시 만들고 `sessionId`를 갱신합니다.

### CWD snapshot

- 30초마다 모든 탭의 CWD를 snapshot
- graceful shutdown 때도 마지막 snapshot과 flush 수행

## 현재 중요한 저장 책임 분리

| 저장 위치 | 현재 책임 |
|---|---|
| 서버 JSON | 여러 클라이언트가 공유해야 하는 워크스페이스/탭/CWD 상태 |
| 브라우저 localStorage | 브라우저별 active workspace, 그리드 표현, 토큰, 폰트, local snapshot |

## 현재 구조에서 중요한 불일치

### 서버도 `gridLayouts`를 저장하지만, 메인 UI는 localStorage를 더 강하게 사용한다

현재 `WorkspaceService`는 `gridLayouts`를 저장하고, `useWorkspaceManager`도 이를 메모리에 보관합니다.

하지만 실제 그리드 렌더와 복원은 `useMosaicLayout()`이 `mosaic_layout_<workspaceId>`를 읽는 방식이 중심입니다.

따라서 다음 사실이 동시에 맞습니다.

- 서버 `workspaces.json`에 `gridLayouts`가 있다
- 현재 사용자가 보는 그리드는 브라우저 저장값이 더 직접적인 원본이다

## 런타임 캐시와 비영속 상태

### 서버 메모리

- `SessionManager.sessions`
- headless terminal state
- replay pending queue
- observability counters
- `FileService` CWD cache

### 브라우저 메모리

- WebSocket `clientId`
- session handler registry
- workspace handler registry
- terminal debug event store

## 진단용 또는 로컬 실행 부산물

다음 경로는 현재 메인 런타임의 진실 원본이 아닙니다.

- `server/data/snapshots/*.json`
- `server/data/buffer-dump/*.bin`
- `server/server/data/buffers/*.buf`

이 경로들은 현재 코드 기준으로 진단 실험 산출물 또는 로컬 부산물에 가깝습니다.

## 작업 시 주의할 점

- `server/data/workspaces.json`은 로컬 세션 상태라 작업 중 자연스럽게 더러워질 수 있습니다
- `server/data/totp.secret`는 비밀값이므로 커밋 대상이 아닙니다
- 터미널 snapshot localStorage는 세션 재시작이나 lineage 변경 시 제거 마커가 함께 관리됩니다
