# 기능 맵

## 현재 사용자에게 실제 노출되는 기능

### 인증

- 비밀번호 로그인
- TOTP 검증
- JWT refresh
- 로그아웃

### 워크스페이스

- 생성
- 선택
- 이름 변경
- 삭제
- 드래그 재정렬

### 탭

- 생성
- 이름 변경
- 닫기
- 재시작
- 드래그 재정렬
- 셸 타입 선택

### 터미널

- 실시간 입력/출력
- workspace 전환 후 runtime 유지
- snapshot 기반 reconnect 복구
- 선택 텍스트 복사
- 붙여넣기
- status 표시
- CWD 표시
- 폰트 크기 조절/저장

### 그리드

- 탭 모드와 그리드 모드 전환
- splitter drag
- equal / focus / auto 레이아웃 모드
- stale layout 복구
- 모바일에서는 자동 비활성화

### 설정

- 세션 duration
- 비밀번호 교체
- TOTP on/off
- `externalOnly`
- issuer / accountName
- CORS
- PTY 기본값
- idle delay
- 파일 제한 일부
- 로컬 그리드 비율

## 현재 UI에서 실제 제공되는 터미널 메뉴

현재 컨텍스트 메뉴는 다음만 제공합니다.

- 새 세션
- 세션 닫기
- 복사
- 붙여넣기

즉 과거 pane split 메뉴나 zoom 배지는 현재 기능 맵에 넣으면 안 됩니다.

## 현재 동작 제약

### 모바일

- `768px` 미만이면 모바일
- 모바일에서는 그리드 모드를 강제로 탭 모드로 되돌림
- 워크스페이스 목록은 드로어로 표시

### 기본 제한

현재 기본 설정값 기준:

- 워크스페이스 최대 10개
- 워크스페이스당 탭 최대 8개
- 전체 세션 최대 32개

### LRU scaffold는 현재 비활성

`App.tsx`에는 workspace alive/LRU 구조가 있지만 `MAX_ALIVE_WORKSPACES = 0`이라 현재 eviction이 발생하지 않습니다.

### 플랫폼 편향

현재 최근 검증과 회귀 테스트는 Windows + PowerShell 경로에 특히 많이 맞춰져 있습니다. 다른 셸과 OS도 지원 범위에 있지만, 문제 재현과 안정화 중심축은 Windows 경로입니다.

## 코드에는 있지만 메인 UX에 연결되지 않은 기능

### 파일 매니저 UI

- `components/FileManager/*` 존재
- 서버 파일 API도 존재
- 하지만 현재 `App.tsx` 메인 셸에서 마운트하지 않음

### 구형 pane/session 기반 UX

- `hooks/useSession.ts`
- `hooks/useTabManager.ts`
- `components/Sidebar/*`
- `components/StatusBar/*`

### 문서/코드 뷰어용 의존성

다음 의존성은 깔려 있지만 현재 메인 화면에서 활발히 쓰이지 않습니다.

- `react-markdown`
- `remark-gfm`
- `rehype-highlight`
- `mermaid`
- `highlight.js`

### 서버 `gridLayouts` API

- 서버는 `PUT /api/workspaces/:id/grid`와 `grid:updated`를 지원
- 현재 메인 그리드 UI는 localStorage 중심이라 실제 제품 기능으로는 연결이 약함

## 운영 문서에 적으면 안 되는 주장

현재 코드 기준으로 아래 표현은 부정확합니다.

- 이메일 OTP 기반 2FA가 현재 활성이라는 설명
- 파일 매니저가 현재 메인 UI에서 사용 가능하다는 설명
- pane split UI가 현재 주 사용자 경험이라는 설명
- Playwright spec가 모두 현재 UI를 검증한다는 설명
