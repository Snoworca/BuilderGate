# 실행과 설정

## 현재 기준 실행 포트

`AGENTS.md`, `dev.js`, `server/config.json5`, `frontend/playwright.config.ts`를 함께 보면 현재 기본 개발 포트는 다음입니다.

| 용도 | 포트 |
|---|---|
| HTTP redirect | `http://localhost:2001` |
| 앱 본체 HTTPS | `https://localhost:2002` |
| Vite dev server | `http://localhost:2003` |

중요한 점:

- 수동 검증과 Playwright 검증 대상은 `https://localhost:2002`
- `2001`은 앱이 아니라 HTTPS 리다이렉트 포트
- `2003`은 프런트 원본 서버이지만 평소 직접 붙는 대상이 아니라 HTTPS reverse proxy 뒤쪽

## 개발 진입점

### 루트에서 전체 실행

`node dev.js`

- 서버 기본 포트: `2002`
- 프런트 기본 포트: `2003`
- HTTP redirect 포트: `2001`
- 브라우저 접속 주소: `https://localhost:2002`

`dev.js`는 `--port`, `--fport`를 받습니다.

- `--port`: HTTPS 서버 포트
- `--fport`: Vite 포트
- `--fport`를 생략하면 `serverPort + 1`

### 서버 단독 실행

`server/`에서:

- `npm run dev`
- `npm run build && npm start`

이 경로에서는 `PORT` 환경변수가 없으면 `config.server.port`를 사용합니다. 현재 기본값은 `2002`입니다.

### 프런트 단독 실행

`frontend/`에서:

- `npm run dev`

Vite 설정은 `DEV_SERVER_PORT` 기본 `2002`, `DEV_FRONTEND_PORT` 기본 `2003`을 사용합니다. 즉 프런트만 단독으로 띄워도 기본 프록시 대상은 `https://localhost:2002`입니다.

## 서버 부팅 순서

`server/src/index.ts` 기준 현재 부팅 순서는 아래와 같습니다.

1. `config.json5` 로드와 검증
2. `CryptoService` 생성
3. `RuntimeConfigStore`, `ConfigFileRepository` 생성
4. `AuthService` 생성
5. `twoFactor.enabled`가 켜져 있으면 `TOTPService` 생성/초기화
6. `FileService`, `SettingsService` 생성
7. `WorkspaceService` 생성, 상태 로드, orphan tab 복구
8. 주기적 CWD snapshot 타이머 시작
9. HTTP 라우트 등록
10. 개발 모드면 Vite reverse proxy 등록
11. `SSLService`로 인증서 로드 또는 생성
12. HTTPS 서버 생성
13. `WsRouter` 생성 후 `SessionManager`와 연결
14. HTTP redirect 서버 시작
15. 터미널 observability 타이머 시작

## 현재 설정 파일 진실

### 주 설정 파일

- 실제 파일: `server/config.json5`
- 예제 파일: `server/config.json5.example`
- 없으면 example에서 생성

### 자동 암호화

현재 자동 암호화 대상은 `auth.password`입니다.

- 평문이면 서버 시작 중 `enc(...)` 형식으로 덮어써 저장
- `config.json5.bak`를 만들어 백업
- 암호화 키는 `hostname + platform + arch` 기반

### TLS 동작

- `ssl.certPath`, `ssl.keyPath`가 비어 있으면 `server/certs/`에 자체 서명 인증서 생성
- 값이 있으면 해당 PEM 파일 로드
- HTTPS 서버는 TLS 1.2~1.3 범위를 사용

## 현재 로컬 설정 파일의 실제 의미

현재 `server/config.json5`는 대략 아래와 같은 운영 의도를 갖습니다.

- 서버 포트 `2002`
- `useConpty: true`
- `scrollbackLines: 1000`
- `maxSnapshotBytes: 2097152`
- `twoFactor.enabled: true`
- `twoFactor.externalOnly: true`

즉, 기본 로컬 개발 환경은 "비밀번호는 항상 필요하지만 localhost에서는 TOTP를 생략하는" 구성입니다.

## 런타임에서 편집 가능한 설정

설정 화면과 `PATCH /api/settings`는 전체 `config.json5`를 노출하지 않습니다. `RuntimeConfigStore`가 허용한 subset만 수정됩니다.

- `twoFactor.enabled`
- `twoFactor.externalOnly`
- `twoFactor.issuer`
- `twoFactor.accountName`
- `security.cors.allowedOrigins`

### 적용 시점

| 적용 범위 | 의미 |
|---|---|
| `immediate` | 저장 즉시 현재 런타임에 반영 |

## 현재 설정 계층에서 중요한 함정

### 1. `pty.maxBufferSize`는 레거시 호환 alias다

스키마는 `pty.maxBufferSize`를 읽으면 `pty.maxSnapshotBytes`로 변환합니다. 오래된 문서가 `maxBufferSize`를 언급해도 현재 내부 런타임은 `maxSnapshotBytes`를 사용합니다.

### 2. `pty.bufferFilter`는 현재 런타임에서 사용되지 않는다

`server/config.json5`에는 설명과 값이 남아 있지만 현재 서버 스키마와 런타임 로직은 이 값을 소비하지 않습니다. 즉 현재는 사실상 no-op 키입니다.

### 3. TOTP secret은 설정 파일이 아니라 별도 파일에 있다

현재 구현은 `server/data/totp.secret`을 사용합니다. 설정 파일에는 켜짐/issuer/accountName만 있고 실제 secret은 따로 저장됩니다.
