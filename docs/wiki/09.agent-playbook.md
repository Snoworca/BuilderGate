# 에이전트 작업 가이드

## 먼저 믿어야 할 것

이 저장소에서 가장 먼저 믿어야 하는 것은 현재 코드입니다.

권장 우선순위:

1. `server/src/*`, `frontend/src/*`
2. `docs/wiki/*`
3. 최신 `docs/report/*`
4. `docs/plan/*`, `docs/spec/*`, `docs/srs/*`
5. `README.md`, `CLAUDE.md`

`README.md`와 `CLAUDE.md`는 과거 포트, 과거 통신 구조, 과거 UI 설명이 섞여 있어 빠른 개요용 참고 이상으로 쓰면 안 됩니다.

## 작업 전에 알아야 할 함정

### 1. 서버 `gridLayouts`가 현재 메인 UI의 진실 원본이 아니다

현재 그리드 UI는 `localStorage`의 `mosaic_layout_<workspaceId>`를 사용합니다. 서버 `gridLayouts`를 고쳐도 사용자가 보는 그리드가 바로 바뀐다고 가정하면 안 됩니다.

### 2. `pane-*` 개념은 현재 활성 UI 기준으로 레거시다

현재 UI는 `.grid-cell`, `.terminal-view`, `WorkspaceTabBar`, `MosaicContainer` 중심입니다. `.pane-leaf`, `.pane-resizer`, `.status-prefix` 같은 선택자는 오래된 테스트/구조에서 온 것입니다.

### 3. 터미널은 이제 "화면 컴포넌트 = runtime"이 아니다

현재는 `TerminalRuntimeLayer`가 실제 xterm 인스턴스를 유지하고, tab/grid는 host slot만 바꿉니다. 탭 전환이나 그리드 전환 시 컴포넌트가 다르게 보인다고 해서 같은 세션의 runtime이 새로 생기는 구조가 아닙니다.

### 4. 로컬 데이터가 자주 더럽다

다음 경로는 로컬 실행 결과나 진단 산출물일 수 있으므로 커밋 전에 구분해야 합니다.

- `server/data/workspaces.json`
- `server/data/workspaces.json.bak`
- `server/data/totp.secret`
- `server/data/snapshots/`
- `server/data/buffer-dump/`
- `server/server/`
- `.playwright-mcp/`

### 5. 설정 파일의 모든 키가 live는 아니다

- `pty.bufferFilter`는 현재 no-op
- `auth.localhostPasswordOnly`는 설정 파일로만 관리되고 설정 UI에는 없음
- `fileManager.maxCodeFileSize`는 제외 필드
- `pty.maxBufferSize`는 레거시 alias

## 권장 검증 순서

### 기본 검증

1. `server`에서 `npm test`
2. `frontend`에서 `npm run build`

### UI 변경 시 추가 검증

- 검증 대상은 반드시 `https://localhost:2002`
- 헬스체크는 `curl -k https://localhost:2002/health`
- Playwright는 `frontend/playwright.config.ts` 기준으로 실행

### 터미널 런타임 변경 시 추가 검증

- 워크스페이스 전환
- `tab -> grid`
- `grid -> tab`
- splitter drag
- 재시작 후 snapshot lineage
- PowerShell 기반 입력/echo/backspace

## 디버깅 도구

### 서버 관측

- `GET /api/sessions/telemetry`
- `GET /api/sessions/debug-capture/:id`
- `POST /api/sessions/debug-capture/:id/enable`
- `DELETE /api/sessions/debug-capture/:id`

debug capture는 localhost 요청만 허용됩니다.

### 브라우저 쪽 터미널 디버그

클라이언트는 `window.__buildergateTerminalDebug` 전역 스토어를 노출합니다.

예:

- `window.__buildergateTerminalDebug.start(sessionId)`
- `window.__buildergateTerminalDebug.getEvents(sessionId)`

이 기능은 최근 Playwright 회귀 테스트에서도 사용됩니다.
