## 개요

세션을 추가하는 4개 경로의 셸 선택 UX를 통일한다.  
현재 WorkspaceSidebar만 셸 선택 ContextMenu를 지원하고, 나머지 3개 경로(TabBar, 그리드 컨텍스트 메뉴, EmptyState/EmptyCell)는 기본 셸로만 세션을 생성한다.

### 목표 UX 요약

| 경로 | 클릭 | 롱프레스 |
|------|------|---------|
| 워크스페이스 사이드바 | 셸 선택 메뉴 (현행 유지) | — |
| 탭바 + 버튼 | 현재 활성 탭과 같은 셸 | 셸 선택 메뉴 |
| 그리드 컨텍스트 메뉴 | 해당 셀과 같은 셸 | (컨텍스트 메뉴 내 서브메뉴로 처리) |
| 그리드 EmptyCell | 기본 셸 | 셸 선택 메뉴 |
| 빈 화면 EmptyState | 기본 셸 | 셸 선택 메뉴 |

### CWD 처리 규칙

| 경우 | CWD |
|------|-----|
| 클릭 (같은 셸) | 현재 탭/셀의 CWD 전달 |
| 롱프레스 → 같은 계열 셸 선택 | 현재 탭/셀의 CWD 전달 |
| 롱프레스 → 다른 계열 셸 선택 | `~/` (홈 디렉토리, cwd 생략) |

**셸 계열 분류**:
- Windows: `powershell`, `cmd`
- Unix: `bash`, `zsh`, `sh`, `wsl`, `auto`

## 롱프레스 구현 전략

**결정: 커스텀 훅 `useLongPress` 신규 생성**

이유:
- `useDragReorder`는 드래그/드롭 상태 관리와 결합되어 있어 "롱프레스 후 셸 메뉴" 단독 용도로 재사용하기 어렵다.
- `useDragReorder` 내부의 `longPressMs` 타이머 로직은 이미 드래그 시작 전 취소 조건(포인터 이동 5px 이상)이 내재되어 있어, 단순히 롱프레스만 감지하는 독립 훅을 만드는 것이 더 명료하다.
- 3개 컴포넌트(WorkspaceTabBar, EmptyCell, EmptyState)에서 동일한 패턴이 필요하므로 훅으로 추출이 타당하다.

**`useLongPress` 훅 명세**:
```typescript
interface UseLongPressOptions {
  onLongPress: (event: React.PointerEvent) => void;
  onPress?: (event: React.PointerEvent) => void;  // 일반 클릭 (pointerup 시 short press)
  delay?: number;  // 기본 500ms
  moveThreshold?: number;  // 포인터 이동 취소 임계값, 기본 5px
}

function useLongPress(options: UseLongPressOptions): {
  onPointerDown: (e: React.PointerEvent) => void;
}
```

동작:
1. `pointerdown` → 타이머 시작
2. 포인터 이동 > moveThreshold → 타이머 취소 (드래그 허용)
3. delay 내 `pointerup` → 일반 클릭(short press)으로 처리
4. delay 경과 → `onLongPress` 호출

여기서 `getSameFamilyCwd` 헬퍼:
```typescript
function getShellFamily(shell: string): 'windows' | 'unix' {
  return ['powershell', 'cmd'].includes(shell) ? 'windows' : 'unix';
}

function getSameFamilyCwd(selectedShell: string, activeTab?: WorkspaceTabRuntime): string | undefined {
  if (!activeTab?.cwd) return undefined;
  const currentFamily = getShellFamily(activeTab.shellType);
  const selectedFamily = getShellFamily(selectedShell);
  return currentFamily === selectedFamily ? activeTab.cwd : undefined;
}
```

이 헬퍼는 `frontend/src/utils/shell.ts`에 배치한다.

### Phase 1: 공통 유틸리티 및 훅 생성

**변경 파일**: 없음 (신규 파일 2개)

#### 1-1. `frontend/src/utils/shell.ts` 신규 생성

```typescript
import type { ShellType } from '../types';

export type ShellFamily = 'windows' | 'unix';

const WINDOWS_SHELLS: ShellType[] = ['powershell', 'cmd'];

export function getShellFamily(shell: string): ShellFamily {
  return WINDOWS_SHELLS.includes(shell as ShellType) ? 'windows' : 'unix';
}

/**
 * 선택된 셸이 현재 탭과 같은 계열이면 cwd를 반환, 다른 계열이면 undefined 반환.
 */
export function resolveCwd(
  selectedShell: string,
  currentShell: string,
  currentCwd: string | undefined
): string | undefined {
  if (!currentCwd) return undefined;
  return getShellFamily(selectedShell) === getShellFamily(currentShell)
    ? currentCwd
    : undefined;
}
```

#### 1-2. `frontend/src/hooks/useLongPress.ts` 신규 생성

```typescript
import { useRef, useCallback } from 'react';

interface UseLongPressOptions {
  onLongPress: (e: React.PointerEvent) => void;
  delay?: number;
  moveThreshold?: number;
}

export function useLongPress({ onLongPress, delay = 500, moveThreshold = 5 }: UseLongPressOptions) {
  const timerRef = useRef<ReturnType<typeof setTimeout> | null>(null);
  const startPos = useRef({ x: 0, y: 0 });
  const firedRef = useRef(false);

  const cancel = useCallback(() => {
    if (timerRef.current) {
      clearTimeout(timerRef.current);
      timerRef.current = null;
    }
    firedRef.current = false;
  }, []);

  const onPointerDown = useCallback((e: React.PointerEvent) => {
    if (e.button !== 0) return;
    startPos.current = { x: e.clientX, y: e.clientY };
    firedRef.current = false;

    const capturedE = e;

    const handleMove = (ev: PointerEvent) => {
      const dx = ev.clientX - startPos.current.x;
      const dy = ev.clientY - startPos.current.y;
      if (Math.abs(dx) > moveThreshold || Math.abs(dy) > moveThreshold) {
        cancel();
        window.removeEventListener('pointermove', handleMove);
        window.removeEventListener('pointerup', handleUp);
      }
    };

    const handleUp = () => {
      cancel();
      window.removeEventListener('pointermove', handleMove);
      window.removeEventListener('pointerup', handleUp);
    };

    window.addEventListener('pointermove', handleMove);
    window.addEventListener('pointerup', handleUp);

    timerRef.current = setTimeout(() => {
      timerRef.current = null;
      firedRef.current = true;
      window.removeEventListener('pointermove', handleMove);
      window.removeEventListener('pointerup', handleUp);
      onLongPress(capturedE);
    }, delay);
  }, [onLongPress, delay, moveThreshold, cancel]);

  return { onPointerDown };
}
```

### Phase 4: EmptyCell 수정

**변경 파일**: `frontend/src/components/Grid/EmptyCell.tsx`

#### 변경 내용

1. Props 인터페이스 변경:
   ```typescript
   interface Props {
     onAdd: (shell?: string) => void;
     availableShells?: ShellInfo[];
   }
   ```

2. `useLongPress` 적용:
   - 클릭: `onAdd()` (기본 셸)
   - 롱프레스: 셸 선택 메뉴 표시

3. EmptyCell은 현재 onClick만 있으므로, 클릭 이벤트를 `pointerdown` 기반으로 전환해야 한다.  
   단, onClick 이벤트와 useLongPress의 onPointerDown을 함께 사용하면 충돌 가능성이 있으므로, onClick을 제거하고 useLongPress의 onPointerDown만 사용하되 short-press(pointerup 내 delay 미만)는 onLongPress가 아닌 일반 클릭으로 처리한다.

   `useLongPress` 훅에 `onClick` 옵션 추가 (Phase 1에서 같이 설계):
   ```typescript
   interface UseLongPressOptions {
     onLongPress: (e: React.PointerEvent) => void;
     onClick?: () => void;  // short press 핸들러
     delay?: number;
     moveThreshold?: number;
   }
   ```

   훅 내부에서 `pointerup`이 delay 이전에 발생하면 `onClick()` 호출.
