## Phase 2: Runtime Load Normalization And PTY Policy Hardening

### 목표

기존 stale config가 있더라도 non-Windows 부팅이 실패하지 않게 하고, PTY platform 정책이 runtime 경계에서 일관되게 재사용되도록 만든다.

## 선행 조건

- Phase 1 완료

## 기존 코드 재사용 힌트

- `server/src/utils/config.ts`
  - `loadConfig()`가 normalization을 적용할 가장 안전한 진입점이다.
- `server/src/services/SessionManager.ts`
  - `assertRuntimePtyCapabilities()`, `updateRuntimeConfig()`, `resolveShell()`을 방어 계층으로 재사용한다.
- `server/src/test-runner.ts`
  - 기존 non-Windows runtime validation 테스트를 확장해 stale config load와 normalization 시나리오를 추가한다.

## 코드 중복 주의

- IMPL-002 / IMPL-004
  - platform check를 `config.ts`, `SessionManager`, `SettingsService`, `RuntimeConfigStore`에 다시 복붙하지 않는다.
- IMPL-003
  - shell fallback 로직을 bootstrap normalization과 spawn 시점에 따로따로 다르게 구현하지 않는다.

## 구현 체크리스트

- [ ] Phase 2-1: PTY platform 정책 helper 또는 동등한 단일 정책 소스를 도입한다 `IMPL-004`
- [ ] Phase 2-2: `loadConfig()`에서 non-Windows stale PTY 값을 정규화한다 `IMPL-002`
- [ ] Phase 2-3: non-Windows file shell 값이 `powershell`/`wsl`/`cmd`일 때 안전한 값으로 정규화한다 `IMPL-003`
- [ ] Phase 2-4: `SessionManager.assertRuntimePtyCapabilities()`와 `updateRuntimeConfig()`가 같은 정책 소스를 재사용하도록 정리한다 `IMPL-004`
- [ ] Phase 2-5: `createSession()`/spawn 경로가 상위 normalization 실패 시에도 defensive 하게 동작하도록 보강한다 `IMPL-003`
- [ ] Phase 2-6: 로드 시 normalization은 우선 메모리 적용으로 끝내고 자동 파일 rewrite는 하지 않는 정책을 명문화한다 `IMPL-002`

## 언어+버전 주의사항

- ESM import cycle을 만들지 않도록 helper 위치를 조심한다.
- `config.ts`는 process-wide singleton config를 export하므로 normalization은 deterministic 해야 한다.

## LLM 코딩 주의사항

- Simplicity: runtime-safe normalization만 구현하고 config migration framework까지 만들지 않는다.
- Surgical: `SessionManager` 전체를 리팩터링하지 말고 policy seam만 추출한다.

## 테스트 시나리오

### 정상 케이스

- Given: non-Windows에서 `useConpty=true`가 들어 있는 기존 config 파일이 있다
  When: 서버가 부팅된다
  Then: config는 런타임에서 정규화되고 서버는 정상 기동한다

- Given: non-Windows에서 `windowsPowerShellBackend='conpty'`인 기존 config 파일이 있다
  When: 서버가 부팅된다
  Then: backend override는 `'inherit'`로 정규화되고 startup hard-fail이 발생하지 않는다

### 예외 케이스

- Given: non-Windows에서 `shell='powershell'`인 기존 config 파일이 있다
  When: 새 세션이 생성된다
  Then: Windows executable spawn 시도 대신 안전한 fallback shell 정책이 적용된다

- Given: Windows에서 `useConpty=true` 기본 정책이다
  When: 서버가 부팅된다
  Then: 기존 Windows 선호 동작은 유지된다

### 경계값 테스트

- Given: legacy `maxBufferSize`가 남아 있는 config 파일이다
  When: load normalization이 수행된다
  Then: PTY normalization은 legacy buffer key 호환성과 충돌하지 않는다
