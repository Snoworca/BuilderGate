# Tech Decisions

## ADR-2201: 저장소 공용 예제와 실제 생성 설정을 분리한다

### 상태

Accepted

### 결정

- 저장소에 있는 `server/config.json5.example`는 플랫폼 중립 예제로 유지한다.
- 실제 `server/config.json5` 자동 생성 시점에만 OS별 PTY 기본값을 적용한다.

### 근거

- 예제 파일은 모든 플랫폼 사용자가 보게 되는 공용 기준점이다.
- Windows 최적값을 공용 예제에 박아 두면 non-Windows 부팅 회귀가 반복된다.
- 사용자는 이미 Windows에서 `ConPTY` 선호 정책을 원하므로, 실제 생성 파일에서는 Windows 최적값을 계속 줄 수 있다.

## ADR-2202: non-Windows stale PTY 값은 로드 시 정규화한다

### 상태

Accepted

### 결정

- `loadConfig()` 경계에서 non-Windows용 PTY normalization을 적용한다.
- 최소 대상:
  - `useConpty -> false`
  - `windowsPowerShellBackend -> 'inherit'`
  - `shell in ['powershell', 'wsl', 'cmd'] -> 'auto'`

### 근거

- 초기 생성만 고쳐서는 이미 잘못된 `config.json5`를 가진 사용자까지 복구되지 않는다.
- 현재 부팅 실패는 세션 생성 이전의 bootstrap validation에서 터지므로, 로드 경계가 가장 안전한 seam이다.

## ADR-2203: 로드 시 자동 파일 rewrite는 이번 단계에서 하지 않는다

### 상태

Accepted

### 결정

- normalization은 우선 메모리 기준으로만 적용한다.
- `config.json5` 원본을 조용히 rewrite하지 않는다.

### 근거

- `ConfigFileRepository`는 comment/legacy key 보존 성격이 강하다.
- 자동 rewrite는 예기치 않은 diff와 운영 혼란을 만들 수 있다.
- 부팅 안정성 확보가 이번 단계의 핵심이며, 파일 migration은 후속 작업으로 분리 가능하다.

## ADR-2204: PTY platform 정책은 공유 helper로 수렴한다

### 상태

Accepted

### 결정

- capability, validation, bootstrap defaults, runtime assertions가 같은 정책 helper를 재사용하도록 설계한다.

### 근거

- 현재 정책이 `RuntimeConfigStore`, `SettingsService`, `SessionManager`, `config.ts`에 흩어져 있다.
- 동일 정책이 여러 군데 복사되면 다시 drift가 생긴다.

## ADR-2205: Settings는 OS와 맞지 않는 Windows 전용 옵션을 숨긴다

### 상태

Accepted

### 결정

- non-Windows에서 `Use ConPTY`, `PowerShell backend`는 표시하지 않는다.
- 숨겨진 값 때문에 save validation이 실패하는 구조는 제거한다.

### 근거

- 사용자가 직접 제안한 UX 방향이다.
- 이미 capability 기반 숨김 구조가 있어 확장 비용이 낮다.
- 숨긴 값을 backend가 다시 validation error로 잡으면 UX가 자기모순이 된다.

## ADR-2206: 새로운 shell 엔진 추가는 범위 밖으로 두되, 기존 런타임 지원 shell 계약은 정렬한다

### 상태

Accepted

### 결정

- 이번 Step22는 cross-platform PTY bootstrap과 Windows-only 옵션 정렬에 집중한다.
- 다만 이미 런타임이 지원하던 `cmd`/`zsh`/`sh`는 host capability 기준으로 Settings/session route 계약과 맞춘다.
- 새로운 shell provider 추가나 shell UX 재설계는 후속 과제로 분리한다.

### 근거

- 현재 핵심 장애는 shell 확장보다 bootstrap 부팅 실패다.
- 범위를 넓히면 검증 매트릭스가 급격히 커진다.
