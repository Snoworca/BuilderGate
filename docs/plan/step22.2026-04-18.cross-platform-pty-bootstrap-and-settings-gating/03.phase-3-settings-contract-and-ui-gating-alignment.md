## Phase 3: Settings Contract And UI Gating Alignment

### 목표

non-Windows에서 Windows 전용 PTY 옵션을 숨기고, 숨겨진 값이나 stale 값 때문에 Settings 저장이 실패하지 않도록 capability/UI/save 계약을 정렬한다.

## 선행 조건

- Phase 2 완료

## 기존 코드 재사용 힌트

- `server/src/services/RuntimeConfigStore.ts`
  - `buildFieldCapabilities()`를 그대로 capability 기준점으로 사용한다.
- `server/src/services/SettingsService.ts`
  - `validatePlatformPatch()`와 patch schema를 platform policy와 정렬하는 방향으로 재사용한다.
- `frontend/src/components/Settings/SettingsPage.tsx`
  - 이미 capability 기반 conditional render가 있으므로 확장 비용이 낮다.

## 코드 중복 주의

- IMPL-005 / IMPL-006
  - frontend에서 별도 OS 판별 로직을 만들지 말고 서버 capability snapshot을 기준으로 숨긴다.
- IMPL-004
  - shell/backend allowed values를 서버 타입, settings schema, frontend 타입에 각각 따로 유지하지 않는다.

## 구현 체크리스트

- [ ] Phase 3-1: `RuntimeConfigStore` capability가 PTY platform helper를 재사용하도록 정리한다 `IMPL-004`, `IMPL-005`
- [ ] Phase 3-2: `SettingsService` patch validation이 capability와 같은 allowed values를 보도록 정리한다 `IMPL-004`, `IMPL-006`
- [ ] Phase 3-3: non-Windows에서 `Use ConPTY`, `PowerShell backend` UI가 확실히 숨겨지도록 검증한다 `IMPL-005`
- [ ] Phase 3-4: 숨겨진/stale PTY 값 때문에 unrelated settings save가 막히지 않도록 payload 또는 validation 계약을 보강한다 `IMPL-006`
- [ ] Phase 3-5: frontend settings 타입과 backend settings schema의 shell/backend 계약을 이번 범위 안에서 최소 일치 상태로 맞춘다 `IMPL-004`

## 언어+버전 주의사항

- React 19 코드베이스이며 local state와 snapshot state가 함께 존재한다.
- Settings save failure 복구 경로가 이미 있으므로, capability refresh 후 state 보정 흐름을 깨지 말아야 한다.

## LLM 코딩 주의사항

- Simplicity: OS 판별을 프런트에서 다시 하지 말고 capability snapshot만 신뢰한다.
- Surgical: Settings 페이지 전체를 재구성하지 말고 PTY 관련 field와 save contract만 손댄다.

## 테스트 시나리오

### 정상 케이스

- Given: non-Windows capability snapshot이다
  When: Settings 페이지가 렌더링된다
  Then: `Use ConPTY`와 `PowerShell backend` 필드는 표시되지 않는다

- Given: stale PTY 값이 있는 기존 설정이 로드되었다
  When: 사용자가 unrelated setting만 수정해 저장한다
  Then: 숨겨진 PTY 값 때문에 save validation이 실패하지 않는다

### 예외 케이스

- Given: Windows이고 winpty probe가 실패했다
  When: Settings 페이지를 연다
  Then: 축소된 backend 옵션과 save validation이 서로 모순되지 않는다

### 경계값 테스트

- Given: capability snapshot이 non-Windows인데 draft에 Windows 전용 값이 남아 있다
  When: draft reconcile 또는 save error recovery가 실행된다
  Then: UI는 회복 가능한 상태를 유지하고 self-block 되지 않는다
