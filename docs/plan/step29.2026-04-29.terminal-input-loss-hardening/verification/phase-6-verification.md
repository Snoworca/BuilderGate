## Regression Fixes Found During Phase 6

- `TerminalRuntimeProvider` was unmounted when switching to an empty workspace because the provider was gated by `activeWorkspaceTabs.length > 0`. This disposed parked terminal runtimes during rapid workspace handoff. The provider now remains mounted for an active workspace, while `EmptyState` renders inside the runtime stage when the active workspace has no tabs.
- `TerminalHostSlot` now performs an additional animation-frame measurement after visibility/layout changes. This prevents a visible terminal host from remaining parked after fast workspace transitions.
- `terminal-authority.spec.ts` now creates a fresh source workspace for handoff preservation and evicts only known test workspaces when the workspace limit is reached.
- `grid-equal-mode.spec.ts` now shares the same test-workspace cleanup policy and waits for the initial grid visible-repair timer before assertions that are specifically about idle/manual repair behavior.

## Rollout Decision

## Feature Flag And Rollback

- Server mode: `BUILDERGATE_INPUT_RELIABILITY_MODE=observe|queue|strict`.
- Frontend localhost override: `window.__buildergateTerminalDebug.setInputReliabilityMode('observe' | 'queue' | 'strict' | null)`.
