# Phase 1: Observability and Reproduction Baseline

## Phase 1 목표

동작 변경을 최소화하고, 빠른 입력 손실을 어디서 구분할 수 있는지 먼저 만든다. 이 Phase가 끝나면 "브라우저/xterm에서 사라졌는지", "프론트 gate에서 버렸는지", "WS 전송에서 사라졌는지", "서버 replay에서 막혔는지", "PTY write 후 화면 redraw가 늦은 것인지"를 구분할 수 있어야 한다.

## 구현 범위

### Phase 1-0: input reliability mode plumbing

대상:
- `server/src/utils/config.ts` 또는 기존 runtime config 로딩 지점
- `server/src/index.ts`의 frontend runtime config 전달 지점이 있다면 해당 경로
- `frontend/src/contexts/WebSocketContext.tsx` 또는 runtime config 소비 지점
- terminal debug/test helper

작업:
- `inputReliabilityMode: 'observe' | 'queue' | 'strict'` 타입을 정의한다.
- 기본값은 `observe`다.
- 서버 env `BUILDERGATE_INPUT_RELIABILITY_MODE`를 읽되, 알 수 없는 값은 startup error 또는 명시적 warning 후 `observe`로 처리한다. 조용한 coercion은 금지한다.
- localhost debug/test에서만 localStorage override를 허용한다.
- Phase 2~5 테스트가 `queue` mode를 명시적으로 켤 수 있는 helper를 제공한다.
- UI 설정 화면은 추가하지 않는다.

완료 조건:
- Phase 2 시작 전에 코드에서 현재 mode를 읽고 debug capture에 기록할 수 있다.
- mode가 없어서 Phase 2 queue 동작을 테스트하지 못하는 상태가 아니어야 한다.

### Phase 1-1: privacy-safe input metadata 유틸 추가

대상:
- `frontend/src/utils/terminalDebugCapture.ts`
- `server/src/services/SessionManager.ts` 또는 별도 shared 불가 시 각 레이어에 동일 의미의 작은 함수

추가 metadata:
- `captureSeq` for browser/xterm event tape
- `inputSeqStart`/`inputSeqEnd` only when already provided by upstream transport in later phases
- `byteLength`
- `codePointCount`
- `graphemeCount`
- `graphemeApproximate`
- `hasHangul`
- `hasCjk`
- `hasEnter`
- `controlCount`
- `printableCount`
- `inputClass`
- `safePreview`

규칙:
- printable 원문은 저장하지 않는다.
- safe control preview는 기존 정책을 유지한다.
- `Intl.Segmenter`가 있으면 grapheme count에 사용하고, 없으면 code point count로 fallback한다.

### Phase 1-2: captureSeq 생성 및 debug capture 연결

대상:
- `TerminalView`
- `TerminalContainer`
- `WebSocketContext`
- `WsRouter`
- `SessionManager`

작업:
- Phase 1에서는 `TerminalView`에 `captureSeqRef`를 추가하고 xterm/helper textarea event tape에만 사용한다.
- sequence range는 아직 생성하지 않는다. `inputSeqStart`/`inputSeqEnd`의 단일 소유자는 Phase 4의 `TerminalContainer.InputSequencer`다.
- `xterm_data_emitted`, `xterm_data_dropped_not_ready`, `ws_input_sent`, `ws_input_dropped_not_ready`, `input_blocked`, server `input` debug event에 원문 없는 byte/count/class metadata를 포함한다.
- Phase 3 protocol 변경 전까지는 `captureSeq`, count, UTF-8 byte length, timestamp, sessionId로 browser/server 상관관계를 추적한다.
- server가 legacy 또는 future client로부터 sequence range를 받지 못한 경우에도 임의 sequence를 만들지 않는다.

### Phase 1-3: helper textarea event tape

대상:
- `TerminalView`
- `terminalDebugCapture`

debug capture가 켜진 경우에만 아래 event를 ring buffer에 기록한다.

- `keydown`
- `beforeinput`
- `input`
- `compositionstart`
- `compositionupdate`
- `compositionend`
- `xterm.onData`

각 event에 포함:
- `captureSeq` 또는 `compositionSeq`
- `eventType`
- `safeKeyName` for keydown only when the key is in the explicit control/navigation allowlist
- `keyCategory`, `keyUtf8ByteLength`, `keyHasHangul`, `keyHasCjk` for printable keydown
- `keyCode` only for IME sentinel values such as `229`
- `inputType` for input/beforeinput
- `dataLength`, `hasHangul`, `hasCjk`
- `inputReady`
- `captureState`가 아직 없으면 `inputReady`
- `serverReady`
- `geometryReady`
- `restorePending`
- `visible`
- `helperDisabled`
- `helperReadOnly`
- `isComposingRef`
- `activeElementIsHelper`

주의:
- event.data 원문은 저장하지 않는다.
- printable `KeyboardEvent.key`와 printable `KeyboardEvent.code`는 저장하지 않는다. `key`는 실제 입력 문자일 수 있고, `code`도 비밀번호/명령 입력 패턴을 유추하게 만들 수 있다.
- control/navigation allowlist 예: `Enter`, `Backspace`, `Tab`, `Escape`, `ArrowLeft`, `ArrowRight`, `ArrowUp`, `ArrowDown`, `Home`, `End`, `PageUp`, `PageDown`, `Delete`, `Insert`.
- `compositionupdate`는 빈번하므로 debug capture enabled일 때만 기록한다.

### Phase 1-4: 현재 hard drop 지점 관측 강화

대상:
- `TerminalView.term.onData`
- `TerminalContainer.handleInput`
- `WebSocketContext.send`
- `WsRouter.handleInput`

작업:
- printable/non-ASCII 입력도 원문 없이 drop event가 남도록 한다.
- `WebSocketContext.send`가 socket closed로 no-op될 때 `ws_send_rejected_not_open` debug event를 남기는 경로를 준비한다.
- Phase 1에서 API를 바꾸기 어렵다면 `send()` 내부 console/debug capture만 추가하고, Phase 4에서 반환값을 바꾼다.
- debug/replay detail은 primitive flat field만 사용한다. nested `gateState` 객체를 넣지 않는다.

## 코드 중복 주의

- 기존 `buildTerminalInputDebugPayload()`를 확장하거나 내부 helper를 추출한다.
- `SessionManager`의 `buildInputDebugDetails()`와 의미가 다르면 안 된다. 필요하면 Phase 1에서 별도 문서 주석으로 frontend/server metadata field correspondence를 남긴다.
- debug capture endpoint는 이미 `server/src/index.ts`에 있으므로 새 endpoint를 만들지 않는다.
- byte length는 browser `TextEncoder`, Node `Buffer.byteLength(data, 'utf8')` 기준으로 계산한다.

## 테스트 시나리오

### Unit

- Hangul printable input `"가나다"`는 preview 없이 metadata만 남는다.
- ASCII safe control input `" \r\x7f"`는 기존 preview를 유지한다.
- mixed input `"가\r"`는 원문 없이 `hasEnter=true`, `hasHangul=true`, `inputClass=mixed-printable-control`로 기록된다.

주의:
- 현재 frontend에는 unit test runner가 없다. 프론트 helper를 pure TS로 분리해 테스트하려면 먼저 명시적 test script를 추가하거나, 같은 내용을 Playwright debug hook으로 결정적으로 검증한다.
- 기존 테스트 인프라가 없는 상태에서 "Unit 통과"라고 표시하지 않는다.

### E2E

- `terminal-keyboard-regression.spec.ts`
  - debug capture start 후 `A`, `Space`, `Enter` 입력의 event sequence가 `xterm_data_emitted -> ws_input_sent -> server input -> raw_output` 순서로 관측된다.
- `terminal-korean-ime.spec.ts`
  - synthetic composition event tape가 `compositionstart -> compositionupdate* -> compositionend`를 기록한다.

## Phase 1 완료 게이트

- [ ] printable/Hangul 입력 원문이 debug capture에 저장되지 않는다.
- [ ] `inputReliabilityMode` 기본값과 test override가 존재한다.
- [ ] `xterm_data_dropped_not_ready`, `ws_input_dropped_not_ready`, `input_blocked`, `ws_send_rejected_not_open` 중 어느 경로든 metadata로 관측 가능하다.
- [ ] sequence range는 Phase 1에서 새로 생성되지 않으며, browser event tape는 `captureSeq`를 사용한다.
- [ ] 모든 byte field가 UTF-8 byte length 기준이다.
- [ ] 기존 safe-control debug test가 깨지지 않는다.
- [ ] `npm --prefix server run test`
- [ ] 관련 Playwright: `terminal-keyboard-regression.spec.ts`, `terminal-korean-ime.spec.ts`

## LLM 구현 주의

- 관측성 Phase에서 입력 전달 동작을 바꾸지 말 것.
- 이벤트 원문을 로그에 넣지 말 것.
- debug capture disabled 상태의 runtime overhead를 최소화할 것.
