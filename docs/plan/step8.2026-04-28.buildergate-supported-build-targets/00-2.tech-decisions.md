# 전체 지원 대상 빌드 기술 결정

## ADR-001: `npm run build`는 전체 지원 대상 5개를 생성한다

| 항목 | 결정 |
|---|---|
| 상태 | Accepted |
| 맥락 | SRS는 Windows/Linux amd64를 필수로 요구하고 ARM64를 추가 지원으로 둔다. 사용자는 `npm run build`가 전체 지원 대상을 호출하기를 원했다. |
| 결정 | `build`와 `build:daemon-all`은 `--all-supported`를 호출한다. `--all-supported`는 `win-amd64`, `linux-amd64`, `win-arm64`, `linux-arm64`, `macos-arm64`를 생성한다. |
| 영향 | 기본 빌드 시간이 증가하지만 release build 의미가 명확해진다. |

## ADR-002: 사용자-facing profile은 `amd64`, 내부 pkg/npm arch는 `x64`를 사용한다

| 항목 | 결정 |
|---|---|
| 상태 | Accepted |
| 맥락 | 사용자와 SRS는 `amd64`를 사용하지만 `pkg` target과 npm `--cpu`는 `x64`를 사용한다. |
| 결정 | profile name은 `win-amd64`, `linux-amd64`로 두고 target metadata의 `arch`는 `x64`로 둔다. |
| 검증 | resolver test에서 profile, pkg target, npm install `--cpu x64`를 확인한다. |

## ADR-003: `build:daemon-exe` 단일 기본 output은 유지한다

| 항목 | 결정 |
|---|---|
| 상태 | Accepted |
| 맥락 | SRS의 `AC-8-012A`는 `npm run build:daemon-exe`와 `node tools/build-daemon-exe.js`가 기존 `dist/bin` 호환 경로를 유지해야 한다. |
| 결정 | 단일 기본 build path는 변경하지 않고, 복수 대상 build만 `dist/bin/<target>` 구조를 사용한다. |
| 영향 | 기존 로컬 단일 패키징 흐름과 신규 release build 흐름이 공존한다. |

## ADR-004: Linux는 SVG만 필수 icon artifact로 검증한다

| 항목 | 결정 |
|---|---|
| 상태 | Accepted |
| 맥락 | SRS는 Linux amd64/ARM64 배포본에 `BuilderGate.svg`를 요구한다. Windows는 `.ico`, macOS는 `.icns`와 `.app` 적용이 필요하다. |
| 결정 | `validateBuildOutput()`은 공통 `BuilderGate.svg`, Windows `BuilderGate.ico`, macOS `BuilderGate.icns` 및 `BuilderGate.app`을 플랫폼별 필수로 검증한다. Linux에서 `.ico`/`.icns` 존재를 필수로 보지 않는다. |
| 영향 | 실제 copy helper가 추가 파일을 생성해도 Linux 검증은 SRS와 일치한다. |

## ADR-005: macOS는 ARM64만 지원하고 `.app` bundle은 필수다

| 항목 | 결정 |
|---|---|
| 상태 | Accepted |
| 맥락 | SRS는 macOS ARM64 산출물이 raw 실행파일과 `BuilderGate.app`을 함께 포함해야 한다고 명시한다. |
| 결정 | `macos-arm64`만 macOS profile로 유지하고 `validateBuildOutput()` 및 tests에서 `.app` bundle 구조를 필수 검증한다. |
| 영향 | macOS x64 target은 추가하지 않는다. |

## ADR-006: 기존 `build:daemon-*` ARM64 scripts는 호환 alias로 유지한다

| 항목 | 결정 |
|---|---|
| 상태 | Accepted |
| 맥락 | README와 기존 자동화가 `build:daemon-win-arm64` 계열을 참조할 수 있다. |
| 결정 | 새 표준 script는 `build:windows-amd64`, `build:linux-amd64`, `build:windows-arm64`, `build:linux-arm64`, `build:macos-arm64`이고, 기존 ARM64 script는 alias로 유지한다. |
| 영향 | 신규 문서는 새 script명을 우선 표기하고 기존 호환성은 보존한다. |
