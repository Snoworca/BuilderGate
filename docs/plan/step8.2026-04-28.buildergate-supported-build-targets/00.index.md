# BuilderGate 전체 지원 대상 빌드 구현 계획

## 목표

`npm run build`가 전체 지원 대상 5개(`win-amd64`, `linux-amd64`, `win-arm64`, `linux-arm64`, `macos-arm64`)를 생성하도록 빌드 스크립트, 산출물 검증, README/docs 정책을 SRS와 일치시킨다. Windows/Linux amd64는 필수 지원 대상이고 ARM64는 추가 지원 대상이며, macOS는 ARM64만 지원한다.

## 현재 코드 요약

| 영역 | 현재 상태 | 계획상 조치 |
|---|---|---|
| `package.json` | `build`가 `build:daemon-all`을 호출하고 `build:daemon-all`은 `--all-arm64`만 실행 | `build`와 `build:daemon-all`을 전체 지원 대상 5개로 변경 |
| `tools/build-daemon-exe.js` | ARM64 profile과 `--all-arm64`만 명명된 그룹으로 제공 | amd64 profile, `--all-supported`, `--required-amd64` 추가 |
| `validateBuildOutput()` | `server/dist/public/index.html` 검증이 명시적이지 않고 icon 요구가 플랫폼별로 덜 분리됨 | frontend asset, PM2 부재, 플랫폼별 icon/app bundle 검증 명확화 |
| `tools/daemon/build-daemon-exe.test.js` | ARM64 중심 script/profile 테스트 | 전체 5개 대상과 amd64 필수 대상 테스트로 갱신 |
| `README.md`, `tools/daemon/docs.test.js` | README가 ARM64 3개 기본 빌드를 설명 | 전체 5개 대상과 macOS ARM64-only 설명으로 갱신 |

## 공통 구현 원칙

- 기존 native daemon, stop utility, config path, TOTP QR, PM2 제거 계약은 변경하지 않는다.
- `build:daemon-exe`와 `node tools/build-daemon-exe.js`의 단일 기본 output `dist/bin` 호환 경로를 유지한다.
- `amd64`는 사용자-facing profile 이름이고 `pkg`/npm CPU 값은 `x64`로 매핑한다.
- macOS amd64는 지원 대상에 추가하지 않는다.
- 빌드 output root cleanup이 `dist` root를 지우지 않도록 기존 safe guard를 유지한다.

## 최종 검증 명령

| 범위 | 명령 |
|---|---|
| 빌드 unit/docs | `node --test tools/daemon/build-daemon-exe.test.js tools/daemon/docs.test.js` |
| daemon 전체 회귀 | `npm run test:daemon` |
| docs 정책 | `npm run test:docs` |
| 빠른 빌드 smoke | `node tools/build-daemon-exe.js --required-amd64 --skip-runtime-install` |
| 전체 빌드 | `npm run build` |
| diff hygiene | `git diff --check` |

`npm run build`는 대상별 Node runtime/pkg 다운로드와 5개 패키징을 수행하므로 시간이 오래 걸릴 수 있다. 실행하지 못하면 이유와 대체 검증 결과를 최종 보고에 남긴다.
