# React Mosaic Option A Slot Reorder Patch Plan

Date: 2026-04-22  
Scope: `react-mosaic-component` 내부 patch를 통해 grid mode의 split DnD를 full-cell guide + slot reorder semantics로 전환한다  
Mode: plan only

## Goal

grid mode에서 현재 split 기반 DnD를 slot reorder UX로 치환한다.

핵심 목표:

- 셀 전체 blue guide target 도입
- drop semantics를 split insertion에서 move/insert reorder로 전환
- 이 reorder UX를 `equal` 모드에만 한정
- `equal` 모드에서 reorder 후 canonical grid 재적용
- persistence/reload에서도 reorder 결과 일관성 유지
- Option A 기준으로 vendor patch 관리 경로 확정

## Requirements

- `IMPL-001` drag target guide는 split 조각이 아니라 대상 셀 전체를 파란 반투명 영역으로 표시해야 한다.
- `IMPL-002` drop semantics는 split 생성이 아니라 source 제거 후 target slot 삽입(move)이어야 한다.
- `IMPL-003` `[1,2,3,4,5]`에서 `1 -> 5` 예시는 `[2,3,4,5,1]`로 동작해야 한다.
- `IMPL-004` reorder는 `swap`이 아니라 순서 보존형 `move/insert`로 정의해야 한다.
- `IMPL-005` Option A 기준으로 `react-mosaic-component` 내부 patch/fork 경로를 사용해야 한다.
- `IMPL-006` patch는 실제 runtime에 반영되도록 `lib`/type/CSS 산출물까지 고려해야 한다.
- `IMPL-007` `equal` 모드는 reorder 후 canonical equal grid를 다시 적용해야 한다.
- `IMPL-008` `none/focus/auto` 모드에서는 이 reorder UX가 시작되지 않아야 하며 기존 동작을 유지해야 한다.
- `IMPL-009` reorder 시작점은 셀 body 전체가 아니라 창 이동 버튼/그립(handle)이어야 한다.
- `IMPL-010` desktop에서는 마우스 좌측 버튼 drag로 reorder를 시작하고, drop은 좌측 버튼 release로 정의해야 한다. 우클릭은 기존 컨텍스트 메뉴로 유지해야 한다.
- `IMPL-011` persistence/reload 후에도 reorder 결과와 mode semantics가 유지되어야 한다.
- `IMPL-012` 회귀 테스트는 `1->5`, `5->1`, `middle->middle`, self-drop, persistence를 포함해야 한다.
- `IMPL-013` `target slot index`는 source 제거 전 visible order에서 계산하고, source 제거 후 그 인덱스에 source를 삽입해야 한다.
- `IMPL-014` clean install 후에도 vendor patch가 자동 재적용되어야 한다.
- `IMPL-015` root-level drop path는 reorder mode에서 split semantics를 노출하지 않도록 비활성화 또는 no-op 처리되어야 한다.
- `IMPL-016` touch reorder는 현재 범위 밖이다.
- `IMPL-017` source를 동일 slot에 drop하면 no-op 이어야 한다.
- `IMPL-018` 어떤 셀에도 해당하지 않는 바깥 영역에 drop하면 no-op 이어야 한다.

## Assumptions

1. Option A의 1차 구현은 `patch-package` 기반 vendor patch로 진행한다.
2. v1에서는 “drop on cell = move to that slot index”로 정의하고 before/after split semantics는 도입하지 않는다.
3. `equal` 외 모드에서는 reorder UX를 비활성화하고 기존 semantics를 유지한다.
4. reorder 입력은 그립(handle) 기반 desktop mouse 좌클릭 drag/release로 한정한다.

## Recommended Execution Order

1. patch 관리 방식과 reorder 계약을 먼저 고정한다.
2. vendor library 내부에서 full-cell target과 reorder payload를 구현한다.
3. 앱에서 `equal` 전용 gate와 persistence를 정합시킨다.
4. Playwright/E2E와 수동 검증 포인트를 확장한다.

## Success Criteria

- drag 중 split guide가 아니라 hovered cell 전체가 blue guide로 표시된다.
- drop 후 leaf order가 move semantics로 재배치된다.
- `equal` 모드는 reorder 후 canonical grid를 유지한다.
- `none/focus/auto`에서는 reorder UX가 시작되지 않는다.
- reorder는 그립(handle)에서만 시작된다.
- persistence/reload 후 reordered tree와 mode 상태가 일치한다.
- vendor patch 적용 방식이 저장소 안에서 재현 가능하다.
- `target slot index` 기준 시점과 삽입 규칙이 예시와 일반 케이스에서 일관된다.
- self-drop은 no-op로 유지된다.
- target outside drop은 no-op로 유지된다.

## Out Of Scope

- tab bar / sidebar reorder UX 재설계
- `equal` 외 모드의 reorder UX 확장
- touch long-press reorder 지원
- `react-mosaic-component` 자체 교체
