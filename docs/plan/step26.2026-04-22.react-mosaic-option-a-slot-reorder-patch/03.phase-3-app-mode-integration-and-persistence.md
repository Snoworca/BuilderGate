## Non-Equal Mode Rule

- `none/focus/auto` 모드에서는 Option A reorder branch가 시작되지 않는다.
- 따라서 이 모드들의 기존 topology/ratio semantics는 현재 구현을 그대로 유지한다.
- app layer는 이 모드들에서 reorder drag source를 연결하지 않는다.
- rejected reorder attempt는 persistence와 tree를 다시 쓰지 않는다.
- custom toolbar surface 전체가 draggable이 되지 않도록 vendor wrapper 억제와 app wiring이 같이 필요하다.

## Equal Mode Persistence Rule

- `equal` 모드의 reordered leaf order는 persistence/reload 이후에도 canonical source order여야 한다.
- 이후 equal tree 재구성은 이 persisted reordered order를 기준으로 해야 한다.
- 탭 추가 시에는 persisted reordered order 뒤에 새 tab을 append한다.
- 탭 제거 시에는 persisted reordered order에서 해당 leaf만 제거한다.
- 즉, workspace tab sort order가 persisted equal reorder order를 다시 덮어쓰면 안 된다.
