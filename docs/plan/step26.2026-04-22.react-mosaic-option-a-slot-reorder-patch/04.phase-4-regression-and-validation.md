## Test

### Scenario 1: move semantics are correct

Given a grid of sessions  
When the source cell is dropped on a target cell  
Then the final order matches the rule \"compute target slot from pre-removal visible order, remove source, insert at that slot\" rather than swap

### Scenario 1-1: self-drop is a no-op

Given a grid of sessions  
When the source is dropped on its own slot  
Then the final order does not change

### Scenario 1-2: outside-target drop is a no-op

Given a grid of sessions  
When the source is released outside any valid target cell  
Then the final order does not change

### Scenario 1-3: invalid vendor reset branch restores the pre-drag tree

Given an equal-mode drag that ends without a valid target  
When the vendor drag end path resolves the drop  
Then the previous split reset branch is replaced by an explicit restore path that clears source hide and restores the pre-drag tree

### Scenario 2: right click is not hijacked

Given a grid cell  
When the user right-clicks  
Then the context menu still opens and reorder does not start

### Scenario 2-1: non-primary pointer cannot start reorder

Given a grid cell  
When the user presses a non-primary pointer button  
Then reorder does not start and grid order remains unchanged

### Scenario 2-2: drag only starts from the handle

Given an equal-mode grid  
When the user drags from outside the handle  
Then reorder does not start

### Scenario 3: non-equal modes do not enter reorder

Given `none`, `focus`, or `auto` mode  
When the user drags from the handle  
Then the Option A reorder branch does not start
