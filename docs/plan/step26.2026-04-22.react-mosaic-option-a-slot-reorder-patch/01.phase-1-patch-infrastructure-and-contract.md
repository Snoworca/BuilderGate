# Phase 1: Patch Infrastructure And Contract

## Objective

Option A를 실제로 적용할 수 있는 patch 관리 경로와 reorder 계약을 먼저 고정한다.

## Target Files

- `frontend/package.json`
- `frontend/patches/` 또는 동등한 patch 저장 위치
- `frontend/node_modules/react-mosaic-component/package.json` (참고)
- planning/reference 문서

## Planned Changes

1. `patch-package` 도입과 적용 방식을 확정
2. `frontend/package.json`에 `patch-package` + `"postinstall": "patch-package"` wiring 계획 명시
3. patch 파일명(`frontend/patches/react-mosaic-component+6.1.1.patch`)과 생성 절차 명시
4. runtime 기준 patch 대상(`lib/*.js`, `lib/index.d.ts`, 루트 CSS) 명시
5. 이 저장소에서는 `src -> lib` 재생성 없이 runtime 산출물 patch를 우선한다는 원칙 명시
6. reorder payload와 slot index semantic을 문서/계획에 고정
7. right-click non-reorder 정책 명시

## Implementation Checklist

## Authoring Note

- patch authoring 기본 절차:
  1. `frontend/node_modules/react-mosaic-component/lib/*.js`, `lib/index.d.ts`, `react-mosaic-component.css`를 우선 수정
  2. 필요하면 대응 `src/*.tsx`는 이해 보조용으로만 함께 맞춘다
  3. `frontend`에서 `npx patch-package react-mosaic-component` 실행
  4. 생성된 `frontend/patches/react-mosaic-component+6.1.1.patch`를 저장소에 포함한다
