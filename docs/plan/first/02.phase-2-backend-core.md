### 2.3 SSE (Server-Sent Events) Implementation

- [ ] **2.3.1** SSE 유틸리티 함수 생성 (`server/src/utils/sse.ts`)
  ```typescript
  export function sendSSE(res: Response, event: string, data: object) {
    res.write(`event: ${event}\n`);
    res.write(`data: ${JSON.stringify(data)}\n\n`);
  }

  export function initSSE(res: Response) {
    res.setHeader('Content-Type', 'text/event-stream');
    res.setHeader('Cache-Control', 'no-cache');
    res.setHeader('Connection', 'keep-alive');
    res.flushHeaders();
  }
  ```
- [ ] **2.3.2** SSE 클라이언트 관리 메서드 구현
  ```typescript
  addSSEClient(sessionId: string, res: Response): void
  removeSSEClient(sessionId: string, res: Response): void
  ```
- [ ] **2.3.3** 브로드캐스트 메서드 구현
  ```typescript
  private broadcast(sessionId: string, event: string, data: object): void {
    const sessionData = this.sessions.get(sessionId);
    if (sessionData) {
      sessionData.sseClients.forEach(client => {
        sendSSE(client, event, data);
      });
    }
  }
  ```

## Test Scenarios

### TS-2.1: Create Session

**Given**: 서버가 실행 중
**When**: `POST /api/sessions` 요청
**Then**: 새 세션이 생성되고 세션 정보 반환

```bash
curl -X POST http://localhost:4242/api/sessions \
  -H "Content-Type: application/json"
# Expected: {"id":"uuid","name":"Session-1","status":"idle",...}
```

### TS-2.2: List Sessions

**Given**: 세션이 1개 이상 존재
**When**: `GET /api/sessions` 요청
**Then**: 세션 목록 배열 반환

```bash
curl http://localhost:4242/api/sessions
# Expected: [{"id":"...","name":"Session-1",...}]
```

### TS-2.3: Delete Session

**Given**: 세션 ID가 존재
**When**: `DELETE /api/sessions/:id` 요청
**Then**: 204 No Content 응답, 세션 삭제됨

```bash
curl -X DELETE http://localhost:4242/api/sessions/{id}
# Expected: 204 No Content
```

### TS-2.4: SSE Stream Connection

**Given**: 세션이 존재
**When**: `GET /api/sessions/:id/stream` 요청
**Then**: SSE 연결 수립, 이벤트 수신 가능

```bash
curl -N http://localhost:4242/api/sessions/{id}/stream
# Expected: SSE stream with events
```

### TS-2.5: Command Execution

**Given**: SSE 스트림 연결 상태
**When**: `POST /api/sessions/:id/input` with `{ "data": "echo hello\n" }`
**Then**:
1. status 이벤트: `{ "status": "running" }`
2. output 이벤트: `{ "data": "echo hello\n" }` (echo)
3. output 이벤트: `{ "data": "hello\n" }` (결과)
4. status 이벤트: `{ "status": "idle" }`

```bash
curl -X POST http://localhost:4242/api/sessions/{id}/input \
  -H "Content-Type: application/json" \
  -d '{"data":"echo hello\n"}'
```

### TS-2.6: Status Change Detection

**Given**: 세션이 idle 상태
**When**: 명령 실행 시작
**Then**: running → idle 상태 전환이 SSE로 브로드캐스트됨
