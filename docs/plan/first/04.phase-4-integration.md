## Test Scenarios (E2E)

### TS-4.1: Complete User Flow

**Given**: 앱이 처음 로드됨 (세션 없음)
**When**:
1. "New Session" 클릭
2. `pwd` 입력
3. `ls` 입력
4. 새 세션 생성
5. 첫 세션 삭제

**Then**:
1. 세션 생성 성공
2. 현재 디렉토리 출력
3. 파일 목록 출력
4. 두 번째 세션 생성 및 선택
5. 첫 세션 제거, 두 번째 세션 유지

### TS-4.2: Status Indicator Sync

**Given**: 세션이 idle 상태 (초록색)
**When**: 장시간 명령 실행 (`sleep 3` on Unix, `timeout 3` on Windows)
**Then**:
1. 즉시 빨간색으로 변경
2. 3초 후 초록색으로 복귀

### TS-4.3: Session Isolation

**Given**: 세션 A, B 존재
**When**:
1. A 선택 → `export TEST=A` (또는 Windows: `set TEST=A`)
2. B 선택 → `echo $TEST` (또는 Windows: `echo %TEST%`)

**Then**: B에서는 TEST 변수가 비어있음 (세션 격리 확인)

### TS-4.4: Rapid Input

**Given**: 세션 선택됨
**When**: 빠르게 여러 문자 타이핑
**Then**: 모든 문자가 순서대로 터미널에 표시됨

### TS-4.5: Large Output

**Given**: 세션 선택됨
**When**: `seq 1 1000` (Unix) 또는 대량 출력 명령 실행
**Then**:
1. 모든 출력이 표시됨
2. 스크롤 가능
3. UI 프리징 없음
