## Implementation Checklist

### 1.1 Backend Project Setup (server/)

- [ ] **1.1.1** `server/` 디렉토리 생성
- [ ] **1.1.2** `npm init` 실행 및 package.json 설정
  ```json
  {
    "name": "claude-web-shell-server",
    "version": "0.1.0",
    "type": "module",
    "scripts": {
      "dev": "ts-node-dev --respawn src/index.ts",
      "build": "tsc",
      "start": "node dist/index.js"
    }
  }
  ```
- [ ] **1.1.3** 의존성 설치
  ```bash
  npm install express cors uuid node-pty
  npm install -D typescript ts-node-dev @types/express @types/cors @types/uuid @types/node
  ```
- [ ] **1.1.4** `tsconfig.json` 설정
  ```json
  {
    "compilerOptions": {
      "target": "ES2022",
      "module": "NodeNext",
      "moduleResolution": "NodeNext",
      "outDir": "./dist",
      "rootDir": "./src",
      "strict": true,
      "esModuleInterop": true,
      "skipLibCheck": true
    },
    "include": ["src/**/*"]
  }
  ```
- [ ] **1.1.5** 디렉토리 구조 생성
  ```
  server/src/
  ├── index.ts
  ├── routes/
  ├── services/
  ├── types/
  └── utils/
  ```
- [ ] **1.1.6** 기본 Express 서버 코드 작성 (`src/index.ts`)
  ```typescript
  import express from 'express';
  import cors from 'cors';

  const app = express();
  const PORT = process.env.PORT || 4242;

  app.use(cors());
  app.use(express.json());

  app.get('/health', (req, res) => {
    res.json({ status: 'ok' });
  });

  app.listen(PORT, () => {
    console.log(`Server running on http://localhost:${PORT}`);
  });
  ```
- [ ] **1.1.7** 서버 실행 테스트 (`npm run dev`)
  - http://localhost:4242/health 접속 확인

### 1.2 Frontend Project Setup (frontend/)

- [ ] **1.2.1** Vite + React + TypeScript 프로젝트 생성
  ```bash
  npm create vite@latest frontend -- --template react-ts
  ```
- [ ] **1.2.2** 의존성 설치
  ```bash
  cd frontend
  npm install
  npm install xterm xterm-addon-fit
  ```
- [ ] **1.2.3** Vite 프록시 설정 (`vite.config.ts`)
  ```typescript
  import { defineConfig } from 'vite';
  import react from '@vitejs/plugin-react';

  export default defineConfig({
    plugins: [react()],
    server: {
      port: 3000,
      proxy: {
        '/api': {
          target: 'http://localhost:4242',
          changeOrigin: true,
        },
      },
    },
  });
  ```
- [ ] **1.2.4** 디렉토리 구조 생성
  ```
  frontend/src/
  ├── main.tsx
  ├── App.tsx
  ├── components/
  │   ├── Sidebar/
  │   ├── Terminal/
  │   ├── Header/
  │   └── StatusBar/
  ├── hooks/
  ├── services/
  ├── types/
  └── styles/
  ```
- [ ] **1.2.5** 기본 레이아웃 컴포넌트 스캐폴딩 (`App.tsx`)
  ```tsx
  function App() {
    return (
      <div className="app">
        <header className="header">Claude Web Shell</header>
        <div className="main">
          <aside className="sidebar">Sidebar</aside>
          <main className="terminal">Terminal</main>
        </div>
        <footer className="statusbar">Status</footer>
      </div>
    );
  }
  export default App;
  ```
- [ ] **1.2.6** 기본 CSS 스타일 설정 (`src/styles/globals.css`)
- [ ] **1.2.7** 프론트엔드 실행 테스트 (`npm run dev`)

## Test Scenarios

### TS-1.1: Backend Health Check

**Given**: 백엔드 서버가 실행 중
**When**: `GET /health` 요청
**Then**: `{ "status": "ok" }` 응답

```bash
curl http://localhost:4242/health
# Expected: {"status":"ok"}
```

### TS-1.2: Frontend Dev Server

**Given**: 프론트엔드 개발 서버가 실행 중
**When**: 브라우저에서 `http://localhost:3000` 접속
**Then**: "Claude Web Shell" 텍스트가 표시됨

### TS-1.3: Proxy Configuration

**Given**: 백엔드와 프론트엔드 모두 실행 중
**When**: 프론트엔드에서 `/api/health` fetch
**Then**: 백엔드의 health 응답이 프록시되어 수신됨

## Notes

- node-pty는 네이티브 모듈이므로 빌드 도구(Python, Visual Studio Build Tools 등)가 필요할 수 있음
- Windows에서는 `windows-build-tools` 설치가 필요할 수 있음:
  ```bash
  npm install --global windows-build-tools
  ```
