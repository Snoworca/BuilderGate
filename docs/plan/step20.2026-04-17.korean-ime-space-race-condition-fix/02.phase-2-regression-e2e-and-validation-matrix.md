## 언어+버전 주의사항

- Playwright는 실제 한글 IME 조합을 완전하게 에뮬레이트하지 못한다.
- 따라서 자동 테스트는 debug-event 기반 증거 수집용으로 쓰고, 최종 PASS는 수동 검증을 포함해야 한다.

## LLM 코딩 주의사항

- Simplicity: 테스트는 IME race 가드와 영문 회귀 방지에 집중한다. backend replay/ConPTY 재설계 테스트까지 확장하지 않는다.
- Surgical: 기존 `terminal-keyboard-regression.spec.ts`를 과도하게 흔들지 말고, IME 전용 spec 파일을 별도로 추가한다.

## 테스트 시나리오

### 정상 케이스

- Given: 새 PowerShell 워크스페이스
  When: IME 조합 상태에서 `Space`
  Then: `ime_guard_delegated` 이벤트가 생기고 수동 `Space` 이벤트는 없다

- Given: 새 PowerShell 워크스페이스
  When: IME 조합 상태에서 `Backspace`
  Then: `ime_guard_delegated` 이벤트가 생기고 수동 `Backspace` 이벤트는 없다

- Given: IME 비활성 상태
  When: 영문 `Space`/`Backspace`
  Then: 기존 keyboard regression이 계속 통과한다

### 예외 케이스

- Given: Playwright가 실제 IME 합성 자체를 완전히 재현하지 못한다
  When: 테스트 실행
  Then: helper textarea composition 이벤트와 debug-event 증거로 우회 검증한다

  Then: 동일 증상이 재발하지 않아야 한다
