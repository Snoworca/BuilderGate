## Phase 1: Frontend IME Composition Guard

### 목표

`TerminalView`에서 한글 IME 조합 중/조합 종료 직후 `Space`와 `Backspace`가 수동 입력 경로로 빠지지 않도록 guard를 추가한다.

## 선행 조건

- 없음

## 기존 코드 재사용 힌트

- `frontend/src/components/Terminal/TerminalView.tsx`
  - 현재 `attachCustomKeyEventHandler` 구조를 그대로 재사용하고, guard만 최소 보강한다.
- `frontend/src/utils/terminalDebugCapture.ts`
  - 기존 디버그 이벤트 기록기 재사용. 신규 IME guard 이벤트도 같은 패턴으로 넣는다.
- 미병합 참조 커밋: `13e68c8`
  - semantic reference로만 사용하고, blind cherry-pick은 금지한다.

## 코드 중복 주의

- IMPL-001 / IMPL-002
  - 별도 IME helper 모듈을 새로 만들지 않는다. `TerminalView` 내부 보조 state로 해결한다.
- IMPL-003
  - 영문 `Space`/`Backspace` 경로 전체 제거는 금지. IME 상태일 때만 네이티브 위임으로 분기한다.

## 언어+버전 주의사항

- TypeScript 5.x + React 19 코드베이스다.
- 브라우저 이벤트 객체는 `KeyboardEvent`/`CompositionEvent` 표준 속성을 우선 사용한다.
- DOM 리스너 등록/해제는 동일 ref/node 기준으로 정확히 cleanup 해야 한다.

## LLM 코딩 주의사항

- Simplicity: root cause fix는 `TerminalView` 하나에 국한한다. backend나 websocket 경계 수정으로 범위를 넓히지 않는다.
- Surgical: 기존 `manual_input_forwarded` 영문 경로는 유지하고, IME 상태일 때만 수동 경로를 차단한다.

## 테스트 시나리오

### 정상 케이스

- Given: helper textarea가 composition 상태다
  When: 사용자가 `Space`를 누른다
  Then: `manual_input_forwarded` 대신 `ime_guard_delegated`가 기록되고 xterm 네이티브 처리로 위임된다

- Given: helper textarea가 composition 상태다
  When: 사용자가 `Backspace`를 누른다
  Then: 수동 `\x7f` 전송 없이 xterm 네이티브 처리로 위임된다

- Given: IME 비활성 상태의 영문 prompt
  When: 사용자가 `Space`를 누른다
  Then: 기존 수동 `Space` 경로는 유지된다

### 예외 케이스

- Given: Chrome이 `ev.isComposing === false`를 너무 빨리 반환한다
  When: composition 종료 직전 `Space` keydown이 들어온다
  Then: `isComposingRef.current` 또는 `keyCode === 229` guard가 race를 흡수한다

### 경계값 테스트

- Given: compositionend와 `Space` keydown이 같은 event loop turn에 인접해 있다
  When: 사용자가 마지막 음절 확정과 동시에 `Space`를 입력한다
  Then: 공백이 단어 내부로 이동하지 않는다
