## 상세 구현 지침

### SessionManager 통합 규칙

- headless write 는 PTY output 발생 직후, WS broadcast 이전에 수행한다.
- write 실패 시 세션 전체를 죽이지 않고 `replayMode = 'legacy'` 로 낮춘다.
- `replayMode` 가 `legacy` 인 세션은 snapshot 대신 기존 history path 를 쓴다.

### runtime config 변경

- 기존 `pty.maxBufferSize` 의미를 `headless scrollback upper bound` 와 정렬한다.
- 값이 작아지면 다음 output 까지 기다리지 말고 즉시 re-truncate 한다.
- 세션 메트릭에 `snapshotBytes`, `scrollbackLines`, `serializeMsAvg` 를 노출한다.
- `config.schema.ts` 에 feature flag 와 scrollback 관련 필드를 명시적으로 추가한다.

### restart semantics

- session restart 시:
  - 새로운 PTY
  - 새로운 headless instance
  - `generation + 1`
  - old snapshot/replay state 폐기

## 테스트 섹션

### 시나리오 1: PTY output 와 emulator state 동기화
- Given: 일반 shell 세션이 출력한다
- When: `SessionManager` 가 output 를 처리한다
- Then: WS broadcast 된 내용과 snapshot 근원이 같은 output stream 을 소비한다

### 시나리오 2: resize 동기화
- Given: 세션이 존재한다
- When: `resize(sessionId, cols, rows)` 호출
- Then: PTY 와 emulator 가 같은 크기를 가진다

### 시나리오 3: runtime config 축소
- Given: 큰 scrollback 을 가진 세션과 작은 새 maxBufferSize 가 있다
- When: `updateRuntimeConfig()` 호출
- Then: snapshot/scrollback 상태가 즉시 새 상한으로 줄어든다

### 시나리오 4: headless failure fallback
- Given: emulator write 또는 snapshot 생성 실패가 발생한다
- When: 세션이 계속 동작한다
- Then: `replayMode` 가 legacy 로 전환되고 경고가 남는다

### 시나리오 5: restart generation 증가
- Given: 세션이 재시작된다
- When: 새 snapshot 을 요청한다
- Then: `generation` 이 증가하고 stale snapshot 과 구분된다
