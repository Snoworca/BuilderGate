## 테스트 섹션

### 시나리오 1: reconnect race
- Given: snapshot 전송 직후 live output 와 reconnect 가 겹친다
- When: 새 socket 가 resubscribe 한다
- Then: stale output 이 먼저 보이지 않는다

### 시나리오 2: Codex/TUI
- Given: alternate-screen 기반 TUI 가 실행 중이다
- When: refresh 또는 reconnect 가 일어난다
- Then: 서버 snapshot 이 동일한 화면 수준으로 복구한다

### 시나리오 3: 긴 scrollback + resize
- Given: 매우 긴 출력과 여러 번의 resize 가 있다
- When: snapshot 을 만들고 다시 적용한다
- Then: truncate 정책과 wrap 결과가 수용 범위 안이다

### 시나리오 4: fallback telemetry
- Given: headless failure 를 강제로 발생시킨다
- When: 세션이 legacy 모드로 강등된다
- Then: fallback count/log 가 기록된다

### 시나리오 5: cleanup gate
- Given: 일정 기간 fallback count 가 0 이다
- When: cleanup 단계로 진입한다
- Then: legacy replay/local snapshot 제거 PR 을 시작할 수 있다
