## 상세 구현 지침

### 모드별 규칙

- tab mode:
  - 모든 활성 workspace terminal instance 를 DOM 에 유지
  - visibility 변경만 수행
  - 재subscribe 금지
- grid mode:
  - tile remount 허용
  - remount 시 subscribe -> snapshot -> live 경로 사용
- workspace switch:
  - `activeWorkspaceId` 변경은 UI 초점 변경
  - 세션 상태는 server authored state 로 유지

### restart / disconnected

- restart 시 old sessionId 관련 local artifacts 전부 제거
- disconnected overlay 는 화면 상태를 덮을 뿐, stale snapshot source 가 되면 안 된다
- 새 sessionId 가 발급되면 새 generation + 새 subscribe barrier 로만 복구한다
- `useWorkspaceManager.restartTab()` 는 old session cleanup 과 new session activation 을 분리해서 처리한다

## 테스트 섹션

### 시나리오 1: tab mode workspace switch
- Given: 두 workspace 와 긴 scrollback 이 있다
- When: workspace 를 전환했다가 돌아온다
- Then: 같은 terminal instance 또는 server snapshot 기준으로 원상 복구된다

### 시나리오 2: grid remount
- Given: grid mode 에서 타일이 remount 된다
- When: 사용자가 다시 해당 pane 을 연다
- Then: 최근 화면 상태가 server snapshot 으로 복구된다

### 시나리오 3: header cwd 유지
- Given: 여러 cwd 를 가진 pane 이 있다
- When: pane focus 를 바꾼다
- Then: header/meta row 가 실제 active pane 과 일치한다

### 시나리오 4: restart 후 stale state 금지
- Given: 세션이 재시작되어 새 sessionId 를 받았다
- When: UI 가 다시 표시된다
- Then: 이전 session 의 snapshot/local data 가 섞이지 않는다

### 시나리오 5: feature flag rollback
- Given: `terminal.authoritativeState = false`
- When: 앱이 실행된다
- Then: 기존 legacy replay 경로로 동작한다
