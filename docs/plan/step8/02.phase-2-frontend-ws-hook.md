## 언어+버전 주의사항

- React 18: `useSyncExternalStore`는 이 경우 과도함. `useState` + `useEffect`로 충분
- 네이티브 `WebSocket` API: `ws.readyState`로 연결 상태 확인 (0=CONNECTING, 1=OPEN, 2=CLOSING, 3=CLOSED)
- `ws.send()`는 `readyState === 1`일 때만 호출 (그 외 무시 또는 큐잉)

## LLM 코딩 주의사항

- **Simplicity**: 메시지 큐잉은 구현하지 않는다. 연결이 끊긴 동안의 메시지는 버린다 (터미널 출력은 서버 버퍼에 남아있으므로 재연결 시 subscribe로 받음)
- **Surgical**: 이 Phase에서는 useSSE.ts, useWorkspaceSSE.ts를 수정하지 않는다. 새 훅만 생성. 기존 훅 교체는 Phase 3-4에서 수행

## 상세 구현 지침

### 2-1 + 2-7. WebSocketProvider + useWebSocket

**파일**: `frontend/src/contexts/WebSocketContext.tsx` (신규)

```typescript
interface WebSocketContextValue {
  // 연결 상태
  status: 'connected' | 'reconnecting' | 'disconnected';
  clientId: string | null;

  // 메시지 전송
  send: (msg: ClientWsMessage) => void;

  // 세션 구독
  subscribeSession: (sessionId: string, handlers: SessionHandlers) => () => void;

  // 워크스페이스 이벤트 핸들러 등록
  setWorkspaceHandlers: (handlers: WorkspaceHandlers) => void;
}
```

핵심: `subscribeSession`은 **클린업 함수를 반환**한다. TerminalContainer가 unmount될 때 자동 해제.

```typescript
// 사용 예시 (Phase 3에서 구현)
useEffect(() => {
  const unsubscribe = ws.subscribeSession(sessionId, {
    onOutput: (data) => terminalRef.current?.write(data),
    onStatus: (status) => onStatusChange(sessionId, status),
    onError: (msg) => onStatusChange(sessionId, 'disconnected'),
    onCwd: (cwd) => onCwdChange?.(sessionId, cwd),
  });
  return unsubscribe;
}, [sessionId]);
```

### 2-2. 재연결 로직

```typescript
const RECONNECT_MAX = 10;
const RECONNECT_BASE_MS = 1000;
const RECONNECT_MAX_MS = 30000;

function getReconnectDelay(attempt: number): number {
  return Math.min(RECONNECT_BASE_MS * Math.pow(2, attempt), RECONNECT_MAX_MS);
}
```

재연결 시 `send({ type: 'subscribe', sessionIds: [...activeSubscriptions] })`로 모든 활성 구독을 복원한다.

### 2-3. 메시지 라우팅

```typescript
ws.onmessage = (event) => {
  const msg: ServerWsMessage = JSON.parse(event.data);

  if ('sessionId' in msg) {
    // 세션별 핸들러로 라우팅
    const handlers = sessionHandlersRef.current.get(msg.sessionId);
    if (!handlers) return;

    switch (msg.type) {
      case 'output': handlers.onOutput?.(msg.data); break;
      case 'status': handlers.onStatus?.(msg.status); break;
      case 'cwd': handlers.onCwd?.(msg.cwd); break;
      case 'session:error': handlers.onError?.(msg.message); break;
      case 'session:exited': handlers.onError?.(`Shell exited: ${msg.exitCode}`); break;
    }
  } else {
    // 워크스페이스 핸들러로 라우팅
    workspaceHandlersRef.current[msg.type]?.(msg.data);
  }
};
```

## 테스트 시나리오

### 정상 케이스
- **Given**: WS 서버가 실행 중
  **When**: useWebSocket 훅이 마운트
  **Then**: WS 연결 성공, status === 'connected', clientId 수신

### 예외 케이스
- **Given**: WS 서버가 중단됨
  **When**: useWebSocket 훅이 마운트
  **Then**: status === 'reconnecting', 지수 백오프로 재연결 시도

- **Given**: 재연결 10회 실패
  **When**: 11번째 시도 없음
  **Then**: status === 'disconnected', 사용자 알림 가능

### 경계값 테스트
- **Given**: WS 연결 중
  **When**: 서버가 갑자기 종료 (clean close 아님)
  **Then**: onclose 감지 → 1초 후 재연결 시도

- **Given**: 재연결 후
  **When**: 3개 세션이 구독 중이었음
  **Then**: 자동으로 subscribe 메시지 전송, subscribed 응답 수신
