## 1. 아키텍처 개요

### 1.1 현재 → 전환 후 비교

```
[현재: 다중 프로토콜]

Browser ─── SSE ──────► /api/sessions/:id/stream   (세션별 × N개)
        ├── SSE ──────► /api/workspaces/stream      (워크스페이스 × 1개)
        ├── POST ─────► /api/sessions/:id/input     (키 입력, 매 키스트로크)
        ├── POST ─────► /api/sessions/:id/resize    (크기 변경)
        └── GET ──────► /api/sessions/:id/cwd       (3초 폴링)
        총: N+1 SSE 연결 + 반복 HTTP 요청

[전환 후: 하이브리드]

Browser ═══ WebSocket ═══► /ws                      (단일 양방향 채널)
        │
        ├─► 수신: output, status, cwd, session:error, workspace:*, tab:*, grid:*
        ├─◄ 송신: input, resize, subscribe, unsubscribe
        │
        └── REST ─────► /api/sessions (CRUD)         (요청-응답 패턴 유지)
                  ────► /api/workspaces (CRUD)
                  ────► /api/auth/*
                  ────► /api/sessions/:id/files/*
                  ────► /api/settings
        총: 1 WS 연결 + REST (CRUD만)
```

### 1.2 핵심 설계 원칙

| 원칙 | 적용 방법 |
|------|----------|
| 단일 연결 | 모든 실시간 통신을 1개 WebSocket으로 통합 |
| 하이브리드 유지 | CRUD/파일 API는 REST로 유지 (도구 호환성) |
| 메시지 라우팅 | JSON 엔벨로프의 `type` + `sessionId`로 라우팅 |
| 구독 기반 | 클라이언트가 명시적으로 세션 구독/해제 |
| 장애 격리 | 개별 세션 에러가 WS 연결 전체에 영향 안 줌 |

### 2.2 컴포넌트 책임

| 컴포넌트 | 책임 | Phase |
|----------|------|-------|
| **WsRouter** (신규) | WS 메시지 수신 → type별 핸들러 디스패치 | Phase 1 |
| **SessionManager** (수정) | SSE broadcast → WS broadcast, 구독 관리 | Phase 3 |
| **useWebSocket** (신규) | 단일 WS 연결, 재연결, 메시지 라우팅 | Phase 2 |
| **TerminalContainer** (수정) | useSSE → useWebSocket 구독 | Phase 3 |
| **workspaceRoutes** (수정) | SSE broadcast → WS broadcast | Phase 4 |

## 3. 메시지 프로토콜

### 3.1 클라이언트 → 서버

```typescript
// 공통 엔벨로프
interface ClientMessage {
  type: string;
  sessionId?: string;
  requestId?: string;  // 응답 매칭용 (선택)
}

// 구체 메시지 타입
type ClientWsMessage =
  | { type: 'subscribe';   sessionIds: string[] }
  | { type: 'unsubscribe'; sessionIds: string[] }
  | { type: 'input';       sessionId: string; data: string }
  | { type: 'resize';      sessionId: string; cols: number; rows: number }
  | { type: 'ping' };
```

### 3.2 서버 → 클라이언트

```typescript
type ServerWsMessage =
  // 세션 이벤트
  | { type: 'output';         sessionId: string; data: string }
  | { type: 'status';         sessionId: string; status: 'running' | 'idle' }
  | { type: 'cwd';            sessionId: string; cwd: string }
  | { type: 'session:error';  sessionId: string; message: string }
  | { type: 'session:exited'; sessionId: string; exitCode: number }
  // 구독 응답
  | { type: 'subscribed';     sessions: Array<{ sessionId: string; status: string; cwd?: string }> }
  // 워크스페이스 이벤트
  | { type: 'workspace:created';   data: Workspace }
  | { type: 'workspace:updated';   data: { id: string; changes: Partial<Workspace> } }
  | { type: 'workspace:deleted';   data: { id: string } }
  | { type: 'workspace:deleting';  data: { id: string } }
  | { type: 'workspace:reordered'; data: { workspaceIds: string[] } }
  // 탭 이벤트
  | { type: 'tab:added';        data: WorkspaceTab }
  | { type: 'tab:updated';      data: { id: string; workspaceId: string; changes: Partial<WorkspaceTab> } }
  | { type: 'tab:removed';      data: { id: string; workspaceId: string } }
  | { type: 'tab:reordered';    data: { workspaceId: string; tabIds: string[] } }
  | { type: 'tab:disconnected'; data: { id: string; workspaceId: string } }
  // 그리드 이벤트
  | { type: 'grid:updated'; data: GridLayout }
  // 연결 이벤트
  | { type: 'connected'; clientId: string }
  | { type: 'pong' };
```

## 4. 시퀀스 다이어그램

### 4.1 WS 연결 + 세션 구독 (핵심 플로우)

```
┌──────┐     ┌──────────┐     ┌─────────────┐     ┌──────────────┐
│Client│     │ WsRouter │     │SessionManager│     │   PTY        │
└──┬───┘     └────┬─────┘     └──────┬───────┘     └──────┬───────┘
   │              │                   │                     │
   │ 1. WS upgrade (?token=JWT)      │                     │
   │─────────────▶│                   │                     │
   │              │ 2. JWT 검증       │                     │
   │              │───────────────────│                     │
   │              │ 3. { connected }  │                     │
   │◀─────────────│                   │                     │
   │              │                   │                     │
   │ 4. { subscribe, sessionIds }    │                     │
   │─────────────▶│                   │                     │
   │              │ 5. 구독 등록      │                     │
   │              │──────────────────▶│                     │
   │              │ 6. 세션 상태 조회  │                     │
   │              │◀──────────────────│                     │
   │ 7. { subscribed, sessions }     │                     │
   │◀─────────────│                   │                     │
   │              │                   │                     │
   │              │     [PTY 출력 발생]│                     │
   │              │                   │◀────────────────────│
   │              │ 8. broadcast      │                     │
   │◀─────────────│◀──────────────────│                     │
   │ { output, sessionId, data }     │                     │
   │              │                   │                     │
   │ 9. { input, sessionId, data }   │                     │
   │─────────────▶│                   │                     │
   │              │ 10. PTY write     │                     │
   │              │──────────────────▶│─────────────────────▶
```

### 4.2 서버 재시작 시 재연결 + 좀비 감지

```
┌──────┐     ┌──────────┐     ┌─────────────┐
│Client│     │ WsRouter │     │SessionManager│
└──┬───┘     └────┬─────┘     └──────┬───────┘
   │              │                   │
   │ [WS 연결 끊김 감지]              │
   │──X           │                   │
   │              │                   │
   │ [지수 백오프 재연결 시도]          │
   │ (1s → 2s → 4s → ...)            │
   │              │                   │
   │ 1. WS 재연결 성공                │
   │─────────────▶│                   │
   │ 2. { connected }                │
   │◀─────────────│                   │
   │              │                   │
   │ 3. { subscribe, 이전 세션 목록 } │
   │─────────────▶│                   │
   │              │ 4. 세션 존재 확인  │
   │              │──────────────────▶│
   │              │ 5. 일부 세션 없음  │
   │              │◀──────────────────│
   │              │                   │
   │ 6. { subscribed, sessions: [    │
   │      { id1, status: "running" },│  ← 생존 세션
   │      { id2, status: "error" }   │  ← 좀비 → disconnected
   │    ]}                           │
   │◀─────────────│                   │
   │              │                   │
   │ [id2 → DisconnectedOverlay 표시]│
```
