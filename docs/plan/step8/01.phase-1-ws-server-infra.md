## 언어+버전 주의사항

- TypeScript + Node.js: `ws` 라이브러리의 WebSocket 타입은 `import { WebSocket, WebSocketServer } from 'ws'`
- `server.on('upgrade')` 핸들러에서 Express의 `req` 객체가 아닌 `http.IncomingMessage`를 받음
- `ws.send()`는 문자열 또는 Buffer를 받음 — JSON.stringify 필수

## LLM 코딩 주의사항

- **Simplicity**: WsRouter는 단순한 switch/case 또는 Map<type, handler>로 구현. 이벤트 에미터 패턴이나 미들웨어 체인 불필요
- **Surgical**: `server/src/index.ts`에 ws 부착 코드만 추가, 기존 Express 라우트 수정 없음

### 1-3. WebSocketServer 생성

**파일**: `server/src/ws/WsRouter.ts` (신규)

```typescript
// 핵심 구조
export class WsRouter {
  private wss: WebSocketServer;
  private clients: Map<WebSocket, ClientMeta>;

  constructor(server: https.Server, authService: AuthService) {
    this.wss = new WebSocketServer({ noServer: true });
    this.setupUpgradeHandler(server, authService);
    this.setupConnectionHandler();
  }
}
```

`noServer: true`로 생성하여 `server.on('upgrade')` 이벤트에서 수동으로 핸들링. 이렇게 하면 Express 미들웨어와 충돌하지 않는다.

### 1-5. ping/pong 하트비트

```typescript
// 30초마다 ping 전송, pong 미수신 시 연결 종료
private startHeartbeat() {
  setInterval(() => {
    for (const [ws, meta] of this.clients) {
      if (!meta.isAlive) { ws.terminate(); continue; }
      meta.isAlive = false;
      ws.ping();
    }
  }, 30000);
}
```

## 테스트 시나리오

### 정상 케이스
- **Given**: HTTPS 서버가 실행 중
  **When**: `wss://localhost:4242/ws?token=유효JWT`로 WebSocket 연결
  **Then**: 연결 성공, `{ type: "connected", clientId: "uuid" }` 수신

### 예외 케이스
- **Given**: HTTPS 서버가 실행 중
  **When**: 토큰 없이 `wss://localhost:4242/ws`로 연결 시도
  **Then**: HTTP 401 응답 후 소켓 종료

- **Given**: 유효한 WS 연결
  **When**: 잘못된 JSON 메시지 전송 `"not json"`
  **Then**: 에러 로그 출력, 연결 유지 (끊지 않음)

### 경계값 테스트
- **Given**: 유효한 WS 연결
  **When**: 30초 동안 메시지 없음
  **Then**: 서버 ping → 클라이언트 pong → 연결 유지

- **Given**: 유효한 WS 연결
  **When**: 60초 동안 pong 미응답
  **Then**: 서버가 연결 종료 (terminate)
