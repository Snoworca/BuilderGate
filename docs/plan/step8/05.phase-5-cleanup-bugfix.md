## LLM 코딩 주의사항

- **Simplicity**: 삭제할 코드를 정확히 식별하고 제거. 빈 파일이나 주석으로 남기지 않는다
- **Surgical**: SessionManager에서 SSE 관련 필드/메서드만 정확히 제거. PTY 관리, 버퍼링, 상태 관리 등 핵심 로직은 손대지 않는다

### 5-4. SessionManager SSE 정리

**삭제 대상 (SessionData 인터페이스에서):**
```typescript
// 삭제
sseClients: Set<Response>;
```

**삭제 대상 (메서드):**
- `addSSEClient()` (라인 223-239)
- `removeSSEClient()` (라인 241-246)
- 기존 `broadcast()` — SSE 전용 (라인 364-371). WS용 `broadcastWs()`만 남기고 이름을 `broadcast()`로 변경

**수정 대상 (PTY onData):**
```typescript
// 변경 전 (SSE 경로)
if (sData.sseClients.size === 0) {
  this.appendBufferedOutput(sData, data);
} else {
  this.broadcast(id, 'output', { data });
}

// 변경 후 (WS 경로)
const hasSubscribers = this.wsRouter?.hasSubscribers(id);
if (!hasSubscribers) {
  this.appendBufferedOutput(sData, data);
} else {
  this.broadcast(id, 'output', { data });  // WS broadcast
}
```

### 5-9. Focus 탈취 버그 수정

**TerminalView.tsx:178-189**

```typescript
// 변경 전
requestAnimationFrame(() => {
  requestAnimationFrame(() => {
    fitAddon.fit();
    onResize(term.cols, term.rows);
    term.focus();  // ← 모든 터미널이 focus를 탈취

    const bg = term.options.theme?.background || '#1e1e1e';
    document.documentElement.style.setProperty('--terminal-bg', bg);
  });
});

// 변경 후
requestAnimationFrame(() => {
  requestAnimationFrame(() => {
    fitAddon.fit();
    onResize(term.cols, term.rows);
    // term.focus() 제거 — 사용자 클릭(handleClick)에서만 focus

    const bg = term.options.theme?.background || '#1e1e1e';
    document.documentElement.style.setProperty('--terminal-bg', bg);
  });
});
```

기존 `handleClick` (라인 277-279)은 그대로 유지 — 사용자가 터미널 클릭 시 focus 부여.

## 테스트 시나리오

### 정상 케이스
- **Given**: 서버 빌드 완료
  **When**: `tsc --noEmit` 실행
  **Then**: SSE import 에러 없음, 빌드 성공

- **Given**: 그리드 모드 8탭
  **When**: 새 터미널 추가
  **Then**: 기존 터미널의 focus가 탈취되지 않음, 각 터미널에 클릭으로 입력 가능

### 예외 케이스
- **Given**: SSE 엔드포인트 삭제 후
  **When**: 구 버전 프론트엔드가 SSE 연결 시도
  **Then**: 404 응답 (서버 크래시 없음)

### 경계값 테스트
- **Given**: 8개 터미널 동시 WS 구독
  **When**: 모든 터미널에서 빌드 실행 (대량 출력)
  **Then**: 단일 WS 채널로 모든 출력 정상 전달, 입력 지연 없음

### 통합 검증 시나리오
- **Given**: 전체 시스템 (WS + REST 하이브리드)
  **When**: 다음 시퀀스 실행:
    1. 로그인
    2. 워크스페이스 생성
    3. 탭 4개 추가
    4. 그리드 모드 전환
    5. 각 탭에서 명령 실행
    6. cd로 디렉토리 변경 → CWD 실시간 갱신 확인
    7. 서버 재시작
    8. 자동 재연결 + 좀비 감지 확인
  **Then**: 모든 단계 정상 동작
