## 코드 중복 금지

| 위험 | 계획상 방지책 |
| --- | --- |
| runtime path resolver가 start/stop/build에 중복 | `tools/daemon/runtime-paths.js` 공통 모듈을 먼저 만든다. |
| state/PID/token masking이 start/stop/sentinel에 중복 | `tools/daemon/state-store.js`, `process-info.js`, `log.js`로 분리한다. |
| QR preflight가 TOTP secret 로직을 복제 | `TOTPService`에 옵션을 추가하고 preflight helper는 이를 호출만 한다. |
| signal shutdown과 internal route가 flush를 중복 구현 | `performGracefulShutdown(reason)` 공통 함수로 추출한다. |
| production config parser가 launcher regex와 서버 Zod로 분기 | launcher preflight는 가능한 서버 config loader 또는 동일 schema 기반 helper를 사용한다. |

## 최종 성공 기준

- FR/NFR/IR/AC/TEST unmapped 항목 0개.
- code/docs/build output에서 production PM2 사용 경로 0개.
- 기본 실행은 daemon, `--foreground`/`--forground`만 foreground.
- TOTP daemon QR은 parent detach 전 1회 출력되고 app child에서 중복 출력되지 않는다.
- native stop은 foreground process를 종료하지 않고 daemon state/PID 검증을 통과한 대상만 종료한다.
- `dist/bin`에는 `BuilderGate.exe`, `BuilderGateStop.exe`, `server/`, `config.json5`, `config.json5.example`, `README.md`, bundled Node가 존재한다.
