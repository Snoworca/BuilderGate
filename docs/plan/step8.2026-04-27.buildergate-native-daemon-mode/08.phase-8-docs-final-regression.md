## 기존 코드 재사용 힌트

- 기존 README의 TOTP/2FA 설명은 유지하되 daemon QR preflight 설명을 추가한다.
- 기존 build output 설명의 `dist/bin`과 config 위치 설명은 유지하고 PM2 설명만 native stop 설명으로 대체한다.

## 코드 중복 주의

- root README와 dist README 내용이 달라지지 않도록 build copy 정책을 유지한다.
- docs test의 forbidden PM2 pattern은 production 실행 안내를 대상으로 하되, 과거 migration 설명을 넣어야 한다면 테스트 예외가 명확해야 한다. 기본 계획은 PM2 문자열 0개다.

## 언어+버전 주의사항

- 문서 명령은 Windows PowerShell 기준과 non-Windows shell 기준을 구분한다.
- 프로젝트 규칙에 따라 수동 검증과 Playwright E2E는 `https://localhost:2002`를 대상으로 한다.

## LLM 코딩 주의사항

- Simplicity: PM2 migration guide나 legacy cleanup 절차를 새 default path처럼 작성하지 않는다.
- Surgical: UI/UX 변경 설명을 추가하지 않는다. 이 SRS 범위는 runtime/build/docs다.
- Safety: 문서가 `--daemon`을 기본 실행법으로 안내하면 SRS 위반이다.

## 테스트 시나리오

### 정상 케이스

- Given: README와 `dist/bin/README.md`가 생성되어 있다.
  When: docs test를 실행한다.
  Then: required keywords가 있고 forbidden PM2 patterns가 없다.

### 예외 케이스

- Given: README에 `npm install -g pm2`가 남아 있다.
  When: docs test를 실행한다.
  Then: failure를 반환한다.

### 경계값 테스트

- Given: user가 문서만 보고 source production을 실행한다.
  When: `node tools/start-runtime.js`, `node tools/start-runtime.js --foreground`, `node stop.js` 절차를 따른다.
  Then: daemon/foreground/stop 차이를 이해할 수 있다.
