## 구현 결과

- README에 source/package 기본 daemon 실행, `--foreground`/`--forground`, native stop, config 위치, TOTP QR preflight, `--reset-password`, `--bootstrap-allow-ip`, `--help` 동작을 문서화했다.
- `tools/daemon/docs-policy.js`와 `tools/daemon/docs.test.js`를 추가해 root README와 `dist/bin/README.md`의 required/forbidden 문서 정책을 자동 검증한다.
- `tools/daemon/native-daemon.integration.test.js`를 추가해 source daemon, foreground, native stop 계약을 검증한다. Foreground Ctrl+C는 실제 PTY에서 `node tools/start-runtime.js --foreground`를 실행해 app child flush marker와 daemon state/sentinel 부재를 확인한다.
- `tools/build-daemon-exe.js`가 배포본 README와 모든 packaged daemon runtime 파일을 검증하도록 보강했다.
- packaged sentinel은 `dist/bin/tools/daemon/sentinel-entry.js`를 bundled Node로 실행하도록 조정했고, stop/process 검증은 `--internal-sentinel`과 sentinel entry path를 모두 유효한 sentinel marker로 인정한다.
- Node Web Crypto global이 없는 런타임에서도 TOTP secret 생성이 실패하지 않도록 `TOTPService.initialize()` 회귀 테스트와 fallback을 추가했다.
- foreground 실행은 parent가 받은 `SIGINT`/`SIGTERM`을 app child로 전달하고 child 종료를 기다리도록 보강했다.
- `--target` cross-OS/cross-architecture build는 bundled Node/runtime config가 host OS/CPU와 불일치할 수 있어 명시적으로 거부하고, README에 대상 OS/CPU에서 빌드해야 한다는 제약을 문서화했다.
- PTY 기반 foreground Ctrl+C 통합 테스트를 추가해 실제 터미널에서 app child가 flush marker를 기록하고 daemon state/sentinel이 생성되지 않는지 검증했다.
