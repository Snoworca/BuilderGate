# Phase 7: Build Output, Bundled Runtime, PM2 Dependency 제거

## Phase 7: Build output, bundled runtime, PM2 dependency 제거

### 목표

`npm run build:daemon-exe`가 `dist/bin`에 native daemon 실행에 필요한 전체 runtime을 만들고, PM2 runtime dependency를 포함하지 않도록 한다.

## 선행 조건

- [x] Phase 3 native launcher 동작 확인
- [x] Phase 5 stop utility 동작 확인
- [x] Phase 6 sentinel internal mode 동작 확인

## Target Files

| 파일 | 변경 |
| --- | --- |
| `tools/build-daemon-exe.js` | PM2 install 제거, output validation 추가 |
| `package.json` | build script 유지 또는 필요 시 검증 script 추가 |
| `tools/daemon/runtime-paths.js` | packaged build output contract 검증 재사용 |
| `dist/bin/README.md` | build 시 root README copy 후 Phase 8 내용 반영 |
| tests | build output smoke/docs dependency tests |

## 기존 코드 재사용 힌트

- `OUTPUT_DEFAULT = path.join(ROOT, 'dist', 'bin')`은 이미 SRS 요구와 일치하므로 유지한다.
- `copyRuntimeFiles()`의 config/example/README copy 구조는 유지한다.
- `installRuntimeDependencies()`의 bundled Node copy는 유지한다.

## 코드 중복 주의

- build script에서 native daemon path contract를 별도 하드코딩하지 말고 resolver의 packaged expectations와 일치시키는 테스트를 작성한다.
- PM2 제거 후 bundled Node까지 제거하면 SRS 위반이다. PM2 dependency만 제거한다.

## 구현 결과

- `tools/build-daemon-exe.js`에서 PM2 runtime install을 제거하고 production dependency install을 `npm install --omit=dev`로 고정했다.
- `dist/bin` 산출물 검증을 추가해 app/stop 실행파일, `server/dist/index.js`, bundled Node, `config.json5`, `config.json5.example`, `README.md`, PM2 부재를 확인한다.
- `server/config.json5`가 없는 clean checkout build에서는 `server/dist/utils/configTemplate.js`의 `renderBootstrapConfigTemplate(platform)`을 재사용해 OS-aware bootstrap config를 생성한다.
- `--skip-runtime-install`은 dependency install만 건너뛰고 bundled Node copy는 유지하도록 수정했다.
- packaged README에 PM2 문구가 남으면 build validation이 실패하도록 방어했다.

## 언어+버전 주의사항

- `pkg@5.8.1` target은 현재 `node18-*`를 사용한다. Node API 선택 시 packaged 실행 호환성을 확인한다.
- Windows에서 copied `node.exe` 실행 권한과 path에 공백이 있는 경우를 고려한다.

## LLM 코딩 주의사항

- Simplicity: 설치 크기 최적화나 single-file 배포는 이번 Phase 범위가 아니다.
- Surgical: `dist/bin` 전체 배포 원칙을 깨지 않는다.
- Safety: output clean은 `dist/` 하위만 허용하는 기존 guard를 유지한다.

## 테스트 시나리오

### 정상 케이스

- Given: frontend/server build가 성공한다.
  When: `npm run build:daemon-exe`를 실행한다.
  Then: `dist/bin`에 app/stop exe, server runtime, config, README, bundled Node가 존재한다.

### 예외 케이스

- Given: `server/dist/index.js`가 build 후 없다.
  When: build script가 output validation을 수행한다.
  Then: build failure로 종료한다.

### 경계값 테스트

- Given: PM2가 global/local에 설치되어 있다.
  When: build를 실행한다.
  Then: output runtime dependency에는 PM2가 포함되지 않는다.
