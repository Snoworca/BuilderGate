# Phase 2: Foreground 실행 계약

## Phase 2: Foreground 실행 계약

### 목표

`--foreground`와 `--forground` 실행을 현재 콘솔 lifecycle에 연결하고, foreground process가 daemon state나 stop utility 대상이 되지 않는 계약을 확정한다.

## 선행 조건

- [ ] Phase 1 CLI parser, runtime path resolver, strict config 기반 완료
- [ ] `server/dist/index.js` 실행 전 build artifact 준비 절차 확인

## Target Files

| 파일 | 변경 |
| --- | --- |
| `tools/start-runtime.js` | foreground mode branch 추가 |
| `tools/daemon/launcher.js` | 신규 또는 확장. foreground app spawn/stdio 연결 |
| `tools/daemon/runtime-paths.js` | foreground env 구성 재사용 |
| `server/src/index.ts` | foreground 환경에서도 strict config와 TOTP env path를 존중하도록 준비 |
| `server/src/test-runner.ts` 또는 launcher tests | foreground no-sentinel/no-active-state tests |

## 기존 코드 재사용 힌트

- `ensureDependenciesAndBuild()`는 source production foreground에서도 build artifact 준비에 재사용한다.
- `resolvePort()`와 `CONFIG_ENV_KEY` 기존 의미를 유지한다.
- `start-runtime.js`의 `withLocalBinPath()`는 app child PATH 구성에 재사용 가능하다.

## 코드 중복 주의

- foreground와 daemon의 app env 구성은 같은 helper에서 만들고, mode별 차이만 인자로 분리한다.
- foreground는 active daemon state를 쓰지 않아야 하므로 daemon state writer helper를 무조건 호출하는 구조로 만들지 않는다.

## 언어+버전 주의사항

- Windows에서 child spawn `shell`/stdio 옵션은 현재 콘솔과 Ctrl+C propagation을 깨지 않도록 테스트한다.
- pkg packaged foreground와 source foreground의 `process.execPath` 의미가 다르므로 resolver 결과를 직접 사용한다.

## LLM 코딩 주의사항

- Simplicity: Phase 2에서는 sentinel, daemon readiness, QR preflight를 구현하지 않는다.
- Surgical: foreground mode 추가 때문에 daemon default를 아직 변경하지 않는다. default daemon 전환은 Phase 3에서 수행한다.
- Compatibility: `--forground` 오타 alias는 문서화된 호환 요구사항이므로 제거하거나 경고-only로 만들지 않는다.

## 테스트 시나리오

### 정상 케이스

- Given: `node tools/start-runtime.js --foreground -p 2002`
  When: launcher를 실행한다.
  Then: app stdout/stderr가 현재 콘솔로 흐르고 sentinel/state active daemon이 생성되지 않는다.

### 예외 케이스

- Given: `server/dist/index.js`가 없다.
  When: foreground를 실행한다.
  Then: build artifact 준비 또는 명확한 startup failure가 발생하고 active daemon state는 남지 않는다.

### 경계값 테스트

- Given: `--forground` alias다.
  When: foreground 실행 경로를 resolve한다.
  Then: `--foreground`와 동일하게 처리된다.
