# Phase 3. Grid Mode Cutover And Canonical Mosaic Layout

## 목적

grid mode를 새 shared runtime 구조에 연결하고, `react-mosaic` tree를 workspace layout의 정본으로 확정한다. tab mode는 이제 projection이고, grid mode만 canonical tree를 직접 그린다.

## 범위

- `MosaicContainer`가 terminal JSX가 아니라 host slot descriptor를 렌더하도록 전환
- canonical mosaic tree와 tab projection의 분리
- persisted layout, stale recovery, Workspace-2 같은 edge case 유지

## 구현 체크리스트

1. `renderTerminal(tab)` 구조를 host slot 구조로 대체한다.
2. canonical tree는 계속 `useMosaicLayout`가 소유하되 tab mode는 tree를 파괴하지 않게 한다.
3. stale persisted tree recovery는 유지한다.
4. grid pane close/restart/focus 동작을 registry 경유로 맞춘다.
5. layout drag, equal/focus/auto mode가 host 구조에서도 그대로 동작하게 맞춘다.
6. grid 진입 시 canonical tree leaf topology와 tab bar 순서의 관계를 명시한다.
7. unassigned leaf append 또는 stale recovery 시에는 현재 tab bar 순서를 사용해 새 leaf를 채운다.

## 테스트

- stale mosaic layout recovery 회귀 테스트
- grid 진입 시 검은 화면 재현 방지 테스트
- grid pane focus/selection/context menu 회귀 테스트
- grid <-> tab 왕복 시 runtime identity 유지 테스트
- tab reorder 이후 grid 재진입 시 split topology는 유지되고 새 leaf 보충 순서만 tab order를 따른다는 테스트

## 종료 조건

- grid mode가 더 이상 자체 terminal subtree를 새로 만들지 않는다.
- `Workspace-2` stale layout 복구가 유지된다.
- tab/grid 왕복이 same session runtime handoff로 동작한다.

## 리스크

- `react-mosaic`와 host slot 경계가 어설프면 pane reorder/drag 시 host 유실이 생길 수 있다.
- tab projection과 canonical tree를 섞어 저장하면 persistence가 깨질 수 있다.
