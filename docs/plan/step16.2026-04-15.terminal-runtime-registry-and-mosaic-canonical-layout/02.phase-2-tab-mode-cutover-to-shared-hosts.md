# Phase 2. Tab Mode Cutover To Shared Hosts

## 목적

tab mode를 새 shared runtime 구조 위로 먼저 올린다. 현재 tab mode는 absolute stack keep-alive와 가장 비슷한 성질을 갖고 있으므로, 가장 낮은 리스크로 registry를 실전 검증할 수 있다.

## 범위

- tab mode direct `TerminalContainer` 렌더 제거
- active tab slot + hidden parking slot 도입
- `WorkspaceTabBar`와 active tab selection 경로 보존
- metadata row와 terminal body 분리 상태 유지
- tab bar 순서와 canonical mosaic tree의 역할 분리 명문화

## 구현 체크리스트

1. tab mode가 terminal JSX를 직접 만들지 않고 active tab host slot만 렌더하게 바꾼다.
2. 비활성 tab는 hidden parking slot으로 이동시킨다.
3. active tab 변경 시 runtime 재생성 대신 host reassignment만 일어나게 한다.
4. context menu, selection, paste, focus가 registry 경유로 계속 동작하도록 맞춘다.
5. metadata row show/hide가 host geometry를 흔들지 않도록 경계를 고정한다.
6. tab reorder의 진실원은 workspace tabs 배열이며, canonical tree topology는 즉시 재작성하지 않는다고 문서와 구현을 맞춘다.

## 테스트

- tab 전환 시 same session runtime identity 유지 테스트
- 빠른 tab 전환 시 subscribe/unsubscribe churn이 늘지 않는 테스트
- PowerShell 입력, selection, paste, fit 회귀 테스트
- tab reorder 후 tab bar 순서는 유지되지만 canonical tree topology가 불필요하게 초기화되지 않는 테스트

## 종료 조건

- tab mode 단독 사용에서는 기존과 동일하거나 더 안정적이다.
- same session runtime이 tab 전환으로 재생성되지 않는다.
- grid를 아직 건드리지 않아도 old grid path와 병행 동작한다.
- tab reorder 정책이 문서와 구현 모두에서 모호하지 않다.

## 리스크

- hidden parking slot 처리 미흡 시 입력 포커스가 사라질 수 있다.
- tab mode host rect 계산이 꼬이면 resize noise가 되살아날 수 있다.
