## Reproduction Procedure

### Scenario A: Workspace switch baseline

1. Open a terminal session.
2. Produce long-running vertical output.
3. Switch to another workspace.
4. Return to the original workspace.

Expected:

- output should keep flowing
- no blank gap should appear between lines
- no duplicate replay block should appear

Observed:

- workspace switch is significantly more stable than mode switch

### Scenario B: Tab to grid during active output

1. Start from `tab` mode.
2. Run long-running vertical output, for example:

```powershell
1..200 | ForEach-Object { Write-Output $_; Start-Sleep -Milliseconds 25 }
```

3. While output is still progressing, switch to `grid` mode.

Expected:

- current output should remain visible with no replay corruption
- both panes should stay interactive

Observed:

- current output may collapse to a prompt-heavy snapshot
- blank or partial-looking view can appear immediately after switch
- later redraw/live output may make the screen appear "normal again"

### Scenario C: Grid to tab during active output

1. Start from `grid` mode.
2. Run long-running vertical output in one pane.
3. Switch back to `tab` mode.

Expected:

- same session should continue with no blank gap and no duplicate chunk

Observed:

- mode switch remains less stable than workspace switch

### Scenario D: Grid pane focus routing

1. Open a 2-pane `PowerShell` grid workspace.
2. Click inside the left pane body and type a command.
3. Click inside the right pane body and type a command.

Expected:

- left command goes to left session
- right command goes to right session

Observed:

- textarea-direct focus works reliably
- pane-surface click is still less reliable than textarea focus

### Scenario E: Splitter hover and drag

1. Hover the mouse over the splitter between two panes.
2. Check whether the cursor changes to resize shape.
3. Drag the splitter and verify pane size changes.

Expected:

- hover shows resize cursor
- drag changes pane sizes

Observed:

- splitter styling was partially restored through CSS
- drag/hover still needs dedicated final verification after runtime-layer fixes

## Current Technical Interpretation

The remaining mode-switch issue is not primarily a workspace-subscribe problem anymore.

Current strongest hypothesis:

1. `tab <-> grid` causes a large `resize`
2. PTY redraw follows the resize
3. authoritative replay snapshot is produced too early relative to redraw
4. the client applies a prompt-heavy or partial snapshot
5. later live redraw/output makes the screen appear normal again

This matches the observed symptom:

- "looks broken right after the switch"
- "becomes normal again a little later"

## Current Mitigations Already Applied

- delayed replay refresh after resize
- host/runtime split to avoid direct terminal recreation on every mode change
- stale tree fallback for grid layout
- hidden/hostless snapshot deferral
- pane focus forwarding through runtime wrapper
- splitter cursor/z-index CSS hardening

### 2026-04-16 Final Retest After Drain-Safe Deadline Adjustment

One more review pass found that forcing a deadline refresh while `pendingHeadlessWrites`
was still non-zero could theoretically discard queued replay output before the headless
snapshot had caught up. The scheduler was adjusted again so that:

- the quiet window is ignored after the deadline
- but replay refresh still waits for headless write drain before sending the new snapshot

Retest path:

1. Restart server on `https://localhost:2002`
2. Re-login and open `PW-MODE-1776287966303`
3. Switch to `tab` mode
4. Run:

```powershell
1..120 | ForEach-Object { Write-Output $_; Start-Sleep -Milliseconds 10 }
```

5. Switch to `grid` while output is still active

Observed:

- visible output resumed in the `31+` range immediately after the switch
- output remained contiguous through `120`
- the prompt returned in the same pane without a blank reset or duplicate block

Captured screenshot:

- `pw-mode-grid-after-final-deadline-drain-fix.png`

Interpretation:

- the drain-safe deadline logic preserved the improved `tab -> grid` behavior in the
  dedicated PowerShell-only workspace while removing the queued-output drop risk

### 2026-04-16 Final Clamp Retest
