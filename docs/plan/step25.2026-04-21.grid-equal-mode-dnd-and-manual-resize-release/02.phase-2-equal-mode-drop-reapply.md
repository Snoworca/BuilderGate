# Phase 2: Equal Mode Drop Reapply

## Objective

equal 모드에서 tile drop이 끝나면 즉시 균등 배치를 다시 적용한다.

## Target Files

- `frontend/src/components/Grid/MosaicContainer.tsx`
- `frontend/src/utils/mosaic.ts`
- 필요 시 `react-mosaic` completion hook 사용 경로

## Planned Changes

1. tile drop completion 시점 사용
2. 현재 mode가 `equal`이면 dropped tree를 기준으로 grid-shaped equal layout 재구성
3. equal arrangement는 버튼 클릭 시점에 정해진 `rows/cols`를 그대로 사용
4. persistence와 layout refresh 동기화

## Implementation Note

- `react-mosaic`는 tile drag 시작 시 hidden placeholder를 만들기 위해 intermediate `onChange/onRelease`를 먼저 발생시킬 수 있다.
- 따라서 `equal` 재격자화는 drag-start hide release가 아니라 **실제 drop commit 이후**에만 실행되어야 한다.
- split resize는 계속 `onRelease` 기반으로 처리하되, tile drag는 실제 drop completion 신호와 마지막 committed tree를 기준으로 재격자화해야 한다.

## Implementation Checklist

- [ ] drop completion hook 연결
- [ ] equal mode일 때만 재적용
- [ ] 기본 2행 / portrait 예외 2열 규칙 반영
- [ ] arrangement를 resize마다 다시 계산하지 않음
- [ ] layout refresh와 save 동기화
- [ ] grid-shaped equal 재구성 유지

## Test

### Scenario 1: equal mode drop reapplies immediately

Given equal mode grid  
When user drags a tile to a different drop target  
Then final tree is rebuilt into the equal grid immediately

### Scenario 2: 5 sessions use top 3 / bottom 2

Given 5 sessions in equal mode  
When equal is applied  
Then the final arrangement is 2 fixed rows with top 3 and bottom 2

### Scenario 3: portrait chooses 2 fixed columns once

Given the viewport is taller than wide when equal is pressed  
When equal is applied and later the window is resized  
Then the arrangement stays on the 2 fixed columns rule until equal is explicitly recomputed
