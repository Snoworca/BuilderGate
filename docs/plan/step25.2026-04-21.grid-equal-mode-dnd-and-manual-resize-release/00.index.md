# Grid Equal Mode DnD And Manual Resize Release Plan

Date: 2026-04-21  
Scope: Grid mode에서 `equal` 모드의 드래그앤드롭 재정렬과 수동 split resize 후 mode release를 일관되게 구현한다  
Mode: plan only

## Goal

Grid mode의 레이아웃 모드와 실제 트리를 다시 일치시킨다.

핵심 목표:

- `equal` 모드에서 타일 이동 후 즉시 균등 재적용
- `equal`은 기본 2행 고정, portrait 예외 시 2열 고정
- split 수동 리사이즈 시 `equal` 모드 해제
- 모든 모드 버튼을 토글로 지원
- `focus/auto` 기존 의미 유지
- 저장/복원/툴바 표시와 실제 tree state 일치

## Requirements

- `IMPL-001` equal 모드에서 타일 이동 드롭 완료 시 즉시 균등 배치를 다시 적용해야 한다.
- `IMPL-002` equal은 기본적으로 2행 고정(예: 5개면 위 3 / 아래 2)으로 재구성해야 한다.
- `IMPL-003` 예외적으로 equal 버튼을 누른 시점에 세로가 더 길면 2열 고정으로 재구성해야 한다.
- `IMPL-004` 2행/2열 기준은 equal 버튼을 누른 시점에만 계산하고 resize 이벤트마다 재평가하지 않아야 한다.
- `IMPL-005` split 수동 리사이즈 완료 시 equal 모드를 해제해야 한다.
- `IMPL-006` 타일 이동과 split 수동 리사이즈는 별도 경로로 구분해야 한다.
- `IMPL-007` `equal/focus/auto` 버튼은 이미 선택된 상태에서 다시 누르면 `none`으로 해제되어야 한다.
- `IMPL-008` mode 표시, persistence, restore 결과와 실제 tree 상태가 일치해야 한다.
- `IMPL-009` focus/auto 모드는 기존 동작을 유지해야 한다.
- `IMPL-010` 수동 resize 후에는 사용자가 만든 splitPercentage를 보존해야 한다.

## Assumptions

1. `react-mosaic`의 `onRelease` 또는 drag-end completion 시그널을 활용할 수 있다.
2. 현재 `equal/focus/auto` 3모드만으로는 “모드 없음” 상태를 표현하지 못하므로 `none` 또는 `null` 기반 상태가 필요하다.
3. 최신 요구사항 기준으로 equal은 topology 보존보다 “항상 격자형 재구성”이 우선이다.
4. equal의 2행/2열 기준은 버튼을 누를 때 한 번만 계산하고, 이후 resize 이벤트에서는 유지해야 한다.

## Recommended Execution Order

1. layout mode 상태 모델에 `none`을 도입해 토글 해제/저장/복원 계약을 먼저 정리한다.
2. equal 모드에서 tile drop 완료 시 2행/2열 고정 기준의 격자 재적용 경로를 구현한다.
3. split 수동 리사이즈 완료 시 equal release 경로를 구현한다.
4. 마지막에 회귀와 실사용 검증으로 mode/tree 동기화를 확인한다.

## Success Criteria

- equal 모드에서 tile drop 후 화면이 즉시 2행 고정 또는 portrait 예외 시 2열 고정 격자형으로 수렴한다.
- equal 버튼을 누른 이후 resize 이벤트만으로는 2행/2열 기준이 다시 바뀌지 않는다.
- split 수동 리사이즈 후 equal 표시가 해제되고 사용자 비율이 유지된다.
- 선택된 모드 버튼을 다시 누르면 `none`으로 해제된다.
- 새 mode 상태가 localStorage 저장/복원과 툴바 표시에서 일관된다.
- focus/auto의 기존 흐름은 회귀 없이 유지된다.

- react-mosaic 라이브러리 자체 교체
- drag handle UX 재디자인
