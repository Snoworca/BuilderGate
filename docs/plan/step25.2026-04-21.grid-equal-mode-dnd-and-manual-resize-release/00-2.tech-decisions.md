# Technical Decisions

## ADR-2501: equal release와 toolbar toggle을 위해 none state를 도입한다

- 상태: Accepted
- 이유:
  - 현재 3모드만으로는 “모드 없음” 상태를 정확히 표현할 수 없다.
  - 툴바 표시와 persistence를 일관되게 만들려면 별도 상태가 필요하다.

## ADR-2502: split resize와 tile drop은 구분된 interaction으로 처리한다

- 상태: Accepted
- 이유:
  - 요구사항이 두 행위를 다르게 취급한다.
  - split resize 후 equal 해제와, tile drop 후 equal 유지/재적용은 상반된 정책이다.

## ADR-2503: equal drop 재적용은 completion 시점에만 수행한다

- 상태: Accepted
- 이유:
  - drag 중간 onChange마다 재적용하면 UX가 불안정해진다.
  - release/drop completion 시점이 자연스럽다.

## ADR-2504: split resize는 사용자 비율을 그대로 보존한다

- 상태: Accepted
- 이유:
  - 사용자가 의도적으로 split을 조절한 뒤에는 그 비율이 보존돼야 한다.
  - 따라서 equal 해제 후 tree는 raw 상태를 유지하고 mode만 `none`으로 바뀐다.

## ADR-2505: 모든 mode 버튼은 toggle로 동작한다

- 상태: Accepted
- 이유:
  - 최신 요구사항이 `equal/focus/auto` 전부의 재클릭 해제를 요구한다.
  - 이 동작은 toolbar와 persistence에 동일하게 반영돼야 한다.

## ADR-2506: equal 모드는 topology 보존보다 grid 재구성을 우선한다

- 상태: Accepted
- 이유:
  - 최신 요구사항은 “항상 격자형, 항상 동일한 넓이와 높이”를 우선한다.
  - 따라서 drop 후에는 topology-preserving보다 `applyEqualMode()` 계열의 grid 재구성이 더 적합하다.

## ADR-2507: equal arrangement는 버튼 클릭 시점에만 결정한다

- 상태: Accepted
- 이유:
  - 최신 요구사항은 2행/2열 기준을 resize 이벤트마다 재평가하지 말라고 명시한다.
  - 따라서 equal 버튼 클릭 시점에 `rows` 또는 `cols`를 한 번 결정하고 latch해야 한다.
