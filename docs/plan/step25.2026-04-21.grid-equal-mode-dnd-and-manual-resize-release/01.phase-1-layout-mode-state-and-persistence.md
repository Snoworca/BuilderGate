# Phase 1: Layout Mode State And Persistence

## Objective

mode 없음(`none`)과 toolbar toggle을 표현할 수 있도록 mode state와 persistence 계약을 확장한다.

## Target Files

- `frontend/src/hooks/useLayoutMode.ts`
- `frontend/src/hooks/useMosaicLayout.ts`
- `frontend/src/components/Grid/MosaicToolbar.tsx`
- 관련 타입 참조 파일

## Planned Changes

1. `LayoutMode`에 `none` 또는 `null` 기반 해제 상태 추가
2. `useLayoutMode`와 `useMosaicLayout` 저장/복원 경로 동기화
3. toolbar가 새 mode와 toggle 해제를 정확히 표시하도록 정리

## Implementation Checklist

- [ ] `none` 상태 타입 추가
- [ ] persistence schema에 새 mode 저장 가능
- [ ] restore 시 새 mode 정상 복원
- [ ] toolbar 표시와 active/toggle 상태 정리
