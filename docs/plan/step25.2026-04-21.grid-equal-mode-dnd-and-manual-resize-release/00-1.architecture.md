# Architecture

## 핵심 문제

현재는 `layoutMode`와 실제 `mosaicTree`가 쉽게 어긋난다.

- 드래그앤드롭으로 tile 위치를 바꾸면 tree는 raw drop 결과로 저장되지만 mode는 여전히 `equal`
- split 수동 리사이즈를 해도 mode는 여전히 `equal`

즉, “사용자 의도”와 “mode state”와 “저장된 tree”가 분리되어 있다.

## 제안 구조

### 1. mode state

현재:

- `equal`
- `focus`
- `auto`

제안:

- `equal`
- `focus`
- `auto`
- `none`

이 상태가 있어야 split 수동 조절 이후의 의미와 버튼 재클릭으로 해제된 상태를 저장/복원/표시까지 일관되게 담을 수 있다.

### 2. drop path

- tile drop 완료 시점 감지
- 현재 mode가 `equal`이면:
  - dropped tree를 기준으로 즉시 grid-shaped equal layout으로 재구성
- 사용자의 최신 요구사항 기준으로 topology 보존은 우선순위가 아니다.
- 즉, “일단 그 위치에 넣은 뒤 다시 격자형으로 바꾼다”가 기준이다.
- grid-shaped equal의 구체 규칙:
  - 기본: 2행 고정
  - 예외: equal 버튼을 누른 시점에 세로가 더 길면 2열 고정
  - 이 기준은 버튼 클릭 시점에만 결정되고 이후 resize 이벤트로는 바뀌지 않는다

### 3. split resize path

- split handle interaction start 감지
- release 시점에 split interaction이었으면:
  - 현재 mode가 `equal`일 때 `none`으로 전환
  - tree는 사용자가 만든 비율 그대로 보존

### 4. toolbar toggle path

- 현재 선택된 `equal/focus/auto` 버튼을 다시 누르면:
  - mode를 `none`으로 전환
  - tree는 현재 상태를 그대로 유지

## 권장 구조

- drop은 grid-shaped equal 재적용
- split resize는 equal 해제
- button re-toggle은 `none` 해제
- 둘을 분리하기 위해 interaction source를 저장하는 transient flag가 필요하다
- equal arrangement(`rows`/`cols`)는 버튼 클릭 시점에 latch하고, drop 재적용은 그 latched arrangement를 사용한다
