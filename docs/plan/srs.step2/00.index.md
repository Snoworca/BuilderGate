## 7. Implementation Order

### 7.1 Recommended Order

```
Week 1:
├── Phase 1: Security Infrastructure (1-2 days)
│   └── SSL/TLS, Security Headers, Config Extension
│
├── Phase 2: Authentication Core (2-3 days)
│   └── Password Management, JWT

Week 2:
├── Phase 4: Session Management (1-2 days)
│   └── Heartbeat, Ownership
│
├── Phase 5: Defense Systems (2 days)
│   └── Rate Limiting, Lockout, Blacklist

Week 3:
├── Phase 3: Two-Factor Authentication (1-2 days)
│   └── OTP, SMTP (optional feature)
│
├── Phase 6: Additional Security (2 days)
│   └── CORS, Validation, Logging

Week 4:
├── Phase 7: Frontend Security (2-3 days)
│   └── Login UI, Token Management
│
└── Integration Testing (2 days)
```

### 7.2 Critical Path

```
Phase 1 → Phase 2 → Phase 4 → Phase 7 → Integration Test
```

이 경로의 기능들이 완료되어야 기본적인 인증 시스템이 동작합니다.

## 9. Quality Gates

### 9.1 Per-Phase Gates

각 Phase 완료 시 아래 기준 충족 필수:

| Gate | Criteria |
|------|----------|
| Code Coverage | 핵심 로직 80% 이상 |

| Type Safety | TypeScript strict 모드 에러 없음 |
