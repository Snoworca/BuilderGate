### 7.2 Data Flow Security Requirements

| From | To | Security Requirement |
|------|-----|---------------------|
| Browser | Server | TLS 1.2+, JWT in Authorization header |
| Server | Browser | TLS 1.2+, Security headers, No sensitive data in response |
| Server | Config | AES-256-GCM encrypted secrets |
| Server | PTY | Filtered environment variables |
| Server | Logs | No passwords, OTPs, or tokens in logs |
| Server | SMTP | TLS connection, encrypted credentials |

## 8. Deployment Considerations

### 8.1 Environment Variables (Optional Override)

```bash
# Server
PORT=4242
NODE_ENV=production

# Security (optional - prefer config.json5)
JWT_SECRET_OVERRIDE=<base64-encoded-secret>  # Emergency override only

# SMTP (for containerized deployments)
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=sender@gmail.com
SMTP_PASSWORD=<app-password>
```

### 8.2 File Permissions

```bash
# Configuration file (contains encrypted secrets)
chmod 600 config.json5

# SSL certificates
chmod 600 certs/self-signed.key
chmod 644 certs/self-signed.crt

# Log directory
chmod 700 logs/
```

### 8.3 Startup Sequence

```
1. Load config.json5
   ├── Validate schema
   ├── Encrypt plaintext passwords → save back
   └── Generate JWT secret if empty → save back

2. Initialize SSL
   ├── Load certificates OR generate self-signed
   └── Verify cert-key pair

3. Initialize Services
   ├── CryptoService
   ├── AuthService
   ├── TwoFactorService (if enabled)
   ├── RateLimitService
   ├── AuditService
   └── EnvFilterService

4. Setup Middleware Stack
   └── Rate Limiter → CORS → Security Headers → Auth → Validator → Audit

5. Register Routes
   ├── /api/auth/*
   └── /api/sessions/*

6. Start HTTPS Server
   └── Log startup message (no sensitive data)
```
