  export function createOwnershipMiddleware(sessionManager: SessionManager) {
    return (req: Request, res: Response, next: NextFunction) => {
      const sessionId = req.params.id;
      const jti = req.user?.jti;

      if (!sessionManager.checkOwnership(sessionId, jti)) {
        return res.status(403).json({

      next();

### 3.6 Heartbeat Token Chain

- [ ] **3.6.1** Handle token refresh and ownership
  - 토큰 갱신 시 새 jti 발급
  - PTY 세션의 ownerJti 업데이트 필요?

  **Decision**: ownerJti는 첫 생성 시 고정, 토큰 갱신 시에도 유지
  - 이유: 세션 라이프사이클이 토큰 갱신보다 길 수 있음
  - 대안: 토큰 체인 추적 (복잡)

- [ ] **3.6.2** Alternative: Track original jti
  ```typescript
  interface SessionData {
    // ...
    originalJti: string;  // First token's jti
    currentJti: string;   // Latest token's jti (updated on heartbeat)
  }
  ```

  **Simpler Approach**:
  - 토큰 payload에 `originalJti` 필드 추가
  - 하트비트 시 originalJti 유지
  - 세션 소유권은 originalJti로 확인

## 4. Detailed Specifications

### 4.1 Session Duration (FR-901)

```json5
{
  session: {
    durationMs: 1800000,      // 30분 (default)
    maxDurationMs: 86400000   // 24시간 (max limit)
  }
}
```

**Validation**:
- 60000 (1분) <= durationMs <= 86400000 (24시간)
- durationMs > maxDurationMs → maxDurationMs로 조정

### 4.2 Heartbeat (FR-902)

**Endpoint**: POST /api/auth/heartbeat

**Request**:
```http
POST /api/auth/heartbeat HTTP/1.1
Authorization: Bearer {current_token}
```

**Response (200)**:
```json
{
  "token": "eyJhbGciOiJIUzI1NiIs...(new)",
  "expiresIn": 1800000
}
```

**Processing**:
1. 현재 토큰 검증
2. 새 토큰 발급 (동일 sub, 동일 originalJti, 새 jti, 갱신된 exp)
3. 이전 토큰 jti를 블랙리스트에 추가
4. 새 토큰 반환

### 4.3 JWT Payload Extension

```typescript
interface JWTPayload {
  sub: string;          // "admin"
  iat: number;          // issued at
  exp: number;          // expiry
  jti: string;          // unique token ID
  originalJti?: string; // first token's jti (for ownership)
}
```

### 4.4 Session Ownership (FR-1601)

**Session Creation**:
- 세션 생성 시 현재 토큰의 `originalJti || jti`를 ownerJti로 저장

**Ownership Check**:
- 요청 토큰의 `originalJti || jti`가 세션의 ownerJti와 일치하는지 확인
