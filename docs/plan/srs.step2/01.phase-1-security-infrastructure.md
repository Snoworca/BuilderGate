## 4. Detailed Specifications

### 4.1 SSL Certificate Configuration (FR-1101)

**Config Schema**:
```json5
{
  ssl: {
    certPath: "/path/to/cert.pem",   // 빈 문자열이면 자동 생성
    keyPath: "/path/to/key.pem",     // 빈 문자열이면 자동 생성
    caPath: ""                       // 선택적 CA 체인
  }
}
```

**Validation Rules**:
- certPath, keyPath 모두 비어있거나 모두 설정
- 파일이 존재하면 유효성 검증
- 파일이 없으면 자동 생성 트리거

### 4.2 Self-Signed Certificate Generation (FR-1102)

**Generated Certificate Spec**:
- Algorithm: RSA 2048-bit
- Signature: SHA-256
- Validity: 365 days
- Subject: CN=localhost
- SAN (Subject Alternative Names):
  - DNS: localhost
  - IP: 127.0.0.1
  - IP: ::1

**Output**:
```
certs/
├── self-signed.crt
└── self-signed.key
```

### 4.3 TLS Configuration (FR-1104)

**TLS Options**:
```typescript
{
  minVersion: 'TLSv1.2',
  maxVersion: 'TLSv1.3',
  ciphers: [
    'ECDHE-RSA-AES256-GCM-SHA384',
    'ECDHE-RSA-AES128-GCM-SHA256',
    'ECDHE-RSA-CHACHA20-POLY1305'
  ].join(':'),
  honorCipherOrder: true
}
```

### 4.4 Security Headers (FR-1301)

**Header Configuration**:

| Header | Value | Purpose |
|--------|-------|---------|
| Strict-Transport-Security | max-age=31536000; includeSubDomains | HTTPS 강제 |
| X-Content-Type-Options | nosniff | MIME 스니핑 방지 |
| X-Frame-Options | DENY | 클릭재킹 방지 |
| X-XSS-Protection | 0 | 레거시 XSS 필터 비활성화 |
| Content-Security-Policy | (아래 참조) | XSS 방지 |
| Referrer-Policy | strict-origin-when-cross-origin | 리퍼러 제한 |
| Permissions-Policy | geolocation=(), microphone=(), camera=() | 기능 제한 |
| Cache-Control | no-store | 민감 데이터 캐싱 방지 |

**CSP Directive**:
```
default-src 'self';
script-src 'self';
style-src 'self' 'unsafe-inline';
connect-src 'self';
img-src 'self' data:;
font-src 'self';
frame-ancestors 'none';
```

## 6. Test Scenarios

### 6.1 Unit Tests

#### TC-1101: SSL Certificate Loading
```
Given: 유효한 cert.pem, key.pem 파일이 존재
When: SSLService.loadCertificates() 호출
Then: SSLCredentials 객체 반환
```

#### TC-1102: Self-Signed Certificate Generation
```
Given: ssl.certPath, ssl.keyPath가 빈 문자열
When: 서버 시작
Then:
  - certs/self-signed.crt 생성됨
  - certs/self-signed.key 생성됨
  - config.json5에 경로 업데이트됨
```

#### TC-1103: Certificate-Key Pair Validation
```
Given: cert.pem과 다른 키의 key.pem
When: SSLService.loadCertificates() 호출
Then: Error("Certificate and key pair mismatch") 발생
```

#### TC-1104: TLS Version Enforcement
```
Given: HTTPS 서버 실행 중
When: TLS 1.1로 연결 시도
Then: 연결 거부됨
```

#### TC-1105: TLS 1.2/1.3 Connection
```
Given: HTTPS 서버 실행 중
When: TLS 1.2 또는 TLS 1.3으로 연결
Then: 연결 성공
```

### 6.2 Integration Tests

#### TC-1106: Security Headers Present
```
Given: HTTPS 서버 실행 중
When: GET /health 요청
Then: 모든 보안 헤더가 응답에 포함됨
  - Strict-Transport-Security 헤더 존재
  - X-Content-Type-Options: nosniff
  - X-Frame-Options: DENY
  - Content-Security-Policy 존재
```

#### TC-1107: HTTPS Redirect (Optional)
```
Given: HTTP 및 HTTPS 서버 실행 중
When: HTTP로 요청
Then: HTTPS로 301 리다이렉트
```

#### TC-1108: Certificate Expiry Warning
```
Given: 20일 후 만료되는 인증서
When: 서버 시작
Then: 경고 로그 출력: "Certificate expires in 20 days"
```

## 10. Notes

### 10.1 Development Environment

로컬 개발 시 자체 서명 인증서로 인한 브라우저 경고:
- Chrome: `chrome://flags/#allow-insecure-localhost` 활성화
- Firefox: "위험을 감수하고 계속" 클릭
- 또는 시스템에 인증서 신뢰 추가

### 10.2 Production Considerations

프로덕션 환경에서는:
- Let's Encrypt 또는 상용 인증서 사용
- certPath, keyPath에 실제 경로 설정
- 인증서 자동 갱신 스크립트 구성
