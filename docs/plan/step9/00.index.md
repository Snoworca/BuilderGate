# Step 9: Shell Idle Detection Enhancement

## Overview

세션 idle 상태를 "셸이 프롬프트에서 대기 중"으로 정확하게 감지하도록 개선한다.
현재는 PTY 출력이 멈추면 타이머 기반으로 idle 전환하는데, 이 방식은 타이핑 중에도 에코 출력이 running을 트리거하는 문제가 있다.

## Goals

- 타이핑 중(에코 출력)에는 idle 유지
- Enter 키 입력(명령 제출) 시에만 running 전환
- bash/zsh에서 OSC 133 마커 기반 정밀 감지 지원
- PowerShell/sh 등 OSC 133 미지원 셸은 에코 휴리스틱 fallback

## Architecture

```
Tier 1 (Heuristic)    ← 모든 셸 기본 적용
  writeInput() 시 타임스탬프+길이 기록
  onData()에서 50ms 이내 짧은 출력 = 에코 → running 억제
  Enter 입력 시 즉시 running

Tier 2 (OSC 133)      ← bash/zsh 자동 승격
  셸 초기화 시 OSC 133 마커 주입 스크립트 로드
  onData()에서 마커 파싱 → 정확한 idle/running 전환
  첫 마커 감지 시 heuristic → osc133 자동 전환
```
