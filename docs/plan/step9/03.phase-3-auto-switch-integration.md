# Phase 3: 자동 전환 로직 + 통합

## 목적

Phase 1(에코 휴리스틱)과 Phase 2(OSC 133)를 통합한다. 세션 시작 시 heuristic 모드로 동작하다가, OSC 133 마커가 처음 감지되면 자동으로 osc133 모드로 승격하여 정밀 감지로 전환한다.

## 변경 파일 목록

| File | Action | Description |
|------|--------|-------------|
| `server/src/services/SessionManager.ts` | **수정** | onData 통합 로직, 자동 모드 전환 |

## 1. onData 핸들러 통합 (최종 버전)

Phase 1과 Phase 2의 onData 로직을 통합한 최종 형태:

```typescript
// createSession() 내부, ptyProcess.onData 핸들러

ptyProcess.onData((rawData: string) => {
  const sData = this.sessions.get(id);
  if (!sData) return;

  // ========================================
  // Step 9: OSC 133 마커 처리 (항상 수행)
  // ========================================
  const { stripped, foundMarker } = sData.oscDetector.process(rawData);

  // 자동 모드 승격: 첫 OSC 133 마커 감지 시 heuristic → osc133
  if (foundMarker && sData.detectionMode === 'heuristic') {
    sData.detectionMode = 'osc133';
    console.log(`[Session ${id}] Idle detection upgraded to osc133 mode`);
    // idle 타이머 해제 (osc133 모드에서는 불필요)
    if (sData.idleTimer) {
      clearTimeout(sData.idleTimer);
      sData.idleTimer = null;
    }
  }

  // ========================================
  // Step 9: 모드별 상태 전환
  // ========================================
  if (sData.detectionMode === 'heuristic') {
    // Tier 1: 에코 휴리스틱
    const isEcho = this.isEchoOutput(sData, stripped);
    if (!isEcho) {
      this.updateStatus(id, 'running');
      this.scheduleIdleTransition(id);
    }
  }
  // osc133 모드: OscDetector 콜백에서 직접 상태 전환 (아래 setupOscCallback)

  // ========================================
  // 출력 브로드캐스트/버퍼링 (stripped 사용)
  // ========================================
  // OSC 133 마커가 제거된 출력을 전달
  const outputData = sData.detectionMode === 'osc133' ? stripped : rawData;

  const hasWsSubscribers = this.wsRouter?.hasSubscribers(id) ?? false;
  if (!hasWsSubscribers) {
    this.appendBufferedOutput(sData, outputData);
  } else {
    if (outputData.length > 0) {
      this.broadcastWs(id, 'output', { data: outputData });
    }
  }
});
```

### 설계 결정: 출력 스트리핑

| 모드 | 출력 데이터 | 근거 |
|------|-----------|------|
| `heuristic` | `rawData` (원본) | OSC 마커 없으므로 strip 불필요 |
| `osc133` | `stripped` (마커 제거) | 마커가 xterm.js에 표시되지 않도록 |

**전환 순간 (heuristic → osc133) 주의:** 전환이 일어나는 바로 그 청크부터 `stripped`를 사용한다. 위 코드에서 `detectionMode` 체크 시점이 승격 코드 이후이므로 자연스럽게 처리됨.

## 2. OscDetector 콜백 연결 (setupOscCallback)

`createSession()`에서 OscDetector 생성 후 콜백을 등록한다:

```typescript
// createSession() 내부, sessionData 생성 직후

oscDetector.setCallback((status, event) => {
  // osc133 모드에서만 콜백 기반 상태 전환
  const sd = this.sessions.get(id);
  if (!sd || sd.detectionMode !== 'osc133') return;

  switch (event.type) {
    case 'prompt-start':  // A 마커
    case 'command-end':   // D 마커
      // idle 전환
      this.updateStatus(id, 'idle');
      // osc133 모드에서는 idle 타이머 불필요
      if (sd.idleTimer) {
        clearTimeout(sd.idleTimer);
        sd.idleTimer = null;
      }
      break;

    case 'command-start': // C 마커
      // running 전환
      this.updateStatus(id, 'running');
      break;

    case 'prompt-end':    // B 마커
      // 정보용, 상태 변경 없음 (이미 idle)
      break;
  }
});
```

## 3. writeInput() 최종 버전

Phase 1에서 작성한 writeInput에 osc133 모드 분기 추가:

```typescript
writeInput(id: string, input: string): boolean {
  const data = this.sessions.get(id);
  if (!data) return false;

  // Step 9: 에코 추적 (heuristic 모드에서 사용)
  const hasEnter = input.includes('\r') || input.includes('\n');
  data.echoTracker.lastInputAt = Date.now();
  data.echoTracker.lastInputLen = input.length;
  data.echoTracker.lastInputHasEnter = hasEnter;

  // heuristic 모드: Enter 시 즉시 running
  // osc133 모드: C 마커가 running을 트리거하므로 여기서 상태 변경하지 않음
  if (hasEnter && data.detectionMode === 'heuristic') {
    this.updateStatus(id, 'running');
  }

  data.pty.write(input);
  data.session.lastActiveAt = new Date();
  return true;
}
```

## 4. 모드 전환 상태 다이어그램

```
세션 생성
  │
  ▼
┌─────────────┐
│  heuristic  │  ← 초기 모드 (모든 셸)
│             │
│ 에코 필터링  │
│ + idle 타이머│
└──────┬──────┘
       │ OSC 133 마커 첫 감지
       ▼
┌─────────────┐
│   osc133    │  ← 승격 (bash/zsh만)
│             │
│ A → idle    │
│ C → running │
│ D → idle    │
│ idle타이머X │
└─────────────┘
```

### 전환 규칙

| 조건 | 현재 모드 | 동작 |
|------|----------|------|
| PTY 출력 + 에코 판정 | heuristic | running 억제, idle 유지 |
| PTY 출력 + 에코 아님 | heuristic | running + idle 타이머 |
| Enter 입력 | heuristic | 즉시 running |
| OSC 133 A 마커 | osc133 | idle 전환 |
| OSC 133 C 마커 | osc133 | running 전환 |
| OSC 133 D 마커 | osc133 | idle 전환 |
| Enter 입력 | osc133 | 무시 (C 마커가 처리) |
| 일반 타이핑 | 모두 | 상태 변경 없음 |

## 5. 전체 createSession() 최종 형태 (변경 부분만)

```typescript
createSession(name?: string, shell?: ShellType, cwd?: string): SessionDTO {
  const id = uuidv4();
  this.sessionCounter++;
  const sessionName = name || `Session-${this.sessionCounter}`;

  const { shell: shellCmd, args: shellArgs, shellType } = this.resolveShell(shell);
  const initialCwd = cwd || process.env.HOME || process.env.USERPROFILE || '/';

  // Step 9: OSC 133 셸 환경변수
  const env = this.buildShellEnv(shellType);

  const ptyProcess = pty.spawn(shellCmd, shellArgs, {
    name: this.runtimePtyConfig.termName,
    cols: this.runtimePtyConfig.defaultCols,
    rows: this.runtimePtyConfig.defaultRows,
    cwd: initialCwd,
    env,  // Step 9: 확장된 env
    useConpty: this.runtimePtyConfig.useConpty,
  });

  const session: Session = {
    id,
    name: sessionName,
    status: 'idle',
    createdAt: new Date(),
    lastActiveAt: new Date(),
    sortOrder: this.sessions.size,
  };

  // Step 9: OscDetector 생성
  const oscDetector = new OscDetector();

  const sessionData: SessionData = {
    session,
    pty: ptyProcess,
    idleTimer: null,
    outputBuffer: '',
    initialCwd,
    // Step 9: Idle Detection
    echoTracker: { lastInputAt: 0, lastInputLen: 0, lastInputHasEnter: false },
    detectionMode: 'heuristic',
    oscDetector,
  };

  this.sessions.set(id, sessionData);

  // Step 9: OSC 133 콜백 등록
  oscDetector.setCallback((status, event) => {
    const sd = this.sessions.get(id);
    if (!sd || sd.detectionMode !== 'osc133') return;

    switch (event.type) {
      case 'prompt-start':
      case 'command-end':
        this.updateStatus(id, 'idle');
        if (sd.idleTimer) { clearTimeout(sd.idleTimer); sd.idleTimer = null; }
        break;
      case 'command-start':
        this.updateStatus(id, 'running');
        break;
      case 'prompt-end':
        break;
    }
  });

  // CWD 훅 주입 (기존)
  this.injectCwdHook(id, sessionData, ptyProcess, shellType);

  // PTY 출력 핸들러 (Step 9 통합)
  ptyProcess.onData((rawData: string) => {
    const sData = this.sessions.get(id);
    if (!sData) return;

    // OSC 133 파싱 (항상)
    const { stripped, foundMarker } = sData.oscDetector.process(rawData);

    // 자동 모드 승격
    if (foundMarker && sData.detectionMode === 'heuristic') {
      sData.detectionMode = 'osc133';
      console.log(`[Session ${id}] Idle detection upgraded to osc133 mode`);
      if (sData.idleTimer) { clearTimeout(sData.idleTimer); sData.idleTimer = null; }
    }

    // 모드별 상태 전환
    if (sData.detectionMode === 'heuristic') {
      const isEcho = this.isEchoOutput(sData, stripped);
      if (!isEcho) {
        this.updateStatus(id, 'running');
        this.scheduleIdleTransition(id);
      }
    }
    // osc133: 콜백에서 처리됨

    // 출력 전달 (osc133이면 stripped, 아니면 rawData)
    const outputData = sData.detectionMode === 'osc133' ? stripped : rawData;
    const hasWsSubscribers = this.wsRouter?.hasSubscribers(id) ?? false;
    if (!hasWsSubscribers) {
      this.appendBufferedOutput(sData, outputData);
    } else if (outputData.length > 0) {
      this.broadcastWs(id, 'output', { data: outputData });
    }
  });

  // PTY 종료 핸들러 (기존)
  ptyProcess.onExit(({ exitCode }) => {
    this.broadcastWs(id, 'session:exited', { exitCode });
  });

  return this.toDTO(session);
}
```

## 7. Edge Case 처리

### 7-1. BASH_ENV 주입 실패 (스크립트 경로 오류)

bash 셸에서 OSC 133 스크립트 로드에 실패하면 마커가 출력되지 않는다.
→ `detectionMode`가 `heuristic`으로 유지되어 Tier 1 fallback 자동 적용.
→ 별도 에러 처리 불필요.

### 7-2. 이미 OSC 133을 설정한 사용자 환경

사용자의 `.bashrc`에 이미 OSC 133 마커가 있는 경우:
- `__BUILDERGATE_OSC133` 환경변수로 중복 주입 방지
- 사용자의 마커와 BuilderGate 마커가 중복 출력될 수 있음
  → OscDetector가 모든 마커를 처리하므로 기능적 문제 없음
  → strip이 중복 마커를 모두 제거

### 7-3. 빈 명령 (Enter만 누름)

```
프롬프트 → Enter → PS0(C마커) → 즉시 PROMPT_COMMAND(D+A마커)
```
→ C → running → D → idle → A → idle (중복 idle은 updateStatus에서 무시)
→ 정상 동작

### 7-4. Ctrl+C로 명령 중단

```
running 상태 → Ctrl+C → SIGINT → PROMPT_COMMAND(D+A마커)
```
→ D 마커의 exitCode=130 → idle 전환
→ 정상 동작
