# Phase 1: Tier 1 — Echo Heuristic (에코 휴리스틱)

## 목적

모든 셸에서 동작하는 기본 idle 감지. 사용자 타이핑의 에코 출력을 식별하여 running 전환을 억제하고, Enter 키 입력 시에만 running으로 전환한다.

### 변경 후

```typescript
writeInput(id: string, input: string): boolean {
  const data = this.sessions.get(id);
  if (!data) return false;

  // Step 9: 에코 추적 정보 기록 (pty.write 전에 기록)
  const hasEnter = input.includes('\r') || input.includes('\n');
  data.echoTracker.lastInputAt = Date.now();
  data.echoTracker.lastInputLen = input.length;
  data.echoTracker.lastInputHasEnter = hasEnter;

  // Enter 입력 시 heuristic 모드에서 즉시 running 전환
  if (hasEnter && data.detectionMode === 'heuristic') {
    this.updateStatus(id, 'running');
  }

  data.pty.write(input);
  data.session.lastActiveAt = new Date();
  return true;
}
```

**설계 근거:**
- Enter(\r, \n)는 명령 제출을 의미하므로 즉시 running 전환
- 일반 문자 입력은 에코 추적만 하고 상태 변경하지 않음
- `pty.write()` 전에 tracker를 기록해야 onData 콜백에서 시간차 비교 가능

## 4. onData 핸들러 수정 (에코 필터링)

### 현재 코드 (createSession 내부)

```typescript
ptyProcess.onData((data: string) => {
  this.updateStatus(id, 'running');
  // ... broadcast/buffer ...
  this.scheduleIdleTransition(id);
});
```

### 변경 후

```typescript
ptyProcess.onData((data: string) => {
  const sData = this.sessions.get(id);
  if (!sData) return;

  // Step 9: 에코 휴리스틱 필터링 (heuristic 모드에서만)
  if (sData.detectionMode === 'heuristic') {
    const isEcho = this.isEchoOutput(sData, data);
    if (!isEcho) {
      // 에코가 아닌 출력 → running 전환 + idle 타이머 시작
      this.updateStatus(id, 'running');
      this.scheduleIdleTransition(id);
    }
    // 에코인 경우: running 전환하지 않고, idle 타이머도 건드리지 않음
  }
  // osc133 모드: Phase 3에서 처리 (마커 기반 전환)

  // 출력 브로드캐스트/버퍼링 (상태와 무관하게 항상 수행)
  const hasWsSubscribers = this.wsRouter?.hasSubscribers(id) ?? false;
  if (!hasWsSubscribers) {
    this.appendBufferedOutput(sData, data);
  } else {
    this.broadcastWs(id, 'output', { data });
  }
});
```

**주의:** `sData` 변수를 onData 콜백 시작에서 가져오도록 수정. 기존 코드에서 `this.sessions.get(id)` 호출이 `updateStatus` 뒤에 있었으나, 리팩터링하여 맨 위로 이동.

## 5. isEchoOutput() 메서드 추가

```typescript
/**
 * PTY 출력이 사용자 입력의 에코인지 판정.
 *
 * 에코 조건 (모두 충족 시):
 * 1. 마지막 입력으로부터 50ms 이내
 * 2. 출력 길이가 입력 길이의 2배 이하 (ANSI 색상 코드 여유)
 * 3. 마지막 입력에 Enter가 없었음
 *
 * Enter 입력 후의 출력은 명령 실행 결과이므로 에코가 아님.
 */
private isEchoOutput(sData: SessionData, output: string): boolean {
  const tracker = sData.echoTracker;
  if (tracker.lastInputAt === 0) return false; // 아직 입력 없음

  const elapsed = Date.now() - tracker.lastInputAt;
  const ECHO_TIME_THRESHOLD_MS = 50;
  const ECHO_LENGTH_MULTIPLIER = 2;

  return (
    elapsed < ECHO_TIME_THRESHOLD_MS &&
    output.length <= tracker.lastInputLen * ECHO_LENGTH_MULTIPLIER &&
    !tracker.lastInputHasEnter
  );
}
```

### 에코 판정 임계값 상수 (클래스 상단 또는 메서드 내)

| 상수 | 값 | 근거 |
|------|---|------|
| `ECHO_TIME_THRESHOLD_MS` | 50 | 로컬 PTY 에코는 1-5ms 내 도착. 50ms면 네트워크 지연 없는 로컬 환경에서 충분 |
| `ECHO_LENGTH_MULTIPLIER` | 2 | 컬러 프롬프트의 ANSI 이스케이프 시퀀스로 길이 증가 가능. 2배면 안전 |

## 테스트 방법

### 단위 테스트 (SessionManager.test.ts)

```typescript
describe('Step 9: Echo Heuristic', () => {
  test('일반 타이핑 시 status가 idle 유지', async () => {
    const dto = sessionManager.createSession('test', 'bash');
    // 초기 idle 대기
    await sleep(1000);

    // 일반 문자 입력 (Enter 아님)
    sessionManager.writeInput(dto.id, 'l');

    // 짧은 대기 후 상태 확인
    await sleep(100);
    const session = sessionManager.getSession(dto.id);
    expect(session?.status).toBe('idle');
  });

  test('Enter 입력 시 즉시 running 전환', async () => {
    const dto = sessionManager.createSession('test', 'bash');
    await sleep(1000);

    sessionManager.writeInput(dto.id, 'ls\r');

    const session = sessionManager.getSession(dto.id);
    expect(session?.status).toBe('running');
  });

  test('명령 완료 후 idle 타이머로 idle 복귀', async () => {
    const dto = sessionManager.createSession('test', 'bash');
    await sleep(1000);

    sessionManager.writeInput(dto.id, 'echo hi\r');
    await sleep(100);
    expect(sessionManager.getSession(dto.id)?.status).toBe('running');

    // idleDelayMs 대기 후 idle 복귀
    await sleep(config.session.idleDelayMs + 200);
    expect(sessionManager.getSession(dto.id)?.status).toBe('idle');
  });
});
```

## 알려진 한계

- 컬러 프롬프트 재출력 시 ANSI 시퀀스가 길면 `ECHO_LENGTH_MULTIPLIER=2`로 부족할 수 있음 → Tier 2에서 해결
- 탭 완성 첫 청크가 50ms 이내 도착하면 에코로 오탐할 수 있음 → 탭 완성은 보통 100ms+ 소요하므로 대부분 정상 동작
- PowerShell의 PSReadLine 에코 패턴이 다를 수 있음 → 테스트 후 `ECHO_LENGTH_MULTIPLIER` 조정 가능
