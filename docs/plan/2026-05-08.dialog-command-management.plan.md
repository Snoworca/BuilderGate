# Dialog Infrastructure and Command Preset Manager Implementation Plan

## 개요

이 계획은 BuilderGate 상단 네비게이션의 도구 모음 버튼과 “명령줄 관리” Dialog를 구현한다. 첫 단계는 재사용 가능한 `frontend/src/components/dialog` 공용 shell이며, 이후 서버 저장형 command preset API와 첫 소비자 UI를 붙인다.

스펙 출처는 `docs/srs/buildergate.srs.md`의 `FR-DLG-001`부터 `FR-DLG-010`, `AC-019`, `CON-007`, `NFR-UX-005`, `NFR-MAINT-005`, 그리고 회의 요구사항이다. 이 계획은 서버 저장 범위를 전역 단일 사용자 범위로 둔다. 현재 인증 모델이 로컬 단일 사용자 앱에 가까우므로 workspace별 분리는 이 계획 범위가 아니다.

전체 Phase 수는 4개다. 구현 난이도는 Medium이며, 리스크는 “Dialog focus와 overlay 계약”, “터미널 입력 상태 invariant”, “서버 JSON 저장 원자성”에 집중된다. Infeasible 항목은 없다.

## 선행 조건 및 전제

- 모든 파일 읽기와 쓰기는 UTF-8로 수행한다.
- 수동 검증과 Playwright E2E는 `https://localhost:2002`를 대상으로 한다.
- `frontend/src/components/Modal`은 legacy로 둔다. 새 명령줄 관리 모달은 `frontend/src/components/dialog/WindowDialog.tsx`를 사용한다.
- `react-rnd`는 drag, resize, controlled geometry를 담당한다. 2026-05-08 npm metadata 확인 결과 `react-rnd@10.5.3`은 React 19와 peer range가 충돌하지 않는다. `@radix-ui/react-dialog@1.1.15`도 React 19를 지원하지만, 이번 범위에서는 drag/resize shell과 title-bar handle 통합이 핵심이므로 `react-rnd`를 선택한다.
- Prompt 실행은 붙여넣기만 수행하고 Enter를 전송하지 않는다. Command와 directory 실행은 Enter를 포함해 현재 선택된 tab의 terminal로 전송한다.

## 프로젝트 온보딩 컨텍스트

BuilderGate는 React/Vite 프런트엔드와 Express/TypeScript 서버로 구성된 로컬 터미널 관리 앱이다. 프런트엔드는 Workspace, Tab, Grid, Terminal runtime을 관리하며, 서버는 PTY session과 workspace state를 보관한다.

주요 디렉터리 맵:

| 경로 | 역할 |
|---|---|
| `frontend/src/components/Header` | 상단 네비게이션 버튼과 workspace/cwd 표시 |
| `frontend/src/components/ContextMenu` | portal 기반 context menu 공용 구현 |
| `frontend/src/components/dialog` | 이번 작업에서 새로 만드는 공용 Dialog shell |
| `frontend/src/components/CommandPresetManager` | 이번 작업에서 새로 만드는 명령줄 관리 feature UI |
| `frontend/src/components/Terminal` | terminal handle, input 전송, runtime layer |
| `frontend/src/services/api.ts` | 인증 header를 포함한 REST API client |
| `server/src/services` | 서버 상태와 파일 저장 service |
| `server/src/routes` | Express route mount 지점 |
| `server/src/test-runner.ts` | 서버 regression test registry |

핵심 규칙:

- Interactive AI TUI에서 사용자 입력, prompt redraw, cursor movement는 session status를 `running`으로 전환하면 안 된다. 이번 기능은 `TerminalHandle.sendInput()`만 사용하고 상태를 직접 바꾸지 않는다.
- Modal mode에서는 overlay click과 Escape가 닫기를 수행하지 않는다. X 버튼만 닫는다.
- 서버 저장 preset은 JSON 파일에 저장하며 tmp write, backup, rename 흐름을 유지한다.
- `localStorage`의 Dialog geometry는 browser preference다. 서버 canonical state로 취급하지 않는다.

```ts
export type DialogMode = 'modal' | 'modeless';

export interface DialogRect {
  x: number;
  y: number;
  width: number;
  height: number;
}

export interface DialogSize {
  width: number;
  height: number;
}

export interface WindowDialogProps {
  dialogId: string;
  title: string;
  mode: DialogMode;
  defaultRect: DialogRect;
  minSize: DialogSize;
  onClose: () => void;
  children: React.ReactNode;
}

export function getDialogGeometryKey(dialogId: string): string;
export function clampDialogRect(rect: DialogRect, viewport: DialogSize, minSize: DialogSize): DialogRect;
export function readDialogGeometry(dialogId: string, defaultRect: DialogRect, viewport: DialogSize, minSize: DialogSize): DialogRect;
export function writeDialogGeometry(dialogId: string, rect: DialogRect): void;
```

구현 가이드:

1. `types.ts`에 public prop과 rect type을 둔다.
2. `dialogGeometry.ts`에 pure clamp 함수와 localStorage IO 함수를 분리한다.
3. `getDialogGeometryKey()`는 `buildergate.dialog.${dialogId}.geometry`를 반환한다.
4. `clampDialogRect()`는 width/height를 minSize 이상 viewport 이하로 제한한다.
5. x와 y는 0 이상이며 dialog의 right/bottom이 viewport 밖으로 나가지 않게 계산한다.
6. `readDialogGeometry()`는 JSON parse 실패, nonnumeric 값, 음수 크기, viewport와 맞지 않는 값을 defaultRect 기반 clamp 결과로 대체한다.
7. `writeDialogGeometry()`는 localStorage quota와 private browsing 예외를 catch하고 `console.warn('[WindowDialog] Failed to persist geometry', error)`로 관측 가능하게 만든다.
8. `index.ts`는 `WindowDialog` 추가 전에도 type과 geometry utility를 export한다.
9. unit test는 성공, 손상 JSON, 작은 viewport, minSize 경계, quota 예외 경로를 포함한다.

Rationale: Dialog component에서 geometry 계산을 분리하면 `WindowDialog`와 future modeless shell이 같은 clamp와 persistence 계약을 공유한다.

함정 / 주의사항:

- `localStorage` 값은 보안 state가 아니다. UI preference로만 사용한다.
- viewport보다 큰 minSize가 들어와도 함수가 NaN을 반환하면 안 된다.
- geometry 저장 실패가 Dialog 닫기를 막으면 안 된다.
