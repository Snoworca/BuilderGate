# Terminal Shortcut Manager 구현 계획

## 1. 개요

이 계획은 연구 문서 `docs/research/2026-05-09.buildergate-tui-keyboard-compatibility-research.md`를 구현 착수 가능한 Phase 계획으로 변환한다. 목표는 BuilderGate 웹 터미널에 사용자 등록형 단축키 호환 계층과 `WindowDialog` 기반 관리 UI를 추가하는 것이다.

핵심 산출물은 다음 네 가지다.

- 서버에 저장되는 terminal shortcut profile 및 binding 모델
- xterm.js native 입력 앞단에서 동작하는 순수 shortcut matcher와 얇은 `TerminalView` 통합
- 상단 Tools 메뉴에서 여는 `터미널 키보드` 관리 다이얼로그
- `Shift+Enter -> LF`를 포함한 AI TUI compatibility opt-in profile과 회귀 테스트

기본값은 현행 xterm 동작 보존이다. `Shift+Enter -> LF`는 자동 전역 활성화가 아니라 profile 또는 binding 저장을 통해 활성화한다.

## 7. 아키텍처

```text
KeyboardEvent
  -> TerminalView keydown handler
  -> existing Ctrl+C / Ctrl+V / IME guards
  -> terminalShortcutBindings resolver
  -> action send | block | pass-through
  -> submitCapturedInput(data, debug, "shortcut-binding") for send
  -> TerminalContainer input sequencer
  -> WebSocket input
  -> server PTY
```

Persisted state:

```ts
export type TerminalShortcutScope = 'global' | 'workspace' | 'session';
export type TerminalShortcutProfile = 'xterm-default' | 'ai-tui-compat' | 'custom';

export type TerminalShortcutAction =
  | { type: 'send'; data: string; label?: string }
  | { type: 'pass-through' }
  | { type: 'block' };

export interface TerminalShortcutBinding {
  id: string;
  scope: TerminalShortcutScope;
  workspaceId?: string;
  sessionId?: string;
  profile?: TerminalShortcutProfile;
  key: string;
  code: string;
  ctrlKey: boolean;
  shiftKey: boolean;
  altKey: boolean;
  metaKey: boolean;
  location: number;
  action: TerminalShortcutAction;
  enabled: boolean;
  allowRepeat: boolean;
  description?: string;
  sortOrder: number;
  createdAt: string;
  updatedAt: string;
}

export interface TerminalShortcutProfileSelection {
  scope: TerminalShortcutScope;
  workspaceId?: string;
  sessionId?: string;
  profile: TerminalShortcutProfile;
  updatedAt: string;
}

export interface TerminalShortcutState {
  version: 1;
  profileSelections: TerminalShortcutProfileSelection[];
  bindings: TerminalShortcutBinding[];
}
```

Resolution order:

1. Existing hard guards in `TerminalView`: `Ctrl+C` selection copy, `Ctrl+V` paste-only block, IME active delegation.
2. Reserved combination guard: `Ctrl+C` and `Ctrl+V` skip user mapping.
3. User binding with scope order `session > workspace > global`.
4. Built-in profile binding for the active profile only when no user binding matches.
5. `code + modifiers + location` exact match.
