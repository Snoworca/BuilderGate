# Phase 4: Packaged Runtime First-Run Smoke And Internal Entrypoint

## 목표

GitHub Release ZIP을 fresh extraction한 뒤 EXE를 처음 실행하면 새 서버가 정상 부팅하고 bootstrap 상태를 반환하는지 자동 검증한다.

## 발견된 문제

`BuilderGate.exe --foreground` 검증 중 packaged child가 다음 오류를 냈다.

```text
Error: Cannot find module '<release-dir>\\--internal-app'
```

현재 packaged command는 `BuilderGate.exe --internal-app` 형태다. 반면 `tools/start-runtime.js`는 이미 `process.pkg && BUILDERGATE_INTERNAL_MODE === 'app'` 경로를 지원한다. 따라서 packaged child는 CLI arg보다 env mode 중심으로 실행하는 방향이 안전하다.

## 변경 대상

| 파일 | 변경 |
|---|---|
| `tools/daemon/launcher.js` | packaged app/sentinel launch args 제거, env mode 사용 |
| `tools/start-runtime.js` | foreground packaged internal app env 전달 보장 |
| `tools/daemon/launcher.test.js` | packaged foreground/daemon/sentinel launch contract 갱신 |
| `tools/daemon/start-runtime-compat.test.js` | start-runtime compatibility 갱신 |
| `tools/daemon/process-info.js` | arg marker 의존성이 깨지는지 검토, 필요 시 env/state/log 기반 보강 |
| 신규 smoke script 또는 docs command | release ZIP fresh extraction first-run 검증 |

## 구현 상세

1. packaged `resolveAppLaunchCommand()`는 args를 비우고 command만 EXE로 둔다.
2. daemon app env는 기존처럼 `BUILDERGATE_INTERNAL_MODE=app`를 전달한다.
3. foreground packaged app env도 `BUILDERGATE_INTERNAL_MODE=app`를 전달한다.
4. packaged `resolveSentinelCommand()`도 args를 비우고 `BUILDERGATE_INTERNAL_MODE=sentinel`를 사용한다.
5. sentinel state/start 정보는 args가 아니라 env로 전달하거나, 현재 env/state 파일 경로로 충분한지 검증한다.
6. process discovery가 `--internal-app` arg에만 의존하는 경우 state file의 PID/contract와 runtime root 기준으로 보강한다.

## first-run smoke

```text
precondition:
  ports 2001/2002 free
  no BUILDERGATE_CONFIG_PATH override
  fresh temp extraction directory

steps:
  download release ZIP
  verify sha256
  extract to temp
  assert config.json5 has auth.password: ""
  run BuilderGate.exe --foreground or daemon start
  curl -k https://localhost:2002/api/auth/bootstrap-status

expected:
  setupRequired: true
  requesterAllowed: true for localhost
```

## 테스트

| ID | 시나리오 |
|---|---|
| TC-028-4A | packaged foreground launch env에 `BUILDERGATE_INTERNAL_MODE=app` 포함 |
| TC-028-4B | packaged app launch args는 `--internal-app`를 module path로 전달하지 않음 |
| TC-028-4C | packaged sentinel launch env에 `BUILDERGATE_INTERNAL_MODE=sentinel` 포함 |
| TC-028-4D | fresh release extraction에서 `/api/auth/bootstrap-status`가 setup required 반환 |
| TC-028-4E | 기존 daemon이 떠 있으면 smoke가 먼저 실패하고 stop 안내를 출력 |

## 완료 조건

- packaged runtime이 foreground와 daemon mode 모두에서 내부 app/sentinel 진입에 성공한다.
- fresh release ZIP smoke로 bootstrap first-run 계약을 재현 가능하게 검증한다.
- stale daemon/config override와 release asset 문제를 구분할 수 있는 진단 메시지가 있다.
