# Phase 1: Auth Token Storage Quota Recovery

## 목표

localStorage quota가 `terminal_snapshot_*` 때문에 소진된 상태에서도 login, 초기 비밀번호 설정, 2FA verify, token refresh가 복구 가능한 경우 성공하도록 한다.

## 변경 대상

| 파일 | 변경 |
|---|---|
| `frontend/src/services/tokenStorage.ts` | token/expires 저장 실패 시 cleanup/retry/partial state 제거 |
| `frontend/src/utils/terminalSnapshot.ts` 또는 신규 `frontend/src/utils/storageQuota.ts` | quota 판별, snapshot 목록화, recoverable cache cleanup |
| `frontend/tests/e2e/auth-bootstrap.spec.ts` | bootstrap token 저장 quota 회귀 테스트 |
| 필요 시 `frontend/src/contexts/AuthContext.tsx` | 저장 실패 메시지 관측성 보강, UI 변경은 최소화 |

## 구현 상세

1. `isQuotaExceededError(error: unknown): boolean` 유틸을 추가한다.
2. `collectTerminalSnapshotEntries()`를 추가해 `terminal_snapshot_*`, `terminal_snapshot_remove_*`를 수집한다.
3. snapshot JSON에서 `savedAt`, value length, parse 상태를 추출한다.
4. `evictRecoverableTerminalSnapshots(options)`를 추가한다.
5. `tokenStorage.setToken(token, expiresIn)`을 다음 구조로 변경한다.

```text
try write token/expires
catch error
  clear token/expires
  if quota
    evict recoverable terminal snapshots
    retry once
  if retry fails or non-quota
    clear token/expires
    throw
```

## 정리 대상 정책

| Key | Auth quota cleanup 대상 | 이유 |
|---|---:|---|
| `terminal_snapshot_*` | 예 | terminal replay cache, 오래된 항목 삭제 가능 |
| `terminal_snapshot_remove_*` | 예 | 제거 marker cache, 오래된 marker 삭제 가능 |
| `cws_auth_token` | 아니오 | 현재 저장 대상이며 partial clear만 수행 |
| `cws_auth_expires` | 아니오 | 현재 저장 대상이며 partial clear만 수행 |
| `mosaic_layout_*` | 아니오 | 사용자가 구성한 레이아웃 상태 |
| `tab_state_*` | 아니오 | 탭 상태 의미가 있어 별도 정책 필요 |
| `active_workspace_id` | 아니오 | 작은 값이며 삭제 UX 영향 있음 |

## 테스트

| ID | 시나리오 |
|---|---|
| TC-028-1A | `cws_auth_token` 첫 저장에서 합성 `QuotaExceededError` 발생, 오래된 terminal snapshot 정리 후 retry 성공 |
| TC-028-1B | quota cleanup 후에도 실패하면 `cws_auth_token`/`cws_auth_expires`가 남지 않음 |
| TC-028-1C | non-quota error는 snapshot을 삭제하지 않고 기존 오류 경로로 실패 |
| TC-028-1D | bootstrap password 설정 성공 시 quota recovery 후 token이 저장됨 |

## 완료 조건

- auth token 저장 경로는 quota recovery를 한 번 수행한다.
- recovery가 실패해도 partial auth key가 남지 않는다.
- Playwright 회귀 테스트가 `https://localhost:2002` 기준으로 통과한다.
