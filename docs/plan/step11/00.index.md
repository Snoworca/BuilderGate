**신규 파일: `frontend/src/hooks/useInlineRename.ts`**

```typescript
import { useState, useEffect, useRef, useCallback } from 'react';

interface UseInlineRenameOptions {
  onRename: (name: string) => void;
}

export interface UseInlineRenameReturn {
  isEditing: boolean;
  editName: string;
  inputRef: React.RefObject<HTMLInputElement | null>;
  startEdit: (currentName: string) => void;
  handleChange: (e: React.ChangeEvent<HTMLInputElement>) => void;
  handleKeyDown: (e: React.KeyboardEvent<HTMLInputElement>) => void;
  handleBlur: () => void;
}

export function useInlineRename({ onRename }: UseInlineRenameOptions): UseInlineRenameReturn {
  const [isEditing, setIsEditing] = useState(false);
  const [editName, setEditName] = useState('');
  const inputRef = useRef<HTMLInputElement>(null);

  useEffect(() => {
    if (isEditing) inputRef.current?.focus();
  }, [isEditing]);

  const startEdit = useCallback((currentName: string) => {
    setEditName(currentName);
    setIsEditing(true);
  }, []);

  const confirm = useCallback(() => {
    const trimmed = editName.trim();
    if (trimmed) onRename(trimmed);
    setIsEditing(false);
  }, [editName, onRename]);

  const cancel = useCallback(() => {
    setIsEditing(false);
  }, []);

  const handleChange = useCallback((e: React.ChangeEvent<HTMLInputElement>) => {
    setEditName(e.target.value);
  }, []);

  const handleKeyDown = useCallback((e: React.KeyboardEvent<HTMLInputElement>) => {
    e.stopPropagation(); // 터미널 포커스 방지
    if (e.key === 'Enter') confirm();
    if (e.key === 'Escape') cancel();
  }, [confirm, cancel]);

  const handleBlur = useCallback(() => {
    confirm();
  }, [confirm]);

  return { isEditing, editName, inputRef, startEdit, handleChange, handleKeyDown, handleBlur };
}
```

## Phase 3: 워크스페이스 전환 시 터미널 DOM 유지 (FR-004)

이 Phase가 가장 복잡한 구조 변경이다.

### 현재 구조의 문제

`App.tsx`의 탭 모드 렌더링:
```tsx
activeTab ? (
  <TerminalContainer
    key={`ws-${activeTab.id}-${activeTab.sessionId}`}
    ...
  />
) : null
```

워크스페이스 전환 시 `wm.activeWorkspaceTabs`가 교체되므로 이전 워크스페이스의 탭들이 unmount → xterm dispose → 스크롤백 소멸.

### 해결 방향: 전체 탭 렌더링 + display:none 토글

모든 워크스페이스의 탭을 항상 DOM에 유지하고, 활성 워크스페이스만 표시한다.

### 변경 파일: `frontend/src/App.tsx`

**핵심 변경**: `wm.activeWorkspaceTabs` 렌더링 → `wm.tabs` (전체 탭) 렌더링

```tsx
{/* Tab mode: render ALL tabs, hide inactive */}
{viewMode !== 'grid' && wm.tabs.map(tab => {
  const isActiveWs = tab.workspaceId === wm.activeWorkspaceId;
  const ws = wm.workspaces.find(w => w.id === tab.workspaceId);
  const isActiveTab = ws?.activeTabId === tab.id;
  const isVisible = isActiveWs && isActiveTab;

  return (
    <div
      key={`ws-${tab.id}-${tab.sessionId}`}
      style={{
        display: isVisible ? 'flex' : 'none',
        flexDirection: 'column',
        width: '100%',
        height: '100%',
        '--tab-color': TAB_COLORS[tab.colorIndex] || TAB_COLORS[0],
      } as React.CSSProperties}
    >
      {tab.status === 'disconnected' ? (
        isVisible ? (
          <div style={{ position: 'relative', width: '100%', height: '100%' }}>
            <DisconnectedOverlay onRestart={() => handleRestartTab(tab.id)} />
          </div>
        ) : null  // 비활성 워크스페이스의 disconnected 탭은 렌더링 불필요
      ) : (
        <TerminalContainer
          sessionId={tab.sessionId}
          isVisible={isVisible}
          onStatusChange={handleTerminalStatusChange}
          onCwdChange={handleCwdChange}
          onAuthError={handleAuthError}
        />
      )}
      {isVisible && (
        <MetadataRow
          tab={tab}
          isOdd={false}
          onRename={(name) => handleRenameTab(tab.id, name)}
        />
      )}
    </div>
  );
})}
```

**중요**: `key`에서 `tab.id`를 사용 (탭이 재시작되면 `sessionId`가 바뀌므로 `tab.id`만으로 keying하여 DOM 노드를 유지). 단, 세션 재시작 시 (`restartTab`) 새 sessionId로 TerminalContainer를 재마운트해야 하므로 key는 `tab.id-tab.sessionId`를 유지하되, 워크스페이스 전환은 key를 바꾸지 않는다.

실제로 현재 key `ws-${tab.id}-${tab.sessionId}`는 세션 재시작 시 자동으로 변경되므로 그대로 유지해도 된다. 워크스페이스 전환 시에는 key가 그대로이므로 DOM 유지가 보장된다.

### ResizeObserver 0-size 가드

**변경 파일: `frontend/src/components/Terminal/TerminalView.tsx`**

현재 ResizeObserver 콜백:
```typescript
const resizeObserver = new ResizeObserver(() => {
  if (rafId !== null) cancelAnimationFrame(rafId);
  rafId = requestAnimationFrame(() => {
    fitAddon.fit();
    // ...
    resizeTimer = setTimeout(() => {
      onResize(term.cols, term.rows);
    }, 100);
  });
});
```

**isVisible prop을 TerminalView까지 전달하는 방안** vs **크기 체크 가드 방안**:

TerminalView는 현재 `isVisible` prop이 없다. `TerminalContainer`에서 `display: none`으로 감싸고 있으므로, TerminalView 입장에서는 자신의 크기가 0이 된다는 것을 ResizeObserver로 감지할 수 있다. 

더 단순한 해결: **크기가 0일 때 ResizeObserver 콜백에서 skip**:

```typescript
const resizeObserver = new ResizeObserver(() => {
  // 0-size 가드: display:none 상태에서는 fit 및 PTY resize 스킵
  const container = containerRef.current;
  if (!container || container.offsetWidth === 0 || container.offsetHeight === 0) return;

  if (rafId !== null) cancelAnimationFrame(rafId);
  rafId = requestAnimationFrame(() => {
    fitAddon.fit();
    rafId = null;
    if (resizeTimer !== null) clearTimeout(resizeTimer);
    resizeTimer = setTimeout(() => {
      onResize(term.cols, term.rows);
      resizeTimer = null;
    }, 100);
  });
});
```

이 방식은 `isVisible` prop을 TerminalView까지 drilling할 필요 없이, `offsetWidth/offsetHeight` 체크만으로 `display: none` 상태를 감지한다.

### display:none → flex 전환 시 fitAddon.fit() 타이밍

**변경 파일: `frontend/src/components/Terminal/TerminalContainer.tsx`**

현재 `TerminalContainer`에서 `isVisible`이 변경될 때 fit을 트리거하는 로직이 없다. `isVisible`이 `true`로 변경될 때 `requestAnimationFrame`으로 fit을 호출해야 한다.

그러나 `TerminalContainer`는 `TerminalHandle`을 통해서만 TerminalView를 제어한다. `TerminalHandle`에 `fit()` 메서드를 추가하는 방안:

```typescript
// TerminalView.tsx — TerminalHandle 확장
export interface TerminalHandle {
  write: (data: string) => void;
  clear: () => void;
  focus: () => void;
  hasSelection: () => boolean;
  getSelection: () => string;
  clearSelection: () => void;
  fit: () => void;  // 추가
}

// useImperativeHandle 내부 추가
fit: () => {
  requestAnimationFrame(() => {
    fitAddonRef.current?.fit();
  });
},
```

**TerminalContainer.tsx** — isVisible 변경 감지:
```typescript
// isVisible이 true로 변경될 때 fit 호출
const prevVisibleRef = useRef(isVisible);
useEffect(() => {
  if (isVisible && !prevVisibleRef.current) {
    // display:none → flex 전환: DOM 레이아웃 완료 후 fit
    requestAnimationFrame(() => {
      terminalRef.current?.fit();
    });
  }
  prevVisibleRef.current = isVisible;
}, [isVisible]);
```

이 방식은 ResizeObserver가 0-size 상태에서 이미 가드되어 있으므로, 표시 전환 시 명시적으로 fit을 한 번 호출해야 한다.

### 그리드 모드의 세션 유지

그리드 모드는 `renderTerminal` 함수를 통해 `MosaicContainer`가 렌더링한다. 현재 `renderTerminal`도 `activeWorkspaceTabs`만을 대상으로 하므로, 워크스페이스 전환 시 그리드 내 터미널도 재마운트된다.

그리드 모드의 세션 유지는 Mosaic 레이아웃 구조상 탭 모드보다 복잡하다. **Phase 3에서는 탭 모드만 커버하고, 그리드 모드는 현 동작 유지**로 범위를 제한한다. SRS FR-004도 탭 모드 위주로 기술되어 있다.

## Phase 4: 세션 유지 상한 — LRU (FR-005)

### 변경 파일: `server/config.json5`

`pty` 섹션에 추가:
```json5
pty: {
  // ... 기존 ...
  maxAliveWorkspaces: 0,  // 0 = 제한없음, 1~100 = 최대 유지 워크스페이스 수
}
```

### 설정값 제공 방법

`config.json5`의 `maxAliveWorkspaces`를 서버 API(`/health` 또는 `/api/config`)로 프론트엔드에 전달하거나, 프론트엔드 독자 설정으로 관리한다.

**단순화 결정**: 이 값은 순수 프론트엔드 메모리 관리이므로, 서버 설정 없이 **프론트엔드 내부 상수 또는 localStorage**로 관리한다. `config.json5`의 `pty.maxAliveWorkspaces`는 향후 Settings UI 연동을 위한 예약이고, Phase 4에서는 **프론트엔드 상태**로 구현한다.

### LRU 구현: `frontend/src/App.tsx`

```typescript
// LRU 방문 순서 (앞이 가장 오래된, 뒤가 가장 최근)
const [workspaceVisitOrder, setWorkspaceVisitOrder] = useState<string[]>([]);
const [aliveWorkspaceIds, setAliveWorkspaceIds] = useState<Set<string>>(new Set());

// maxAliveWorkspaces: 0 = 제한없음 (기본값)
const MAX_ALIVE_WORKSPACES = 0; // TODO: Settings UI 연동 예정
```

**워크스페이스 전환 시 LRU 업데이트**:
```typescript
const handleSelectWorkspace = useCallback((id: string) => {
  wmRef.current.setActiveWorkspaceId(id);
  if (isMobile) closeSidebar();

  // LRU 업데이트
  setWorkspaceVisitOrder(prev => {
    const next = prev.filter(wid => wid !== id);
    next.push(id); // 가장 최근으로 이동
    return next;
  });

  // alive set 갱신 + LRU 초과 시 가장 오래된 것 제거
  setAliveWorkspaceIds(prev => {
    const next = new Set(prev);
    next.add(id);
    return next;
  });
}, [isMobile, closeSidebar]);
```

**LRU 초과 감지 및 해제** (`useEffect`):
```typescript
useEffect(() => {
  if (MAX_ALIVE_WORKSPACES <= 0) return; // 제한없음
  if (aliveWorkspaceIds.size <= MAX_ALIVE_WORKSPACES) return;

  // 현재 활성 워크스페이스를 제외한 가장 오래된 것 제거
  const evictCandidate = workspaceVisitOrder.find(
    wid => wid !== wm.activeWorkspaceId && aliveWorkspaceIds.has(wid)
  );
  if (!evictCandidate) return;

  setAliveWorkspaceIds(prev => {
    const next = new Set(prev);
    next.delete(evictCandidate);
    return next;
  });
}, [workspaceVisitOrder, aliveWorkspaceIds, wm.activeWorkspaceId]);
```

**렌더링 조건에 aliveWorkspaceIds 반영**:
```typescript
// Phase 3의 전체 탭 렌더링에서
{wm.tabs.map(tab => {
  const isAlive = MAX_ALIVE_WORKSPACES <= 0 || aliveWorkspaceIds.has(tab.workspaceId);
  if (!isAlive) return null; // 해제된 워크스페이스 — 렌더링 안 함 (다음 방문 시 재마운트)

  const isVisible = tab.workspaceId === wm.activeWorkspaceId && ws?.activeTabId === tab.id;
  // ...
})}
```

### 설정값 유효성 처리

`maxAliveWorkspaces` 파싱 함수:
```typescript
function parseMaxAliveWorkspaces(raw: unknown): number {
  const n = Number(raw);
  if (!Number.isInteger(n) || n < 0 || n > 100) return 0; // 폴백: 제한없음
  return n;
}
```

규칙:
- 0 → 제한없음 (기본값)
- 1~100 → 해당 수
- 음수, 비정수, 100 초과 → 0으로 폴백

## 위험도 분석

### HIGH: FR-004 렌더링 구조 변경

**리스크**: `activeWorkspaceTabs` → `wm.tabs` 전체 렌더링으로 변경 시 성능 저하 가능성.

- 현재 탭 수: 워크스페이스당 최대 8탭, 워크스페이스 최대 10개 → 최대 80개 TerminalContainer 동시 마운트
- 완화책: `display: none` 상태의 터미널은 화면을 그리지 않으므로 렌더링 부하 최소. xterm 인스턴스의 메모리 사용은 증가하나, `maxAliveWorkspaces`(FR-005)로 제한 가능.

**리스크**: `key` 변경 없이 `isVisible`만 토글하므로 기존 `key` 기반 언마운트/재마운트 패턴이 깨질 수 있음.

- 완화책: `restartTab` 시에는 탭의 `sessionId`가 바뀌므로 `key={ws-${tab.id}-${tab.sessionId}}`가 변경되어 올바르게 재마운트됨. 워크스페이스 전환만으로는 key가 변경되지 않으므로 DOM 유지 보장.

### MEDIUM: FR-005 LRU 해제 후 재마운트

**리스크**: 해제된 워크스페이스에 복귀 시 TerminalContainer가 재마운트되며 서버 PTY에 재연결. 이 과정에서 이전 PTY 프로세스가 아직 살아있다면 정상 재연결, 종료되었다면 DisconnectedOverlay 표시.

- 완화책: 기존 `restartTab` 흐름이 이미 disconnected 상태를 처리하므로 추가 구현 불필요.

### LOW: FR-001/FR-002 컨텍스트 메뉴 공유

**리스크**: `onCloseTab` 콜백 시그니처가 그리드 모드(tabId 전달)와 탭 모드(단순 close)에서 다름.

- 완화책: `BuildTerminalMenuOptions.onCloseTab`을 `() => void` (인자 없음)로 정의하고, 호출 측에서 tabId를 클로저로 바인딩.

### LOW: FR-003 MetadataRow 이름 편집

**리스크**: `onBlur`로 편집 확정 시, 사용자가 다른 UI 요소를 클릭할 때 의도치 않게 편집이 확정될 수 있음.

- 완화책: WorkspaceTabBar의 기존 동작(`onBlur` → confirm)과 동일하게 구현. SRS에서도 "포커스 잃으면(onBlur) 편집 확정"으로 명시.
