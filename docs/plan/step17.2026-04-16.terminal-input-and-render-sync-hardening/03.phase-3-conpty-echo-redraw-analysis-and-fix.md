# Phase 3: ConPTY Echo Redraw Analysis And Fix

## Objective

Address the remaining lower-layer issue if Phase 1 proves input arrives correctly but rendered output still corrupts or lags.

## Hypotheses

- ConPTY emits redraw chunks that are valid but visually late for `Codex TUI`
- PSReadLine / shell integration contributes to line redraw overlap under rapid `Enter`
- xterm/ConPTY settings still mis-handle some cursor/erase behavior under fast repeated commands

## Implementation Checklist

- Compare safe input capture timestamps with PTY raw output timestamps.
- Identify whether overlap is already present in PTY raw output.
- If overlap is already in raw output:
  - treat it as PTY/shell/TUI redraw behavior
  - adjust settings or lifecycle assumptions instead of frontend-only fixes
- If raw output is fine but visible output corrupts:
  - audit xterm rendering path and any client-side replacement logic

## Candidate Fix Areas

- ConPTY / Windows PTY config
- shell integration side effects
- xterm rendering options
- client-side output replacement or buffering boundaries

## Review Gate

- reviewer checks that the chosen fix matches the proven failing layer
- reviewer rejects any speculative patch that is not backed by capture evidence

## Exit Condition

The chosen fix is attached to the layer where the corruption actually begins.
