# Phase 1: Input Vs Redraw Separation

## Objective

Prove whether the remaining failures are caused by:

- input loss before PTY write
- PTY input delivery with delayed redraw
- redraw corruption after correct PTY input

## Implementation Checklist

- Add safe input capture metadata on the server side if missing.
- Keep raw secret-bearing input out of persisted debug previews.
- Record enough metadata to distinguish:
  - repeated `Space`
  - `Backspace`
  - `Enter`
  - fast command bursts
- Ensure browser-side debug events can be correlated with server-side input/output capture for the same session.

## Required Evidence

- Browser-side `manual_input_forwarded`
- Server-side `input`
- Server-side `raw_output`
- Timing relation between `input` and `raw_output`

## Review Gate

- reviewer checks that no secret input is stored
- reviewer checks that capture data is sufficient to distinguish loss vs redraw lag

## Exit Condition

For one real repro, we can answer:

- did the repeated keys reach `writeInput()`?
- did PTY emit output immediately after?
- did the visible corruption happen after raw output already contained the problem?
