# Phase 2: Focus And Repeat Contract Hardening

## Objective

Remove frontend ambiguity around which DOM node actually owns keyboard focus and how repeated keys are forwarded.

## Implementation Checklist

- Audit `TerminalRuntimeLayer` focus routing.
- Ensure runtime-layer click focus and fallback focus target are valid for the current DOM.
- Audit `TerminalView.attachCustomKeyEventHandler()` for:
  - plain key forwarding
  - IME composition safety
  - repeat semantics
- Keep shell prompt fixes while avoiding regressions for IME or regular xterm behavior.

## Regression Coverage

- held `Space` repeat
- plain `Backspace`
- active input element is actually the xterm helper textarea after user-facing focus action

## Review Gate

- reviewer checks no dead fallback selectors
- reviewer checks no IME regression
- reviewer checks tests actually match user-visible behavior

## Exit Condition

Frontend can no longer be blamed for silently dropping repeated keydown events in the validated path.
