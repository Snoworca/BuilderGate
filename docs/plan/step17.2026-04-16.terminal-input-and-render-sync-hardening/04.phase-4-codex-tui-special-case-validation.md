# Phase 4: Codex TUI Special-Case Validation

## Objective

Validate the fix against the actual workload where the user still sees the failure.

## Required Scenarios

### Scenario A: Held Space In Codex

1. Start PowerShell.
2. Launch `codex`.
3. Hold `Space`.
4. Observe whether the visible cursor advances in real time.

### Scenario B: Rapid Type + Enter

1. In a PowerShell-like prompt, type a short invalid token.
2. Press `Enter`.
3. Repeat rapidly.
4. Confirm output stacks line-by-line without overlap corruption.

### Scenario C: Codex From PowerShell

1. Start PowerShell.
2. Launch `codex`.
3. Repeat the held-space scenario and compare with plain prompt behavior.

## Playwright Expectation

- Keep automated coverage for shell-level regressions.
- Use browser/server capture to support the `Codex TUI` manual path when direct automation is insufficient.

## Review Gate

- reviewer checks that final claims do not overstate shell-only fixes as `Codex TUI` fixes

## Exit Condition

The remaining issue is either:

- fixed in both plain shell and `Codex TUI`
- or narrowed to a specific TUI redraw limitation with supporting evidence
