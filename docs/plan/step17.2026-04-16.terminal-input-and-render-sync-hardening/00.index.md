# Terminal Input And Render Sync Hardening

Date: 2026-04-16
Scope: `Codex TUI` held-space responsiveness, rapid `input + Enter` overlap corruption, PowerShell/ConPTY/xterm input-render synchronization
Mode: plan first, then phased implementation

## Goal

The next work item is no longer the main `tab <-> grid` replay problem.

The remaining user-visible failures are now:

1. Holding `Space` inside `Codex TUI` does not move the visible cursor in real time.
2. Repeating `type some chars -> Enter` very quickly can make the screen appear vertically overlapped instead of printing line-by-line.

These symptoms still appear even when the terminal is opened from PowerShell and then `codex` is started inside it, so the problem must be treated as an input/render synchronization issue rather than a simple workspace-switch problem.

## Current Findings

- Plain shell prompt behavior improved after direct `Space` / `Backspace` input forwarding.
- `Codex TUI` still behaves differently from the shell prompt.
- This strongly suggests a lower-layer mismatch involving some combination of:
  - browser key repeat handling
  - xterm helper textarea focus path
  - WebSocket input transport timing
  - ConPTY / PSReadLine redraw behavior
  - `Codex` TUI repaint cadence

## Required Constraints

- Implementation must follow this plan order.
- Every substantial code change must be reviewed by a strict code-review subagent.
- If a reviewer finds a concrete issue, fix it and re-run review.
- Validation must include real `https://localhost:2002` Playwright coverage where possible.

## Success Criteria

- Holding `Space` in `Codex TUI` produces visible cursor advancement with no obvious freeze caused by client-side repeat loss.
- Rapid `type + Enter` repetition no longer creates overlapping vertical output in PowerShell sessions.
- Debug capture can prove whether input reached PTY and whether redraw lag happened after input arrival.
- New regression coverage exists for the shell-level symptom and the capture tooling is safe for secrets.

## Execution Order

1. Separate “input not sent” from “input sent but redraw late”.
2. Harden focus and repeat semantics at the frontend boundary.
3. Fix the server/PTY/render boundary only after step 1 proves it is necessary.
4. Revalidate in `Codex TUI`, not just plain PowerShell.
