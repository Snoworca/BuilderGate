## Test

### 정상 시나리오 1

Given 세션이 생성되고 PTY 출력이 발생할 때  
When `SessionManager` 가 raw output 을 처리하면  
Then headless state 와 live broadcast 가 같은 순서로 반영되어야 한다

### 정상 시나리오 2

Given lease owner 가 resize 를 보낼 때  
When 서버가 resize 를 처리하면  
Then PTY 와 headless terminal 의 `cols/rows` 가 동일해야 한다

### 예외 시나리오 1

Given non-owner viewer 가 resize 를 보낼 때  
When 메시지가 도착하면  
Then PTY/headless resize 는 적용되지 않고 viewer local fit 만 유지되어야 한다

### 예외 시나리오 2

Given headless write 가 특정 control sequence 에서 예외를 던질 때  
When 세션 runtime 이 그 chunk 를 처리하면  
Then 세션은 `degraded` 로 마킹되고 live output 자체는 계속 흘러야 한다

### 경계값 시나리오

Given 25개 세션이 동시에 idle 상태로 유지될 때  
When headless runtime 이 모두 살아 있으면  
Then 메모리 증가량과 event loop lag 이 예산 내인지 측정 가능해야 한다

### 테스트 코드 템플릿

```ts
function testResizeUpdatesPtyAndHeadless(): void {
  const runtime = createRuntimeHarness();
  runtime.acquireGeometryLease('client-a');

  runtime.resizeFromClient('client-a', 132, 40);

  assert.equal(runtime.ptyResizeCalls.at(-1)?.cols, 132);
  assert.equal(runtime.headless.cols, 132);
  assert.equal(runtime.headless.rows, 40);
}
```
