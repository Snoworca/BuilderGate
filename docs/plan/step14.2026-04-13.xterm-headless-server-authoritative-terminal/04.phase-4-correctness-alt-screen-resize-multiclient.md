## Test

### 정상 시나리오 1

Given full-screen TUI 가 alt-screen 에 들어간 상태일 때  
When reconnect 후 서버 snapshot 을 적용하면  
Then TUI 화면이 prompt tail 이 아니라 현재 alt-screen 상태로 복원되어야 한다

### 정상 시나리오 2

Given owner client 가 빠르게 resize 를 여러 번 보낼 때  
When headless 와 PTY 가 이를 처리하면  
Then 최종 geometry 가 일치하고 snapshot 이 최신 geometry 기준이어야 한다

### 예외 시나리오 1

Given duplicate subscribe 가 StrictMode 또는 remount 로 발생할 때  
When 동일 socket-session pair 가 다시 subscribe 하면  
Then snapshot 은 한 번만 적용되어야 한다

### 예외 시나리오 2

Given reconnect 중 live output burst 가 동시에 발생할 때  
When snapshot 이 아직 apply 중이면  
Then output 은 정확히 queue 되고 snapshot 뒤에만 flush 되어야 한다

### 경계값 시나리오

Given owner 와 viewer 가 번갈아 포커스를 빠르게 바꾸는 상황일 때  
When lease transfer 가 연속 발생하면  
Then stale owner resize 는 무시되고 마지막 owner 의 geometry 만 살아야 한다

### 테스트 코드 템플릿

```ts
async function testDuplicateSubscribeIsIdempotent(): Promise<void> {
  const harness = createWsSnapshotHarness();

  harness.subscribe('session-1');
  harness.subscribe('session-1');

  const snapshots = harness.clientMessages.filter(m => m.type === 'screen-snapshot');
  assert.equal(snapshots.length, 1);
}
```
