## Test

### 정상 시나리오 1

Given 단일 PowerShell 세션이 있고 일반 prompt 출력이 누적되어 있을 때  
When PTY output 을 headless terminal 에 write 하고 snapshot 을 serialize 하면  
Then snapshot 이 비어 있지 않고 reconnect 복구 입력으로 재사용 가능해야 한다

### 정상 시나리오 2

Given WSL Bash 또는 Zsh 에서 긴 스크롤 출력이 발생했을 때  
When headless terminal 에 동일한 raw output 을 적용하면  
Then 줄바꿈, wrap, cursor 위치가 문자열 tail replay 보다 안정적으로 유지되어야 한다

### 예외 시나리오 1

Given serializer addon 이 Node 환경에서 로드되지만 DOM 접근 오류를 일으킬 때  
When PoC 가 실행되면  
Then 즉시 실패로 기록하고 custom serializer spike 로 분기해야 한다

### 예외 시나리오 2

Given alt-screen 기반 TUI 가 snapshot serialize 후 복구되지 않을 때  
When smoke test 를 수행하면  
Then primary migration 은 보류되고 fallback 설계가 먼저 확정되어야 한다

### 경계값 시나리오

Given scrollback 이 목표 최대치인 `10k` lines 근처까지 누적됐을 때  
When snapshot serialize 를 수행하면  
Then serialize latency 와 snapshot bytes 가 예산 안에 있어야 한다

### 테스트 코드 템플릿

```ts
import assert from 'node:assert/strict';

async function testHeadlessSnapshotPoc(): Promise<void> {
  const runtime = await createHeadlessPocRuntime();
  runtime.write('echo hello\\r\\n');
  const snapshot = await runtime.serialize();

  assert.ok(snapshot.data.length > 0);
  assert.equal(snapshot.source, 'headless');
}
```
