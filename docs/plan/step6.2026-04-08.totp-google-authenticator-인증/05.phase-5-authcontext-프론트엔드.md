- TypeScript 5.x — 기존 `AuthState` 타입에 optional 필드 추가 시 기존 `useState<AuthState>` 초기값 타입 에러 발생 여부 확인

## LLM 코딩 주의사항

- **Simplicity**: `verify2FA()`에서 중간 단계 판별은 `response.nextStage && !response.token` 조건 하나로 충분. 별도 flag 변수 추가 금지.
- **Surgical**: `AuthContext.tsx`에서 기존 `verify2FA()` 로직을 재작성하지 말고, COMBO-4 중간 단계 처리 블록만 **삽입**

### Phase 5-4: verify2FA() 콜백 수정

```typescript
const verify2FA = useCallback(async (otpCode: string): Promise<boolean> => {
  // ... 기존 코드 ...
  const response = await authApi.verify(state.tempToken!, otpCode, state.nextStage ?? undefined);

  // [신규] COMBO-4 중간 단계 처리 (이메일 OTP 통과, TOTP 미완)
  // 서버가 새 tempToken + nextStage: 'totp' 반환 (FR-803)
  if (response.nextStage && !response.token) {
    setState(s => ({
      ...s,
      isLoading: false,
      nextStage: response.nextStage!,              // 'totp'로 업데이트
      tempToken: response.tempToken ?? s.tempToken, // [중요] TOTP용 새 tempToken으로 교체
      maskedEmail: null                             // 이메일 마스킹 정보 제거
    }));
    return false; // 아직 인증 완료 아님
  }

  // 기존 JWT 발급 처리 (변경 없음)
  if (response.token && response.expiresIn) {
    tokenStorage.setToken(response.token, response.expiresIn);
    setState(s => ({
      ...s,
      isAuthenticated: true,
      isLoading: false,
      requires2FA: false,
      tempToken: null,
      maskedEmail: null,
      nextStage: null,       // [신규] 초기화
      emailFallback: false,  // [신규] 초기화
      expiresAt: Date.now() + response.expiresIn!
    }));
    return true;
  }
  // ...
}, [state.tempToken, state.nextStage]);
```

### Phase 5-5: api.ts — verify() 함수 수정

verify: (tempToken: string, otpCode: string, stage?: 'email' | 'totp') =>

## 테스트 시나리오

  When: login() 호출 후 응답 처리
  Then: `state.nextStage === 'totp'`, `state.requires2FA === true`

### 예외 케이스
- Given: COMBO-4, 이메일 OTP 검증 성공 (서버가 nextStage: 'totp', token 없음 반환)
  When: verify2FA(emailOTP) 호출
  Then: 반환값 `false` (인증 미완), `state.nextStage === 'totp'`, `state.maskedEmail === null`

### 경계값 테스트
- Given: emailFallback: true 응답 수신
  When: login() 처리
  Then: `state.emailFallback === true`, `state.nextStage === 'totp'`

- Given: COMBO-4, 이메일 OTP 성공 (서버가 `{ nextStage: 'totp', tempToken: '<new-token>', token: undefined }` 반환)
  When: verify2FA(emailOTP) 반환값 및 상태 확인
  Then: 반환값 `false`, `state.nextStage === 'totp'`, `state.tempToken === '<new-token>'` (기존 tempToken에서 교체됨), `state.maskedEmail === null`
