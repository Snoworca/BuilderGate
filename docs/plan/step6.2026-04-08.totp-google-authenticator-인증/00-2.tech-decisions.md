# 기술 결정 기록 (ADR)

**프로젝트**: BuilderGate TOTP 인증  
**생성일**: 2026-04-08

---

## ADR-001: TOTP 라이브러리 선택 — otplib

**상태**: 확정  
**결정**: `otplib` v12 사용  
**이유**:
- 순수 Node.js crypto 기반 — 외부 API, 비용 없음
- RFC 6238 완전 구현 (Google/Microsoft Authenticator, Authy 호환)
- TypeScript 타입 지원 내장
- `authenticator.options = { window: 1 }` 설정으로 ±30초 허용 구현 용이
- SRS 제약사항 §6.1에 명시됨

---

## ADR-002: QR 코드 출력 — qrcode-terminal

**상태**: 확정  
**결정**: `qrcode-terminal` v0.12 사용  
**이유**:
- 콘솔 아스키아트 출력 전용 — 웹 UI 변경 없이 서버 시작 시 바로 스캔 가능
- 추가 프론트엔드 변경 불필요
- SRS 제약사항 §6.1에 명시됨

---

## ADR-003: TOTP Secret 저장 방식 — data/totp.secret 파일

**상태**: 확정  
**결정**: `data/totp.secret` 단일 파일, CryptoService 암호화  
**이유**:
- 기존 CryptoService(AES-256-GCM) 재사용 — 새 암호화 로직 불필요
- 서버 재시작 후에도 동일 secret 유지 (사용자가 앱 재등록 불필요)
- `.gitignore`에 추가하여 버전 관리 제외
- SRS Q&A에서 사용자 결정: "data/totp.secret 별도 파일"

---

## ADR-004: TOTPService를 TwoFactorService와 분리

**상태**: 확정  
**결정**: 별도 클래스 `TOTPService`로 구현  
**이유**:
- 단일 책임 원칙(SRP): 이메일 OTP와 TOTP는 독립적 활성화/비활성화 가능해야 함
- 향후 이메일 OTP 제거 시 TOTPService는 영향 없음
- 테스트 독립성
- SRS 제약사항 §6.4에 명시됨

---

## ADR-005: COMBO-4 중간 단계 — 동일 tempToken 재사용

**상태**: 확정  
**결정**: 이메일 OTP 통과 후 새 tempToken 발급하지 않고 기존 tempToken의 stage만 `'totp'`로 변경  
**이유**:
- 구현 단순성 — 클라이언트가 토큰을 교체할 필요 없음
- 만료 타이머 연속성 — OTP 타이머가 리셋되지 않음
- SRS FR-305 [자동 보완] 항목에 명시됨

---

## ADR-006: localhostPasswordOnly 기본값 — false

**상태**: 확정  
**결정**: `auth.localhostPasswordOnly: false`  
**이유**:
- SRS Q&A에서 사용자 결정: "기본값 false"
- 기본적으로 2FA가 모든 접속에 적용 (보안 우선)
- 개발 편의를 원할 때만 수동으로 `true`로 설정

---

## ADR-007: TOTP 미등록 시 서버 기동 허용

**상태**: 확정  
**결정**: `totp.enabled = true`이지만 `data/totp.secret` 없어도 서버 기동 허용, 로그인 시 503 반환  
**이유**:
- SRS Q&A에서 사용자 결정: "서버 시작 시 자동 생성" (FR-201)
- 실제로는 파일이 없으면 자동 생성되므로 이 케이스는 생성 실패 시에만 발생
- 서버 완전 중단 방지

---

## ADR-008: stage 미입력 시 하위 호환

**상태**: 확정  
**결정**: `POST /api/auth/verify`에서 `stage` 미입력 시 `OTPData.stage ?? 'email'` 사용  
**이유**:
- 기존 이메일 OTP 클라이언트가 stage 없이 verify 요청 → 깨지지 않음
- SRS 제약사항 §6.3에 명시됨

---

## ADR-009: npm 패키지 설치 위치 — server/

**상태**: 확정  
**결정**: `otplib`, `qrcode-terminal`은 `server/package.json`에만 설치  
**이유**:
- TOTP 로직은 전적으로 서버 사이드
- 프론트엔드에서 TOTP secret을 다루지 않음
