## 2. 4가지 인증 조합 시퀀스 다이어그램

### COMBO-3: TOTP 전용 플로우

```
Client          authRoutes        TwoFactorService    TOTPService
  │                  │                   │                  │
  │─── POST /login ─▶│                   │                  │
  │                  │─── verifyPwd ────▶│                  │
  │                  │◀── pwd OK ────────│                  │
  │                  │─── isRegistered? ─────────────────▶ │
  │                  │◀── true ──────────────────────────── │
  │                  │─── createPendingAuth('totp') ────▶  │
  │                  │◀── tempToken ──────────────────────  │
  │◀── 202 {requires2FA, tempToken, nextStage:'totp'} ─────│
  │                  │                   │                  │
  │─── POST /verify {tempToken, otpCode, stage:'totp'} ───▶│
  │                  │─── verifyTOTP(otpCode) ─────────────▶│
  │                  │◀── {valid: true} ───────────────────  │
  │                  │─── issueJWT ──────────────────────▶  │
  │◀── 200 {success, token, expiresIn} ────────────────────│
```

### COMBO-4: 이메일 OTP + TOTP 순차 플로우

```
Client          authRoutes        TwoFactorService    TOTPService
  │                  │                   │                  │
  │─── POST /login ─▶│                   │                  │
  │                  │─── verifyPwd ────▶│                  │
  │                  │◀── pwd OK ────────│                  │
  │                  │─── sendEmailOTP ─▶│                  │
  │                  │◀── {tempToken} ───│                  │
  │◀── 202 {requires2FA, tempToken, nextStage:'email'} ─── │
  │                  │                   │                  │
  │─── POST /verify {tempToken, emailOTP, stage:'email'} ──▶│
  │                  │─── verifyOTP ────▶│                  │
  │                  │◀── {valid, stage→'totp'} ────────── │
  │◀── 202 {success, nextStage:'totp'} (JWT 없음) ─────── │
  │                  │                   │                  │
  │─── POST /verify {tempToken, totpCode, stage:'totp'} ──▶│
  │                  │─── verifyTOTP(totpCode) ─────────── ▶│
  │                  │◀── {valid: true} ─────────────────── │
  │◀── 200 {success, token, expiresIn} ────────────────── │
```

## 3. 데이터 모델

### TOTPService 내부 상태

```typescript
class TOTPService {
  private secret: string | null = null;      // 복호화된 TOTP secret (메모리)
  private lastUsedStep: number = 0;          // TOTP 재사용 방지 (NFR-105)
  private registered: boolean = false;       // isRegistered() 상태
  
  // config
  private issuer: string;       // 기본: 'BuilderGate'
  private accountName: string;  // 기본: 'admin'
}
```

### OTPData 확장 (TwoFactorService의 otpStore)

```typescript
interface OTPData {
  otp: string;                    // 이메일 OTP 코드 (TOTP-only 시 빈 문자열)
  email: string;                  // 이메일 주소 (TOTP-only 시 빈 문자열)
  expiresAt: number;              // 만료 타임스탬프
  attempts: number;               // 검증 시도 횟수
  stage: TwoFAStage;              // [신규] 'email' | 'totp'
  totpLastUsedStep?: number;      // [신규] TOTP 재사용 방지 (자동 보완)
}
```

### data/totp.secret 파일 형식

```
enc(AAAA...base64...)   ← CryptoService.encrypt()으로 암호화된 BASE32 secret
```

## 5. 보안 아키텍처

### TOTP Secret 보호 계층

```
저장: data/totp.secret
  ↓ CryptoService.encrypt() (AES-256-GCM)
  
메모리: TOTPService.secret (런타임에만 보관)
  ↓ TOTPService.destroy() 호출 시 null로 클리어
  
검증: otplib.authenticator.check()
  ↓ timing-safe 비교 (otplib 내장 또는 crypto.timingSafeEqual)
```

### 다단계 인증 보안 게이트

```
1단계: 비밀번호 인증 (bcrypt compare)
2단계-A: 이메일 OTP 검증 (TwoFactorService)
2단계-B: TOTP 검증 (TOTPService)
  - 최대 3회 시도 (NFR-104)
  - 타임스텝 재사용 방지 (NFR-105)
  - ±1 타임스텝 허용 (NFR-304)
```
