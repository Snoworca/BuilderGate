# Phase 4: authRoutes — 4가지 인증 플로우 구현

## 목표

`authRoutes.ts`에서 4가지 COMBO 인증 플로우를 모두 구현한다.
TOTP 미등록 차단, localhostPasswordOnly 바이패스, 이메일 폴백, COMBO-4 순차 인증까지 포함한다.
이 Phase가 가장 복잡하고 위험도가 높으므로 철저한 회귀테스트가 필수다.

## 선행 조건

- Phase 1 완료 (타입: TwoFAStage, LoginResponse.nextStage 등)
- Phase 2 완료 (TOTPService.isRegistered(), verifyTOTP())
- Phase 3 완료 (AuthRouteAccessors.getTOTPService, TwoFactorService.createPendingAuth(stage))

## 기존 코드 재사용 힌트

- `server/src/routes/authRoutes.ts` — 기존 `POST /api/auth/login` 핸들러 구조 유지, COMBO 분기만 추가
- `server/src/routes/authRoutes.ts` — 기존 `POST /api/auth/verify` 핸들러에서 stage 검증 추가
- `server/src/services/TwoFactorService.ts` — 기존 `sendOTP()`, `verifyOTP()` 재사용 (수정 최소화)

## 코드 중복 주의

- `req.ip` 타입: `string | undefined` → `req.ip ?? ''` 사용
- `uuid` import 방식 확인: 기존 authRoutes에서 이미 사용 중인 방식 그대로 따름
- `const isLocalhost = ['127.0.0.1', '::1', '::ffff:127.0.0.1'].includes(req.ip ?? '')` — 정확한 배열 리터럴 사용

## LLM 코딩 주의사항

- **Simplicity**: COMBO 결정 로직을 하나의 헬퍼 함수 `detectCombo(config, totpService)` 로 추출하면 가독성 향상. 단, 인라인으로 써도 무방 — 함수 추출 강제 금지.
- **Surgical**: 기존 COMBO-1(2FA 없음) 코드에 TOTP 관련 코드를 추가하지 말 것. 기존 분기 아래에 새 분기 추가.
- `otpStore`에서 가져온 `OTPData.stage ?? 'email'` — null-coalescing 필수 (기존 데이터 하위 호환, `FR-802` [자동 보완])

## 상세 구현 가이드

### POST /api/auth/login — 전체 분기 구조

```typescript
router.post('/login', async (req, res) => {
  // 1. 비밀번호 검증
  const { password } = loginSchema.parse(req.body);
  const authService = accessors.getAuthService();
  const isValid = await authService.verifyPassword(password);
  if (!isValid) {
    res.status(401).json({ success: false, message: 'Invalid password' });
    return;
  }

  // 2. localhostPasswordOnly 바이패스 (FR-602)
  // ⚠️ authService.getConfig() 메서드 없음 — AuthService에 getLocalhostPasswordOnly() 추가 필요
  // Phase 3에서 AuthService에 아래 메서드 추가:
  //   getLocalhostPasswordOnly(): boolean { return this.config.localhostPasswordOnly ?? false; }
  const localhostPasswordOnly = authService.getLocalhostPasswordOnly();
  const isLocalhost = ['127.0.0.1', '::1', '::ffff:127.0.0.1'].includes(req.ip ?? '');
  if (localhostPasswordOnly && isLocalhost) {
    const { token, payload } = authService.issueToken();
    res.status(200).json({ success: true, token, expiresIn: payload.exp - payload.iat });
    return;
  }

  // 3. twoFactor 비활성화 시 COMBO-1 (변경 없음)
  const twoFactorService = accessors.getTwoFactorService();
  const totpService = accessors.getTOTPService();
  if (!twoFactorService || !twoFactorService.isEnabled()) {
    const { token, payload } = authService.issueToken();
    res.status(200).json({ success: true, token, expiresIn: payload.exp - payload.iat });
    return;
  }

  // 4. TOTP 활성화 여부 및 등록 여부 확인
  const totpEnabled = totpService != null;
  const totpRegistered = totpService?.isRegistered() ?? false;
  
  if (totpEnabled && !totpRegistered) {
    // FR-401: TOTP 미등록 시 503 차단 + AC-401: console.warn 필수
    console.warn('[TOTP] Login blocked: TOTP is enabled but secret is not registered.');
    // NFR-106: 에러 응답에 내부 정보 최소화 (stack trace, 경로 등 노출 금지)
    res.status(503).json({
      success: false,
      message: 'TOTP is enabled but not configured. Delete data/totp.secret and restart to re-register.'
    });
    return;
  }

  // 5. 이메일 OTP 가능 여부 확인
  const emailEnabled = twoFactorService.hasEmailConfig();  // smtp + email 있는지 확인

  // 6. COMBO 분기
  if (totpEnabled && emailEnabled) {
    // COMBO-4: 이메일 + TOTP
    await handleCombo4(req, res, twoFactorService, totpService!);
  } else if (totpEnabled) {
    // COMBO-3: TOTP 전용
    await handleCombo3(req, res, twoFactorService, totpService!);
  } else {
    // COMBO-2: 이메일 OTP 전용
    await handleCombo2(req, res, twoFactorService);
  }
});
```

async function handleCombo2(req, res, twoFactorService) {
  const { tempToken, otp } = twoFactorService.createPendingAuth('email');
  await twoFactorService.sendOTP(otp);

  });

### COMBO-4 핸들러 (이메일 폴백 포함)

```typescript
async function handleCombo4(req, res, twoFactorService, totpService) {
  const { tempToken, otp } = twoFactorService.createPendingAuth('email');
  
  try {
    await twoFactorService.sendOTP(otp);  // 이메일 전송 시도
    res.status(202).json({
      success: true,
      requires2FA: true,
      tempToken,
      maskedEmail: twoFactorService.maskEmail(twoFactorService.config.email ?? ''),
      nextStage: 'email',  // FR-304
    });
  } catch (emailErr) {
    // FR-501: 이메일 전송 실패 → TOTP 폴백
    console.warn(`[Auth] Email OTP failed, falling back to TOTP for tempToken=${tempToken}`);
    // stage를 'totp'로 변경
    twoFactorService.updateStage(tempToken, 'totp');
    res.status(202).json({
      success: true,
      requires2FA: true,
      tempToken,
      nextStage: 'totp',
      emailFallback: true,  // FR-501 + FR-502
    });
  }
}
```

> TwoFactorService에 `updateStage(tempToken, stage)` 메서드가 없으면 추가 필요:
> ```typescript
> updateStage(tempToken: string, stage: TwoFAStage): void {
>   const otpData = this.otpStore.get(tempToken);
>   if (otpData) { otpData.stage = stage; }
> }
> ```

### POST /api/auth/verify — stage 검증 및 COMBO-4 중간 단계

```typescript
const verifySchema = z.object({
  tempToken: z.string().uuid('Invalid temporary token format'),
  otpCode: z.string().min(4, 'OTP code is required').max(8, 'OTP code too long'),
  stage: z.enum(['email', 'totp']).optional(),  // [신규] FR-802
});

router.post('/verify', async (req, res) => {
  const { tempToken, otpCode, stage: requestedStage } = verifySchema.parse(req.body);
  const twoFactorService = accessors.getTwoFactorService();
  const totpService = accessors.getTOTPService();
  const authService = accessors.getAuthService();
  
  if (!twoFactorService) {
    res.status(400).json({ success: false, message: '2FA not configured' });
    return;
  }

  const otpData = twoFactorService.getOTPData(tempToken);  // OTPData 조회
  if (!otpData) {
    res.status(401).json({ success: false, errorCode: 'OTP_EXPIRED' });
    return;
  }

  // FR-802: stage 검증
  const currentStage = otpData.stage ?? 'email';  // 하위 호환
  const targetStage = requestedStage ?? currentStage;
  if (targetStage !== currentStage) {
    res.status(400).json({ success: false, message: 'Unexpected verification stage' });
    return;
  }

  if (currentStage === 'email') {
    // 이메일 OTP 검증
    // ⚠️ verifyOTP()가 내부적으로 attempts 증가 + 성공/만료 시 otpStore 삭제를 모두 처리함
    // result.success (not result.valid), result.attemptsRemaining 사용
    const result = twoFactorService.verifyOTP(tempToken, otpCode);
    if (!result.success) {
      res.status(401).json({ success: false, errorCode: result.errorCode, attemptsRemaining: result.attemptsRemaining ?? 0 });
      return;
    }

    // COMBO-4: TOTP가 있으면 다음 단계로 (FR-803)
    // verifyOTP() 성공 시 otpStore에서 tempToken 삭제됨
    // → TOTP 단계용 새 OTPData를 createPendingAuth('totp')로 생성 (새 tempToken)
    // → 프론트엔드가 응답의 새 tempToken으로 state.tempToken 업데이트 (Phase 5에서 처리)
    if (totpService?.isRegistered()) {
      const { tempToken: totpTempToken } = twoFactorService.createPendingAuth('totp');
      res.status(202).json({
        success: true,
        tempToken: totpTempToken,  // TOTP 단계용 새 tempToken (FR-803)
        nextStage: 'totp',
        message: 'Email OTP verified. Please enter your TOTP code.'
      });
      return;
    }

    // COMBO-2: 최종 JWT 발급 (FR-804) — verifyOTP가 이미 tempToken 삭제했으므로 invalidate 불필요
    const { token, payload } = authService.issueToken();
    res.status(200).json({ success: true, token, expiresIn: payload.exp - payload.iat });

  } else {
    // TOTP 검증 (COMBO-3, COMBO-4 2단계)
    if (!totpService) {
      res.status(400).json({ success: false, message: 'TOTP not configured' });
      return;
    }

    otpData.attempts += 1;
    const result = totpService.verifyTOTP(otpCode, otpData);
    if (!result.valid) {
      if (otpData.attempts >= 3) twoFactorService.invalidatePendingAuth(tempToken);
      res.status(401).json({
        success: false,
        errorCode: 'OTP_INVALID',
        attemptsRemaining: Math.max(0, 3 - otpData.attempts)
      });
      return;
    }

    // TOTP 성공 — NFR-105: lastUsedStep 업데이트
    otpData.totpLastUsedStep = Math.floor(Date.now() / 30000);
    twoFactorService.invalidatePendingAuth(tempToken);
    const { token, payload } = authService.issueToken();
    res.status(200).json({ success: true, token, expiresIn: payload.exp - payload.iat });  // FR-804
  }
});
```

### 정상 케이스

  When: POST /api/auth/login (올바른 비밀번호)
  Then: 202, `{ requires2FA: true, tempToken, nextStage: 'totp' }`

### 예외 케이스
- Given: `totp.enabled = true`, `data/totp.secret` 없음 (등록 안 됨)
  When: POST /api/auth/login (올바른 비밀번호)
  Then: 503, `{ success: false, message: "TOTP is enabled but not configured..." }`

### 예외 케이스 (COMBO-4 이메일 폴백)
- Given: `twoFactor.enabled = true`, `totp.enabled = true`, smtp 설정 있음, SMTP 서버 다운
  When: POST /api/auth/login (올바른 비밀번호)
  Then: 202, `{ nextStage: 'totp', emailFallback: true, tempToken }` (이메일 전송 실패 → TOTP 직접)

### 경계값 테스트
- Given: COMBO-4, 이메일 OTP 통과 후 `stage: 'email'`로 재요청
  When: POST /api/auth/verify `{ stage: 'email' }` (현재 stage는 'totp')
  Then: 400, `{ message: 'Unexpected verification stage' }`

- Given: COMBO-3, 잘못된 TOTP 코드 3회 연속 입력 (NFR-104)
  When: POST /api/auth/verify 3회 (각각 잘못된 otpCode)
  Then: 3번째 요청 후 401, `{ attemptsRemaining: 0 }`, 이후 동일 tempToken으로 재시도 불가 (otpStore 삭제됨)

- Given: localhostPasswordOnly=true, 외부 IP 접속 (FR-603)
  When: POST /api/auth/login (올바른 비밀번호)
  Then: 2FA 플로우 계속 진행 (localhostPasswordOnly 바이패스 적용 안 됨)
