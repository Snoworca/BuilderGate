# Phase 1: 기반 작업 — 스키마/타입/패키지

## 목표

TOTP 기능 전체의 기반이 되는 Zod schema, TypeScript 타입, config.json5 필드, npm 패키지를 먼저 확보한다.
이 Phase가 완료되어야 서버가 TOTP-only 설정을 정상 파싱할 수 있다.

## 언어+버전 주의사항

- TypeScript 5.x — `satisfies` 연산자 사용 가능하지만 단순 인터페이스 확장은 기존 패턴 유지
- Zod v3 — `.optional()`, `.default()` 체이닝 패턴 기존과 동일
- Node.js — `process.platform` 타입은 `string`이지만 `!== 'win32'` 비교로 충분

## LLM 코딩 주의사항

- **Simplicity**: `totpSchema`를 `twoFactorSchema`에 중첩하되 별도 `z.object()` 블록으로 정의. 인라인 정의 금지.
- **Surgical**: `twoFactorSchema`에서 **refine 블록만 교체**. 다른 필드(email, otpLength, smtp 등) 건드리지 않음.
- `auth.types.ts`에서 `OTPData.stage` 필드 타입은 `TwoFAStage` (string literal union). `string` 타입 사용 금지.

### Phase 1-2 & 1-3: config.schema.ts 수정

**수정 대상**: `twoFactorSchema` 내 refine 블록

현재 코드 (변경 전):
```typescript
.refine(
  (data) => !data.enabled || (data.email && data.smtp),
  { message: '2FA enabled requires email and smtp configuration' }
)
```

변경 후:
```typescript
.refine(
  (data) => {
    if (!data.enabled) return true;
    const hasEmail = !!(data.email && data.smtp);
    const hasTotp = !!(data.totp?.enabled);
    return hasEmail || hasTotp;
  },
  { message: '2FA enabled requires either email+smtp or totp configuration' }
)
```

**추가할 totpSchema** (twoFactorSchema 위에 배치):
```typescript
const totpSchema = z.object({
  enabled: z.boolean().default(false),
  issuer: z.string().default('BuilderGate'),
  accountName: z.string().default('admin'),
});
```

**twoFactorSchema에 totp 필드 추가**:
```typescript
const twoFactorSchema = z.object({
  enabled: z.boolean().default(false),
  email: z.string().email().optional(),   // 기존 optional로 변경
  otpLength: z.number().int().min(4).max(8).default(6),
  otpExpiryMs: z.number().int().positive().default(300000),
  smtp: smtpSchema.optional(),
  totp: totpSchema.optional(),            // [신규]
})
```

**authSchema에 localhostPasswordOnly 추가**:
```typescript
const authSchema = z.object({
  password: z.string(),
  durationMs: z.number().int().positive().default(1800000),
  maxDurationMs: z.number().int().positive().default(86400000),
  jwtSecret: z.string().default(''),
  localhostPasswordOnly: z.boolean().default(false),  // [신규]
});
```

## 테스트 시나리오

### 정상 케이스
- Given: `twoFactor.enabled = true`, `twoFactor.totp.enabled = true`, smtp 없음
  When: 서버 시작 (config.json5 파싱)
  Then: Zod 유효성 검사 통과, 서버 정상 기동

### 예외 케이스
- Given: `twoFactor.enabled = true`, smtp 없음, `totp.enabled = false`
  When: 서버 시작 (config.json5 파싱)
  Then: Zod 유효성 검사 실패 (`'2FA enabled requires either email+smtp or totp configuration'`)

### 경계값 테스트
- Given: `twoFactor.enabled = false`, `twoFactor.totp.enabled` 미설정
  When: 서버 시작
