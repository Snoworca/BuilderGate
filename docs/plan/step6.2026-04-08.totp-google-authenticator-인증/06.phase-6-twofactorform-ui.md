# Phase 6: Frontend — TwoFactorForm UI 확장

## 목표

`TwoFactorForm.tsx`에서 `nextStage`에 따라 이메일 OTP 화면과 TOTP 화면을 조건부 렌더링한다.
이메일 폴백 경고 배너, TOTP 화면의 카운트다운 타이머 비표시도 포함한다.

## 선행 조건

- Phase 5 완료 (AuthContext.nextStage, emailFallback 상태 준비)

## 기존 코드 재사용 힌트

- `frontend/src/components/Auth/TwoFactorForm.tsx` — 기존 이메일 OTP 화면 구조 유지, TOTP 화면은 조건부로 전환
- 기존 5분 카운트다운 타이머 컴포넌트/로직 재사용 (이메일 OTP 화면)
- `useAuth()` 훅에서 `nextStage`, `emailFallback` 읽기

## 코드 중복 주의

- 이메일 OTP 화면 코드를 완전히 재작성하지 말 것 — `nextStage === 'email'` 분기에서 기존 화면 그대로 사용
- TOTP 화면은 기존 이메일 화면과 매우 유사 — 공통 입력 UI를 **별도 컴포넌트로 분리하지 말 것**. 기존 인라인 코드에 `stageInfo` 조건 분기만 추가.
- `<OTPInput>`, `<OTPCountdownTimer>` 는 기존 TwoFactorForm 내부 인라인 코드를 지칭하는 **pseudocode**임. 실제로 별도 파일/컴포넌트로 만들지 말 것.

- CSS — 경고 배너는 기존 CSS 클래스 패턴 따름 (인라인 스타일 최소화)

## LLM 코딩 주의사항

- **Simplicity**: `stageInfo` 객체 리터럴 패턴으로 stage별 문구 관리 (SRS §5.3.1 예시 코드 참고). 별도 컴포넌트 분리 불필요.
- **Surgical**: 기존 이메일 OTP 제출 핸들러 변경 없음. `verify2FA(otpCode)` 호출 방식 유지.

## 상세 구현 가이드

### nextStage 조건부 렌더링 구조

```typescript
// frontend/src/components/Auth/TwoFactorForm.tsx
function TwoFactorForm() {
  const { nextStage, maskedEmail, emailFallback, verify2FA, error } = useAuth();

  const stageInfo = nextStage === 'totp'
    ? {
        title: 'Authenticator Code',
        description: emailFallback
          ? 'Email unavailable. Please enter your Authenticator code.'
          : 'Enter the 6-digit code from your authenticator app.',
        showCountdown: false,   // TOTP는 30초 자동 갱신이므로 타이머 불필요
        placeholder: '000000',
      }
    : {
        title: 'Verification',
        description: `Verification code sent to ${maskedEmail}`,
        showCountdown: true,    // 이메일 OTP는 5분 타이머 유지
        placeholder: '000000',
      };

  return (
    <div className="two-factor-form">
      {/* 이메일 폴백 경고 배너 */}
      {emailFallback && (
        <div className="alert alert-warning">
          Email delivery failed. Please use your Authenticator app instead.
        </div>
      )}

      <h2>{stageInfo.title}</h2>
      <p>{stageInfo.description}</p>

      {/* 카운트다운 타이머 (이메일 OTP만) */}
      {stageInfo.showCountdown && <OTPCountdownTimer />}

      {/* OTP 입력 (기존 코드 재사용) */}
      <OTPInput placeholder={stageInfo.placeholder} onSubmit={verify2FA} />
      
      {error && <p className="error">{error}</p>}
    </div>
  );
}
```

## 테스트 시나리오

### 정상 케이스
- Given: `state.nextStage === 'totp'`, `state.emailFallback === false`
  When: TwoFactorForm 렌더링
  Then: 제목 "Authenticator Code", 설명 "Enter the 6-digit code...", 타이머 없음

### 예외 케이스
- Given: `state.emailFallback === true`
  When: TwoFactorForm 렌더링
  Then: 상단에 경고 배너 "Email delivery failed. Please use your Authenticator app instead." 표시

### 경계값 테스트
- Given: `state.nextStage === 'email'`
  When: TwoFactorForm 렌더링
  Then: 기존 이메일 OTP 화면 (5분 타이머 표시, "Verification code sent to ..." 메시지)
