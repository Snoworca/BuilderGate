## 기존 코드 재사용 힌트

- `server/src/services/CryptoService.ts` — **반드시 재사용**:
  - `.encrypt(plaintext: string): string` — TOTP secret 파일 저장 시
  - `.decrypt(encryptedString: string): string` — 파일에서 secret 로드 시
  - `.isEncrypted(value: string): boolean` — 파일 내용이 암호화 형식인지 확인
  - `.timingSafeEqual(a: string, b: string): boolean` — TOTP 코드 비교 (NFR-101)
- `server/src/services/TwoFactorService.ts` — `destroy()` 패턴, 로깅 패턴 참고
- `server/src/utils/errors.ts` — `AppError`, `ErrorCode` 패턴 참고

## 코드 중복 주의

- `CryptoService.timingSafeEqual()` 이 이미 구현됨 → **절대 새로 구현하지 말 것**. 반드시 재사용.
- `generateSecureRandom()` 이 이미 CryptoService에 있으나, TOTP secret은 `otplib.authenticator.generateSecret()`을 사용 (BASE32 포맷이어야 함)
- `TOTPService`에 이메일 관련 코드를 추가하지 말 것 — 이메일 OTP는 `TwoFactorService` 전담

## 언어+버전 주의사항

- TypeScript 5.x — `private readonly` 필드 사용 권장
- `fs.promises.readFile` / `fs.writeFileSync` — 동기(writeFile)로 초기화, 비동기는 필요 없음
- `process.platform !== 'win32'` 분기로 `fs.chmodSync()` 호출 (FR-203)
- `otplib` import: `import { authenticator } from 'otplib'`
- `qrcode-terminal` import: `import qrcode from 'qrcode-terminal'`

## LLM 코딩 주의사항

- **Simplicity**: `initialize()` 메서드 하나로 로드 or 생성 분기. 별도 `tryLoad()`, `tryCreate()` 메서드 분리 금지.
- **Surgical**: 이 파일은 신규 생성이므로 기존 코드 수정 없음. 단, `CryptoService`를 생성자 인자로 받아야 함 — 직접 import + 생성 금지.
- **NFR-105 재사용 방지**: `lastUsedStep`은 `OTPData.totpLastUsedStep` 필드로 관리. `TOTPService` 클래스 멤버로 추가하지 말 것 (tempToken별 독립 추적 필요).
- `verifyTOTP(code, otpData)` — 두 번째 인자는 `otpData: OTPData`. `attempts: number`를 두 번째 인자로 받는 형태로 구현하지 말 것.

```typescript
// server/src/services/TOTPService.ts
import { authenticator } from 'otplib';
import qrcode from 'qrcode-terminal';
import * as fs from 'fs';
import * as path from 'path';
import type { CryptoService } from './CryptoService';
import type { TOTPConfig } from '../types/config.types';
import type { OTPData } from '../types/auth.types';  // NFR-105: totpLastUsedStep 접근 위해 필요

const SECRET_FILE_PATH = path.join(process.cwd(), 'data', 'totp.secret');
const BASE32_REGEX = /^[A-Z2-7]+=*$/;

export class TOTPService {

  private registered: boolean = false;
  // ⚠️ lastUsedStep은 TOTPService 멤버가 아님 — OTPData.totpLastUsedStep 필드로 관리 (NFR-105)
  // 이유: tempToken마다 별도 추적 필요, TOTPService 멤버로 관리 시 단일 세션만 지원

  constructor(
    private readonly config: TOTPConfig,
    private readonly cryptoService: CryptoService
  ) {

    authenticator.options = { window: 1 };  // NFR-304: ±30초 허용

  initialize(): void { /* ... */ }

  // Phase 2-2: secret 자동 생성 (FR-201)
  private createAndSaveSecret(): void { /* ... */ }

  // Phase 2-3: 기존 secret 로드 (FR-202)
  private loadSecret(): void { /* ... */ }

  // Phase 2-5: QR 코드 출력 (FR-102)
  printQRCode(): void { /* ... */ }

  // Phase 2-6: 등록 여부 확인 (FR-402)
  isRegistered(): boolean { return this.registered; }

  // Phase 2-6: TOTP 코드 검증 (FR-101, NFR-101, NFR-104, NFR-105)
  // ⚠️ otpData를 받아서 totpLastUsedStep을 읽음 — verifyTOTP(code, attempts: number) 형태로 구현하지 말 것
  verifyTOTP(code: string, otpData: OTPData): { valid: boolean } { /* ... */ }

  // 정리
  destroy(): void {
    this.secret = null;
    this.registered = false;
  }
}
```

### FR-201: secret 자동 생성

```typescript
private createAndSaveSecret(): void {
  const newSecret = authenticator.generateSecret(); // BASE32, 20바이트

  // data/ 디렉토리 자동 생성
  const dir = path.dirname(SECRET_FILE_PATH);
  if (!fs.existsSync(dir)) {
    fs.mkdirSync(dir, { recursive: true });
  }

  const encrypted = this.cryptoService.encrypt(newSecret);
  fs.writeFileSync(SECRET_FILE_PATH, encrypted, 'utf-8');

  // FR-203: 파일 권한 설정 (Linux/Mac 전용)
  if (process.platform !== 'win32') {
    fs.chmodSync(SECRET_FILE_PATH, 0o600);
  }

  this.secret = newSecret;
  this.registered = true;
  console.log('[TOTP] New secret generated and saved.');
}
```

### FR-202: 기존 secret 로드

```typescript
private loadSecret(): void {
  const content = fs.readFileSync(SECRET_FILE_PATH, 'utf-8').trim();
  const decrypted = this.cryptoService.decrypt(content); // 복호화 실패 시 AppError throw
  
  if (!BASE32_REGEX.test(decrypted)) {
    throw new Error('[TOTP] Secret file has invalid BASE32 format.');
  }

  this.secret = decrypted;
  this.registered = true;
}
```

### initialize() 통합 메서드

```typescript
initialize(): void {
  if (!fs.existsSync(SECRET_FILE_PATH)) {
    console.log('[TOTP] No secret file found. Generating new secret...');
    this.createAndSaveSecret();
    this.printQRCode();  // 최초 생성 시 QR 출력
  } else {
    try {
      this.loadSecret();
      this.printQRCode();  // FR-102: 매 시작마다 출력 (Q&A 결정: 항상 출력)
    } catch (err) {
      console.error('[TOTP] TOTP secret file is corrupted or cannot be decrypted.');
      console.error('[TOTP] Delete data/totp.secret and restart to re-register.');
      throw err; // 서버 시작 중단 (FR-202 요구사항)
    }
  }
}
```

### FR-102: QR 코드 출력

```typescript
printQRCode(): void {
  if (!this.secret) return;
  const issuer = this.config.issuer ?? 'BuilderGate';
  const accountName = this.config.accountName ?? 'admin';
  const uri = authenticator.keyuri(accountName, issuer, this.secret);

  console.log('[TOTP] Google Authenticator QR Code:');
  qrcode.generate(uri, { small: true });
  console.log(`[TOTP] Manual entry key: ${this.secret}`);
  console.log(`[TOTP] Issuer: ${issuer} | Account: ${accountName}`);
}
```

### NFR-101 + NFR-105: verifyTOTP 구현

```typescript
verifyTOTP(code: string, otpData: OTPData): { valid: boolean } {
  if (!this.secret || !this.registered) return { valid: false };

  // NFR-104: 최대 3회 시도
  if (otpData.attempts >= 3) return { valid: false };

  // otplib 내부 timing-safe 검증 + ±1 window
  const isValid = authenticator.check(code, this.secret);
  if (!isValid) return { valid: false };

  // NFR-105: 재사용 방지 (동일 타임스텝 2회 사용 불가)
  const currentStep = Math.floor(Date.now() / 30000);
  const lastUsed = otpData.totpLastUsedStep ?? 0;
  if (currentStep === lastUsed) {
    console.warn('[TOTP] Code replay detected');
    return { valid: false };
  }

  return { valid: true };
}
```

## 테스트 시나리오

### 정상 케이스
- Given: `data/totp.secret` 파일 없음, `totp.enabled = true`
  When: `initialize()` 호출
  Then: 파일 생성됨, `isRegistered() === true`, QR 코드 콘솔 출력됨

### 예외 케이스
- Given: `data/totp.secret` 파일이 손상됨 (유효하지 않은 enc 포맷)
  When: `initialize()` 호출
  Then: 에러 throw, 서버 시작 중단, 오류 메시지 출력

### 경계값 테스트
- Given: 유효한 TOTP 코드를 연속 2회 동일 타임스텝에서 검증
  When: `verifyTOTP(sameCode)` 2회 호출
  Then: 첫 번째 호출 true, 두 번째 호출 false (재사용 방지)
