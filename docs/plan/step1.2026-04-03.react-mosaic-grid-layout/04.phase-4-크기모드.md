## Phase 4: 크기 모드 상태 머신 (균등/포커스/오토)

### 목표
3가지 레이아웃 모드(균등/포커스/오토)의 상태 머신과 트리 조정 알고리즘을 구현한다.

## 선행 조건
- Phase 2 완료 필요 (MosaicContainer의 Mosaic 트리 구조)
- Phase 1 완료 필요 (getMinPercentage, countLeaves 등 헬퍼)

## 기존 코드 재사용 힌트
- `useWorkspaceManager.updateTabStatus()`: tabs의 `status` 필드(`idle`/`running`) 변경을 이미 관리. 오토 모드는 이 데이터를 소비만 함
- Phase 1의 `getMinPercentage()`, `countLeaves()`, `containsLeaf()`, `findLeafSide()`: `applyFocusMode` 내부에서 직접 호출

## 코드 중복 주의
- `applyEqualMode(tree)`: `buildEqualMosaicTree(extractLeafIds(tree))`를 호출하면 되므로 별도 함수 구현 불필요. 래퍼 함수로만 정의
- 오토 모드 idle 감시 useEffect: `useWorkspaceManager`의 `updateTabStatus`가 이미 `tabs` 상태를 갱신하므로, MosaicContainer는 `tabs` prop만 의존. 별도 WS 구독 금지

## 구현 체크리스트
- [ ] Phase 4-1: `applyFocusMode(tree, focusTabId, minPercent)` 함수 구현 — 루트→타겟 경로 조상 splitPercentage 재계산. `findLeafSide/countLeaves` 활용 `FR-5.1`
- [ ] Phase 4-2: `applyEqualMode(tree)` 함수 구현 — `buildEqualMosaicTree(extractLeafIds(tree))` 래퍼 `FR-4.3`
- [ ] Phase 4-3: `applyMultiFocusApprox(tree, idleIds, minPercent)` 함수 구현 — idle/running 비율 ±30% 보정 근사 `FR-6.2`
- [ ] Phase 4-4: `useLayoutMode` 훅 신규 생성 — `equal/focus/auto` 상태, `setMode(mode, focusTabId?)`, `applyToTree(tree, tabs)` `FR-5.2`
- [ ] Phase 4-5: `MosaicContainer`에 `useLayoutMode` 통합 — MosaicToolbar 클릭 → `setMode()` → `applyToTree()` → `setMosaicTree()` `FR-4.1~4.6`
- [ ] Phase 4-6: 오토 모드 idle 감시 useEffect — `tabs.status` 변경 → `applyToTree()` 자동 호출 `FR-6.1`
- [ ] Phase 4-7: 상태 전이 엣지 케이스 — 포커스 대상 탭 닫힘 → equal 전환, auto 중 수동 리사이즈(isUserDrag) → equal 전환 `SRS 상태 전이표`

## 언어+버전 주의사항
- useEffect 의존성: `tabs.map(t => \`${t.id}:${t.status}\`).join(',')` — 객체 참조 대신 문자열 비교로 불필요한 재실행 방지
- 즉시 전환: CSS transition 사용 금지 (Mosaic inline style 제약, SRS ADR)

## LLM 코딩 주의사항
- **Simplicity**: `applyMultiFocusApprox`의 `±30% 보정`은 고정 상수. 동적 가중치 알고리즘 불필요
- **Surgical**: `MosaicContainer`에 `useLayoutMode`를 추가할 때, 기존 `handleMosaicChange`의 isUserDrag 감지 로직에 `if (layoutMode === 'auto') onLayoutModeChange('equal')` 1줄만 추가. 다른 핸들러 수정 금지

## 테스트 시나리오

### 정상 케이스
- Given: 4개 세션 균등 모드
  When: 포커스 버튼 클릭 (세션 A)
  Then: 세션 A = 76%, 나머지 각 8%

- Given: 오토 모드, 전부 running
  When: 세션 C가 idle로 전환
  Then: 300ms 이내 세션 C 확대, 나머지 축소

- Given: 오토 모드, 세션 C/E가 idle
  When: 다중 idle 상태
  Then: C/E가 running보다 큰 비율

### 예외 케이스
- Given: 포커스 모드 (세션 A 확대)
  When: 세션 A 닫기
  Then: 자동으로 equal 모드 전환

- Given: 오토 모드
  When: 사용자가 분할선 수동 드래그
  Then: equal 모드로 자동 전환

### 경계값 테스트
- Given: 8세션 포커스 모드
  When: `applyFocusMode(tree, 'target', 5)`
  Then: 타겟 = 65%, 나머지 각 5%

- Given: 오토 모드, 전부 idle
  When: 8개 세션 모두 idle
  Then: 모든 세션이 포커스 확대 비율 근사 (≈ 균등)
