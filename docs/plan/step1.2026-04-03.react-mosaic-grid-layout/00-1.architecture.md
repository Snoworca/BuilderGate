# 아키텍처 설계: React Mosaic 그리드 레이아웃

## 컴포넌트 계층

```
App.tsx
  └── WorkspaceTabBar (viewMode 전환 버튼)
  └── [viewMode === 'grid']
      └── MosaicContainer (신규)
          ├── <Mosaic<string>> (react-mosaic-component)
          │   └── renderTile → MosaicTile (신규)
          │       ├── MosaicToolbar (신규, position:absolute 오버레이)
          │       ├── TerminalContainer (기존 재사용)
          │       │   └── TerminalView (기존 재사용)
          │       ├── MetadataRow (기존 재사용)
          │       └── DisconnectedOverlay (기존 재사용)
          ├── ContextMenu (기존 재사용, portal to body)
          └── ConfirmModal (기존 재사용)
  └── [viewMode === 'tab']
      └── TabBar + TerminalContainer (기존, 변경 없음)
```

## 훅 계층

```
MosaicContainer
  ├── useMosaicLayout (신규)
  │   ├── buildEqualMosaicTree()
  │   ├── saveLayout() / loadLayout()
  │   └── debouncedSave (1초)
  ├── useLayoutMode (신규)
  │   ├── applyFocusMode()
  │   ├── applyMultiFocusApprox()
  │   └── applyEqualMode()
  ├── useFocusHistory (신규)
  │   └── recordFocus() / getPrevious()
  ├── useContextMenu (기존 재사용)
  └── useResponsive (기존 재사용, isMobile)
```

## 기술 결정 기록 (ADR)

### ADR-1: Mosaic leaf ID = tabId
- **결정**: `MosaicNode<string>`의 leaf 값으로 `tabId`를 사용
- **근거**: 워크스페이스 내 고유, 서버 `activeTabId`와 직접 매핑

### ADR-2: 포커스 확대 = 트리 유지 + splitPercentage 조정
- **결정**: 트리 구조를 재구성하지 않고 조상 노드의 splitPercentage만 재계산
- **근거**: 사용자가 드래그로 만든 배치를 보존

### ADR-3: 오토 모드 = 최선 근사치
- **결정**: 이진 트리에서 비인접 idle 세션의 true 균등 확대는 불가 → 비율 근사
- **근거**: 트리 재구성은 UX 훼손

### ADR-4: 서버 스키마 Breaking Change
- **결정**: `columns/rows/cellSizes` 필드를 제거하고 `mosaicTree` 필드로 교체
- **근거**: 레거시 호환 유지 비용 > 마이그레이션 비용

### ADR-5: localStorage 전용 (IndexedDB 미사용)
- **결정**: `beforeunload`에서 동기 저장 필요 → localStorage만 사용
- **근거**: IndexedDB는 비동기 API라 `beforeunload`에서 사용 불가
