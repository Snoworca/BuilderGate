## Phase 3: 컨텍스트 메뉴 + 세션 생명주기

### 목표
사용자가 세션을 생성/닫기/복사/붙여넣기할 수 있는 인터랙션 계층을 완성한다.

## 선행 조건
- Phase 2 완료 필요 (MosaicContainer, MosaicTile)

## 기존 코드 재사용 힌트
- `ContextMenu` + `useContextMenu`: 그대로 재사용. `items` prop에 `buildTerminalContextMenuItems()` 결과 전달
- `ConfirmModal`: 세션 닫기 확인 모달로 직접 재사용. `destructive` prop 활용
- `TerminalView`의 Ctrl+C/Ctrl+V 핸들러: `attachCustomKeyEventHandler` 내 기존 복사/붙여넣기 로직 참조. 컨텍스트 메뉴 복사/붙여넣기는 별도 경로이므로 충돌 없음

## 코드 중복 주의
- `closeTab` 포커스 이동: `useWorkspaceManager.closeTab()`의 `nextActiveTabId` 계산(right→left 인접)과 `useFocusHistory.getPrevious()`(마지막 사용 순서)는 **다른 전략**. 그리드 모드에서는 `useFocusHistory`만 사용하고, `useWorkspaceManager.closeTab()`의 포커스 로직은 무시 (서버 `activeTabId`는 별개로 동기화)
- `TerminalHandle`에 `hasSelection()/getSelection()/clearSelection()` 추가 시 기존 `write/clear/focus`만 있는 인터페이스를 확장. 기존 코드에서 `TerminalHandle`을 사용하는 곳에 영향 없음 (새 메서드는 옵셔널하게 호출)

## 언어+버전 주의사항
- `navigator.clipboard.readText()`: localhost에서 secure context 인정, 사용자 제스처(컨텍스트 메뉴 클릭) 내에서 호출
- `navigator.vibrate()`: 지원하지 않는 브라우저에서 `?.()` 옵셔널 체이닝 사용

## LLM 코딩 주의사항
- **Simplicity**: `buildTerminalContextMenuItems()`는 단순 배열 반환 함수. 클래스나 팩토리 패턴 불필요. `ContextMenuItem[]` 리터럴 반환
- **Surgical**: `TerminalView.tsx` 수정은 `useImperativeHandle` 블록에 3개 메서드 추가만. xterm 설정, ResizeObserver, 이벤트 핸들러 등 다른 부분 수정 금지

## 테스트 시나리오

### 정상 케이스
- Given: 그리드 모드, 세션 B에 텍스트 선택됨
  When: 우클릭 → 복사
  Then: 클립보드에 선택 텍스트 저장, 선택 해제

- Given: 세션 B 우클릭 → "새 세션 열기"
  When: 세션 B의 cwd = "/home/user/project"
  Then: 새 세션 cwd = "/home/user/project", 균등 재배치, 새 세션 포커스

- Given: 세션 D 우클릭 → "세션 닫기"
  When: 확인 모달 "예" 클릭
  Then: 세션 D 삭제, 마지막 사용 세션으로 포커스 이동

### 예외 케이스
- Given: 텍스트 미선택 상태
  When: 우클릭
  Then: 복사 항목 `disabled` 스타일, 클릭 무반응

- Given: 클립보드 권한 거부
  When: 붙여넣기 클릭
  Then: toast "클립보드 붙여넣기 실패" 표시, 에러 throw 없음

### 경계값 테스트
- Given: 세션 1개 남음
  When: 우클릭 → 닫기 → 확인
  Then: Mosaic 트리 null, 빈 화면, JS 에러 없음
