## Phase 2: 핵심 Mosaic 레이아웃 컴포넌트

### 목표
화면에 Mosaic이 실제로 보이고, 드래그 앤 드롭 위치 변경과 분할선 리사이즈가 동작하는 MVP를 구성한다. 레이아웃 저장/복원 기본 기능도 포함한다.

## 선행 조건
- Phase 1 완료 필요 (타입, 헬퍼 함수, API 변경)

## 기존 코드 재사용 힌트
- `TerminalContainer`: MosaicContainer의 `renderTerminal` prop 콜백에서 그대로 반환. sessionId/isVisible 인터페이스 변경 없음
- `MetadataRow`: MosaicTile 하단에 재사용
- `DisconnectedOverlay`: MosaicTile에서 tab.status === 'disconnected' 시 재사용
- `EmptyCell`: MosaicTile에서 tab이 없는 빈 슬롯에 재사용
- `TerminalView`의 ResizeObserver 패턴: Mosaic 분할선 드래그에도 자동 동작, 추가 코드 불필요
- `App.tsx`의 `wmRef.current` 콜백 ref 패턴: MosaicContainer의 콜백 props에도 동일 적용

## 코드 중복 주의
- `tab:added` WS 이벤트: `useWorkspaceManager`가 이미 `tabs` 배열에 추가 → MosaicContainer는 `tabs` prop 변경을 감지하여 트리 재빌드 (별도 WS 리스너 추가 금지)
- `GridCell.tsx`의 레이아웃 스타일: MosaicTile로 이전 시 동일 인라인 스타일 복붙 대신 Mosaic의 내장 스타일 활용

## 언어+버전 주의사항
- `react-mosaic-component` v6: `<Mosaic<string>>` 제네릭 사용 시 `<Mosaic<string> value={...} onChange={...} />` 구문
- `react-dnd`: DndProvider를 Mosaic이 자체 포함하므로 별도 설정 불필요
- Mosaic CSS: `react-mosaic-component/react-mosaic-component.css` import 필요 (또는 커스텀 CSS)

## LLM 코딩 주의사항
- **Simplicity**: MosaicToolbar는 Phase 2에서는 3개 버튼 UI만 구현. 실제 모드 전환 로직은 Phase 4에서 구현. Phase 2의 버튼 클릭은 `onLayoutModeChange` prop 호출만.
- **Surgical**: `App.tsx`에서 `<GridContainer>` 블록만 `<MosaicContainer>`로 교체. 다른 부분(TabBar, Sidebar 등) 수정 금지. `GridContainer.tsx`, `GridCell.tsx`는 삭제하지 말고 남겨둘 것 (Phase 5 완료 후 최종 삭제).

## 테스트 시나리오

### 정상 케이스
- Given: 4개 세션이 있는 워크스페이스
  When: 그리드 모드 활성화
  Then: React Mosaic이 4개 타일을 균등 배치, 각 터미널 정상 렌더링

- Given: 4개 타일 균등 배치
  When: 타일 A를 타일 B 옆으로 드래그
  Then: Mosaic 트리 재구성, 모든 터미널 정상

- Given: 4개 타일
  When: 분할선 드래그
  Then: 크기 변경 + xterm.js fit 정상

### 예외 케이스
- Given: localStorage에 손상된 JSON
  When: 페이지 로드
  Then: 균등 그리드 폴백, 콘솔 warn, UI 크래시 없음

### 경계값 테스트
- Given: 4세션, 분할선 극단 드래그
  When: 한 타일 최소화 시도
  Then: splitPercentage ≥ 8% (4세션 기준), 그 이하 불가
