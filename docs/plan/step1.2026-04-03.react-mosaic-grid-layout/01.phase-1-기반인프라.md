- `useTabManager.ts`의 `loadState/saveState` localStorage 패턴과 신규 `saveLayout/loadLayout`이 유사 — 공통 유틸로 추출하지 말 것 (과도한 추상화), 각자 독립 유지
- `closeTab` 시 포커스 이동 로직이 `useWorkspaceManager`에 이미 있음 — `useFocusHistory`는 그리드 모드 전용으로 분리, 서버 `activeTabId`와 독립적으로 운영

## 언어+버전 주의사항
- TypeScript strict mode 준수
- `MosaicNode` 타입은 `react-mosaic-component`에서 import (`import type { MosaicNode } from 'react-mosaic-component'`)
- 서버는 `MosaicNode` 타입을 직접 의존하지 않음 — `mosaicTree`를 `any | null`로 저장하고 클라이언트가 타입 검증

## LLM 코딩 주의사항
- **Simplicity**: `getMinPercentage`는 단순 if-else로 구현. 설정 파일이나 맵 구조 불필요
- **Surgical**: `useWorkspaceManager.addTab`에 `cwd?` 파라미터 하나만 추가. 다른 메서드 수정 금지. 기존 호출부는 선택적 파라미터이므로 변경 불필요

## 테스트 시나리오

### 정상 케이스
- Given: 빈 상태
  When: `buildEqualMosaicTree(['a','b','c','d'])`
  Then: 2단계 이진 트리, 모든 `splitPercentage` = 50, direction 교차(row→column)

- Given: `getMinPercentage(8)`
  When: 8개 세션
  Then: 반환값 = 5

### 예외 케이스
- Given: `buildEqualMosaicTree(['a'])`
  When: 1개 ID
  Then: 문자열 'a' 반환 (MosaicParent 아님)

- Given: `migrateGridLayout({workspaceId: 'ws1', columns: 2, rows: 2, tabOrder: ['a','b','c','d']})`
  When: 레거시 형식
  Then: `{workspaceId: 'ws1', mosaicTree: buildEqualMosaicTree(['a','b','c','d'])}` 반환

### 경계값 테스트
- Given: `removeFromMosaicTree(tree, 'nonexistent')`
  When: 존재하지 않는 tabId
  Then: 원본 트리 그대로 반환
