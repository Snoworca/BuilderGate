## Phase 5: 레이아웃 영속화 완성 + 통합 검증

### 목표
서버 재시작 후 세션 소멸 복원, 손상 데이터 폴백 등 엣지 케이스를 처리하고 전체 통합 검증을 수행한다. 이전 Phase의 `GridContainer.tsx`, `GridCell.tsx`를 최종 정리한다.

## 선행 조건
- Phase 1~4 모두 완료 필요

## 기존 코드 재사용 힌트
- `useWorkspaceManager.addTab()`: 세션 소멸 복원 시 빈 슬롯에 새 세션 생성할 때 호출
- `useMosaicLayout`의 `loadLayout()`: Phase 2에서 구현한 기본 복원 로직 위에 소멸 처리 계층 추가

## 코드 중복 주의
- `useTabManager.ts`의 localStorage 저장: 탭 모드에서만 사용. 그리드 모드의 `mosaic_layout_*` 키와 충돌 없음. 두 저장소 병존은 정상
- `restoreLayoutWithSessionRecovery`는 Phase 2의 `loadLayout`을 **확장**하는 것이지 대체가 아님. `loadLayout`을 호출한 후 세션 매핑 검증 추가

## 언어+버전 주의사항
- `restoreLayoutWithSessionRecovery`에서 새 세션 생성은 비동기(`addTab`). async/await로 순차 처리하거나 `Promise.all`로 병렬 처리
- `replaceLeafId`는 순수 함수 (트리 불변성 유지, spread operator로 새 객체)

## LLM 코딩 주의사항
- **Simplicity**: `replaceLeafId`는 단순 재귀 교체. 트리 순회 라이브러리 사용 금지
- **Surgical**: `GridContainer.tsx`/`GridCell.tsx` 삭제는 모든 import가 제거된 것을 `grep`으로 확인한 후에만 수행. 아직 참조가 있으면 삭제 금지

## 테스트 시나리오

### 정상 케이스
- Given: 3세션 레이아웃 저장, 서버 재시작(2세션 소멸)
  When: 페이지 재접속
  Then: 트리 구조(splitPercentage) 유지, 소멸 슬롯에 새 세션 자동 생성

- Given: 레이아웃 변경 후 페이지 새로고침
  When: 동일 브라우저
  Then: splitPercentage 값 동일 복원

### 예외 케이스
- Given: localStorage에 `{invalid json` 주입
  When: 페이지 접속
  Then: 균등 그리드 폴백, 콘솔 warn, React Error Boundary 미발동

- Given: localStorage에 `{"schemaVersion": 99, "tree": ...}` 저장
  When: 페이지 접속
  Then: 알 수 없는 버전 → 균등 폴백

### 경계값 테스트
- Given: 3세션 저장, 전부 소멸
  When: 재접속 (currentTabIds 모두 새 ID)
  Then: `buildEqualMosaicTree(currentTabIds)` 폴백

- Given: 8세션 포커스 모드 저장
  When: 재접속
  Then: 포커스 모드 복원, 타겟 세션 확대
