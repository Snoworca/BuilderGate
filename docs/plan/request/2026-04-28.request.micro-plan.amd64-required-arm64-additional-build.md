# amd64 필수 및 ARM64 추가 빌드 구현 요구사항

## 1. 원문 요구

사용자는 ARM64가 추가 지원 대상이며, Windows amd64와 Linux amd64 빌드는 필수로 지원해야 한다고 요구했다.

## 2. SRS 근거

- `docs/srs/buildergate.srs.md`의 `FR-RUN-012`: build output은 Windows amd64와 Linux amd64 배포본을 필수 지원해야 하며, ARM64 배포본은 추가 지원 대상으로 제공할 수 있다.
- `docs/srs/buildergate.srs.md`의 `FR-RUN-013`: 단일 기본 output `dist/bin`은 호환 경로로 유지하고, 대상별 배포본은 `dist/bin/<target>` 구조를 사용한다. 필수 대상은 `win-amd64`, `linux-amd64`이고 추가 ARM64 대상은 `win-arm64`, `linux-arm64`, `macos-arm64`이다.
- `docs/srs/buildergate.srs.md`의 `AC-015`: `npm run build`는 `dist/bin/win-amd64`, `dist/bin/linux-amd64`를 생성해야 하며, 각 배포본은 실행파일, stop utility, `server/dist/public/index.html`, config 파일, README, 대상별 아이콘 산출물을 포함해야 한다.
- `docs/archive/srs/step8.srs.buildergate-native-daemon-mode.2026-04-27.md`의 `AC-8-012`: Windows amd64와 Linux amd64 산출물은 `npm run build`의 필수 결과이고 ARM64 산출물은 추가 ARM64 빌드 실행 시 요구된다.
- `docs/archive/srs/step8.srs.buildergate-native-daemon-mode.2026-04-27.md`의 `AC-8-012A`: `npm run build:daemon-exe` 또는 `node tools/build-daemon-exe.js`의 단일 기본 산출물은 기존 호환 경로 `dist/bin`에 생성되어야 한다.

## 3. 구현 요구사항

1. `npm run build`는 Windows amd64와 Linux amd64 대상별 배포본을 필수 생성해야 한다.
2. ARM64 대상별 배포본은 추가 지원으로 유지하되 기본 `npm run build`의 필수 결과로 취급하지 않는다.
3. 기존 단일 기본 output인 `dist/bin`은 `npm run build:daemon-exe`와 `node tools/build-daemon-exe.js` 경로에서 유지한다.
4. 대상별 배포본은 `dist/bin/<target>` 구조를 사용한다.
5. 필수 amd64 배포본은 실행파일, stop utility, server runtime, frontend assets, config 파일, README, 대상별 아이콘 산출물, bundled Node runtime, PM2 부재를 검증해야 한다.

## 4. 제약

- 기존 native daemon, stop utility, config path, TOTP QR, PM2 제거 계약은 변경하지 않는다.
- UI 시각 요소, 레이아웃, WebSocket/API 계약은 변경하지 않는다.
- 모든 파일 읽기와 쓰기는 UTF-8로 수행한다.
- 구현 단계에서는 관련 회귀 테스트를 추가하고 최종 시점에 다시 실행해야 한다.
