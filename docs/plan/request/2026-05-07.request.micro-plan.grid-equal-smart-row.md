# Grid Equal Smart Row 요구사항 정리

## 원문 요약

Grid Mode의 Equal mode에서 4개 이상 tab을 배치할 때 기존처럼 항상 격자 baseline으로 들어가지 않는다. 화면이 가로형일 때 단일 행으로 나열한 각 session pane의 폭이 격자 baseline으로 나열한 각 pane의 높이보다 큰 경우에는 단일 행을 선택한다. 반대이면 격자 baseline을 유지한다.

## 기준 요구사항

- `FR-GRID-014`: 4개 이상에서는 가로형 단일 행 후보와 격자 후보를 함께 평가한다.
- `FR-GRID-015`: 가로형 화면에서 `singleRowPaneWidth > baselineGridPaneHeight`이면 단일 행을 선택하고, 같거나 작으면 격자 baseline을 선택한다.
- 세로형 또는 정사각형 화면에서는 4개 이상 단일 행 후보를 선택하지 않는다.
- container 치수를 측정할 수 없으면 격자 baseline을 선택한다.

## 구현 제약

- Grid UI의 시각 요소, 레이블, 아이콘, toolbar interaction은 변경하지 않는다.
- React Mosaic tree는 visible empty cell을 만들지 않는다.
- 기존 Equal drag/reorder/repair E2E 흐름을 깨지 않는다.
- 검증은 `https://localhost:2002` 대상 Playwright 설정을 사용하되, 개발 서버 환경 변수를 명시한다.
