## 요약

`97b5b11` 기반의 기존 shell-centered idle 모델 위에 Hermes foreground detector를 최소 범위로 백포트했다.

이번 변경의 핵심은 다음과 같다.

- `hermes` 실행 직후 입력 대기 상태를 `idle`로 유지
- Hermes bootstrap/status repaint는 `idle` 유지
- Hermes semantic output만 `running` 승격
- WSL/bash 부트스트랩은 기존 `BASH_ENV` 경로를 유지하여 rcfile 회귀를 방지
- zsh/macOS 계열에서도 직접 `hermes` 실행은 검출 가능하도록 입력 힌트 기반 경로 추가

## 구현 내용

### 1. Foreground detector 추가

- `server/src/services/ForegroundAppDetector.ts`
- `server/src/services/HermesForegroundDetector.ts`

내부 `ownership/activity` 모델과 Hermes 출력 분류기를 추가했다.

### 2. SessionManager 최소 백포트

- `server/src/services/SessionManager.ts`

아래 기능을 최소 범위로 이식했다.

- `derivedState` 기반 foreground 상태 추적
- `inputBuffer` / `lastSubmittedCommand` / `pendingForegroundAppHint`
- `hermes` submit 시 `foreground_app + waiting_input`
- Hermes output detector observation 반영
- stale CWD prompt refresh 무시
- PowerShell prompt redraw를 heuristic idle로 유지

### 3. 테스트 강화

- `server/src/test-runner.ts`

추가된 회귀 범위:

- WSL bootstrap args 유지
- Windows host에서 `BASH_ENV` 기반 bootstrap 유지
- rcfile bootstrap 미도입 보장
- bash Hermes submit / command echo / bootstrap idle
- bash Hermes semantic output only running
- bash Hermes launch failure fallback
- ordinary bash command의 기존 `running -> idle` 유지
- zsh Hermes submit idle
- stale CWD prompt refresh guard
- zsh prompt return watcher 경로
- PowerShell prompt redraw idle
