**구현 가이드:**

1. `types.ts`에 `DialogRole`과 optional props를 추가한다. 기존 호출자는 optional default를 사용하므로 컴파일 호환을 유지한다.
2. `dialogStack.ts`를 만들고 module-level stack과 subscription 기반 hook을 작성한다. 같은 `dialogId`가 중복되어도 instance token으로 push/remove를 구분한다.
3. `WindowDialog`에서 `useDialogStack(dialogId, mode === 'modal')`을 호출한다.
4. modal inert lifecycle effect와 topmost focus trap effect를 분리한다. inert effect는 `mode === 'modal'`과 mount/unmount만 기준으로 동작하고, `isTopmost` 변화로 cleanup되면 안 된다.
5. focus trap effect는 `mode === 'modal' && isTopmost`일 때만 `keydown`과 `focusin` handler를 등록한다. MessageBox가 열려 lower Dialog가 topmost가 아니게 되어도 lower Dialog의 inert effect는 유지되어야 한다.
6. previous focus restore는 각 modal close 시점에만 수행한다. MessageBox close는 lower Dialog 내부의 이전 focus로 돌려보내고, lower Dialog close는 MessageBox가 열리기 전 원래 focus로 돌려보내야 한다.
7. z-index는 결정적으로 계산한다. `layerZ = 5000 + layerIndex * 20`, `backdropZ = layerZ`, `dialogZ = layerZ + 1`로 고정한다. layer는 full viewport를 덮고, topmost backdrop은 pointer event를 소유해 아래 Dialog와 app click을 차단한다.
8. `showCloseButton === false`이면 title bar close button을 렌더하지 않는다.
9. `resizable === false`이면 `Rnd.enableResizing` 전체를 false로 둔다.
10. `persistGeometry === false`이면 `handleClose`에서 `writeDialogGeometry`를 호출하지 않는다.
11. `role` 기본값은 `dialog`, `ariaDescribedBy`는 존재할 때만 `aria-describedby`로 연결한다.
12. `windowDialogModel.ts`에 `createWindowDialogBehaviorModel`을 두고 role/default prop/z-index 계산을 순수 함수로 분리한다.
13. `dialogStack.test.ts`는 push/remove/topmost 계산을 순수 hook이 아닌 helper 함수 단위로 검증할 수 있게 helper를 export하거나 test-only pure function을 둔다.
14. `windowDialogContract.test.ts`는 Dialog component unit 성격으로 `createWindowDialogBehaviorModel`의 default role, close button 기본 표시, resize 기본 허용, geometry persistence 기본 허용, `layerZ/backdropZ/dialogZ` 계산을 검증한다.
15. 기존 `dialogGeometry.test.ts`는 geometry utility unit test로 유지한다.
16. `frontend/package.json`의 `test:unit:command-management`에 `tests/unit/dialogStack.test.ts`와 `tests/unit/windowDialogContract.test.ts`를 추가한다.
17. `frontend/package.json`의 `lint:command-management`에 `tests/unit/dialogStack.test.ts`와 `tests/unit/windowDialogContract.test.ts`를 추가한다.

**Rationale:**

현재 `WindowDialog`는 모든 modal instance가 document-level keydown/focusin listener를 갖는 구조다. MessageBox가 명령줄 관리 Dialog 위에 뜨면 아래 Dialog의 focus trap이 MessageBox focus를 다시 빼앗을 수 있다. topmost modal 판정은 이 충돌을 제거하고, 이후 modeless 확장에도 같은 stack 토대를 제공한다.

**함정 / 주의사항:**

- 기존 `CommandPresetDialog` 호출부의 props를 깨면 안 된다.
- lower modal의 inert 복원은 MessageBox 닫힘 뒤에도 정상이어야 한다.
- lower modal의 inert cleanup은 MessageBox open/close 때문에 실행되면 안 된다.
- topmost backdrop은 아래 Dialog의 title bar, X 버튼, form field click을 차단해야 한다.
- `Escape`는 계속 닫기 동작을 수행하지 않는다.
- MessageBox geometry는 저장하지 않는다.
- `dialogStack.ts`는 DOM API 없이 테스트 가능해야 한다.
- `windowDialogContract.test.ts`는 TSX/DOM render 없이 component behavior model을 검증한다. 실제 focus trap DOM 동작은 Playwright에서 검증한다.

**테스트 작성 지침:**

- 삭제 성공 toast는 기존 문구를 유지한다.

**테스트 작성 지침:**

### TASK-P3-002: Command Management E2E 삭제 확인 회귀 확장
