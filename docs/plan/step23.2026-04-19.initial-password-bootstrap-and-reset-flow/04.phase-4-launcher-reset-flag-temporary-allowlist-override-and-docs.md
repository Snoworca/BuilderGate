# Phase 4: Launcher Reset Flag, Temporary Allowlist Override, And Docs

## 목표

운영자가 `start.sh` / `start.bat` 실행 옵션만으로 비밀번호를 초기화하고, 필요 시 특정 원격 IP에서만 bootstrap을 허용할 수 있게 만든다.

## 대상 파일

- `tools/start-runtime.js`
- `start.sh`
- `start.bat`
- `README.md`
- 필요 시 신규 helper:
  - `tools/config-reset-password.js`
- `server/src/test-runner.ts` 또는 launcher smoke docs

## 구현 항목

1. `tools/start-runtime.js`에 `--reset-password`를 추가한다.
2. `--reset-password`는 `auth.password` path만 empty string으로 바꾼다.
3. `--bootstrap-allow-ip <ip>` 반복 입력 또는 comma-separated 입력을 지원한다.
4. launcher는 해당 값을 env `BUILDERGATE_BOOTSTRAP_ALLOWED_IPS`로 서버에 전달한다.
5. `--help` 출력에 새 옵션 설명을 추가한다.
6. README에 다음 운영 절차를 문서화한다.
   - 최초 실행
   - localhost bootstrap
   - 원격 bootstrap allowlist
   - password reset

## 코드 재사용/주의

- `resolvePort()` 등 기존 start-runtime flow는 유지한다.
- password reset은 pm2 start 전에 완료되어야 한다.
- 설정 파일 수정은 section-aware helper로 수행해 다른 설정을 손상시키지 않아야 한다.
- Windows/Unix wrapper는 공용 JS helper를 계속 호출하도록 유지한다.

## 테스트

- `node tools/start-runtime.js --help`에 새 옵션이 보이는지
- reset-password 수행 후 `auth.password`만 비워지는지
- reset-password 후 서버가 setup-required 모드로 시작하는지
- bootstrap allowlist flag가 env에 반영되고, 지정되지 않은 주소는 계속 차단되는지
- README의 실행 예시가 실제 동작과 일치하는지

## 완료 조건

- 운영자는 코드 수정 없이 실행 옵션만으로 비밀번호 재설정 흐름에 진입할 수 있다.
- 임시 remote bootstrap 허용이 가능하되 기본값은 안전한 localhost-only다.
