### 핵심 기술 결정 요약

- `config.json5` 최초 생성은 더 이상 `config.json5.example` 복제로 처리하지 않는다.
- 비밀번호 미설정 canonical state는 `auth.password: ""` 이다. `null` 입력은 허용하되 로드/저장 시 `""`로 정규화한다.
- 브라우저 초기 설정 화면은 `password missing && bootstrap requester allowed` 조건에서만 노출한다.
- bootstrap 허용 정책은 `localhost` 기본 허용 + `bootstrap.allowedIps` 설정 + 실행 옵션 기반 임시 allowlist override 조합으로 간다.
- 초기 비밀번호 설정 성공 시 새 비밀번호는 즉시 암호화 저장하고, JWT를 발급해 로그인 화면 재입력 없이 바로 앱으로 진입시킨다.
- `--reset-password`는 `config.json5`의 `auth.password`만 정확히 빈 값으로 만들고 다른 설정은 건드리지 않는다.

### 구현 흐름 요약

```text
No config.json5
  -> config.ts renders built-in JSON5 template
  -> auth.password = ""
  -> bootstrap.allowedIps = []
  -> server starts in setup-required mode

First browser access
  -> GET /api/auth/bootstrap-status
  -> if setupRequired && requesterAllowed
       show BootstrapPasswordForm
  -> POST /api/auth/bootstrap-password
       validate password + confirm
       persist encrypted password
       update AuthService runtime
       issue JWT

Operational reset
  -> start.sh/start.bat --reset-password
  -> tools/start-runtime.js empties only auth.password
  -> optional --bootstrap-allow-ip X.X.X.X
  -> server returns to setup-required mode
```

## 구현 방향 요약

1. `config.json5`는 코드에서 직접 생성하고, 최초 비밀번호는 빈 값으로 둔다.
2. 서버는 “로그인 실패”와 “초기 설정 필요”를 구분하는 bootstrap contract를 제공한다.
3. 초기 설정 API는 기본적으로 localhost만 허용하고, 필요 시 allowlist IP만 추가 허용한다.
4. 프런트는 unauthenticated 진입 시 bootstrap 상태를 먼저 조회하고, 필요하면 비밀번호/확인 폼을 노출한다.
5. 실행 옵션 `--reset-password`는 운영 중 비밀번호를 초기화해 브라우저 bootstrap 화면으로 복귀시키는 관리 도구 역할을 한다.

## 성공 기준

- `config.json5`가 없을 때 서버가 기본 비밀번호 없이 정상 부팅한다.
- unauthenticated 브라우저가 `setup required` 상태를 인식하고 초기 비밀번호 설정 화면을 띄운다.
- bootstrap API는 localhost 또는 명시된 allowlist IP에서만 성공한다.
- 초기 비밀번호 설정 성공 시 비밀번호는 암호화 저장되고, 사용자는 즉시 인증 상태로 진입한다.
- `--reset-password` 실행 후 다음 접속에서 초기 비밀번호 설정 화면이 다시 노출된다.
- `--bootstrap-allow-ip` 실행 옵션이 있으면 지정 IP에서 bootstrap 접근이 허용된다.
- 기존 일반 로그인, 2FA, settings 기반 비밀번호 변경 흐름은 회귀 없이 유지된다.
