## Phase 1: Server Foundation — WorkspaceService + Types + Config

### 목표
Workspace 시스템의 서버측 기반을 구축한다. 데이터 타입, 서비스, 설정을 정의하고, SessionManager에 필요한 인터페이스를 확장한다.

### 선행 조건
- 없음 (첫 번째 Phase)

### 기존 코드 재사용 힌트
- **ErrorCode enum**: `server/src/utils/errors.ts` — 7개 에러 코드 추가 (기존 37개에 append)
- **Config schema**: `server/src/schemas/config.schema.ts` — Zod `z.object()` 패턴으로 workspaceSchema 추가
- **Config loader**: `server/src/utils/config.ts` — `loadConfig()` 싱글턴이 자동으로 새 섹션 인식
- **SessionManager**: `server/src/services/SessionManager.ts` — `sessions: Map<string, SessionData>` 구조, hasSession/deleteMultipleSessions 메서드 추가
- **RuntimeConfigStore**: `server/src/services/RuntimeConfigStore.ts` — in-memory + 파일 영속화 패턴 참고 (WorkspaceService 설계에 활용)

### 구현 체크리스트
- [ ] Phase 1-1: `server/src/types/workspace.types.ts` 생성 — Workspace, WorkspaceTab, GridLayout, WorkspaceState, WorkspaceFile 타입 정의 `FR-7702, FR-7703`

- [ ] Phase 1-4: `server/src/utils/errors.ts`에 7개 ErrorCode 추가 — WORKSPACE_NOT_FOUND, TAB_NOT_FOUND, TAB_LIMIT_EXCEEDED, WORKSPACE_LIMIT_EXCEEDED, SESSION_LIMIT_EXCEEDED, LAST_WORKSPACE, INVALID_NAME. ErrorMessages/ErrorStatusCodes 테이블도 함께 추가 `섹션 5.3`

### 언어+버전 주의사항
- TypeScript 5.x (Node.js backend)
- `fs.promises` 사용 (콜백 API 금지)
- `crypto.randomUUID()` 사용 가능하지만, 기존 코드가 `uuid` 패키지 v4를 사용하므로 일관성 위해 `uuid` 패키지 유지
- Zod 스키마: `.optional().default({...})` 패턴으로 기본값 제공

### LLM 코딩 주의사항
- Simplicity: WorkspaceService는 단일 클래스로 구현. Repository/DAO 패턴 분리 금지. RuntimeConfigStore(`server/src/services/RuntimeConfigStore.ts`)와 동일 수준 복잡도 유지. Phase 1에서는 atomic write/backup/debounce 구현 금지 — 단순 writeFile만
- Surgical: SessionManager 기존 메서드 수정 금지, 새 메서드만 추가. ErrorCode enum은 기존 항목 위치 변경 금지, 끝에 append만

### 테스트 시나리오

#### 정상 케이스
- Given: WorkspaceService 초기화 (파일 없음)
  When: 서비스 시작
  Then: `server/data/` 디렉토리 자동 생성, 빈 Workspace 1개가 자동 생성, workspaces.json에 저장

- Given: 유효한 workspaces.json 존재
  When: 서비스 시작
  Then: 파일에서 상태 로드, in-memory 상태 반영

#### 예외 케이스
- Given: workspaces.json이 손상됨 (invalid JSON)
  When: 서비스 시작
  Then: 로그에 경고 출력, 빈 초기 상태로 시작 (Phase 7에서 .bak 복원 추가)

- Given: `server/data/` 디렉토리 미존재
  When: 서비스 시작
  Then: `fs.mkdir(path, { recursive: true })` 후 정상 진행

#### 경계값 테스트
- Given: config에 maxWorkspaces=10 설정
  When: 11번째 Workspace 생성 시도
  Then: WORKSPACE_LIMIT_EXCEEDED 에러 (AppError, statusCode 409)

- Given: config에 maxTotalSessions=32, 현재 32개 세션
  When: 새 탭 추가 시도 (addTab)
  Then: SESSION_LIMIT_EXCEEDED 에러 (AppError, statusCode 409)

- Given: SessionManager에 세션 ID "abc" 존재
  When: `hasSession("abc")` 호출
  Then: true 반환. `hasSession("nonexistent")` → false 반환
