## Phase 4: Workspace Sidebar + TabBar UI

### 목표
좌측 Workspace 사이드바(데스크톱 고정, 모바일 드로어)와 상단 TabBar(8색 팔레트, 드래그 재배치)를 구현한다. 기존 Session 기반 사이드바를 Workspace 기반으로 전환한다.

### 선행 조건
- Phase 3 완료 필요 (useWorkspaceManager 훅)

### 기존 코드 재사용 힌트
- **useDragReorder**: `frontend/src/hooks/useDragReorder.ts` — 탭, 사이드바 모두에서 롱프레스 드래그 재활용. `isLocked: () => false` 전달. ghostStyle 기반 반투명 고스트 렌더링
- **useContextMenu + ContextMenu**: `frontend/src/hooks/useContextMenu.ts`, `frontend/src/components/ContextMenu/ContextMenu.tsx` — 그대로 재사용. `ContextMenuItem` 배열 빌드 패턴은 `frontend/src/components/PaneSystem/PaneContextMenus.tsx`의 팩토리 함수 참고
- **useResponsive**: `frontend/src/hooks/useResponsive.ts` — 768px 브레이크포인트 감지 + 사이드바 open 상태
- **ConfirmModal**: `frontend/src/components/ConfirmModal.tsx` — Workspace 삭제 확인에 재사용

### 언어+버전 주의사항
- React 18 + 기존 CSS 파일 패턴 유지 (CSS Modules 아닌 일반 CSS import)
- 드로어 스와이프: `touchstart` + `touchmove` + `touchend` 이벤트, `{ passive: true }` 옵션
- ARIA: `role="listbox"`, `role="option"`, `aria-selected` (Workspace); `role="tablist"`, `role="tab"`, `aria-selected`, `aria-controls` (TabBar)
- 포커스 트랩: Tab/Shift+Tab 순환, 드로어 외부 요소 `tabindex=-1`

### LLM 코딩 주의사항
- Simplicity: 모바일 드로어는 별도 라이브러리 없이 CSS transform + touch 이벤트 핸들러로 구현. 제스처 라이브러리 (react-use-gesture 등) 금지. useDragReorder 훅을 반드시 재사용 — 드래그 이벤트 핸들러 직접 구현 금지
- Surgical: TerminalContainer, TerminalView 수정 금지. 기존 컴포넌트를 그대로 렌더링하는 구조. 기존 Sidebar/ 컴포넌트는 이 Phase에서 수정하지 않음 — Phase 8에서 정리

### 테스트 시나리오

#### 정상 케이스
- Given: Workspace 3개 존재
  When: 좌측 패널에서 Workspace 2 클릭
  Then: Workspace 2로 전환, 해당 탭/터미널 표시, activeTabId 터미널에 포커스

- Given: TabBar에 탭 3개 (Blue, Emerald, Amber)
  When: Emerald 탭 롱프레스(300ms) 드래그 → Amber 위치로 이동
  Then: 탭 순서 변경 (Blue, Amber, Emerald), 서버 PUT /api/workspaces/:id/tab-order 호출

#### 예외 케이스
- Given: Workspace 1개만 존재
  When: 컨텍스트 메뉴 열기
  Then: 삭제 항목 숨김

- Given: 탭 8개 존재
  When: `+` 버튼 hover
  Then: 버튼 비활성화 (disabled), "최대 8개" 툴팁 표시

#### 경계값 테스트
- Given: 모바일(320px 너비)
  When: 좌측 가장자리에서 우측으로 50px 스와이프
  Then: 드로어 열림, dim 오버레이 표시, 포커스 드로어 내부로 이동

- Given: 모바일 드로어 열림
  When: 오버레이 영역 터치
  Then: 드로어 닫힘, 포커스 원래 위치로 복귀

- Given: TabBar에서 ArrowRight 키 입력
  When: 현재 탭 이후 탭 존재
  Then: 다음 탭으로 포커스 이동 (aria-activedescendant 업데이트)
