## Phase 5: Grid Mode — 격자 배치 + 리사이즈 + SplitPane 제거

### 목표
Grid Mode를 구현하여 다수의 터미널을 격자로 동시 배치한다. Tab/Grid 전환 토글을 추가하고, 기존 SplitPane 시스템을 완전히 제거한다.

### 선행 조건
- Phase 4 완료 필요 (TabBar, Workspace UI)

### 기존 코드 재사용 힌트
- **useDragReorder**: `frontend/src/hooks/useDragReorder.ts` — 그리드 셀 간 드래그 재배치에 재활용. `isLocked: () => false`
- **TerminalView ResizeObserver**: `frontend/src/components/Terminal/TerminalView.tsx` — `ResizeObserver` → `fitAddon.fit()` 자동 호출. 그리드 셀 리사이즈 시 자동 반영 (추가 코드 불필요)
- **PaneResizer 드래그 패턴**: `frontend/src/components/PaneSystem/PaneResizer.tsx` — `setPointerCapture` + pointer 이벤트 드래그 패턴 참고 (제거 대상이지만 GridResizer 구현 시 동일 패턴 사용)

### 언어+버전 주의사항
- CSS Grid: `grid-template-columns`, `grid-template-rows`로 비율 기반 배치. `fr` 단위 또는 percentage 사용
- `setPointerCapture(e.pointerId)` 기반 드래그 (touch + mouse 통합)
- `ResizeObserver`로 그리드 컨테이너 크기 감시 (FR-7406 반응형)
- `requestAnimationFrame` 사용하여 리사이즈 중 60fps 유지

### LLM 코딩 주의사항
- Simplicity: CSS Grid 네이티브 API만 사용. react-grid-layout, react-resizable 등 라이브러리 도입 금지. 리사이즈는 pointer 이벤트 + CSS custom properties(`--col-1-width` 등)로 구현. useDragReorder 반드시 재활용 — 드래그 로직 직접 구현 금지
- Surgical: TerminalView, TerminalContainer 수정 금지. 그리드 셀이 TerminalContainer를 감싸는 구조. SplitPane 제거 시 위 명시된 파일만 삭제 — 다른 파일은 import 정리만

### 테스트 시나리오

#### 정상 케이스
- Given: 터미널 4개
  When: Grid Mode 전환
  Then: 2×2 격자로 자동 배치, 각 셀에 터미널 렌더링, xterm.js fit() 호출

- Given: Grid Mode에서 열 경계 드래그
  When: 첫 번째 열 너비 60%로 드래그
  Then: 같은 열의 모든 행 너비 조절, 60fps 렌더링, 드래그 종료 시 PUT /api/workspaces/:id/grid 호출

#### 예외 케이스
- Given: 셀 크기가 120×80px 미만으로 축소 시도
  When: 리사이즈 드래그
  Then: 최소 크기에서 클램핑, 더 이상 축소 불가

- Given: 커스텀 cellSizes가 있는 상태
  When: 탭 1개 추가 (격자 재계산 필요)
  Then: cellSizes null로 리셋, 자동 균등 배분

#### 경계값 테스트
- Given: 터미널 1개
  When: Grid Mode 전환
  Then: 1×1 (전체 화면), 리사이저 핸들 없음

- Given: 터미널 3개
  When: Grid Mode
  Then: 2×2 격자, 4번째 셀은 EmptyCell (+ 아이콘 표시)

- Given: portrait 비율 디스플레이 (containerWidth/containerHeight < 1.0)
  When: 터미널 4개, Grid Mode
  Then: cols/rows 전치 (2열×2행 → 2행×2열)

- Given: 그리드 셀 A를 드래그하여 원래 위치에 드롭
  When: 드롭
  Then: no-op (tabOrder 변경 없음, 서버 호출 없음)
