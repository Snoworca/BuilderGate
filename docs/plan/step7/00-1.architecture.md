## 데이터 흐름

```
1. 클라이언트 접속
   GET /api/workspaces → WorkspaceState (전체 상태)
   GET /api/workspaces/stream → SSE 연결 (workspace 이벤트)

2. Workspace 조작
   POST/PATCH/DELETE /api/workspaces/* → WorkspaceService (in-memory 변경)
   → SSE broadcast (다른 클라이언트에 알림)
   → debounce 5초 후 workspaces.json flush

3. 터미널 입출력
   GET /api/sessions/:id/stream → 기존 PTY SSE (변경 없음)
   POST /api/sessions/:id/input → 기존 PTY 입력 (변경 없음)

4. 서버 재시작
   workspaces.json 로드 → sessionId 유효성 체크 → disconnected 표시
```

## 상태 관리 구조

```
useWorkspaceManager(token)
  ├── workspaces: Workspace[]
  ├── tabs: WorkspaceTabRuntime[]    (status, cwd 포함)
  ├── gridLayouts: GridLayout[]
  ├── activeWorkspaceId: string | null  (localStorage)
  │
  ├── CRUD: createWorkspace, deleteWorkspace, renameWorkspace, ...
  ├── Tab: addTab, closeTab, renameTab, reorderTabs, restartTab
  ├── Grid: updateGridLayout
  └── View: setViewMode, setActiveTab

useWorkspaceSSE(token)
  └── workspace:*, tab:*, grid:* 이벤트 → useWorkspaceManager 상태 업데이트

useSSE(sessionId)         ← 기존 훅 재사용 (PTY output/status)
useDragReorder(...)       ← 기존 훅 재사용 (탭/그리드/사이드바)
useContextMenu()          ← 기존 훅 재사용
```
