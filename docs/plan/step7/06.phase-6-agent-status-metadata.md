## Phase 6: Agent Status Display + Metadata Bar

### 목표
에이전트 활성 상태의 시각적 표시(breathing 효과)와 Workspace 하단 메타데이터 바를 구현한다.

### 선행 조건
- Phase 5 완료 필요 (Grid Mode + Tab Mode 모두 동작)

### 기존 코드 재사용 힌트
- **StatusIndicator**: `frontend/src/components/Sidebar/StatusIndicator.tsx` — 기존 running/idle CSS 클래스 패턴 참고. breathing 효과 CSS로 대체
- **TerminalBadges**: `frontend/src/components/Sidebar/TerminalBadges.tsx` — `badge-running` CSS 클래스 패턴 참고
- **CWD 데이터 경로**: 서버 `SessionManager.ts`가 5초 간격으로 PTY CWD를 추적 → 기존 `GET /api/sessions/:id` 응답에 `cwd` 필드 포함. useWorkspaceManager가 각 탭의 sessionId로 CWD를 폴링하여 `WorkspaceTabRuntime.cwd`에 저장. 별도 useCwd 훅 생성 불필요 — useWorkspaceManager 내부에 5초 setInterval CWD 폴링 로직을 인라인으로 구현

### 구현 체크리스트
- [ ] Phase 6-1: Breathing 효과 CSS — `@keyframes breathing` (opacity 0.4↔1.0, 2초 주기, ease-in-out), `will-change: opacity`, `@media (prefers-reduced-motion: reduce)` 시 정적 초록 테두리만 표시 `FR-7501, FR-7502`
- [ ] Phase 6-2: Tab Mode 터미널에 breathing 효과 적용 — `running` 상태 시 2px solid `#22c55e` 테두리 + `inset 0 0 20px rgba(34,197,94,0.15)` 내부 그림자 + breathing 애니메이션 `FR-7501`
- [ ] Phase 6-3: Grid Mode 셀에 breathing 효과 적용 — 각 셀 독립적으로 `running`/`idle`/`disconnected` 상태 반영 `FR-7501`
- [ ] Phase 6-4: Workspace 사이드바 활성 세션 뱃지 — `running` 카운트, 초록 배경 + 흰색 숫자, 0이면 숨김, `aria-live="polite"` 영역에 상태 변경 알림 `FR-7503, NFR-704`
- [ ] Phase 6-5: `frontend/src/components/MetadataBar/MetadataBar.tsx` — Workspace 하단, 탭별 한 줄, 짝수/홀수 행 명암 (`#2a2a2a`/`#2e2e2e`), 탭 0개 시 숨김 `FR-7601, FR-7602`
- [ ] Phase 6-6: MetadataBar 행 렌더링 — 좌측 4px 수직 컬러 바 (팔레트 색상) + 8px 간격 + 흰색 세션명 (max 20자 truncate + ellipsis). 우측 경과시간 `MM:SS` (setInterval 1초) + 복사 아이콘 (16px). CWD는 useWorkspaceManager의 `WorkspaceTabRuntime.cwd`에서 참조 `FR-7603, FR-7604, FR-7605`
- [ ] Phase 6-7: CWD 복사 기능 — `navigator.clipboard.writeText(cwd)`, 성공 시 체크마크 아이콘 1.5초 표시 후 복원. 경과시간 60분 초과 시 `HH:MM:SS` 형식 전환 `FR-7604, FR-7605`

### 언어+버전 주의사항
- CSS `@keyframes` + `animation` 속성 사용 (JS 타이머가 아닌 CSS 애니메이션으로 GPU 가속)
- `navigator.clipboard.writeText()` — HTTPS 환경 필수 (이미 HTTPS 구성됨)
- `setInterval` cleanup: `useEffect` return에서 `clearInterval`
- `@media (prefers-reduced-motion: reduce)` — CSS 미디어 쿼리로 애니메이션 비활성화

### LLM 코딩 주의사항
- Simplicity: Breathing 효과는 순수 CSS `@keyframes`로 구현. JS 타이머/requestAnimationFrame 사용 금지 (GPU 가속을 위해 CSS animation이 적합). MetadataBar 경과시간만 JS setInterval 사용
- Surgical: 기존 `StatusIndicator.tsx`, `TerminalBadges.tsx` 파일 내부 수정 금지 — Phase 8에서 정리. 이 Phase에서는 새 컴포넌트(MetadataBar)를 추가하고, Workspace UI에서 새 컴포넌트를 참조하도록 연결만

### 테스트 시나리오

#### 정상 케이스
- Given: 터미널에서 명령 실행 중 (running 상태)
  When: Tab Mode에서 해당 터미널 확인
  Then: 초록 2px 테두리 + inset 그림자 + opacity 0.4↔1.0 breathing (2초 주기)

- Given: 명령 완료 후 idleDelayMs 경과 (idle 상태)
  When: 터미널 확인
  Then: 테두리/그림자/애니메이션 모두 제거

- Given: 탭 3개 (Blue running, Emerald idle, Amber running)
  When: Metadata Bar 확인
  Then: 3줄 표시, Blue/Amber 행에 경과시간 갱신, 복사 아이콘 표시

#### 예외 케이스
- Given: `prefers-reduced-motion: reduce` 설정
  When: running 상태 터미널
  Then: 정적 초록 테두리만 표시, 애니메이션 없음

- Given: CWD가 아직 로드되지 않음 (초기 폴링 전)
  When: 복사 아이콘 클릭
  Then: 빈 문자열 복사 방지 — 버튼 disabled 또는 "경로 로드 중" 표시

#### 경계값 테스트
- Given: 탭 0개
  When: Metadata Bar 영역 확인
  Then: 메타데이터 바 숨김 (렌더링 없음)

- Given: 세션 경과 시간 59분 59초
  When: 1초 후
  Then: 형식 `59:59` → `01:00:00` 전환 (MM:SS → HH:MM:SS)

- Given: Grid Mode에서 8개 셀 모두 running 상태
  When: 동시 breathing 애니메이션 8개
  Then: CSS animation만 사용하므로 GPU 처리, 메인 스레드 블로킹 없음, 60fps 유지
