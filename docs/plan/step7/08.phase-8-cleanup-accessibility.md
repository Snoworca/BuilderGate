### 언어+버전 주의사항
- 파일 삭제 시 관련 import 정리 필수 — `tsc --noEmit`으로 누락 확인
- `document.addEventListener('keydown', handler)` — `useEffect` cleanup 필수
- exponential backoff: `Math.min(baseDelay * Math.pow(2, attempt), maxDelay)`
- 배너 상태: `useState<'connected'|'disconnected'|'reconnecting'>('connected')`

### LLM 코딩 주의사항
- Simplicity: 네트워크 단절 배너는 `useState` 1개 + 조건부 렌더링. 별도 상태 관리 라이브러리/Context 금지. exponential backoff은 `setTimeout` + 카운터 변수로 구현
- Surgical: 레거시 코드 제거 시 위 체크리스트에 명시된 파일만 삭제. TypeScript 컴파일러가 누락된 import를 감지하므로, 삭제 후 `tsc --noEmit`으로 관련 참조 정리. 관련 없는 파일 수정 금지

### 테스트 시나리오

#### 정상 케이스
- Given: 모든 Phase 완료, 레거시 코드 제거 완료
  When: `cd server && npx tsc --noEmit` + `cd frontend && npx tsc --noEmit`
  Then: 에러 0개

- Given: Ctrl+3 키 입력
  When: 현재 Workspace에 탭 3개 이상
  Then: 3번째 탭으로 전환, 해당 터미널 포커스

- Given: Ctrl+Shift+2 키 입력
  When: Workspace 2개 이상
  Then: 2번째 Workspace로 전환

#### 예외 케이스
- Given: SSE 연결 끊김
  When: 네트워크 복구
  Then: exponential backoff로 재연결 (1초→2초→4초), 상단 "연결 끊김" → "재연결 중..." → 배너 제거, GET /api/workspaces 전체 동기화

- Given: Ctrl+Shift+5 키 입력
  When: Workspace가 3개만 존재
  Then: 아무 동작 없음 (범위 초과 무시)

#### 경계값 테스트
- Given: 레거시 localStorage에 `tab_state_abc`, `child_session_ids` 키 존재
  When: 새 버전 앱 로드
  Then: 레거시 키 무시, 서버에서 상태 로드

- Given: SSE 재연결 시도 10회 실패 (30초 max 도달)
  When: 11번째 재연결 시도
  Then: 30초 간격 유지 (max 초과 안 함), 배너 "재연결 중..." 계속 표시

- Given: running 상태 변경 발생
  When: `aria-live="polite"` 영역 확인
  Then: 스크린 리더에 "터미널 N 활성" 읽힘 (접근성 트리에서 확인)
