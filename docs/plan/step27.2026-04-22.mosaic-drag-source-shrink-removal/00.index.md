# Mosaic Drag Source Shrink Removal Plan

Date: 2026-04-22
Scope: `react-mosaic-component` drag-start source shrink 제거
Mode: implementation plan

## Goal

창 이동 drag를 시작할 때 원본 pane의 geometry가 즉시 접히는 현상을 제거한다.

## Requirements

- `IMPL-001` drag start 시 source pane width/height는 유지되어야 한다.
- `IMPL-002` drop 전까지 source hide를 위한 live tree mutation이 없어야 한다.
- `IMPL-003` split mode와 reorder mode 모두에서 shrink가 제거되어야 한다.
- `IMPL-004` invalid drop/reset 경로는 explicit restore 또는 no-op으로 재정렬되어야 한다.
- `IMPL-005` drag preview와 full-cell/split guide는 계속 보여야 한다.
- `IMPL-006` geometry stability 회귀 테스트가 추가되어야 한다.
- `IMPL-007` right-click, non-primary pointer, handle-only 제약은 유지되어야 한다.

## Recommended Execution Order

1. vendor drag-start hide 경로를 제거한다.
2. split/reorder reset 경로를 source-hide 비가정 구조로 맞춘다.
3. 필요 시 source visual state만 비파괴적으로 추가한다.
4. geometry stability와 기존 negative-path 회귀를 함께 검증한다.

## Success Criteria

- drag 시작 전후 source tile rect가 실질적으로 동일하다.
- invalid drop에서도 source pane이 접히지 않는다.
- split mode와 reorder mode 모두에서 shrink가 사라진다.
- 기존 reorder/drop semantics와 context menu regression이 없다.
