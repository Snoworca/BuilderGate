# Architecture

## Problem

현재 vendor drag-start 경로는 source node를 즉시 `hide()` 처리하여 live mosaic tree를 변경한다.

핵심 경로:

- `ConnectedInternalMosaicWindow.item()`
- `mosaicActions.hide(props.path)`
- `createHideUpdate()`

이 구조는 drop 전 preview 단계에서 layout geometry를 흔든다.

## Target Architecture

### 1. Non-destructive drag start

drag start는 더 이상 tree mutation을 만들지 않는다.

- source tile은 제자리에 남는다
- drag preview는 기존처럼 작동한다
- target guide는 기존처럼 작동한다

### 2. Commit-only tree mutation

실제 tree 변경은 drop commit 시점에만 수행한다.

- split mode: `createDragToUpdates(...)`
- reorder mode: `createReorderedTreePreservingShape(...)`

### 3. Unified invalid-drop handling

invalid drop/reset은 hide 가정 기반 split reset이 아니라 명시적 restore/no-op 구조로 통일한다.

### 4. Optional source visual state

필요하면 source tile에만 비파괴적 visual state를 부여한다.

- opacity
- outline
- drag-source class

이는 geometry mutation과 분리된 순수 시각 상태여야 한다.

## Boundary

- vendor patch: drag-start, end/reset, preview/target lifecycle
- app layer: mode gate 유지, 추가 visual state 연결 여부
- tests: geometry stability + 기존 negative-path 회귀
