# Tech Decisions

## ADR-2701

Drag-start source hide는 제거한다.

Reason:

- 현재 shrink의 직접 원인이다.
- 기능 요구사항이 아니다.
- reorder뿐 아니라 split 경로에도 공통으로 문제를 만든다.

## ADR-2702

Invalid drop/reset은 hide 기반 splitPercentage cleanup이 아니라 explicit restore/no-op로 통일한다.

Reason:

- drag-start tree mutation을 제거하면 예전 reset branch의 전제가 깨진다.
- branch별 예외를 줄이고 reasoning cost를 낮춘다.

## ADR-2703

비주얼 피드백이 필요하면 geometry-preserving visual state만 허용한다.

Reason:

- 사용자 피드백은 필요할 수 있지만 layout collapse는 금지다.

## ADR-2704

Desktop Chrome Playwright에 drag-start geometry stability 회귀를 추가한다.

Reason:

- 이번 변경의 핵심은 drop 결과가 아니라 drag 시작 순간의 비파괴성이다.
