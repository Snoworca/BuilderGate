# Phase 3: Header Metadata Alignment And Final Verification

## Objective

Resolve the remaining strict-mode dead-value errors and close the build with a focused UI verification pass.

## Target Files

- `frontend/src/components/Header/Header.tsx`
- `frontend/src/components/Header/Header.css`
- `frontend/src/components/MetadataBar/MetadataRow.tsx`
- `frontend/src/App.tsx`
- `frontend/src/components/Grid/MosaicTile.tsx`

## Root Cause

- `Header.tsx` computes `displayCwd` but never renders it.
- `MetadataRow.tsx` accepts `isOdd` but never uses it.
- Both issues are harmless at runtime, but strict compiler settings convert them into hard build failures.

## Recommended Decision

Keep `activeCwd` and render it in `Header`.

Reason:
- `App.tsx` still passes the value.
- `Header.tsx` already truncates it.
- `Header.css` already contains dedicated cwd styles.

This is a stronger signal of unfinished UI wiring than dead API.

## Planned Changes

1. Render `displayCwd` in `Header.tsx` using the existing `.header-cwd` and `.header-cwd-path` structure.
2. Preserve mobile/desktop behavior, but ensure the cwd block truncates cleanly.
3. Remove `isOdd` from `MetadataRow.tsx`.
4. Remove `isOdd={false}` from all call sites:
   - `App.tsx`
   - `MosaicTile.tsx`

## Alternate Fallback

If product direction explicitly rejects header cwd display, remove `activeCwd`, `displayCwd`, and unused cwd-specific header markup instead. Do not keep the prop as dormant API.

## Verification

1. `cd frontend && npm run build`
2. Manual smoke test:
   - switch tabs and confirm header cwd updates
   - switch workspaces and confirm header cwd updates
   - verify metadata row still renders in tab mode
   - verify metadata row still renders in grid mode

## Final Gate

When phases 1 through 3 are complete, run the checklist in [verification/final-checklist.md](./verification/final-checklist.md) and do not close the task until all items pass.
