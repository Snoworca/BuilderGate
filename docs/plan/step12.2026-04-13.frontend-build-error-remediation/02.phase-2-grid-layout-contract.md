# Phase 2: Grid Layout Contract

## Objective

Align the `useMosaicLayout` setter contract with its actual React state behavior and remove dead props from `MosaicTile`.

## Target Files

- `frontend/src/hooks/useMosaicLayout.ts`
- `frontend/src/components/Grid/MosaicContainer.tsx`
- `frontend/src/components/Grid/MosaicTile.tsx`

## Root Cause

`MosaicContainer` uses:

```ts
setMosaicTree(prev => prev ? applyMultiFocusApprox(...) : prev)
```

This is valid because the hook internally stores `mosaicTree` with `useState`. The build fails only because the exported type in `UseMosaicLayoutReturn` narrows `setMosaicTree` to `(tree) => void` instead of a normal React setter.

Separately, `MosaicTile` still exposes `onAdd` and `availableShells`, but the component does not use either prop anymore.

## Planned Changes

1. Widen the public setter type in `useMosaicLayout.ts`.
   Recommended contract:

```ts
setMosaicTree: React.Dispatch<React.SetStateAction<MosaicNode<string> | null>>;
```

2. Leave the functional updater call in `MosaicContainer.tsx` intact.
   That preserves the current timer-driven auto-layout logic and avoids inventing a new local workaround.

3. Remove dead props from `MosaicTile.tsx`.
   - delete `onAdd`
   - delete `availableShells`

4. Remove matching prop passing from `MosaicContainer.tsx`.

## Verification

1. `cd frontend && npm run build`
2. Confirm no remaining compiler errors in:
   - `MosaicContainer.tsx`
   - `MosaicTile.tsx`
3. Manual smoke test in grid mode:
   - equal mode
   - focus mode
   - auto mode after status transitions
   - close/restart actions from tile and context menu

## Risks

- Low: setter typing change does not alter runtime behavior.
- Low: dead-prop removal is local and behavior-free.
- Medium-low: grid auto-layout should be checked once because the affected line sits inside a delayed effect.

## Deferred Hardening

Do not mix the current build fix with a broad refactor of `MosaicContainer` effect dependencies. That is a separate stability pass and should stay out of this build-restoration scope.
