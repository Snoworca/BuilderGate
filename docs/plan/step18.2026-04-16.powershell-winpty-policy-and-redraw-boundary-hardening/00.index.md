# Step 18: PowerShell WinPTY Policy And Redraw Boundary Hardening

## 목표

PowerShell에서 `A -> Enter`를 매우 빠르게 반복할 때 발생하는 줄 겹침 문제를, 현재 구조를 해치지 않으면서 근본적으로 해결한다.

유지해야 할 구조:

- Workspace / Tab 구조
- `TerminalRuntimeLayer`
- server-authoritative snapshot
- 단일 WebSocket 채널

## 핵심 결론

이 설계는 문제의 1차 수리 레이어를 다음으로 본다.

- `PowerShell/PSReadLine on ConPTY` redraw 생성/소비 경계

이 설계는 다음 방향을 채택한다.

1. PowerShell 세션은 Windows에서 **전용 backend override**로 제어한다.
2. Step18 1차 릴리스는 기존 설치를 silent cutover 하지 않고 `inherit`를 기본으로 둔다.
3. 검증 완료 후에만 PowerShell 기본 backend를 `winpty`로 전환한다.
4. `ConPTY`는 나머지 셸과 기존 구조에 그대로 유지한다.
5. snapshot apply / ACK / restore drain 및 interactive-ready 경계는 2차 hardening으로 정리한다.

즉, **global architecture를 바꾸지 않고 PowerShell 세션의 producer 조합만 바꾸는 방식**을 중심으로 설계한다.

## 성공 기준

- PowerShell prompt에서 rapid `A + Enter` 반복 시 출력이 줄 단위로 정상 적층된다.
- `tab <-> grid`, workspace switch, reconnect 구조는 그대로 유지된다.
- 기존 non-PowerShell 세션 동작은 깨지지 않는다.
- `useConpty`와 PowerShell override의 운영자-facing 의미가 충돌하지 않는다.
- `winpty` 미가용/생성 실패 시 동작이 결정적이다.
- 브라우저/서버 debug capture로 문제 레이어가 재발하지 않음을 입증할 수 있다.

- server-authoritative snapshot 모델 제거
