# Phase 3: Snapshot Ack Boundary Hardening

## Objective

snapshot apply / restore drain / ACK 경계를 일치시키고, geometry-ready / prompt-ready / input-unblock 순서를 강제해 interactive-ready race를 닫는다.

## Design

### ACK timing change

현재:

- `replaceWithSnapshot()` 완료 직후 ACK

목표:

- restore pending drain 완료 후 ACK

### interactive-ready barrier

입력 허용은 다음 순서 뒤로 미룬다.

1. first fit/resize emitted
2. PowerShell prompt bootstrap installed
3. snapshot replace and buffered drain completed
4. input unblocked

### ownership and signal

- 서버는 `session bootstrap ready` 상태를 authoritative하게 결정한다
- WS `subscribed`는 `ready`를 포함한다
- subscribe 이후 준비되면 `session:ready`를 보낸다
- 프런트는 local readiness를 계산해 helper textarea를 연다

### early input policy

- barrier closed 동안 early input은 queue하지 않는다
- 드롭/차단 정책을 유지해 redraw burst 재주입을 막는다
- open / refresh / reconnect / workspace-restore 모두 동일 정책 적용

### restore semantics

다음 조건이 모두 끝난 뒤에만 authority ready로 간주한다.

1. snapshot replace 완료
2. buffered output drain 완료
3. `restorePendingRef=false`

## Implementation Checklist

- `TerminalView.replaceWithSnapshot()` completion semantics 재정의
- `TerminalContainer`의 ACK 전송 시점 조정
- geometry-ready + prompt-ready + input-unblock contract 문서화
- `subscribed.ready` / `session:ready` 계약 추가
- open / refresh / reconnect / workspace-restore 직후 입력 경계 정리
- duplicate/stale snapshot 방어 유지
- restore 중 입력 허용/금지 상태 재검토

## Test

- restore pending 동안 ACK 미전송
- drain 완료 후 ACK 전송
- stale snapshot ignore 유지
- rapid input 중 snapshot apply가 authority 중간 프레임을 남기지 않음
- 세션 생성 직후 / 재연결 직후 즉시 입력에서도 barrier 유지
