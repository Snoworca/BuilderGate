# Phase 2: Session Contract Truthfulness

## Objective

세션이 실제로 사용한 PTY backend와 `windowsPty` 메타데이터를 완전히 일치시킨다.

## Design

### per-session metadata

현재 전역 `runtimePtyConfig.useConpty` 기반 계산을 제거하고, 세션 생성 시 확정된 backend를 `SessionData`에 저장한다.

저장값:

- `windowsPty.backend`
- `windowsPty.buildNumber`

### headless/snapshot propagation

아래 경로 모두 동일한 per-session backend를 사용한다.

- `SessionData.windowsPty`
- `createHeadlessTerminalState(... windowsPty ...)`
- `getScreenSnapshot()`
- `WsRouter screen-snapshot`

## Implementation Checklist

- `getWindowsPtyInfo()`를 session-aware 버전으로 교체
- `createHeadlessTerminalState()` 호출부는 세션별 backend 사용
- snapshot cached / fresh path 모두 per-session metadata 유지
- 테스트에서 PowerShell-winpty 세션의 snapshot metadata 확인

## Test

- PowerShell winpty 세션의 snapshot에 `backend: 'winpty'`
- PowerShell conpty 세션의 snapshot에 `backend: 'conpty'`
- PowerShell `inherit + pty.useConpty=true` 세션의 snapshot에 `backend: 'conpty'`
- non-PowerShell ConPTY 세션의 snapshot에 `backend: 'conpty'`
- frontend `TerminalView`가 backend truthfulness를 유지
