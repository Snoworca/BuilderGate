## Key Architectural Decisions Summary

| ADR | 결정 | 근거 |
|-----|------|------|
| ADR-001 | CSS Media Query + React Hook 혼합 | 레이아웃은 CSS, 동적 로직은 JS |
| ADR-002 | CSS Grid 2-컬럼 | 정확한 Mdir 레이아웃 재현 |
| ADR-003 | react-markdown + remark-gfm | React 생태계 최적, GFM 지원 |
| ADR-004 | highlight.js | rehype 직접 통합, 선택적 언어 로딩 |
| ADR-005 | Touch Events API (라이브러리 없음) | 핀치줌만 필요, 최소 의존성 |
| ADR-006 | Custom Context Menu | 메뉴 항목 4개로 단순, 라이브러리 과잉 |
| ADR-007 | 독립 FileService 클래스 | 낮은 결합도, 테스트 용이 |
| ADR-008 | OS별 CWD 조회 + 폴백 | Windows 주 대상, 안정적 폴백 |
| ADR-009 | path.resolve + path.relative 검증 | OWASP 준수, 심볼릭 링크 대응 |
| ADR-010 | 인메모리 저장 | 기존 SessionManager 패턴 유지 |

## How to Use This Plan

### For Implementation (코딩 에이전트)

1. **Phase 순서대로** 문서를 읽는다
2. 각 Phase의 **구현 체크리스트**를 따라 코딩한다
3. **테스트 시나리오**의 Given-When-Then을 테스트 코드로 작성한다
4. Phase 완료 후 **verification** 문서를 채운다
5. 모든 Phase 완료 후 **integration-test-guide** 실행
6. **final-validation** 보고서 작성

### For Review (코드 리뷰어)

1. **architecture.md**로 전체 구조 파악
2. 각 Phase **체크리스트** 대비 구현 완료 여부 확인
3. **verification** 문서의 테스트 결과/품질 평가 검토
4. **integration-test-guide**의 E2E 시나리오 통과 여부 확인
