## 🎯 목표 (Objective)

파일 브라우저에서 선택한 파일의 내용을 읽고 표시하는 Viewer 탭 구현:
- **Markdown 뷰어**: CommonMark + GFM + Mermaid 다이어그램 렌더링
- **코드 뷰어**: 16개 언어 문법 강조 표시 (Syntax Highlighting)
- **파일 읽기 API**: 경로 보안 검증, 바이너리 파일 감지, 크기 제한
- **자동 탭 전환**: MdirPanel에서 파일 선택 시 Viewer 탭으로 자동 이동

### 커버하는 요구사항
- **FR-2301**: Markdown 뷰어 (react-markdown + GFM)
- **FR-2302**: Mermaid 다이어그램 렌더링
- **FR-2303**: 코드 뷰어 (highlight.js, 16개 언어)
- **FR-2304**: 파일 내용 읽기 API (UTF-8, 바이너리 감지, 크기 제한)

const ViewerPanel: React.FC = () => {

  const { content, isLoading, error } = useFileContent(selectedFile?.path);

**useFileContent Hook**:
```typescript
// frontend/src/hooks/useFileContent.ts
const useFileContent = (filePath: string | null) => {
  const [content, setContent] = useState<string>('');
  const [isLoading, setIsLoading] = useState(false);
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    if (!filePath) return;

    const fetchContent = async () => {
      setIsLoading(true);
      try {
        const data = await fileApi.readFile(sessionId, filePath);
        setContent(data.content);
        setError(null);
      } catch (err) {
        setError(err.message);
      } finally {
        setIsLoading(false);
      }
    };

    fetchContent();
  }, [filePath]);

  return { content, isLoading, error };
};
```

### 정상 케이스 (Normal Cases)

#### TC-5.1: Markdown 파일 렌더링
```gherkin
Given 세션 ID "abc123"이 생성되어 있고
  And "/home/user/README.md" 파일이 존재하며
  And 파일 내용이 "# 제목\n\n- 항목1\n- 항목2"일 때
When MdirPanel에서 "README.md"를 선택하고
  And Viewer 탭으로 자동 전환되면
Then MarkdownViewer가 렌더링되고
  And "제목"이 <h1> 태그로 표시되며
  And 항목1, 항목2가 <li> 태그로 표시된다
```

#### TC-5.2: 코드 파일 문법 강조
```gherkin
Given 세션 ID "abc123"이 생성되어 있고
  And "/home/user/app.js" 파일이 존재하며
  And 파일 내용이 "const x = 42;"일 때
When MdirPanel에서 "app.js"를 선택하고
  And Viewer 탭으로 자동 전환되면
Then CodeViewer가 렌더링되고
  And "const" 키워드가 파란색으로 강조되며
  And 줄 번호 "1"이 표시된다
```

#### TC-5.3: Mermaid 다이어그램 렌더링
```gherkin
Given Markdown 파일에 "```mermaid\ngraph TD\nA-->B\n```" 블록이 있을 때
When MarkdownViewer가 렌더링되면
Then Mermaid 다이어그램이 SVG로 변환되어 표시되고
  And A → B 화살표가 시각적으로 보인다
```

### 경계 케이스 (Edge Cases)

#### TC-5.4: 바이너리 파일 거부
```gherkin
Given "/home/user/image.png" 파일(바이너리)이 존재할 때
When MdirPanel에서 "image.png"를 선택하면
Then Viewer에 "바이너리 파일은 표시할 수 없습니다" 에러 메시지가 표시되고
  And HTTP 400 에러가 발생한다
```

#### TC-5.5: 파일 크기 초과
```gherkin
Given "/home/user/large.md" 파일 크기가 1.5MB일 때
When 파일을 읽으려고 시도하면
Then Viewer에 "파일 크기가 너무 큽니다 (최대 1MB)" 에러 메시지가 표시되고
  And HTTP 413 에러가 발생한다
```

#### TC-5.6: 미지원 확장자
```gherkin
Given "/home/user/data.bin" 파일이 존재하고
  And 확장자 ".bin"이 EXTENSION_MAP에 없을 때
When 파일을 선택하면
Then PlainTextViewer가 렌더링되고
  And 원본 텍스트가 그대로 표시된다
```

#### TC-5.7: Mermaid 렌더링 실패
```gherkin
Given Markdown 파일에 잘못된 Mermaid 문법 "```mermaid\ninvalid syntax\n```"이 있을 때
When MarkdownViewer가 렌더링되면
Then 원본 코드 블록이 표시되고
  And "다이어그램 렌더링 실패" 메시지가 함께 표시된다
```
