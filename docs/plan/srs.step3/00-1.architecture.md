## 3. Class Diagram (핵심 신규 클래스)

### 3.1 Backend - FileService

```typescript
// server/src/services/FileService.ts
class FileService {
  private sessionManager: SessionManager;
  private config: FileManagerConfig;

  // CWD 조회
  getCwd(sessionId: string): Promise<string>

  // 디렉토리 목록
  listDirectory(sessionId: string, path?: string): Promise<DirectoryListing>

  // 파일 읽기
  readFile(sessionId: string, path: string): Promise<FileContent>

  // 파일 복사
  copyFile(sessionId: string, source: string, destination: string): Promise<void>

  // 파일 이동
  moveFile(sessionId: string, source: string, destination: string): Promise<void>

  // 파일 삭제
  deleteFile(sessionId: string, path: string): Promise<void>

  // 디렉토리 생성
  createDirectory(sessionId: string, path: string, name: string): Promise<void>

  // 경로 보안 검증 (private)
  private validatePath(basePath: string, targetPath: string): string
  private isPathBlocked(path: string): boolean
  private isBinaryFile(buffer: Buffer): boolean
}
```

### 3.2 Backend - Types (신규)

```typescript
// server/src/types/file.types.ts
interface FileManagerConfig {
  maxFileSize: number;          // 1MB
  maxCodeFileSize: number;      // 500KB
  maxDirectoryEntries: number;  // 10000
  blockedExtensions: string[];
  blockedPaths: string[];
}

interface DirectoryEntry {
  name: string;
  type: 'file' | 'directory';
  size: number;
  extension?: string;
  modified: string;  // ISO 8601
}

interface DirectoryListing {
  cwd: string;
  path: string;
  entries: DirectoryEntry[];
  totalEntries: number;
}

interface FileContent {
  path: string;
  content: string;
  size: number;
  encoding: string;
  extension: string;
  mimeType: string;
}
```

## 10. Error Handling Strategy

### 10.1 Backend

```
FileService 에러 → AppError 클래스 사용 (기존 패턴)
                 → ErrorCode enum에 신규 코드 추가
                 → createErrorResponse() 포맷 유지
```

### 10.2 Frontend

```
API 에러 → useFileBrowser hook에서 catch
        → error state 업데이트
        → MdirPanel에서 에러 다이얼로그 표시
        → 자동 복구 (상위 디렉토리 이동 등)
```
