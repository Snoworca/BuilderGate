## ADR-001: Frontend 반응형 전략

**결정**: CSS Media Query + React Hook 혼합 방식

| 선택지 | 장점 | 단점 | 결정 |
|--------|------|------|------|
| CSS-only (media queries) | 성능 우수, JS 무관 | 동적 로직 어려움 | 레이아웃에 사용 |
| JS-only (ResizeObserver) | 유연한 조건 처리 | 리렌더링 비용 | 동적 로직에 사용 |
| **혼합 방식** | **양쪽 장점** | 약간의 복잡성 | **채택** |

- CSS Media Query: 사이드바 표시/숨김, 반응형 레이아웃 전환
- `useResponsive` Hook (matchMedia): 햄버거 버튼 조건부 렌더링, Mdir 컬럼 수 결정

## ADR-002: Mdir 파일 목록 렌더링 방식

**결정**: CSS Grid + Virtual Scroll (필요시)

| 선택지 | 장점 | 단점 | 결정 |
|--------|------|------|------|
| CSS Flexbox column-wrap | 간단한 구현 | 컬럼 높이 불균형 가능 | 부적합 |
| CSS Grid (fixed 2-col) | 정확한 2-컬럼 제어 | 약간 복잡 | **채택** |

- 10,000개 항목 대응: `overflow-y: auto` + lazy rendering 고려

## ADR-008: CWD 추적 방식

**결정**: OS별 프로세스 CWD 조회 + 폴백

| OS | 주요 방식 | 폴백 |
|----|----------|------|
| Linux | `/proc/<pid>/cwd` readlink | `os.homedir()` |
| macOS | `lsof -p <pid> -Fn` | `os.homedir()` |
| Windows | PowerShell `(Get-Process -Id <pid>).Path` 또는 PTY에 `cd` 실행 | `os.homedir()` |

- Windows 환경 주 대상 (ProjectMaster가 Windows에서 실행)
- 캐시: 1초 TTL로 빈번한 조회 최적화

## ADR-009: Path Traversal 방지 전략

**결정**: `path.resolve()` + `path.relative()` 기반 검증

```typescript
function validatePath(basePath: string, targetPath: string): string {
  const resolved = path.resolve(basePath, targetPath);
  const relative = path.relative(basePath, resolved);
  if (relative.startsWith('..') || path.isAbsolute(relative)) {
    throw new AppError(ErrorCode.PATH_TRAVERSAL);
  }
  return resolved;
}
```

- 심볼릭 링크: `fs.realpath()` 추가 검증
- 블랙리스트: `.ssh`, `.gnupg`, `/etc/shadow` 등 접근 차단
- OWASP Path Traversal 가이드라인 준수

## ADR-010: 세션 이름/순서 저장 방식

**결정**: 인메모리 (기존 패턴 유지)

| 선택지 | 영속성 | 복잡도 | 결정 |
|--------|--------|--------|------|
| **인메모리 (SessionManager)** | 서버 재시작 시 초기화 | 최소 | **채택** |
| SQLite | 영속적 | 의존성 추가 | 향후 고려 |
| JSON 파일 | 영속적 | 동시성 이슈 | 부적합 |

- 현재 SessionManager가 `Map<string, SessionData>`로 관리
- sortOrder, name을 동일 Map에 포함
- 서버 재시작 시 세션 자체가 초기화되므로 영속성 불필요
