# Phase 2: Frontend Idle Trigger And Grid Scoping

## Objective

grid mode 세션이 `idle`에 들어간 뒤 quiet window가 지나면 one-shot repair replay를 요청하도록 프런트 경로를 추가한다.

## Prerequisites

- Phase 1 완료

## Target Files

- `frontend/src/components/Terminal/TerminalContainer.tsx`
- `frontend/src/components/Terminal/TerminalRuntimeLayer.tsx`
- `frontend/src/components/Grid/MosaicContainer.tsx`
- `frontend/src/contexts/WebSocketContext.tsx` (필요 시)
- `server/src/test-runner.ts` 또는 frontend 테스트 경로

## Planned Changes

1. 현재 터미널이 grid mode host인지 판정 가능한 정보 연결
2. `status=idle` transition 감지
3. 짧은 quiet window 후 one-shot repair replay 요청
4. 중복 idle transition이나 즉시 running 복귀 시 타이머 취소
5. replay 완료 후 snapshot 재적용 + `fit()` 경로가 자연스럽게 이어지도록 유지

## Implementation Checklist

- [ ] grid mode 한정 조건 추가
- [ ] idle repair timer 추가
- [ ] running 복귀 시 pending timer 취소
- [ ] 동일 idle 구간당 1회만 repair replay 실행
- [ ] snapshot 적용 후 viewport sync가 유지되는지 확인

## Test

### Scenario 1: grid idle triggers one repair replay

Given a grid mode terminal  
When it transitions from running to idle and stays quiet  
Then frontend sends exactly one repair replay request

### Scenario 2: idle blip is cancelled

Given a session transitions to idle briefly  
When it returns to running before quiet window ends  
Then no repair replay is requested

### Scenario 3: tab mode is untouched

Given a tab mode terminal  
When it transitions to idle  
Then no repair replay request is sent
