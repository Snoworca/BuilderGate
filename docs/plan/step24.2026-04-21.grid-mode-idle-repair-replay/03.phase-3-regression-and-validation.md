## Test

### Scenario 1: repair replay does not duplicate output

Given output arrives near idle repair timing  
When repair replay completes  
Then visible content is not duplicated and queued output flushes once

### Scenario 2: codex/hermes grid corruption recovers after idle

Given a real grid mode TUI session with broken line wrapping  
When the session settles into idle  
Then one repair replay recovers the layout without user drag

### Scenario 3: existing resize replay remains green

Given the existing resize replay tests  
When the new repair replay feature is added  
Then prior resize replay behavior remains green
