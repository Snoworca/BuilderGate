# Technical Decisions

## ADR-2401: 1차 구현은 geometry mutation 없는 repair replay를 채택한다

- 상태: Accepted
- 이유:
  - 사용자의 수동 리사이즈가 주는 “화면 재구성 효과”를 가장 안전하게 재현할 수 있다.
  - synthetic resize는 TUI side effect 리스크가 높다.

## ADR-2402: repair replay는 grid mode에서만 시작한다

- 상태: Accepted
- 이유:
  - 현재 문제의 재현 영역이 grid mode에 집중되어 있다.
  - tab mode 회귀 반경을 줄일 수 있다.

## ADR-2403: idle 즉시가 아니라 quiet window 뒤에 1회 수행한다

- 상태: Accepted
- 이유:
  - idle 직후 남은 output drain과 충돌하지 않기 위해서다.
  - busy↔idle 짧은 왕복에서 불필요한 replay를 줄인다.

## ADR-2404: 기존 snapshot ACK 및 `session:ready` 경계를 재사용한다

- 상태: Accepted
- 이유:
  - 새 프로토콜을 최소한으로 유지하고, replay lifecycle을 일관되게 만든다.

## ADR-2405: codex/hermes detector 의존 없이 symptom-based로 시작한다

- 상태: Accepted
- 이유:
  - 문제는 앱 식별보다 “grid mode TUI corruption” 증상에 가깝다.
  - 1차 구현은 grid mode 세션 전체 중 idle repair 조건에 맞는 경우만 다루는 것이 단순하다.
