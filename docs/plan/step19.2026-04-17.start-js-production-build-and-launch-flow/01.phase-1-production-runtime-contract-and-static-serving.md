## Phase 1: Production Runtime Contract And Static Serving

### 목표

production 모드에서 `server`가 `/`, `/assets/*`, `/logo.svg`를 제공하고, 기존 `/api`, `/health`, `/ws` 계약을 깨지 않게 만든다.

## 선행 조건

- 없음
- 입력 연구 문서 2종과 `AGENTS.md` 검토 완료

## 기존 코드 재사용 힌트

- `server/src/index.ts`
  - 이미 dev/prod 분기와 API/WS/TLS 흐름이 있다. 여기에 production static branch를 추가한다.
- `server/src/utils/config.ts`
  - `getServerRoot()`가 이미 존재하므로 public path 계산 시 새 루트 계산 유틸을 중복 만들지 않는다.
- `frontend/src/services/api.ts`
  - API base가 relative `/api`라 same-origin static hosting과 호환된다.
- `frontend/src/contexts/WebSocketContext.tsx`
  - WebSocket URL이 same-origin `/ws`라 production reverse proxy 없이도 맞는다.

## 코드 중복 주의

- IMPL-001, IMPL-002
  - 새 HTTPS server, 새 reverse proxy, 새 frontend server를 만들지 않는다.
- IMPL-002
  - `viteProxy` production replacement를 별도 모듈로 복제하지 말고 `index.ts`의 existing request path 위에 최소 변경으로 올린다.

- [ ] Phase 1-2: `express.static(publicDir, { index: false })` branch 추가 `IMPL-001`

## 언어+버전 주의사항

- Server는 TypeScript + NodeNext ESM 구조다.
  - 새 server-side import는 emitted `.js` 경로 규칙을 지켜야 한다.
- absolute path 계산은 `process.cwd()`에 기대지 말고 기존 server-root helper를 우선 사용한다.
- 기존 Express middleware 순서를 흐트러뜨리지 않는다.

## LLM 코딩 주의사항

- Simplicity: SSR, manifest rendering, template engine을 도입하지 않는다.
- Surgical: `server/src/index.ts`의 API/WS/TLS 부위를 건드리지 말고 production non-API branch만 최소 변경한다.

## 테스트 시나리오

### 정상 케이스

- Given: `server/dist/public/index.html`과 assets가 존재한다
  When: production 서버로 `GET /` 요청
  Then: `index.html`이 반환된다

- Given: `server/dist/public/assets/*`가 존재한다
  When: 해당 자산 URL 요청
  Then: static file이 반환된다

- Given: production 서버가 기동 중이다
  When: `GET /health`, `POST /api/auth/login`, `GET /ws?...`
  Then: 기존 health/API/WS 계약이 유지된다

### 예외 케이스

- Given: `server/dist/public`이 아직 staging되지 않았다
  When: production 서버 기동 또는 `/` 요청
  Then: 원인 파악 가능한 로그/실패 형태가 남는다

- Given: 존재하지 않는 asset path `/assets/missing.js`
  When: 요청
  Then: `index.html`이 아니라 asset 404로 남는다

### 경계값 테스트

- Given: HTML navigation path `/settings` 또는 future deep-link path
  When: 브라우저가 GET HTML 요청
  Then: `index.html` fallback을 받는다

- Given: static branch와 API route가 동시에 등록된 상태
  When: `/api/...` 요청
  Then: static middleware가 API를 가로채지 않는다
