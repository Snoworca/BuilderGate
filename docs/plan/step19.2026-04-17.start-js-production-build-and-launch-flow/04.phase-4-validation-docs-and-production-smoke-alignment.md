## 언어+버전 주의사항

- 문서 예제 명령은 Windows PowerShell 기준과 cross-platform 관점을 혼동하지 않게 적는다.
- Playwright 관련 변경은 default developer loop를 느리게 만들 수 있으므로 opt-in 여부를 명확히 적는다.

## LLM 코딩 주의사항

- Simplicity: production smoke는 최소 성공 경로만 다룬다. 전체 회귀 자동화 욕심을 넣지 않는다.
- Surgical: README/validation docs만 수정하고 테스트 인프라 대공사는 하지 않는다.

## 테스트 시나리오

### 정상 케이스

- Given: 사용자가 일반 로컬 운영 실행만 원한다
  When: `node start.js`
  Then: 문서가 기본 fallback 포트 `2222`를 안내한다

- Given: build 후 start.js로 서버가 기동된다
  When: README 절차대로 검증 수행
  Then: AGENTS 기준 검증은 `--port 2002` 또는 config `server.port=2002`로 재현 가능하다

- Given: Playwright production smoke가 opt-in으로 정의된다
  When: 실행
  Then: `https://localhost:2002`를 대상으로 최소 성공 경로를 검증한다

### 예외 케이스

- Given: start.js flow는 되지만 dev.js와 README가 불일치한다
  When: 문서 검토
  Then: 충돌 지점이 제거되어야 한다

### 경계값 테스트

- Given: 운영 smoke와 개발 E2E가 공존한다
  When: 팀원이 문서를 읽는다
  Then: 어느 상황에서 dev.js를 쓰고 어느 상황에서 start.js를 쓰는지 혼동하지 않는다
