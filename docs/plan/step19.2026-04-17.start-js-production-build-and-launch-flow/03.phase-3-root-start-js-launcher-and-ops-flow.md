## Phase 3: Root start.js Launcher And Ops Flow

### 목표

루트 `start.js`가 production server를 올바른 작업 디렉터리와 환경 변수로 실행하도록 만든다.

## 선행 조건

- [ ] Phase 1 완료
- [ ] Phase 2 완료
- [ ] `server/dist/index.js`와 `server/dist/public`이 생성 가능한 상태

## 기존 코드 재사용 힌트

- `dev.js`
  - `child_process.spawn`, signal cleanup, 기본 port parsing 패턴 참고
- `server/src/index.ts`
  - 서버는 이미 `PORT` 기반으로 HTTPS/redirect 서버를 기동한다. launcher는 이 계약만 주입하면 된다.
- `server/config.json5`
  - `server.port`가 존재하면 launcher 기본값보다 우선하는 설정 입력으로 사용한다.

## 코드 중복 주의

- IMPL-005
  - `start.js`가 TLS, redirect, ws, auth logic를 재구현하면 안 된다.
- IMPL-006
  - `dev.js`와 `start.js`를 하나로 합치지 않는다. 개발/운영 흐름을 분리 유지한다.

## 언어+버전 주의사항

- 루트 script는 CommonJS 기준으로 작성한다.
- `node` literal보다 `process.execPath`를 우선 고려한다.
- launcher는 Windows signal/child lifecycle 차이를 고려해야 한다.
- 작업 디렉터리 계약은 optional이 아니라 mandatory다.

## LLM 코딩 주의사항

- Simplicity: build 포함 auto-magic launcher로 만들지 않는다. `build.js`와 `start.js` 책임을 분리한다.
- Surgical: `dev.js`는 수정이 필요하더라도 최소화하고, 가능하면 untouched로 둔다.

## 테스트 시나리오

### 정상 케이스

- Given: CLI 인자와 config `server.port`가 모두 없다
  When: `node start.js`
  Then: HTTPS는 `2222`, redirect는 `2221`로 기동한다

- Given: `server/config.json5`에 `server.port=2333`이 있다
  When: `node start.js`
  Then: HTTPS는 `2333`, redirect는 `2332`로 기동한다

- Given: `node build.js`가 성공했다
  When: `node start.js --port 2002`
  Then: `https://localhost:2002/health`가 응답하고 `/`가 앱 진입점을 제공한다

- Given: CLI `--port`로 2444를 지정한다
  When: `node start.js --port 2444`
  Then: HTTPS는 2444, redirect는 2443으로 동작한다

### 예외 케이스

- Given: build artifact가 없다
  When: `node start.js`
  Then: missing `server/dist/index.js` 또는 `server/dist/public/index.html`에 대한 actionable error가 나온다

- Given: child process가 crash한다
  When: `start.js`가 server close event를 받는다
  Then: exit code가 상위 프로세스로 전파된다

### 경계값 테스트

- Given: SIGINT/SIGTERM
  When: `start.js` 종료
  Then: child process도 정상 종료 경로를 탄다
