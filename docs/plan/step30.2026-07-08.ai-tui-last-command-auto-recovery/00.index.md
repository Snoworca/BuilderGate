## Goal

마지막으로 shell prompt에서 실행한 복구 가능 AI TUI 명령(`codex`, `claude`, 사용자 등록 명령 등)을 Workspace tab에 영속 저장하고, tab restart 또는 서버 재시작 orphan recovery 후 현재 활성화된 Recovery Option을 다시 조회하여 `codex resume --last`, `claude --continue` 같은 복구 명령을 자동 실행한다.

## Architecture

`SessionManager`는 shell-level command submission event만 발행하고, `WorkspaceService`가 tab 영속 메타데이터와 recovery option lookup을 소유한다. Restore execution은 `RecoveryOptionService`의 현재 enabled option을 다시 조회하고 `buildRecoveryRestoreInput()`으로 shell별 quoting을 적용한 뒤, 새 세션이 durable하게 저장된 후 지연/guard된 `SessionManager.scheduleRestoreInput()` 경로로 주입한다.

## Global Constraints

- UTF-8로 읽고 쓴다.
- AI TUI 내부 사용자 입력, local echo, prompt redraw, cursor movement, ticker output, waiting-for-input repaint는 session status를 `running`으로 전환하지 않는다.
- 자동 실행은 enabled Recovery Option과 이전 shell-level matched command가 모두 있을 때만 수행한다.
- Disabled/deleted option은 절대 자동 실행하지 않고 tab recovery metadata를 clear한다.
- Dangerous Claude flags는 built-in default에 추가하지 않고 사용자가 저장한 option에서만 실행한다.
- Recovery command/argument는 control character를 거부하고 shell-specific quoting을 통과해야 한다.
- UI visual change는 기존 컴포넌트 패턴에 맞춘 최소 연결만 수행한다.
- Phase 완료 시 AGENTS.md 규칙에 따라 서브에이전트 코드 리뷰에서 `No findings` 판정을 받을 때까지 수정/재리뷰한다.

## 5-Subagent Research Synthesis

1. Input path: `SessionManager.writeInput()`에서 `submittedCommand && !isAiForeground`인 경우만 복구 메타데이터 후보로 삼아야 한다. 내부 AI TUI 입력은 상태/메타데이터 갱신 트리거가 아니다.
2. Restart/orphan path: `WorkspaceService.restartTab()`은 replacement session 저장 성공 후, `checkOrphanTabs()`는 final save 성공 후 restore를 예약해야 한다. sessionId guard가 필수다.
3. Recovery option path: `RecoveryOptionService`, routes, command utility, frontend dialog는 존재하지만 server route/init과 App/Header 연결이 빠져 있다.
4. Data contract: `server/data/workspaces.json`에는 이미 flat recovery fields가 남아 있다. 새 nested 구조보다 `recoveryOptionId`, `recoveryCommand`, `recoveryArguments`, `recoveryIcon`, `recoveryUpdatedAt`를 공식화한다.
5. Verification/security: backend recovery tests가 없고 frontend recovery unit tests는 현재 red다. E2E helpers/port도 수정해야 한다. Route auth, lookup, quoting, stale disabled/deleted skip, idle invariant를 직접 검증한다.

## Completion Criteria

1. A shell-level `codex` submission persists tab recovery metadata for the enabled Codex option.
2. A shell-level `claude` submission persists tab recovery metadata for the enabled Claude option.
3. A user-defined enabled command such as `claudep` can be matched and later restored.
4. AI TUI foreground user input does not update recovery metadata and does not change session status to `running`.
5. Restarting a tab with valid metadata recreates PTY with saved shell/CWD and schedules the current enabled restore command.
6. Orphan recovery after server restart performs the same enabled-option lookup and restore scheduling.
7. Disabled/deleted option never auto-executes and clears tab recovery metadata.
8. Recovery option APIs are authenticated and mounted.
9. Metadata icons render in desktop tab/header paths without URL/image/script execution.
10. Required tests and sub-agent review loop pass.
