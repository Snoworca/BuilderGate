# Codex TUI Resize And Replay Remediation Plan

Date: 2026-04-14  
Scope: fix duplicate redraw, blank-gap growth, and replay instability that appear when a Codex TUI session is bounced quickly across workspaces  
Mode: plan only

## Goal

Make Codex-style TUI sessions stable during rapid workspace switching by fixing the real failure boundary:

- redundant resize bursts during visibility changes
- no-op server resize treated as real lifecycle mutation
- non-atomic snapshot/live-output boundaries for TUI redraw workloads
- insufficient telemetry and regression coverage for replay lineage

## Requirements

- `BG-001` Rapid workspace switching must not create visible duplicate Codex TUI blocks.
- `BG-002` Workspace switching must not inject large blank regions into the visible terminal.
- `BG-003` The fix must preserve existing refresh, reconnect, restart, and hidden-workspace recovery behavior.
- `FR-001` The frontend must deduplicate resize emission so identical `cols/rows` are not repeatedly sent.
- `FR-002` Workspace visibility changes must not cause avoidable terminal height oscillation.
- `FR-003` The server must treat no-op resize as a no-op and must not increment replay lineage for unchanged geometry.
- `FR-004` Authoritative snapshots must be fenced behind headless write drain so TUI redraw frames are not split across snapshot and queued live output.
- `FR-005` Replay refresh must not be triggered purely by frontend layout noise.
- `FR-006` Fallback recovery rules for TUI-heavy sessions must avoid compounding duplicate redraw artifacts.
- `FR-007` Telemetry must expose replay lineage at `sessionId/replayToken/snapshotSeq` granularity.
- `FR-008` Automated and manual regression coverage must prove the Codex `1부터 500까지 종 방향으로 출력해줘` scenario.

## Assumptions

1. The current `@xterm/headless` authority model remains in place; this plan hardens it rather than replacing it.
2. Server restart persistence is still out of scope.
3. The primary reproduction target is `https://localhost:2002`.
4. The most important manual reproduction remains a real Codex TUI session, not a synthetic local snapshot stub.

## Recommended Execution Order

1. Freeze the reproduction path and add fine-grained replay telemetry before changing behavior.
2. Stop redundant frontend resize traffic and remove layout churn that should never reach the PTY.
3. Make server resize idempotent and move authoritative snapshot generation behind a write-drain fence.
4. Harden replay/fallback policy specifically for TUI-style redraw workloads.
5. Close with automated regression, Codex-driven validation, and ship gates.

## Success Criteria

- Rapid workspace bounce no longer duplicates visible Codex prompt/status blocks.
- Blank vertical gaps do not grow when moving between workspaces.
- Identical `cols/rows` transitions do not produce `screenSeq` churn or replay refreshes.
- A live Codex session printing `1..500` vertically survives multiple workspace bounces without visible corruption.
- Replay lineage can be explained from telemetry without guessing.
- Existing refresh/reconnect/restart regressions remain green.

## Out Of Scope

- replacing `node-pty`
- redesigning the workspace UX beyond what is required to stabilize terminal geometry
- persistent terminal state across full server restart
- generic terminal beautification unrelated to the duplicate-redraw bug
