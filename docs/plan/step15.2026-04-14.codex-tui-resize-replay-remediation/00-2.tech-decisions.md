# Tech Decisions

## ADR-001: Treat no-op resize as invalid input to the PTY lifecycle

### Decision

`SessionManager.resize()` must immediately return when incoming `cols/rows` matches the current session geometry.

### Why

- current behavior turns frontend noise into replay lineage churn
- Codex TUI responds to resize regardless of whether the geometry effectively changed
- this is the lowest-risk, highest-value correction

### Consequences

- lower redraw pressure
- lower snapshot churn
- easier telemetry interpretation

## ADR-002: Frontend resize emission must be deduplicated before crossing the network

### Decision

`TerminalView` must track the last sent geometry and skip redundant `onResize()` calls from:

- initial mount
- visibility return
- `ResizeObserver`

### Why

- server-side idempotence alone still wastes client work and increases jitter
- multiple frontend resize sources currently overlap

### Consequences

- fewer resize messages
- lower risk of repeated `fit -> resize -> redraw`

## ADR-003: Metadata layout must not resize the terminal accidentally

### Decision

The metadata bar must no longer change the terminal viewport height as a side effect of visibility switching.

Accepted options:

- reserve fixed height whether visible or not
- move metadata into a layout region that does not change the xterm viewport box

Rejected option:

- keep current mount/unmount behavior and rely only on debounce

### Why

- geometry instability is a structural cause, not just a timing issue

## ADR-004: Authoritative snapshot generation must wait for headless write drain

### Decision

`getScreenSnapshot()` must not serialize headless state until prior queued writes relevant to that snapshot lineage are drained.

### Why

- TUI redraw frames are not safe to split across snapshot and queued output
- Codex uses cursor movement and footer redraw patterns that expose partial-frame boundaries

### Consequences

- more coherent authoritative snapshots
- slightly more complex sequencing and telemetry

## ADR-005: Replay telemetry must be lineage-oriented, not only aggregate-oriented

### Decision

Add a recent-event ring buffer for replay events keyed by:

- `timestamp`
- `clientId`
- `sessionId`
- `replayToken`
- `snapshotSeq`
- `event`

### Why

- current `/api/sessions/telemetry` counters are useful for trends but weak for forensic debugging

### Consequences

- easier reproduction and diagnosis
- bounded in-memory telemetry cost if the ring size is explicit

## ADR-006: Codex validation must use a real TUI workload

### Decision

Final validation must include a real Codex session and the prompt:

`1부터 500까지 종 방향으로 출력해줘`

### Why

- synthetic echo tests do not reproduce TUI redraw behavior
- the user-reported corruption happens on a real Codex session

### Consequences

- some validation remains partially manual
- Playwright automation may need helper instrumentation rather than pure DOM assertions
