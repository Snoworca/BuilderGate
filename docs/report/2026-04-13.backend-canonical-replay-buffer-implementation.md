# Backend Canonical Replay Buffer Implementation

Date: 2026-04-13
Branch: `refactor/buffer`
Scope: implement server-primary terminal history replay based on `docs/plan/step13.2026-04-13.backend-canonical-replay-buffer`
Status: implemented

## What Changed

### Backend

- `server/src/services/SessionManager.ts`
  - replaced gap-only `outputBuffer` behavior with canonical `replayBuffer`
  - replay now records for every PTY output chunk
  - added replay snapshot API and runtime re-truncation on `pty.maxBufferSize` changes
  - added lightweight replay filtering for alternate-screen enter/leave control sequences
  - wired session delete cleanup into WS replay state teardown

- `server/src/ws/WsRouter.ts`
  - added `history` replay message handling on subscribe
  - added `history:ready` client ack handling
  - router now owns replay-pending per socket/session state
  - queued live output is held until replay completion
  - duplicate subscribe is idempotent
  - unsubscribe, disconnect, and session cleanup clear replay-pending state

- `server/src/types/ws-protocol.ts`
  - added `history`
  - added `history:ready`
  - extended client metadata for replay-pending state

### Frontend

- `frontend/src/contexts/WebSocketContext.tsx`
  - added `onHistory`
  - added `onSubscribed`

- `frontend/src/components/Terminal/TerminalContainer.tsx`
  - added server-history replay path
  - added initial fallback-to-local-snapshot decision only when no server history arrives
  - sends `history:ready` after replay completes

- `frontend/src/components/Terminal/TerminalView.tsx`
  - stopped mount-time automatic local snapshot restore
  - added explicit `restoreSnapshot()`
  - added explicit `replaceWithHistory()`
  - keeps output buffered while restore or replay is pending

- `frontend/src/types/ws-protocol.ts`
  - mirrored protocol additions

## Known Limits

- this is still a lightweight replay solution, not a headless terminal emulator
- replay is stream-faithful and bounded, not guaranteed to be pixel-for-pixel identical to a fully rendered terminal screen
- current replay filter is conservative and does not attempt full TUI normalization
