# 수정 완료 보고서: twoFactor.externalOnly 바이패스 + Ctrl+V 이중 붙여넣기

**날짜**: 2026-04-09  
**커밋**: `8016a12`

## Issue 1: twoFactor.externalOnly 설정이 무시되는 버그

### 이슈 설명

`server/config.json5`에 `twoFactor.externalOnly: true`로 설정해도 `localhost:4242`로 접속 시 2FA(TOTP) 입력창이 표시되었음.

### 문제 원인

`server/src/routes/authRoutes.ts`의 로그인 플로우에서 `localhostPasswordOnly` 체크는 있었으나, `twoFactor.externalOnly` 설정을 확인하는 로직이 누락되어 있었음.

또한 `createAuthRoutes` 팩토리의 `AuthRouteAccessors` 인터페이스에 해당 설정을 외부에서 주입받는 접근자가 정의되어 있지 않았음.

초기 수정 시 `configStore`(존재하지 않는 변수)를 참조해 `ReferenceError: configStore is not defined` 런타임 오류 발생 → `runtimeConfigStore.getEditableValues()`로 수정.

### 해결 방법

1. `AuthRouteAccessors`에 `getTwoFactorExternalOnly: () => boolean` 추가
2. 로그인 단계 2b: `twoFactor.externalOnly && isLocalhost` 조건 만족 시 JWT 직접 발급, TOTP 건너뜀
3. `index.ts`에서 `runtimeConfigStore?.getEditableValues().twoFactor.externalOnly ?? false`로 접근자 연결

### 변경 파일

- `server/src/routes/authRoutes.ts` — 인터페이스 + 바이패스 로직 추가
- `server/src/index.ts` — getTwoFactorExternalOnly 접근자 연결
- `server/src/test-runner.ts` — makeAuthHarness에 옵션 추가, 테스트 2개 추가

### 테스트

| 테스트 | 조건 | 기대값 | 결과 |
|--------|------|--------|------|
| `testAuthRoutesExternalOnlyBypass` | externalOnly=true + localhost IP | 200 + JWT 발급 | 통과 |
| `testAuthRoutesExternalOnlyDisabled` | externalOnly=false + localhost IP | 202 + tempToken | 통과 |
| 전체 35개 | — | 전체 통과 | 통과 |

## Issue 2: Ctrl+V 이중 붙여넣기 버그

### 이슈 설명

터미널에서 Ctrl+V로 붙여넣기 시 클립보드 텍스트가 두 번 전송되었음.  
예: "테스트"를 복사 후 Ctrl+V → "테스트테스트"가 입력됨.

### 문제 원인

xterm v6의 paste 핸들러는 다음 순서로 동작함:

1. `paste` 이벤트에서 `clipboardData.getData('text/plain')` 읽기
2. `triggerDataEvent` 호출 → PTY로 1차 전송
3. **`preventDefault()` 미호출** → 브라우저가 `<textarea>`에 텍스트를 추가 삽입
4. 브라우저가 `input` 이벤트(타입: `insertText`) 발생
5. xterm의 `_inputEvent` 핸들러가 `triggerDataEvent` 재호출 → 2차 전송

이 경로는 Chrome/Windows 환경에서 일관적으로 재현됨.

### 해결 방법

`termEl`에 **capture 단계** paste 이벤트 리스너를 추가해 `e.preventDefault()` 호출:

```typescript
const onPasteCapture = (e: Event) => { e.preventDefault(); };
termEl.addEventListener('paste', onPasteCapture, { capture: true });
```

- capture 단계이므로 xterm의 내부 paste 핸들러(clipboardData 읽기)는 영향받지 않음
- 브라우저의 textarea 삽입 동작만 차단되어 두 번째 `triggerDataEvent` 경로 차단

### 변경 파일

- `frontend/src/components/Terminal/TerminalView.tsx` — capture 단계 paste 리스너 추가/제거
- `frontend/tests/e2e/terminal-paste.spec.ts` — Playwright E2E 테스트 신규 생성
- `CLAUDE.md` — 테스트 규칙 섹션 추가
