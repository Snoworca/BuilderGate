## 이슈 1: React Mosaic 이중 인스턴스 충돌 + 포커스 모드 미동작

### 문제
- react-mosaic-component가 내부에 react-dom@18을 번들하여 프로젝트의 React 19와 ReactCurrentDispatcher 충돌 발생으로 앱 크래시
- MosaicTile에서 포커스(⊡) 버튼 클릭 시 tabId가 handleLayoutModeChange에 전달되지 않아 포커스 모드가 동작하지 않음

### 원인
- npm의 nested node_modules에 react-dom@18이 별도 설치되어 React 이중 인스턴스 문제
- MosaicTile의 onLayoutModeChange 타입이 `(mode: LayoutMode) => void`로 focusTabId를 받지 않음

### 해결
- vite.config.ts에 resolve.alias로 react/react-dom을 프로젝트의 단일 인스턴스로 강제
- MosaicTile에서 focus 모드 클릭 시 자신의 tabId를 래핑하여 전달
- **커밋**: a0f9b66

## 이슈 2: 터미널 스크롤/줌 토스트/자동스크롤/Ctrl+Wheel 4건

### 문제
1. 그리드 모드에서 터미널 스크롤 불가, 스크롤바 사라짐
2. Ctrl+마우스휠 확대/축소 시 폰트 사이즈 토스트가 간헐적 미표시, 위치가 정중앙이 아닌 위쪽
3. 확대/축소 후 터미널이 맨 아래로 자동 스크롤되지 않음
4. 스크롤바가 내려간 상태에서 Ctrl+휠이 줌이 아닌 스크롤로 동작

### 원인
1. flex child의 기본 `min-height:auto`가 xterm viewport를 팽창시켜 `scrollHeight===clientHeight`
2. TerminalView(1200ms)와 FontSizeToast(1000ms) 두 개의 독립 hide 타이머가 경쟁 조건 발생
3. handleFontSizeChange에서 `fitAddon.fit()` 후 `term.scrollToBottom()` 호출 누락
4. xterm의 `.xterm-viewport`가 wheel 이벤트를 먼저 소비하여 containerRef의 버블 단계 리스너에 도달하지 못함

### 해결
1. MosaicTile, TerminalContainer, TerminalView CSS에 `min-height:0` 추가
2. FontSizeToast를 순수 컴포넌트로 단순화, handleFontSizeChange 내부에서 단일 타이머로 통합
3. `fitAddon.fit()` 직후 `term.scrollToBottom()` 추가
4. wheel 이벤트 리스너를 `{ capture: true }`로 변경하여 캡처 단계에서 가로챔
- **커밋**: 23a84c6

## 이슈 3: 새 세션 추가 시 UI 미갱신

### 문제
- 그리드 모드 컨텍스트 메뉴에서 "새 세션" 클릭 또는 빈 워크스페이스에서 "Add Terminal" 클릭 시 아무 반응 없음
- API는 201로 성공하지만 UI에 세션이 나타나지 않음

### 원인
- `addTab()`이 WS `tab:added` 이벤트에만 의존했으나, x-client-id 제외 정책으로 자기 자신은 WS 이벤트를 수신하지 못함
- EmptyState의 `onClick={onAddTab}`이 React SyntheticEvent를 cwd 파라미터로 전달하여 API body에 이벤트 객체가 포함됨
- handleAddTab의 `useCallback([])`  클로저에서 `wm.activeWorkspaceId`가 stale closure

### 해결
- API 응답 후 직접 `setTabs()`로 탭을 로컬 상태에 추가 (WS 이벤트 의존 제거)
- `typeof cwdOrEvent === 'string'` 체크로 이벤트 객체 필터링
- `wmRef.current.activeWorkspaceId` 사용으로 stale closure 방지
- **커밋**: ac83285, ab39c96

## 이슈 4: 최대 탭 도달 시 새 세션 추가 비활성화

### 문제
- 워크스페이스당 최대 탭 수(8개)에 도달해도 UI에 피드백 없이 추가 버튼이 활성 상태

### 해결
- 그리드 컨텍스트 메뉴 "새 세션"에 `disabled: tabs.length >= 8` 추가
- 워크스페이스 사이드바 컨텍스트 메뉴 "Add Terminal"에 `disabled: tabCount >= maxTabs` 추가
- WorkspaceItem에 `tabCount`, `maxTabs` props 추가
- **커밋**: 8f59cf1
