## 운영 제약

- live BuilderGate 검증이 필요하면 `https://localhost:2002`를 사용하지 않는다.
- `node.exe` 전체 종료는 금지한다. 테스트 서버를 직접 시작한 경우에만 해당 테스트 서버 프로세스 트리만 종료한다.
- `REL-BGSTAB-001`은 non-2002 live split 검증과 장시간 browser memory/input-lag soak 증거가 남을 때까지 `draft`로 유지한다.

## 남은 결정

Live SpecKiwi 반영을 진행하려면 사용자의 target/scope 결정이 필요하다.

1. 새 target/scope `0.5.5-buildergate-stability` / `BGSTAB`를 생성하고 제안 SRS를 `docs/spec`에 반영한다.
2. 또는 active target `0.5.4-ai-tui-recovery`에 추가하도록 명시 승인한다.
3. 또는 SpecKiwi live mutation은 보류하고 운영 검증만 계속한다.

현재 권고는 1번이다. BuilderGate 안정화는 AI TUI recovery와 다른 runtime reliability 범위이므로 별도 target/scope가 가장 일관적이다.
