## Historical Wave0 Findings

Wave0 implements most of the baseline configuration contract, but the independent evaluator found gaps that should be fixed before future waves depend on Wave0 as fully clean.

Findings:

- Medium: `docs/research/plug-leak/wave0.md` describes `write-hidden` as the legacy hidden-output baseline and fallback, while current code defaults to `snapshot-restore` and rejects `write-hidden`.
- Medium: SRS/report evidence over-claims full frontend typecheck. `npm --prefix frontend run typecheck` currently fails because of unrelated untracked `RecoveryOptionManager` export/API gaps.
- Low: `outputCoalesceWindowMs` allows `0`, while the research document says coalesce windows are bounded positive integers.

Verification observed at original evaluation time:

- Passed: focused frontend runtime/backpressure/residency tests, 24 tests.
- Passed: focused server config/settings/runtime tests, 17 tests.
- Passed: full server suite, 234 tests.
- Failed: `npm --prefix frontend run typecheck`, blocked by unrelated untracked `RecoveryOptionManager` work.

Recommended next actions:

1. Decide whether `write-hidden` or `snapshot-restore` is the intended Wave0 legacy baseline.
2. Align `docs/research/plug-leak/wave0.md`, `FR-BGSTAB-008`, config schema/types, and frontend `inputReliabilityMode` with that decision.
3. Correct Wave0 evidence wording so it says full frontend typecheck is currently blocked by unrelated untracked RecoveryOption work unless that work is completed first.
4. Decide whether `outputCoalesceWindowMs = 0` is intentionally allowed as “no coalesce”; if yes, update the research/spec wording. If not, tighten schema/settings constraints and tests.

## Historical Cross-Wave Risks To Fix Before Continuing

4. Full frontend typecheck is blocked by unrelated untracked RecoveryOption work. Either complete that work or explicitly exclude it from Wave0-3 verification claims.
5. Keep shutdown policy in Wave5. Do not let Wave2 or Wave3 silently mark shutdown tabs unrecoverable.

## Remediation Status — 2026-06-26

Follow-up implementation resolved the handoff blockers in the current worktree:

- Wave0 contract drift: `write-hidden` is accepted and restored as the legacy hidden-output default/fallback, `hiddenOutputTailBytes` defaults to a bounded 256 KiB tail, and `outputCoalesceWindowMs` rejects zero.
- Frontend typecheck blocker: the untracked RecoveryOption type/API exports were completed enough for `npm --prefix frontend run typecheck` to pass.
- Wave3 durability: immediate `WorkspaceService.flushToDisk()` failures now propagate to callers; delete workspace, delete tab, restart tab, and direct session delete paths persist lifecycle markers before awaited termination and have save-failure regression tests.
- Wave2 absence: `FR-BGSTAB-011` now specifies verified process-tree termination, and `docs/plans/2026-06-26.projectmaster.bgwave2.plan.md`, `.sidecar.json`, and `.validator.json` exist.
- Wave2 implementation: `server/src/utils/processTreeTerminator.ts` implements verified identity checks, Windows PID-only taskkill, POSIX safe leaf-first fallback, and unverified skip behavior; `SessionManager` now exposes async termination APIs and records best-effort OS start identity.
- Wave2 re-review fixes: ordinary terminator failures now finalize sessions with failed cleanup telemetry instead of breaking delete compatibility, POSIX process-group kill is disabled until stronger ownership verification exists, and batch termination missing/all-session behavior has explicit tests.
- Wave2 final re-review fixes: POSIX cwd must be sampled and match before termination, sampled descendants are rechecked after root exit so surviving children are reported as degraded/unverified instead of false completion, and Wave2 plan/sidecar/SRS evidence now names these cases.
- Wave0 re-review fixes: `SettingsSnapshot` and `terminalHiddenOutput` now accept the restored legacy `write-hidden` terminal hidden-output policy, with focused regressions covering a settings snapshot payload and legacy hidden-output writes.
- Wave0 final re-review fixes: the `SettingsSnapshot` regression fixture now uses a real `Record<EditableSettingsKey, FieldCapability>` capabilities object with `satisfies`, instead of a type assertion that bypasses part of the snapshot shape.
- Wave3 re-review fixes: delete workspace, delete tab, and direct session delete now roll back in-memory tab lifecycle metadata if the pre-termination durable save fails; regression tests assert the lifecycle state, recoverability, reason, cleanup status, exit code, timestamp, and generation remain unchanged after failure.
- Documentation re-review fixes: stale pre-remediation handoff findings are explicitly marked historical, current Wave2 artifacts are linked as current, and Wave3 SRS trace links now reference current files/functions instead of stale line ranges.
