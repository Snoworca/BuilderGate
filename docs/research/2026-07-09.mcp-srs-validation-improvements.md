# MCP SRS 검증 개선사항 취합

작성일: 2026-07-09

## 대상

- 연구 원본: `docs/research/mcp/00.index.md` 및 하위 문서 01~07
- 작성 SRS: `docs/spec/40.mcp-session-orchestration.srs.md`
- 검증 방식: 5개 서브에이전트 독립 검토

## 채택한 개선사항

| ID | 개선사항 | 반영 방식 |
| --- | --- | --- |
| IMP-01 | capability scope 목록, 기본 session-bound grant, leader/follower grant, destructive scope, submit scope, token claim, browser JWT 거부, Origin 누락 조건, empty whitelist 정책, 주요 error code를 SRS 계약으로 고정 | 신규 SRS 요구사항 `SEC-MCP-002` |
| IMP-02 | webhook random key의 최소 32 bytes entropy, key별 target/profile/mode/scope binding, request override 거부, query/header key, source `"0"` 유지, redaction을 검증 가능하게 고정 | 신규 SRS 요구사항 `SEC-MCP-002` |
| IMP-03 | server startup registry reconcile/backfill과 사라진 tab/session binding retired 처리를 수락 기준으로 추가 | 신규 SRS 요구사항 `FR-MCP-006` |
| IMP-04 | `/api/mcp-control/*` 관리 REST 계약, error shape, UI JWT와 MCP runtime credential 분리, `mcpControlApi` response masking을 고정 | 신규 SRS 요구사항 `IR-MCP-002` |
| IMP-05 | agent profile `displayName`, 서버 측 command/args/aliases/kickoff validation, webhook one-time URL copy, live sessions 탭 refresh/search/reply/close confirm UX를 고정 | 신규 SRS 요구사항 `IR-MCP-002` |
| IMP-06 | 연구 문서의 결번 #9와 `leaderSessionId`/`leaderSessionKey`, `aliasSource`/`nameSource` 용어 혼선을 정리 | 연구 문서 보정 |

## 보류 또는 제외한 항목

| ID | 항목 | 처리 |
| --- | --- | --- |
| DEF-01 | 현재 코드에 MCP 구현이 없음 | SRS가 `planned` 상태이므로 정상 gap. 구현 완료 판정 이슈가 아니라 SRS 작성 범위의 blocker가 아님 |
| DEF-02 | active target의 기존 `REL-BGSTAB-005` draft warning | MCP SRS 작성 전부터 존재한 BGSTAB 범위 이슈. 최종 검증에서 기존 warning으로 보고하되 이번 SRS 개선에 섞지 않음 |
| DEF-03 | `REL-BGSTAB-005` trace line drift | BGSTAB 요구사항 정비 범위. 별도 BGSTAB SRS sync 또는 hot-fix 작업에서 처리 필요 |

## 개선 후 재검증 기준

- `speckiwi validate --json`에서 MCP scope 관련 error가 없어야 한다.
- `docs/spec/40.mcp-session-orchestration.srs.md`에 신규 개선 요구사항이 생성되어야 한다.
- 재검증 서브에이전트가 IMP-01~IMP-05가 SRS에 반영됐다고 판정해야 한다.
