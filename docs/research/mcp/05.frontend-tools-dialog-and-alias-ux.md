# 프론트엔드 도구 다이얼로그와 alias UX

## 배치 원칙

요구사항은 "UI의 설정 화면이 아닌 도구 다이얼로그 창"을 명시한다. 따라서 일반 `SettingsPage`에 넣지 않는다. Header의 Tools 메뉴에 `MCP 세션 제어` 항목을 추가하고, 기존 `WindowDialog` 기반 관리 다이얼로그 패턴을 따른다.

현재 근거:

- Header Tools 메뉴는 명령줄 관리, 터미널 키보드, 복구 옵션 같은 운영 도구를 연다.
- App은 각 도구 다이얼로그의 open state를 직접 관리한다.
- `WindowDialog`는 focus trap, resize, geometry persistence를 제공한다.
- `CommandPresetDialog`, `TerminalShortcutDialog`는 tablist/tabpanel 패턴을 이미 사용한다.

## 세션 alias UX

사용자의 alias source는 기존 하단 상태 표시줄 이름이다.

현재 근거:

- `MetadataRow`는 세션 이름을 표시한다.
- 세션 이름 span은 더블클릭으로 인라인 편집을 시작한다.
- `WorkspaceTabBar`에도 tab 이름 더블클릭 rename이 있다.
- type에는 `nameSource: 'default' | 'terminal-title' | 'user'`가 있다.

정책:

- `WorkspaceTab.name`을 primary alias로 사용한다.
- `nameSource='user'`일 때만 "사용자 지정 alias"로 취급한다.
- `nameSource='terminal-title'`이면 display name으로 검색 후보에는 포함하지만 alias exact 우선순위는 낮춘다.
- MCP `set_alias`는 `nameSource='user'`로 저장한다.
- 새 `nameSource='mcp'`는 만들지 않는다. 사용자가 MCP로 변경한 alias도 명시적 사용자 의도이므로 `user`가 적합하다.

## 에이전트 프로파일 UI

필드:

- 이름 또는 alias label.
- command: `claude`, `codex` 등.
- args: 배열 편집.
- aliases: `클로드`, `Claude Code` 등 검색용 별칭.
- default: 하나만 true.
- enabled.
- kickoff prompt.
- MCP client config mode: env, generated-file, manual.

검증:

- command 비어 있음 금지.
- command에 NUL, 제어 문자 금지.
- args는 structured array로 저장하고 shell string으로 저장하지 않는다.
- aliases 중복 금지.
- kickoff prompt size limit.
- generated-file 모드는 파일 경로 노출과 cleanup 정책을 서버가 검증한다.

기존 `CommandPreset`과 다른 점:

- command preset은 UI 삽입용 `kind,label,value`.
- agent profile은 세션 실행 계약이며 default/alias/kickoff/status와 연결된다.
- 그래서 새 타입이 필요하다.

## MCP 서버 탭

표시:

- 현재 listener 상태: stopped, starting, active, rebind failed.
- bind host.
- port.
- endpoint URL.
- last rebind time.
- last error.

조작:

- enable/disable.
- port 변경.
- bind mode 변경: loopback, whitelist.
- apply 버튼은 `PATCH /api/mcp-control/config` 호출.

포트 변경 UX:

- 저장 중 상태 표시.
- 실패 시 이전 포트 유지와 실패 사유 표시.
- 성공 시 새 endpoint URL 표시.
- 앱 전체 refresh나 process restart를 유도하지 않는다.

## 보안 탭

필드:

- external IP/CIDR whitelist 행 편집.
- external mode transport security: direct TLS 또는 trusted TLS proxy.
- allowed origins.
- token rotation.
- webhook query key redaction 설정.

정책:

- loopback 모드에서는 whitelist 입력이 저장되어도 외부 bind가 되지 않는다.
- whitelist 모드로 바꿀 때 경고 문구와 validation이 필요하다.
- whitelist 모드는 TLS/proxy 검증이 없으면 저장 자체를 막는다.
- CIDR parsing은 서버가 최종 검증한다.
- 오류는 기존 dialog 패턴처럼 `role="alert"`로 노출한다.

## 웹훅 탭

기능:

- 새 webhook 생성.
- target session 또는 default agent profile 선택.
- key 최초 표시.
- URL 복사.
- rotate.
- revoke.
- last used, rate limit status 확인.

표시 정책:

- random key와 full URL은 최초 생성 또는 회전 response에서만 전체 표시한다.
- 목록에서는 masked key와 key id만 표시한다.
- URL query 전체를 로그나 화면 history에 지속 저장하지 않는다.

## Live sessions 탭

표시 columns:

- alias.
- agentKind.
- status.
- mcpConnected.
- sessionId.
- sessionKey.
- leader.
- workspace/tab.
- lastSeenAt.

가능한 조작:

- alias 변경.
- search 테스트.
- status refresh.
- leader로 reply 테스트.
- close는 destructive이므로 별도 확인 모달을 둔다.

## API client 타입

```ts
interface McpControlConfig {
  enabled: boolean;
  bindMode: 'loopback' | 'whitelist';
  host: string;
  port: number;
  transportSecurity: 'loopback-http' | 'direct-tls' | 'trusted-tls-proxy';
  trustedProxies: string[];
  externalWhitelist: string[];
  allowedOrigins: string[];
  webhookKeyHeaderName: string;
  status: 'stopped' | 'active' | 'rebinding' | 'failed';
  lastError?: string;
  lastRebindResult?: {
    ok: boolean;
    at: string;
    message?: string;
  };
}

interface McpAgentProfile {
  id: string;
  displayName: string;
  command: string;
  args: string[];
  aliases: string[];
  isDefault: boolean;
  enabled: boolean;
  kickoffPrompt: string;
  mcpClientConfigMode: 'env' | 'generated-file' | 'manual';
  createdAt: string;
  updatedAt: string;
}

interface McpLiveSession {
  sessionId: string;
  sessionKey: string;
  workspaceId: string;
  tabId: string;
  alias: string;
  nameSource: 'default' | 'terminal-title' | 'user';
  agentKind?: string;
  agentStatus: 'unknown' | 'starting' | 'ready' | 'busy' | 'waiting_input' | 'completed' | 'failed';
  bindingLifecycle: 'live' | 'closing' | 'closed' | 'retired' | 'failed' | 'closing-failed';
  mcpConnected: boolean;
  leaderSessionKey?: string;
  lastSeenAt?: string;
}

interface McpWebhookListItem {
  keyId: string;
  maskedKey: string;
  target: string;
  profileId?: string;
  mode: 'send-only' | 'open-or-send';
  scope: 'mcp:webhook.invoke';
  lastUsedAt?: string;
  expiresAt?: string;
  revoked: boolean;
  revokedAt?: string;
  rateLimitStatus: {
    windowSeconds: number;
    burstLimit: number;
    remaining?: number;
    resetAt?: string;
  };
}

interface McpWebhookOneTimeResponse extends McpWebhookListItem {
  fullKey: string;
  fullUrl: string;
}

interface McpWebhookRevokeResponse {
  keyId: string;
  revoked: true;
  revokedAt: string;
  auditId: string;
}

interface McpLiveSessionCloseRequest {
  confirmClose: true;
  expectedSessionKey: string;
  confirmationNonce: string;
}
```

`frontend/src/services/api.ts`에는 기존 `workspaceApi`, `commandPresetApi`, `recoveryOptionApi`와 같은 객체 패턴으로 `mcpControlApi`를 추가한다.

## UI 테스트

- Header Tools 메뉴에 `MCP 세션 제어` 항목이 있다.
- 다이얼로그가 `WindowDialog`로 열리고 geometry/focus contract를 깨지 않는다.
- 탭 전환이 가능하다.
- port 저장 실패가 alert로 표시된다.
- whitelist invalid input이 서버 validation error를 표시한다.
- MetadataRow rename 후 MCP live session alias에 반영된다.
- MCP-origin alias update가 `tab:updated`를 통해 하단 상태 표시줄과 tab bar에 반영된다.
- webhook key와 full URL은 즉시 생성 또는 회전 response에서만 전체 표시되고 그 외에는 masked로 표시된다.
