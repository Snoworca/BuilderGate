# 구현 계획

## 전체 전략

한 번에 MCP를 붙이지 않는다. 먼저 현재 입력/세션 lifecycle 경계를 정리한 뒤, MCP adapter가 그 경계를 호출하게 한다. 이렇게 해야 기존 `SessionManager`, `WorkspaceService`, `WsRouter`, Settings 화면이 MCP 전용 예외로 오염되지 않는다.

## Phase 0 - SRS 정리와 결정 고정

목표:

- 본 연구를 바탕으로 `docs/spec/` SRS를 갱신한다.
- `sessionId`와 `sessionKey`의 역할을 명확히 고정한다.
- webhook query key 허용과 완화책을 명시한다.

산출물:

- SRS 요구사항.
- 보안 decision record.
- 테스트 matrix.

검증:

- SRS-MD 규칙 검증.
- 요구사항 1-13 추적표 완성.

## Phase 1 - 도메인 타입과 저장소

목표:

- MCP control-plane의 데이터 source of truth를 만든다.

작업:

- `server/src/types/mcp.types.ts` 추가.
- `McpConfigService` 추가.
- `AgentCommandProfileService` 추가.
- `McpSessionRegistryService` 추가.
- `AuditLogService` 추가.
- `McpSessionRegistryService.reconcileFromWorkspaceState()` 추가.
- MCP config에 `transportSecurity`, `trustedProxies` 필드 추가.
- data files:
  - `server/data/mcp-config.json`
  - `server/data/agent-command-profiles.json`
  - `server/data/mcp-session-registry.json`
  - `server/data/mcp-audit.jsonl`

주의:

- 기존 `CommandPresetService`와 `RecoveryOptionService`는 수정 최소화.
- 저장은 기존 서비스처럼 atomic write와 UTF-8을 사용한다.
- token/key 원문 저장 금지.

검증:

- validation unit tests.
- atomic persist/rollback tests.
- secret redaction tests.

## Phase 2 - SessionInputGateway 도입

목표:

- WebSocket 입력, restore 입력, MCP 입력의 정책을 통합한다.

작업:

- `SessionReplayCoordinator` 또는 임시 `ReplayStateProvider` 인터페이스 추가.
- `SessionInputGateway` 추가.
- `WsRouter.handleInput`이 gateway를 호출하도록 변경.
- `SessionManager.scheduleRestoreInput`의 직접 `writeInput` 호출을 gateway 경로로 이동.
- gateway에서 replay/Enter/hidden/session ready policy를 다룬다.

주의:

- AI TUI 입력 시 `idle` 유지 invariant를 깨면 안 된다.
- 기존 WebSocket input tests를 모두 유지해야 한다.
- fallback으로 `SessionManager.writeInput` 직접 호출 경로를 남기지 않는다.

검증:

- 기존 WS replay/input tests.
- restore input Enter safety 회귀 테스트.
- MCP source를 흉내낸 gateway unit test.

## Phase 3 - Session binding과 registry 동기화

목표:

- `sessionKey` stable addressing을 도입한다.

작업:

- tab 생성 시 registry binding 생성.
- restart/orphan recovery 시 `currentSessionId`와 generation 갱신.
- tab delete/session finalize 시 binding closed/retired.
- 서버 시작 시 workspace state와 registry 불일치 reconcile/backfill.
- `sessionId` direct lookup의 stale guard 구현.

주의:

- 기존 `WorkspaceTab` contract는 additive 변경만 한다.
- 가능하면 `WorkspaceTab`에는 `sessionKey`만 추가하고 registry 상세는 별도 저장한다.

검증:

- tab create creates binding.
- restart keeps sessionKey and changes sessionId.
- stale sessionId rejected.
- orphan recovery preserves sessionKey.
- startup reconcile backfills existing tabs.

## Phase 4 - MCP HTTP server controller

목표:

- 별도 MCP listener를 runtime에 start/rebind/stop 가능하게 한다.

작업:

- `server/src/mcp/McpHttpServerController.ts`.
- `server/src/mcp/McpRequestGuards.ts`.
- loopback/whitelist/Origin/auth guard.
- external whitelist mode requires TLS or trusted TLS proxy.
- trusted proxy validation: remote address, forwarded proto, proxy bypass rejection.
- `/mcp` Streamable HTTP endpoint stub.
- `tools/list`와 health/status tool.
- rebind API.

주의:

- 앱 HTTPS server와 lifecycle 분리.
- rebind 실패 시 rollback.
- query string logging 제거.

검증:

- loopback default.
- external denied.
- whitelist allowed.
- Origin mismatch denied.
- port rebind success/failure.
- plain HTTP external bind rejected.
- trusted proxy positive/negative cases.

## Phase 6 - Agent launch와 lifecycle close

목표:

- MCP로 다른 agent 세션을 열고 닫을 수 있게 한다.

작업:

- `buildergate.session.open_agent`.
- PTY env injection option.
- MCP client config mode: env, generated-file, manual.
- `WorkspaceService.addTabWithLaunchContext` 또는 additive launch option 추가.
- agent command profile resolve.
- ready wait 또는 explicit status update.
- kickoff prompt injection.
- `buildergate.session.close`.
- deferred self-close job queue.
- launch transaction compensation: draft binding cancel, token revoke, generated-file cleanup, `deleteTab` cleanup on post-spawn failure.

주의:

- ready 전 kickoff를 shell에 흘리면 안 된다.
- 초기 구현에서는 ready timeout 시 kickoff를 주입하지 않고 `kickoffPending=true`로 반환한다.
- self-close는 tool response 이후 실행.
- process tree cleanup과 tab removal 모두 확인.

검증:

- open_agent creates tab/session/binding/token.
- open_agent pre-spawn failure leaves no tab/session.
- open_agent post-spawn failure deletes tab/session and revokes token.
- legacy/manual claim binds session with one-time code.
- kickoff waits for ready.
- kickoff is not injected when ready timeout occurs.
- close removes tab and terminates process tree.
- self-close denied/allowed cases.

## 위험과 완화

| 위험 | 완화 |
| --- | --- |
| SessionManager 비대화 | MCP tool logic을 별도 service로 분리 |
| 입력 정책 분산 | SessionInputGateway 도입 |
| sessionId stale | sessionKey와 generation guard |
| 외부 노출 | loopback default, whitelist, Origin, separate token |
| query key 유출 | hash 저장, redaction, rotate, short expiry |
| self-close response loss | deferred close |
| kickoff 오입력 | ready signal 전 보류 |
| 포트 rebind 실패 | candidate listener 후 swap, rollback |

## Definition of Done

- 요구사항 추적표 1-13이 모두 통과.
- 보안 기본값이 loopback-only.
- MCP tool 전체 schema와 policy 테스트 통과.
- 세션 검색, 메시지 전달, 새 agent launch, reply, self-close E2E 통과.
- webhook query key flow 통과.
- 프론트 도구 다이얼로그에서 port/whitelist/agent/webhook 관리 가능.
- audit log에 secret/raw long prompt가 남지 않음.
- 서브에이전트 또는 리뷰어가 no blocking finding 판정.
