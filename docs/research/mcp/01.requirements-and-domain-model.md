# 요구사항과 도메인 모델

## 목표

이 기능의 목표는 BuilderGate 내부의 터미널 세션을 MCP tool surface로 노출하여 Codex, Claude Code, 기타 코딩 에이전트가 서로를 실행하고 지시하며 결과를 되돌려 받을 수 있게 하는 것이다. 최종 사용 흐름은 다음과 같다.

1. 사용자가 리더 세션에서 "빌더 게이트 세션에 계속 진행해라고 보내"라고 지시한다.
2. 코딩 에이전트는 MCP tool로 세션을 검색한다.
3. 검색 결과가 단일 대상이면 해당 세션에 입력을 전달한다.
4. 필요하면 새 세션을 열고 `claude` 또는 `codex` 같은 에이전트 명령을 실행한다.
5. 새 세션은 자기 `sessionId`, `leaderSessionKey`, MCP endpoint를 알 수 있고, 리더에게 답장을 보낼 수 있다.
6. 외부 웹훅도 같은 control-plane을 통해 작업을 만들 수 있다. 이때 caller session id는 문자열 `"0"`이다.
7. 작업이 끝난 follower 세션은 leader가 있을 때만 자기 자신을 닫을 수 있다.

## 용어

| 용어 | 의미 |
| --- | --- |
| `sessionId` | 현재 PTY runtime 세션 ID. UUID로 생성된다. restart/orphan recovery 후 바뀔 수 있다. |
| `sessionKey` | 탭/역할을 안정적으로 가리키는 MCP control-plane 식별자. 재시작 후에도 유지된다. |
| `tabId` | Workspace 탭 ID. UI와 workspace 저장소의 주소다. |
| `alias` | 사용자가 하단 상태 표시줄 왼쪽 또는 탭 이름을 더블클릭해 설정한 이름. MCP 검색의 primary alias다. |
| `leaderSessionKey` | MCP로 세션을 만든 호출자 세션. follower의 reply/self-close 권한 근거다. |
| `agentKind` | `claude`, `codex`, `hermes` 같은 코딩 에이전트 종류. |
| `agentCommandProfile` | 에이전트 실행 명령, 별칭, 기본값, 기본 인자, kickoff prompt 설정. |
| `assignment` | 한 세션이나 외부 webhook이 다른 세션에 전달한 작업 지시 단위. |
| `capability` | 특정 actor가 특정 tool 범위를 수행할 수 있게 하는 권한 토큰 또는 웹훅 key. |

## 기능 요구사항

### FR-01 MCP HTTP endpoint

- 기본 상태에서 MCP HTTP 서버는 `127.0.0.1`에만 bind한다.
- 외부 IP 접근은 Tools 메뉴의 MCP 다이얼로그 보안 탭에서 IP/CIDR whitelist를 저장했을 때만 가능하다.
- 외부 IP 접근은 TLS listener 또는 신뢰된 TLS reverse proxy가 설정된 경우에만 가능하다. plain HTTP 외부 bind는 금지한다.
- 외부 접근 모드에서는 remote address, forwarded header 신뢰 정책, Origin 검증, 인증을 모두 통과해야 한다.
- 포트는 같은 다이얼로그에서 변경할 수 있고, 변경 시 앱 프로세스가 아니라 MCP listener만 재시작된다.

### FR-02 자기 세션 식별

- MCP client는 `buildergate.session.whoami`로 자신의 `sessionId`를 조회한다.
- `sessionId`는 UUID 형식이어야 한다.
- 응답에는 `sessionKey`, `tabId`, `workspaceId`, `leaderSessionKey`, `agentKind`, `alias`, `bindingLifecycle`, `agentStatus`도 포함한다.

### FR-03 세션 목록과 alias

- `buildergate.session.list`는 현재 살아있는 세션과 workspace tab 정보를 반환한다.
- 기본 옵션은 자기 자신 포함이다. `includeSelf=false`를 지원한다.
- 사용자가 직접 설정한 alias는 `WorkspaceTab.name`과 `nameSource='user'`로 저장한다.
- MCP가 alias를 설정할 때도 같은 저장 경로를 사용한다.

### FR-04 검색과 메시지 전달

- `buildergate.session.search`는 alias, tab name, agentKind, role, cwd, recovery command 후보를 기준으로 검색한다.
- 정확히 하나의 대상이 resolve되면 `buildergate.message.send`가 해당 세션에 메시지를 전달한다.
- 모호한 대상, stopped 세션, retired 세션, 권한 없는 대상은 명시적으로 실패한다.
- newline/Enter 포함 입력은 기본 차단 또는 별도 승인 정책이 필요하다.

### FR-05 코딩 에이전트 명령 프로파일

- Tools 메뉴 MCP 다이얼로그에서 명령, 별칭, 기본값, 기본 인자, enabled, kickoff prompt를 관리한다.
- 예: command `claude`, alias `클로드`, default `true`.
- 기존 recovery option은 "마지막 명령 복구" 용도이며, agent profile의 source of truth로 쓰지 않는다. 필요하면 migration/import만 제공한다.

### FR-06 연결/상태 관리

- MCP control-plane은 MCP로 연결된 세션과 상태를 관리한다.
- 새 세션 생성 시 환경 변수 또는 launch metadata로 MCP endpoint, session binding token, leader id를 전달한다.
- 자동 감지가 불가능하면 코딩 에이전트가 명시적으로 `buildergate.session.update_status`를 호출해 `agentKind`, `ready`, `busy`, `waiting_input`, `completed` 등을 갱신한다.

### FR-07 세션 생성/종료

- `buildergate.session.open_agent`는 workspace tab을 만들고 지정한 에이전트 명령을 주입한다.
- 에이전트가 live 상태가 된 뒤 kickoff prompt를 첫 작업 지시로 전달한다.
- `buildergate.session.close`는 target 세션을 종료하고 tab/window를 닫는다.
- `buildergate.session.close_self`는 `leaderSessionKey`가 있는 follower에서만 허용된다.

### FR-08 웹훅

- Tools 메뉴 MCP 다이얼로그에서 웹훅을 생성/회전/폐기한다.
- URL은 query parameter에 random key를 포함한다. 예: `/webhook/agent?key=<random>&prompt=<encoded>`.
- key는 hash만 저장하고 UI에서는 최초 표시 또는 masked 표시를 사용한다.
- 외부 웹훅의 caller session id는 `"0"`이다.

## 비기능 요구사항

- 보안 기본값은 닫혀 있어야 한다.
- `sessionId`는 식별자일 뿐 권한 증명으로 사용하지 않는다.
- 모든 tool call은 actor, target, scope, 결과, 거부 이유를 audit log에 남긴다.
- raw prompt, token, random key, JWT는 로그에 저장하지 않는다. safe preview와 hash만 저장한다.
- 입력 주입은 WebSocket 입력, 복구 입력, MCP 입력이 같은 safety policy를 공유해야 한다.
- 사용자 키 입력, local echo, prompt redraw, waiting input repaint는 세션 상태를 `running`으로 바꾸면 안 된다.
- 포트 rebind 실패 시 이전 MCP listener로 rollback해야 한다.

## 도메인 데이터 모델

```ts
interface McpEndpointConfig {
  enabled: boolean;
  bindMode: 'loopback' | 'whitelist';
  host: '127.0.0.1' | '0.0.0.0' | string;
  port: number;
  transportSecurity: 'loopback-http' | 'direct-tls' | 'trusted-tls-proxy';
  trustedProxies: string[];
  externalWhitelist: string[];
  allowedOrigins: string[];
  authRequired: true;
}

interface McpSessionBinding {
  sessionKey: string;
  currentSessionId: string;
  workspaceId: string;
  tabId: string;
  generation: number;
  alias: string;
  nameSource: 'default' | 'terminal-title' | 'user';
  agentKind?: string;
  leaderSessionKey?: string;
  createdBy: 'user' | 'mcp' | 'webhook';
  bindingLifecycle: 'live' | 'closing' | 'closed' | 'retired' | 'failed' | 'closing-failed';
  mcpConnected: boolean;
  agentStatus: 'unknown' | 'starting' | 'ready' | 'busy' | 'waiting_input' | 'completed' | 'failed';
  lastSeenAt?: string;
  retiredAt?: string;
}

interface AgentCommandProfile {
  id: string;
  command: string;
  args: string[];
  aliases: string[];
  isDefault: boolean;
  enabled: boolean;
  kickoffPrompt: string;
  mcpClientConfigMode: 'env' | 'generated-file' | 'manual';
  createdAt: string;
  updatedAt: string;
}

interface McpLaunchAttempt {
  id: string;
  requestedBy: string;
  sessionKey: string;
  status: 'draft' | 'spawning' | 'active' | 'cancelled' | 'failed';
  cleanupStatus: 'not-needed' | 'pending' | 'completed' | 'failed';
  errorCode?: string;
  createdAt: string;
  updatedAt: string;
}

interface Assignment {
  id: string;
  sourceSessionKey: string | '0';
  targetSessionKey: string;
  promptHash: string;
  promptPreview: string;
  deliveryMode: 'paste' | 'submit';
  status: 'created' | 'resolved' | 'delivered' | 'acknowledged' | 'completed' | 'failed' | 'cancelled';
  auditId: string;
}
```

## 상태기계

### MCP session binding

```text
live -> closing -> closed
live -> retired
live -> failed
closing -> closing-failed
closed -> retired
failed -> retired
```

### Agent runtime status

```text
unknown -> starting -> ready -> busy -> waiting_input -> ready
starting -> failed
busy -> completed -> ready
```

### Message delivery

```text
draft -> resolved -> policy_checked -> delivered -> acknowledged -> completed
                                   -> rejected
                         delivered -> failed
```

### Port rebind

```text
idle -> validating -> starting_new_listener -> draining_old_listener -> active
                                      -> rollback -> active
                                      -> failed
```

## 수락 기준

- `buildergate.session.whoami`가 UUID `sessionId`와 stable `sessionKey`를 반환한다.
- 새 MCP-launched 세션은 `leaderSessionKey`를 조회할 수 있다.
- 사용자가 하단 상태 표시줄 왼쪽에서 alias를 바꾸면 MCP list/search 결과에 반영된다.
- MCP `set_alias`가 같은 UI 이름을 변경하고 모든 client에 `tab:updated`가 전파된다.
- `빌더 게이트` alias 검색이 단일 결과이면 메시지 전달이 성공한다.
- 모호한 alias는 target 후보 목록과 함께 실패한다.
- 기본 MCP endpoint는 외부 IP에서 접근할 수 없다.
- whitelist를 켠 경우에도 whitelist 밖 IP, 잘못된 Origin, 잘못된 token은 거부된다.
- 포트 변경은 앱 프로세스를 재시작하지 않고 MCP listener만 새 포트로 이동한다.
- self-close는 leader 없는 세션에서 거부되고, leader 있는 세션에서 tool response 이후 tab/session을 닫는다.
