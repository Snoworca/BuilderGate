# MCP 세션 오케스트레이션 연구 인덱스

작성일: 2026-07-09

## 결론 요약

요구사항은 현재 구조에서 구현 가능하다. 다만 기존 REST/WS/세션 서비스에 MCP 로직을 직접 끼워 넣으면 입력 안정성, 세션 종료, 보안 정책이 빠르게 오염된다. 구현은 별도 MCP control-plane을 만들고, 실제 작업은 기존 `WorkspaceService`, `SessionManager`, 향후 분리할 `SessionInputGateway`를 통해 위임하는 방식이 가장 안전하다.

핵심 결정은 다음과 같다.

- HTTP MCP는 별도 `McpHttpServerController`가 관리하는 독립 listener로 둔다. 기본 bind는 `127.0.0.1`이고, 도구 다이얼로그의 보안 탭에서 외부 IP/CIDR whitelist를 켰을 때만 외부 접근을 허용한다. 외부 whitelist 모드는 TLS 또는 TLS reverse proxy가 검증된 경우에만 활성화한다.
- 현재 `SessionManager`의 `sessionId`는 이미 `uuidv4()`로 생성된다. MCP는 이 값을 그대로 `sessionId`로 노출하되, restart/orphan recovery로 바뀌는 런타임 ID이므로 안정 주소용 `sessionKey`를 함께 도입한다.
- 사용자가 하단 상태 표시줄 왼쪽을 더블클릭해 바꾸는 세션명은 MCP의 primary alias로 정의한다. MCP가 alias를 바꿀 때도 같은 `WorkspaceTab.name`/`nameSource='user'` 경로를 사용한다.
- 세션 검색은 MCP tool 이름이 자연어를 처리하는 것이 아니라, 코딩 에이전트가 사용자 문장을 이해한 뒤 `buildergate.session.search`와 `buildergate.message.send`를 순서대로 호출하는 모델이다.
- MCP 서버/tool 간 직접 메시징은 이번 범위의 요구사항이 아니다. 지원 모델은 코딩 에이전트가 `buildergate.session.search`, `buildergate.message.send`, `reply_to_leader`를 호출하고, 실제 전달은 terminal/session orchestration과 shared input gateway가 수행하는 방식이다.
- 코딩 에이전트 명령, 별칭, 기본값, 킥오프 프롬프트, 웹훅, MCP 포트/보안 설정은 일반 Settings 화면이 아니라 Header Tools 메뉴의 새 다이얼로그에서 관리한다.
- 외부 웹훅은 사용자의 요구대로 query key를 지원하되, 키는 고엔트로피 난수로 생성하고 hash만 저장하며 로그에는 query string을 남기지 않는다. 헤더 기반 키도 병행 지원한다.
- 자기 종료는 `leaderSessionKey`가 있는 MCP-launched 세션에서만 허용한다. 응답 전송을 보장하기 위해 즉시 kill이 아니라 짧은 지연 후 tab/session 종료를 예약한다.

| 2 | MCP로 자신의 UUID sessionId 조회 | 01, 03, 04 | 가능. 기존 `SessionManager` UUID 사용, stable `sessionKey` 병행 |
| 3 | MCP로 다른 세션 목록과 사용자 alias 조회 | 01, 03, 04, 05 | 가능. `WorkspaceTab.name`을 primary alias로 사용 |
| 4 | 세션 검색 후 메시지 전달 | 03, 04 | 가능. `search -> resolve -> send` tool chain |
| 4a | MCP 서버/tool 간 직접 메시징 | 03, 04 | 비요구사항. 코딩 에이전트가 tool chain을 호출하고 delivery는 terminal/session orchestration으로 수행 |
| 5 | 코딩 에이전트 명령/별칭/기본값 정의 | 03, 05, 07 | 가능. `AgentCommandProfileService` 신규 도입 |
| 6 | MCP 연결 세션/상태 관리 | 01, 04, 06 | 가능. `McpSessionRegistryService`와 explicit status update |
| 7 | MCP 포트 변경 시 서버 rebind, 프로세스 restart 금지 | 03, 07 | 가능. `McpHttpServerController.rebind()` 필요 |
| 8 | 다른 세션 열기/닫기, leader id, reply | 03, 04, 07 | 가능. leader linkage와 deferred self-close 필요 |
| 9 | 원문 요구사항 결번 | - | 사용자 요구사항 번호에 9번이 없어 결번으로 보존 |
| 10 | 킥오프 프롬프트 설정 및 MCP-launched 세션 첫 주입 | 04, 05 | 가능. launch transaction에 kickoff step 추가 |
| 11 | query key 웹훅, prompt param, caller id `0` | 03, 06 | 가능. 별도 webhook capability와 redaction 필요 |
| 12 | MCP로 세션 alias 설정 | 03, 04, 05 | 가능. workspace tab rename 경로 재사용 |
| 13 | creator/leader 있는 세션만 self-close | 04, 06 | 가능. actor guard와 scheduled close 필요 |

## 공식 MCP 근거

확인일: 2026-07-09.

- MCP Transports: https://modelcontextprotocol.io/specification/2025-11-25/basic/transports
- MCP Tools: https://modelcontextprotocol.io/specification/2025-11-25/server/tools
- MCP Security Best Practices: https://modelcontextprotocol.io/docs/tutorials/security/security_best_practices

공식 사양상 MCP는 JSON-RPC 기반이며 메시지는 UTF-8이어야 한다. Streamable HTTP는 단일 endpoint가 `POST`와 `GET`을 지원하는 구조다. 로컬 HTTP 서버는 DNS rebinding 방지를 위해 `Origin` 검증, 인증, localhost bind가 요구된다. Tool은 `tools/list`로 발견되고 `tools/call`로 호출된다.
