# MCP SRS 2차 검증 개선사항 취합

작성일: 2026-07-09

## 1. 검증 범위

- 대상 연구 문서: `docs/research/mcp/00.index.md` 및 하위 01~07 문서
- 대상 SRS: `docs/spec/40.mcp-session-orchestration.srs.md`
- 선행 개선사항 문서: `docs/research/2026-07-09.mcp-srs-validation-improvements.md`

## 2. 판정 요약

2차 5개 서브에이전트 검증 결과, 보안/운영, 세션 오케스트레이션, SRS 품질 관점은 추가 개선사항이 없었다.
프론트엔드/REST API 관점과 요구사항 추적성 관점에서 아래 5건을 타당한 추가 개선사항으로 채택한다.

## 3. 채택 개선사항

| ID | 개선사항 | 반영 대상 |
| --- | --- | --- |
| IMP2-01 | live sessions 관리 UI가 사용하는 action REST 계약을 명시한다. `GET /api/mcp-control/sessions?query=...`, `POST /api/mcp-control/sessions/search-test`, `POST /api/mcp-control/sessions/:sessionKey/reply-test`, `POST /api/mcp-control/sessions/:sessionKey/close`의 요청, 응답, 오류, audit, close confirmation 계약을 고정한다. | 신규 SRS 요구사항 |
| IMP2-02 | webhook 관리 목록 API를 명시한다. `GET /api/mcp-control/webhooks`와 `mcpControlApi.listWebhooks()`는 masked key, bound target/profile/mode/scope, last used, expiry, revoked, rate-limit 상태만 반환하고 full key/full URL은 반환하지 않는다. | 신규 SRS 요구사항 |
| IMP2-03 | agent profile `displayName` 검증 계약을 명시한다. required/trim/length/control-char 정책과 duplicate displayName 허용 여부를 고정하고, field error shape에 포함한다. | 신규 SRS 요구사항 |
| IMP2-04 | webhook key를 query와 header 양쪽에서 받을 때의 수용 기준을 명시한다. header key 단독 요청, query/header 동일 키, query/header 불일치 거부, redaction/audit/rate-limit 동일 적용을 고정한다. | 신규 SRS 요구사항 |
| IMP2-05 | agent status enum과 invalid status 거부 정책을 명시한다. 최소 `unknown`, `starting`, `ready`, `busy`, `waiting_input`, `completed`, `failed`를 계약화하고 list/search/whoami 반영을 고정한다. | 신규 SRS 요구사항 |

## 4. 제외 항목

- `REL-BGSTAB-005` draft warning은 기존 BGSTAB 범위의 미해결 항목이므로 MCP SRS 개선사항에서 제외한다.
- MCP 구현이 아직 코드에 없다는 사실은 이번 작업의 SRS planned 상태와 모순되지 않으므로 개선사항에서 제외한다.
