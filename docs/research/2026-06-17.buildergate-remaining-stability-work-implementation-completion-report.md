# BuilderGate Remaining Stability Work Implementation Completion Report

작성일: 2026-06-17

대상 계획: `docs/research/2026-06-17.buildergate-remaining-stability-work-implementation-plan.md`

## 완료 범위

이번 구현은 계획 문서의 Phase 0-4, Phase 5 split-shadow foundation, PR-G split WebSocket traffic migration까지 완료했다.

- Phase 0: 프론트엔드 runtime resource limits typed getter와 변경 알림을 추가했다.
- Phase 1: browser WebSocket `bufferedAmount` 기반 input backpressure와 low-water 복구 경로를 추가했다.
- Phase 2: visible xterm write scheduler, queue byte cap, overflow 복구 요청, stale-terminal 차단 상태를 추가했다.
- Phase 3: terminal snapshot/localStorage budget, max entries, tombstone TTL, legacy marker cleanup을 runtime config에 연결했다.
- Phase 4: live terminal runtime residency LRU와 visible pinning, hidden TTL self-refresh를 추가했다.
- Phase 5: `realtime.wsTransportMode` 설정, runtime config 노출, split-shadow control/output handshake, pair token 검증, duplicate output socket replacement를 추가했다.
- PR-G: `split` mode에서 terminal output, replay snapshot, screen repair payload를 paired output socket으로 라우팅하고 input/resize/ACK/control 메시지는 control socket에 유지했다.
- PR-G 보강: output socket close/backpressure/queue overflow 시 queued/current terminal payload를 control socket으로 reroute하고, ACK 전 full split output failure는 control socket으로 fresh snapshot replay를 강제 발행한다. degraded/fallback snapshot recovery가 rerouted output 또는 screen-repair pending output을 덮어 쓰는 경우를 막기 위해 해당 output을 recovery replay queue에 보존해 ACK 이후 다시 flush한다. `split-shadow`는 관찰 모드 격리를 유지하도록 output close가 control replay를 강제로 유발하지 않게 고정했다. frontend는 output socket close 후 control fallback subscription을 허용한다.

## 안정화 효과

구현 후 입력 지연과 메모리 증가를 유발하던 주요 무제한 누적 경로에 상한과 복구 정책이 생겼다.

- 브라우저 송신 buffer가 high-water를 넘으면 입력 메시지만 차단하고 control 메시지는 보존한다.
- visible terminal 출력은 scheduler queue cap을 넘으면 무한 누적하지 않고 authoritative snapshot/repair 경로로 복구한다.
- localStorage snapshot은 전체 budget, per-snapshot cap, entry count, tombstone TTL을 적용한다.
- 숨겨진 terminal runtime은 LRU로 unmount되어 장시간 사용 중 DOM/xterm/runtime 객체가 계속 늘어나는 경로를 줄인다.
- split mode는 terminal output/replay/repair payload를 paired output channel로 분리하고, output channel이 없거나 준비되지 않거나 실패하면 control socket fallback 및 authoritative snapshot recovery를 유지한다.
- fallback/degraded recovery snapshot은 output failure 직전 control로 reroute된 terminal output과 screen-repair pending output을 replay pending queue에 보존해 snapshot 적용 이후 화면 손실을 줄인다.
- split-shadow mode는 control 경로를 기준 동작으로 유지하면서 output channel pairing/handshake만 검증하고, output channel failure가 control replay 상태를 교란하지 않는다.
- unified mode는 기본값으로 유지된다.
