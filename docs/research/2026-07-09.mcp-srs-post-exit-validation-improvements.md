# MCP SRS Post-Exit Validation Improvements

Date: 2026-07-09

## Summary

IMP4 반영 후 독립 검증에서 남은 해석 충돌 3건을 확인했다. 모두 구현 누락이 아니라 SRS/연구 문서의 우선순위와 wording 정합성 문제다.

## Accepted Improvements

| ID | Improvement | Application |
| --- | --- | --- |
| IMP5-01 | `webhookKeyHeaderName` 금지 목록은 일부 `X-Forwarded-*` 예시가 아니라 모든 `X-Forwarded-` prefix header를 금지해야 한다. | SRS MCP 보강 요구사항 |
| IMP5-02 | webhook audit/status의 `promptPreview` 계약과 충돌하지 않도록 raw prompt 저장 허용 문구를 제거하고 preview/hash/metadata 중심으로 좁힌다. | 연구 문서 갱신 |
| IMP5-03 | webhook key/full URL 노출은 최초 생성만이 아니라 즉시 생성 또는 회전 response에서만 허용된다는 테스트 wording으로 정리한다. | 연구 문서 갱신 |
| IMP5-04 | domain model 연구 문서의 `bindingLifecycle` 타입과 상태기계를 최종 wire enum(`live`, `closing`, `closed`, `retired`, `failed`, `closing-failed`)과 맞춘다. | 연구 문서 갱신 |

## Rejected Items

None.
