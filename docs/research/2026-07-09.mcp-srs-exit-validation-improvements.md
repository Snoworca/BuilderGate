# MCP SRS 종료 검증 개선사항 취합

작성일: 2026-07-09

## 1. 검증 범위

- 대상 연구 문서: `docs/research/mcp/00.index.md` 및 하위 01~07 문서
- 대상 SRS: `docs/spec/40.mcp-session-orchestration.srs.md`
- 선행 개선사항 문서:
  - `docs/research/2026-07-09.mcp-srs-validation-improvements.md`
  - `docs/research/2026-07-09.mcp-srs-second-validation-improvements.md`
  - `docs/research/2026-07-09.mcp-srs-final-validation-improvements.md`

## 2. 판정 요약

최종 종료 검증에서 요구사항 추적성과 세션 오케스트레이션 관점은 추가 개선사항이 없었다.
보안/운영, 프론트엔드/REST API, SRS 품질 관점에서 아래 항목을 타당한 추가 개선사항으로 채택한다.

## 3. 채택 개선사항

| ID | 개선사항 | 반영 대상 |
| --- | --- | --- |
| IMP4-01 | whitelist 거부 계열 안정 error code를 명시한다. 빈 whitelist는 `MCP_WHITELIST_EMPTY`, whitelist 불일치는 `MCP_WHITELIST_DENIED`로 관측 가능해야 한다. | 신규 SRS 요구사항 |
| IMP4-02 | webhook `promptPreview`의 redaction/normalization 계약을 명시한다. 원문 prompt를 저장하지 않고, 설정된 최대 길이로 자르며, 제어문자와 개행을 단일 공백으로 정규화한다. | 신규 SRS 요구사항 |
| IMP4-03 | 운영 상태에 redacted recent audit events 노출 계약을 추가한다. | 신규 SRS 요구사항 |
| IMP4-04 | `webhookKeyHeaderName`의 금지 header 목록을 명시한다. `Authorization`, `Cookie`, `Forwarded`, `X-Forwarded-*` 등 auth/proxy 경계 header는 사용할 수 없어야 한다. | 신규 SRS 요구사항 |
| IMP4-05 | webhook full key/full URL 1회 노출 문구를 create와 rotate 모두로 정렬한다. 기존 create-only 문구는 create-or-rotate one-time secret response로 해석되도록 명시한다. | 신규 SRS 요구사항 및 연구 문서 갱신 |
| IMP4-06 | frontend API type 예시의 `agentStatus`를 고정 union 타입으로 정리하고, `webhookKeyHeaderName` config 필드를 추가한다. | 연구 문서 갱신 |
| IMP4-07 | `bindingLifecycle` wire enum을 `live`, `closing`, `closed`, `retired`, `failed`, `closing-failed`로 고정하고 연구 예시의 `active` 표현을 정리한다. | 신규 SRS 요구사항 및 연구 문서 갱신 |
| IMP4-08 | 기존 넓은 표현인 `INPUT_REJECTED_REPLAY_PENDING or equivalent observable status`와 `confirmation token or equivalent UI confirmation marker`에 대해, 보강 요구사항이 우선하는 tightening 계약임을 명시한다. | 신규 SRS 요구사항 |

## 4. 제외 항목

- 기존 `REL-BGSTAB-005` draft warning은 BGSTAB 범위의 기존 미해결 항목이므로 MCP SRS 개선사항에서 제외한다.
- MCP 기능이 아직 구현되지 않았다는 사실은 현재 SRS가 planned 상태인 것과 모순되지 않으므로 개선사항에서 제외한다.
