# MCP SRS 최종 검증 개선사항 취합

작성일: 2026-07-09

## 1. 검증 범위

- 대상 연구 문서: `docs/research/mcp/00.index.md` 및 하위 01~07 문서
- 대상 SRS: `docs/spec/40.mcp-session-orchestration.srs.md`
- 선행 개선사항 문서:
  - `docs/research/2026-07-09.mcp-srs-validation-improvements.md`
  - `docs/research/2026-07-09.mcp-srs-second-validation-improvements.md`

## 2. 판정 요약

최종 5개 서브에이전트 검증 라운드에서 요구사항 추적성 관점은 추가 개선사항이 없었다.
보안/운영, 세션 오케스트레이션, 프론트엔드/REST API, SRS 품질 관점에서 아래 항목을 타당한 추가 개선사항으로 채택한다.

## 3. 채택 개선사항

| ID | 개선사항 | 반영 대상 |
| --- | --- | --- |
| IMP3-01 | webhook key와 URL 노출 정책을 create와 rotate 모두에서 1회 응답으로 정렬하고, list/persist/history에는 full key/full URL을 남기지 않는 계약으로 고정한다. | 신규 SRS 요구사항 |
| IMP3-02 | webhook header key의 기본 header 이름 또는 설정 필드를 명시하고, query/header redaction, audit, rate-limit 처리의 동일성을 고정한다. | 신규 SRS 요구사항 |
| IMP3-03 | webhook rate-limit 기준과 안정 error code를 보강한다. per-key/per-effective-client-IP 기준, window/burst 설정 필드, `WEBHOOK_RATE_LIMITED`, `WEBHOOK_PROMPT_TOO_LARGE`, transport/origin 계열 denial code를 계약화한다. | 신규 SRS 요구사항 |
| IMP3-04 | `DELETE /api/mcp-control/webhooks/:id`의 revoke semantics, 성공 응답, already-revoked 처리, list 반영 상태를 명시한다. | 신규 SRS 요구사항 |
| IMP3-05 | `buildergate.session.list`의 `includeSelf` 생략 기본값을 `true`로 고정하고, `includeSelf=false` 동작을 명시한다. | 신규 SRS 요구사항 |
| IMP3-06 | `buildergate.session.search` 검색 기준에 alias, tab name, agentKind, role, cwd, recovery command를 명시한다. | 신규 SRS 요구사항 |
| IMP3-07 | self-close deferred job 실패 시 leader가 live 상태이고 reply 가능하면 실패 메시지를 leader에게 전달하는 조건을 명시한다. | 신규 SRS 요구사항 |
| IMP3-08 | replay pending 입력 거부 error code를 `INPUT_REJECTED_REPLAY_PENDING`로 고정한다. | 신규 SRS 요구사항 |
| IMP3-09 | live-session close REST 요청의 확인 shape를 `confirmClose`, `expectedSessionKey`, `confirmationNonce`로 고정하고 실패 code를 명시한다. | 신규 SRS 요구사항 |
| IMP3-10 | 연구 문서의 REST endpoint 목록과 frontend API type 예시를 SRS 보강 결과와 맞춘다. | 연구 문서 갱신 |
| IMP3-11 | 연구 테스트 계획의 empty whitelist 정책을 SRS와 동일하게 reject-only로 정리한다. | 연구 문서 갱신 |

## 4. 제외 항목

- 기존 `REL-BGSTAB-005` draft warning은 BGSTAB 범위의 기존 미해결 항목이므로 MCP SRS 개선사항에서 제외한다.
- MCP 기능이 아직 구현되지 않았다는 사실은 현재 SRS가 planned 상태인 것과 모순되지 않으므로 개선사항에서 제외한다.
