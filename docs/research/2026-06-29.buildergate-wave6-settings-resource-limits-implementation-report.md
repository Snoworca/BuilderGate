Wave6 implements the browser Settings path for selected `resourceLimits` fields only. It does not change cleanup policy, terminal runtime residency behavior, visible output scheduling, browser WebSocket backpressure behavior, split WebSocket behavior, or broad process termination behavior.

## Boundary

The worktree already contains unrelated Wave7/recovery changes, including terminal residency, split WebSocket, visible output scheduler, recovery option, and related frontend/runtime files. Those files are outside Wave6 scope and were not used as Wave6 completion evidence. They must be committed or reviewed separately.

Wave6 explicitly keeps these categories out of the Settings UI and Settings PATCH contract:

- `resourceLimits.telemetry.*`
- `stabilityModes.*`
- `resourceLimits.headless.writeLagWarnMs`
- `resourceLimits.headless.writeBatchMaxBytes`
- `resourceLimits.headless.overflowPolicy`
- `resourceLimits.terminal.visible*`
- `resourceLimits.terminal.scrollbackLines`
- `resourceLimits.ws.perClientControlQueueMaxBytes`
- `resourceLimits.ws.outputCoalesceWindowMs`

The server capability snapshot now marks the reserved fields unavailable, so direct Settings PATCH attempts are rejected instead of silently accepted.

## Review Loop

Independent review findings were addressed as follows:

- Reserved resource fields were initially still accepted by direct PATCH. Fixed by marking reserved Wave6 keys unavailable in `RuntimeConfigStore` and validating all patch keys before filtering no-op changes.
- No-op selected resource PATCH initially reported changed keys. Fixed by comparing previous and merged values before persistence/apply.
- Successful Settings save summary could remain visible after a later failed save. Fixed by clearing the summary at save start and adding E2E coverage.
- `stabilityModes.*` was initially still advertised/accepted. Fixed by marking all stability mode keys unavailable for Wave6 Settings and adding a direct PATCH rejection regression test.
- Dirty worktree boundary was documented here so Wave6 completion evidence is limited to the Wave6-scoped files above.

## Result

Wave6 Settings resource limit work is implemented against `FR-BGSTAB-013`. The selected resource limits can be inspected and patched through Settings, reserved fields are hidden or rejected, public runtime config remains browser-safe, and the existing save/error/apply-summary flow is preserved.
