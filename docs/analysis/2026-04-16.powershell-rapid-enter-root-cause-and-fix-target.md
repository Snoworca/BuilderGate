# PowerShell Rapid Enter Root Cause And Fix Target

작성일: 2026-04-16
대상 환경: `https://localhost:2002`

## 문제 정의

재현 절차:

1. PowerShell 세션
2. `A` 입력 후 즉시 `Enter`
3. 이를 매우 빠르게 반복

관찰 결과:

- 출력이 줄 단위로 아래로 쌓이지 않음
- 프롬프트와 에러 메시지 조각이 서로 겹침
- `CommandNotFoundException` 텍스트가 분리되어 끼어듦

## 직접 재현 증거

실제 앱에서 재현 후 브라우저 debug capture와 서버 debug capture를 함께 확인했다.

확인한 사실:

- 브라우저 입력 전송 정상
  - `xterm_data_emitted`
  - `ws_input_sent`
- 서버 입력 수신 정상
  - `input`
- 동일 재현에서 `replayEvents = 0`
  - snapshot/replay 경로가 주된 개입 경로가 아님
- 서버 `raw_output` preview 안에 이미 다음 redraw 조각이 존재
  - `\x1b[?25l`
  - `\x1b[24;19HA`
  - `\x1b[24;80H`
  - `\x1b[K`
  - `CommandNotFoundEx` / `ception` 분리 조각

즉 이번 재현은 프런트 replay/snapshot이 화면을 새로 망가뜨린 것보다, 서버가 관측한 PTY raw output 단계에서 이미 upstream redraw burst가 드러난다고 보는 편이 맞다.

## root cause 재판정

### 주된 실패 레이어

가장 설득력 있는 설명은 다음이다.

- `PowerShell/PSReadLine + ConPTY`가 빠른 입력/Enter 반복에서 current-line redraw와 prompt repaint ANSI burst를 만든다.
- BuilderGate는 그 burst를 semantic하게 분리하지 않고 headless/live output 경로로 그대로 통과시킨다.
- 이전 redraw가 완전히 가라앉기 전에 다음 입력이 들어오면 줄이 세로로 겹쳐 보인다.

### 주원인이 아닌 것

- `shell integration`
  - PowerShell은 `OSC133` 스크립트를 쓰지 않음
- 프런트 중복 입력 전송
  - Enter는 `term.onData()` 한 경로만 탐
- steady-state replay/snapshot 중복
  - 이번 재현에서 `replayEvents = 0`

### 증폭 요인

다음은 주원인보다 증폭 요인에 가깝다.

- PTY 기본 크기에서 시작 후 나중에 `fit/resize`
- `restorePending`과 snapshot ACK 경계
- prompt override의 per-prompt file write

## 시니어 재검증 결과

시니어 3명은 모두 아래에 동의했다.

1. `replay/snapshot/restore`는 1차 수리 레이어가 아니다.
2. 현 구조를 유지하려면 upstream redraw 생성/소비 경계를 먼저 고쳐야 한다.
3. 문제의 중심은 `PowerShell/PSReadLine on ConPTY`다.

세부 의견 차이:

- 강경안: PowerShell에서 `ConPTY` 사용 자체를 끊는다.
- 보수안: `ConPTY`는 유지하되 `ConPTY ↔ renderer` 계약층을 바로잡는다.

공통분모만 남기면 다음 문장이 맞다.

> 현재 구조를 보존한 채 근본 해결을 노린다면, 1차 수리 원인은 `PowerShell/PSReadLine on ConPTY`의 redraw 생성/소비 경계다.
