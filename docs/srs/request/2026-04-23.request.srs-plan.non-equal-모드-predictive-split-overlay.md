## 요약

`equal`을 제외한 `none/focus/auto` 모드에서 창 드래그 시, 실제 split 결과를 미리 예측할 수 있는 predictive split overlay를 도입한다.

## 목표

- 사용자가 non-equal 모드에서도 “어디에 drop되면 어떤 분할 결과가 생기는지”를 drag 중에 직관적으로 알 수 있어야 한다.
- 이번 범위에서는 `split semantics`를 유지한다.
- `equal` full-cell reorder guide는 그대로 유지하고 회귀시키지 않는다.

## 변경 유형

- enhancement

## 제약사항

- `equal`은 기존 full-cell reorder semantics를 유지한다.
- `none/focus/auto`는 이번 범위에서 reorder로 바꾸지 않는다.
- non-equal guide는 reorder처럼 보이면 안 된다.
- root edge drop zone도 시각적으로 드러나야 한다.
- `focus/auto`의 post-drop behavior는 mode별 invariant로 고정하고, drag guide와 실제 immediate split 결과가 모순되지 않아야 한다.
- 기본 구현 경로는 `thin vendor patch + app-owned predictive overlay`이며, CSS-only는 축소안 또는 fallback일 뿐 기본안이 아니다.
- 기존 Playwright 회귀(`grid-equal-mode.spec.ts`)와 `equal` UX를 깨뜨리면 안 된다.

## 메인 해석

- 이번 작업의 본질은 non-equal 모드의 drag semantics를 바꾸는 것이 아니라, 기존 split drop 결과를 더 정확히 예고하는 것이다.
- Option B의 기본 경로는 새로운 tree 시뮬레이션 엔진 대신, thin vendor patch로 hover metadata를 안정적으로 노출하고 앱이 소유한 predictive overlay DOM으로 immediate split preview를 그리는 것이다.
- CSS-only restyling은 root/cell 구분, hover metadata 관찰성, root edge commit assertion을 동일하게 보장할 수 있을 때만 fallback 또는 축소안으로 허용한다.

## Option B 고정 계약

- hover metadata source: thin vendor patch가 hovered target의 `kind(root-edge/cell)`, `edge`, `rect`, `target id` 또는 동등한 commit target metadata를 앱으로 전달해야 한다.
- root/cell 구분 방식: app은 CSS selector 추론이 아니라 명시적 metadata discriminator로 root edge와 cell split target을 구분해야 한다.
- overlay DOM ownership: predictive overlay DOM은 app이 렌더링하고 제어한다. non-equal에서 vendor preview는 숨기거나 중립화할 수 있지만, 의미 해석의 주체는 app이어야 한다.
- fallback exit criteria: CSS-only 경로는 `FR-1/FR-2/FR-5` 수준의 보장을 유지하면서 Playwright가 root edge 실제 drop commit과 최종 tree 또는 DOM geometry를 검증할 수 있을 때만 허용한다. 이 조건을 못 맞추면 기본 경로를 유지한다.
- auto status-change test path: 결정론적 검증은 test-only status injection hook `window.__PM_TEST_API__.grid.injectAutoStatusChange(...)`를 기본 경로로 사용한다. Playwright 공용 helper `runNonEqualSplitDragCase`가 drag hold 중과 drop/cancel 직후 이 hook을 호출해 `TC-NE-5/6`를 수행한다.
- auto status-change hook runtime rule: 이 글로벌 훅은 dev/test 전용이다. namespace는 `window.__PM_TEST_API__ ??= {}`와 `window.__PM_TEST_API__.grid ??= {}` 방식으로 idempotent 초기화하고, `injectAutoStatusChange`는 `MosaicContainer` unmount 또는 workspace change 시 제거한다. `grid` namespace가 비면 함께 정리하고, `Window` 확장은 TypeScript `declare global` 또는 동등한 안전한 narrowing으로만 접근한다. production 동작과 수동 검증 경로는 이 hook을 전혀 사용하지 않는다.

## 모드별 post-drop invariant

| 모드 | drop 직후 raw split 유지 | 후속 트리거 | drag 중 auto timer/status-change | overlay 보장 시점 | FR / AC / Test 매핑 |
|------|--------------------------|------------|----------------------------------|-------------------|---------------------|
| `none` | 유지됨. drop commit 결과와 stabilized layout이 동일해야 한다. | 없음 | 해당 없음 | `immediate = stabilized` | `FR-1`, `FR-4` / `AC-1` / `TC-NE-1`, `TC-NE-2` |
| `focus` | 유지됨. 먼저 raw split commit을 만든 뒤 기존 focus 규칙이 후속 적용될 수 있다. | 성공한 primary-button drop commit이 drag 종료 후 확정될 때만 focus 재적용. cancel/outside/self/right-click/non-primary에서는 재적용 금지 | 해당 없음 | `immediate`만 보장. stabilized focus layout은 별도 후속 결과 | `FR-1`, `FR-4`, `FR-5` / `AC-2` / `TC-NE-3`, `TC-NE-4`, `TC-NE-7`, `TC-NE-8` |
| `auto` | 유지됨. drop 직후 raw split commit을 만든 뒤 auto 규칙이 후속 적용될 수 있다. | 없음 | drag active 동안 timer/status-change 기반 재배치는 일시 정지하고 drop/cancel 후 재개 | `immediate`만 보장. stabilized auto layout은 재개 후 별도 결과 | `FR-1`, `FR-4`, `FR-5` / `AC-3` / `TC-NE-5`, `TC-NE-6`, `TC-NE-7`, `TC-NE-8` |

## 수용 기준 요약

- `AC-1`: `none`의 cell/root edge overlay rect는 실제 drop commit 결과와 stabilized layout에 모두 일치해야 한다.
- `AC-2`: `focus` overlay는 immediate split result만 예고하며, focus 재적용은 성공한 primary drop commit 이후에만 발생해야 한다.
- `AC-3`: `auto`는 drag 중 timer/status-change를 일시 정지해야 하며, overlay는 immediate split result만 예고해야 한다. 이 계약은 `runNonEqualSplitDragCase`가 호출하는 test-only status injection hook으로 `TC-NE-5/6`에서 결정론적으로 검증되어야 한다.
- `AC-4`: root edge는 screenshot만이 아니라 실제 drop commit 후 persisted tree 또는 최종 DOM geometry로 검증되어야 한다.
- `AC-5`: `outside/self/right-click/non-primary`는 `none/focus/auto` 모두 no-op으로 자동화되어야 한다.
- `AC-6`: `equal` guide와 non-equal overlay는 계속 분리되고, equal 회귀 스위트가 그대로 통과해야 한다.
