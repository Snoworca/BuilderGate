# 요구사항 정리 문서: React Mosaic 기반 그리드 레이아웃 시스템

**작성일**: 2026-04-02
**유형**: PRD-QNA (기능 업그레이드)
**대상 프로젝트**: BuilderGate

---

## 원본 요구사항

기존 BuilderGate 프로젝트에 React Mosaic 기반 그리드 레이아웃 시스템을 적용한다.

---

## 의도 요약

BuilderGate의 그리드 모드 터미널 레이아웃을 React Mosaic 라이브러리로 전환하여, 드래그 앤 드롭 기반 창 재배치, 사용자 크기 조절, 자동 크기 모드(균등/포커스/오토)를 지원하고, 세션별 컨텍스트 메뉴를 통한 세션 관리(생성/닫기/복사/붙여넣기)를 제공한다.

---

## 핵심 목표

1. React Mosaic을 활용한 유연한 타일형 그리드 레이아웃 구현
2. 세션 터미널 컨텍스트 메뉴를 통한 직관적 세션 관리
3. 3가지 크기 모드(균등/포커스/오토)로 다양한 작업 흐름 지원
4. 레이아웃 상태의 브라우저 DB 영속화

---

## 제약 조건

- 기존 컨텍스트 메뉴 컴포넌트(`ContextMenu`)를 재사용
- TUI(터미널 UI) 깨짐 방지를 위한 resize 처리 필수
- 최소 창 크기 제한으로 과도한 축소 방지
- 레이아웃은 절대좌표가 아닌 실수 기반 상대 비율로 저장
- 다른 브라우저에서는 기본 균등 그리드로 폴백

---

## 대상 사용자

- 다수의 코딩 에이전트를 병렬 운용하는 개발자
- 여러 터미널 세션을 동시에 모니터링하는 사용자

---

## 구조화된 요구사항

### FR-1: 세션 터미널 컨텍스트 메뉴
- 우클릭(데스크톱) / 롱프레스(모바일)로 컨텍스트 메뉴 표시
- 메뉴 항목: 새 세션 열기, 세션 닫기, 복사, 붙여넣기
- 복사는 텍스트 선택 시에만 활성화
- 기존 `ContextMenu` 컴포넌트 재사용

### FR-2: 세션 생성/닫기
- 새 세션 열기: 현재 세션의 CWD 기반으로 생성
- 그리드 모드: 세션 추가 후 해당 세션으로 포커스 이동
- 탭 모드: 탭 추가 후 해당 탭으로 전환
- 세션 닫기: 모달 확인창 표시 후 삭제
- 닫기 후 마지막(가장 최근) 세션으로 포커스 이동

### FR-3: 크기 모드 아이콘 박스
- 각 세션 왼쪽 상단 모서리에 마우스 호버 시 표시
- 스타일: 회색 반투명, 둥근 모서리, 가로가 긴 직사각형
- 아이콘 버튼 3개:
  1. **균등 분할**: 모든 창 동일 크기
  2. **포커스 확대**: 현재 창만 크고 나머지 작게
  3. **오토**: 기본 균등 + 사용자 입력 대기 세션 자동 확대

### FR-4: 오토 모드 상세
- 기본 상태: 모든 세션 균등 크기
- 사용자 입력 대기 세션 감지 시 해당 세션 확대
- 입력 대기 세션이 여러 개면 해당 세션들만 균등 확대

### FR-5: React Mosaic 레이아웃
- 드래그 앤 드롭으로 창 위치 변경
- 사용자가 직접 크기 조절 가능
- 크기 변경 시 TUI 깨지지 않도록 resize 이벤트 처리
- 각 창에 최소 너비/높이 제한

### FR-6: 레이아웃 영속화
- 실수 기반 상대 비율로 변환하여 저장
- 브라우저 DB(IndexedDB 또는 localStorage)에 저장
- 같은 브라우저 재접속 시 레이아웃 복원
- 다른 브라우저에서는 기본 균등 그리드로 표시
