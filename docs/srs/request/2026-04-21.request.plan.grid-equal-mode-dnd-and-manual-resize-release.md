# 요청 정리: Grid Equal Mode DnD And Manual Resize Release

작성일: 2026-04-21

## 배경

- Grid mode는 `react-mosaic-component` 기반이라 드래그앤드롭으로 타일 위치를 바꿀 수 있다.
- 현재 `균등 모드(equal)`가 선택된 상태에서도:
  - 타일을 다른 위치에 드래그해 넣으면 실제 트리는 균등 배치가 깨지는데 모드 표시는 그대로 남는다.
  - 분할선을 수동으로 조절해도 `equal` 모드가 풀리지 않는다.

## 요구사항

1. `equal` 모드에서 타일을 드래그앤드롭해 다른 위치에 넣으면 즉시 균등 배치가 다시 적용되어야 한다.
2. 사용자가 분할선을 수동으로 조절하면 `equal` 모드는 자동으로 해제되어야 한다.
3. “타일 이동”과 “분할선 수동 리사이즈”는 구분해서 처리해야 한다.
4. 모든 모드 버튼(`equal`, `focus`, `auto`)은 토글로 동작해야 한다.
5. 이미 선택된 버튼을 다시 누르면 해당 모드는 해제되고 “아무 모드도 아님” 상태가 되어야 한다.
6. `focus`, `auto` 모드는 기존 의미를 유지해야 한다.
7. 모드 해제 상태는 저장/복원/툴바 표시까지 일관되게 반영되어야 한다.
8. `equal` 모드는 항상 격자형으로만 동작해야 한다.
9. 기본 배치 규칙은 **2행 고정**이며, 세션이 5개면 위 3개 / 아래 2개처럼 위에서 아래로 채운다.
10. 예외적으로 `equal` 버튼을 누른 시점에 화면이 세로가 더 길면 **2열 고정**으로 계산한다.
11. 이 `2행/2열` 기준은 resize 이벤트마다 다시 계산하지 않고, **equal 버튼을 누른 시점에만** 결정한다.

## 초기 판단

- 타일 이동 후 균등 재적용은 `onRelease` 또는 드롭 완료 시점에 tree를 다시 격자형으로 재구성하는 방식으로 구현 가능하다.
- 수동 분할선 리사이즈 후 `equal` 해제 결과는 `manual/custom`이 아니라 “아무 모드도 아님(`none`)”으로 정의하는 것이 최신 요구와 일치한다.
- 따라서 상태 모델은 `equal/focus/auto`에 더해 `none` 또는 `null` 기반 토글 해제 상태를 공식 지원해야 한다.
- `equal`의 핵심은 topology 보존이 아니라 **항상 2행 고정 / portrait 예외 시 2열 고정 격자 재구성**이다.
