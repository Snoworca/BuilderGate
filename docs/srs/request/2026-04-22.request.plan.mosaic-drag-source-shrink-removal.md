## User Request

Grid mode에서 창 이동 drag를 시작할 때 원본 pane이 즉시 쪼그라드는 현상을 제거한다.

## Confirmed Understanding

- 현재 shrink는 `react-mosaic-component` vendor drag-start 경로가 `hide()`를 호출하기 때문에 발생한다.
- 이 동작은 필수 요구사항이 아니다.
- 사용자가 drag를 시작하는 순간 원본 창의 geometry는 유지되어야 한다.
- drop이 완료되기 전까지 live mosaic tree가 source hide로 변형되면 안 된다.

## In Scope

- vendor drag-start 경로의 source hide 제거 또는 동등한 비파괴 전환
- split mode / reorder mode 모두에서 drag-start shrink 제거
- invalid drop/reset 경로 재정렬
- drag preview/guide는 유지
- geometry stability 회귀 테스트 추가

## Out Of Scope

- 새로운 reorder semantics 설계 변경
- equal 모드 슬롯 재배치 규칙 변경
- 모드 버튼 UI 재설계

## Requirements

- `IMPL-001` drag start 시 source pane width/height가 즉시 축소되지 않아야 한다.
- `IMPL-002` drop 전까지 live mosaic tree는 source hide를 위해 변형되지 않아야 한다.
- `IMPL-003` split mode와 equal reorder mode 모두에서 shrink가 제거되어야 한다.
- `IMPL-004` invalid drop/reset은 이전 tree를 안전하게 유지하거나 복원해야 한다.
- `IMPL-005` drag preview와 blue target guide는 유지되어야 한다.
- `IMPL-006` Desktop Chrome 회귀 테스트가 drag-start geometry stability를 검증해야 한다.
- `IMPL-007` 기존 right-click, non-primary no-op, handle-only 규칙은 유지되어야 한다.

## Risks To Address

- split reset 경로가 기존 hide 가정에 묶여 있을 수 있다.
- source와 preview가 동시에 보여 UX가 어색할 수 있다.
- vendor patch가 커져 patch-package 재현성이 흔들릴 수 있다.
