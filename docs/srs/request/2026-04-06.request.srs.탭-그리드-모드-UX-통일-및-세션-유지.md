# 요구사항 정리 문서

## 원문 보존

> 1. 그리드 모드 세션에서 마우스 오른쪽 버튼을 눌러서 컨텍스트 메뉴를 띄울 수 있습니다. 탭 모드에서도 동일하게 동작해야 합니다.
> 2. 탭 모드에서 탭을 더블클릭하여 이름을 바꿀 수 있습니다. 그리드 모드에서도 하나의 세션 이름을 더블클릭하여 이름을 바꿀 수 있어야 합니다.
> 3. 워크스페이스를 바꿨다가 다시 돌아오면 세션이 초기화 됩니다. 즉, 기존에는 스크롤바가 있었는데 모두 사라져버려요. 워크스페이스를 바꾸어도 스크롤바가 사라지지 않았으면 좋겠습니다. 즉, 위에서 입력한 내용이 그대로 유지 되었으면 합니다.

## 의도 요약

탭 모드와 그리드 모드 간 UX 기능 격차를 해소하고, 워크스페이스 전환 시 터미널 세션 내용이 유실되지 않도록 개선한다.

## 핵심 목표

1. **컨텍스트 메뉴 통일**: 그리드 모드에서 사용 가능한 우클릭 컨텍스트 메뉴를 탭 모드에서도 동일하게 제공
2. **세션 이름 변경 통일**: 탭 모드의 더블클릭 이름 변경을 그리드 모드의 MetadataRow에서도 동일하게 제공
3. **워크스페이스 전환 시 세션 유지**: 워크스페이스를 변경했다가 돌아왔을 때 터미널 스크롤백 버퍼와 입력 내용이 그대로 유지

## 제약사항

- 기존 그리드 모드와 탭 모드의 동작을 깨뜨리지 않아야 함
- 기존 컨텍스트 메뉴 항목(새 세션, 세션 닫기, 복사, 붙여넣기)은 그대로 유지

## 대상 사용자

- BuilderGate 웹 터미널 사용자 (개발자)

## 구조화된 요구사항

| # | 요구사항 | 유형 | 우선순위 |
|---|---------|------|---------|
| R1 | 탭 모드 터미널 영역에서 우클릭 시 컨텍스트 메뉴 표시 | 기능 | HIGH |
| R2 | 그리드 모드 MetadataRow의 세션 이름 더블클릭으로 이름 변경 | 기능 | HIGH |
| R3 | 워크스페이스 전환 후 복귀 시 터미널 스크롤백 버퍼 유지 | 기능 | HIGH |
