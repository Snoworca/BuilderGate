# 요구사항 정리 문서

- 작성일: 2026-04-08

## 원문 요구사항

> - Google Authenticator 도 추가합시다. 설정값에서 true/false 로 통하여 활성화 됩니다. 서버를 기동할 때 등록할 수 있는 QR 코드를 콘솔로 보여줍니다.
> - 이메일과 함께 사용할 수 있습니다. 이 경우 로그인 패스워드 입력 => 이메일 => 구글 otp 순으로 돌아갑니다.
> - 옵션을 하나 더 추가합니다. 로컬 호스트일 경우 패스워드 인증만 사용 옵션 입니다. 로컬 호스트에서는 이메일 인증과 otp 가 활성화 된 경우 사용되지 않습니다.
> - 기본값은 otp 는 활성화 되어있고 이메일은 활성화 되어있지 않습니다.

## 의도 요약

기존 이메일 OTP 2FA 시스템에 Google Authenticator(TOTP) 인증을 추가하고,
localhost 환경에서는 2FA를 모두 건너뛰는 옵션을 제공하여 개발 편의성을 높인다.

## 핵심 목표

1. Google Authenticator (TOTP/RFC 6238) 지원 추가
2. 이메일 OTP와 TOTP의 독립적 활성화/비활성화 지원
3. 두 방식 동시 사용 시 순차 인증 플로우 제공 (비밀번호 → 이메일 → TOTP)
4. localhost 환경에서 2FA 바이패스 옵션 제공
5. 서버 시작 시 콘솔 QR 코드로 간편 등록

## 제약 사항

- 기존 이메일 OTP 구현(TwoFactorService) 유지 (하위 호환)
- TOTP 라이브러리: `otplib` (외부 API 없이 오프라인 동작)
- QR 코드 콘솔 출력: `qrcode-terminal` 패키지
- TOTP secret은 서버 재시작 후에도 유지되어야 함 (영구 저장 필요)
- 단일 사용자 애플리케이션 (admin 계정 하나)

## 우선순위

기능 > 보안 > 편의성

## 대상 사용자

단일 관리자 (본인 전용 웹 터미널)

## 구조화된 요구사항 목록

### 요구사항 1: Google Authenticator TOTP 지원

#### [원문]
Google Authenticator 도 추가합시다. 설정값에서 true/false 로 통하여 활성화 됩니다.
서버를 기동할 때 등록할 수 있는 QR 코드를 콘솔로 보여줍니다.

#### [정리]
- `config.json5`에 `totp.enabled: boolean` 설정 추가
- TOTP secret을 영구 저장하여 서버 재시작 후에도 동일 secret 유지
- 서버 시작 시 콘솔에 ASCII QR 코드 출력 (Google Authenticator로 스캔 가능)
- TOTP 검증: `otplib` 라이브러리 사용 (오프라인, 무료, API 키 불필요)

#### [미정 사항 - 질문 필요]
- QR 코드를 매번 출력? 아니면 최초/미등록 시에만?
- TOTP 미등록 상태에서 로그인 요청 시 처리 방법

---

### 요구사항 2: 이메일 + TOTP 동시 사용

#### [원문]
이메일과 함께 사용할 수 있습니다. 이 경우 로그인 패스워드 입력 => 이메일 => 구글 otp 순으로 돌아갑니다.

#### [정리]
- 이메일 OTP (`twoFactor.enabled: true`) + TOTP (`totp.enabled: true`) 동시 활성화 가능
- 동시 활성화 시 인증 단계: 비밀번호 → 이메일 OTP → TOTP 순서로 모두 통과
- TOTP만 활성화 시: 비밀번호 → TOTP
- 이메일만 활성화 시: 비밀번호 → 이메일 OTP (기존 동작 유지)

#### [미정 사항 - 질문 필요]
- 둘 다 활성화 시 "모두 통과" vs "하나라도 통과" 여부

---

### 요구사항 3: localhost 패스워드 전용 옵션

#### [원문]
옵션을 하나 더 추가합니다. 로컬 호스트일 경우 패스워드 인증만 사용 옵션 입니다.
로컬 호스트에서는 이메일 인증과 otp 가 활성화 된 경우 사용되지 않습니다.

#### [정리]
- 새 config 옵션: `auth.localhostPasswordOnly: boolean`
- 이 옵션이 `true`이고 요청 IP가 localhost인 경우: 2FA(이메일/TOTP) 완전 바이패스
- 해당 IP 범위: `127.0.0.1`, `::1`, `::ffff:127.0.0.1`

#### [미정 사항 - 질문 필요]
- 이 옵션의 기본값 (true/false)

---

### 요구사항 4: 기본값 설정

#### [원문]
기본값은 otp 는 활성화 되어있고 이메일은 활성화 되어있지 않습니다.

#### [정리]
- `totp.enabled` 기본값: `true`
- `twoFactor.enabled` 기본값: `false` (현재와 동일)
- 즉, 기본 인증 플로우: 비밀번호 → TOTP
