## 사용자 원문

- 새로운 이슈: 이제는 무브 버튼이 동작하지 않는다. 무브버튼을 누르고 드래그하여 창을 이동조차 할 수 없다.
- 네, 2번 방향으로 계획하시오. 계획 문서를 최대한 자세히 쓰시오.
- `$snoworca-lite-spec-and-plan` 을 작성하시오. 문서가 작성되면 서브에이전트로 평가하고 개선사항을 받아서 서브에이전트로 개선하고 재평가를 받는 루프를 실행하시오.

## 요약

equal 모드에서 사용자가 보는 “무브 버튼 영역 전체”가 실제 drag source처럼 동작하도록 복원한다.

## 목표

- 사용자가 무브 버튼의 아이콘만 정밀하게 집지 않아도, 버튼의 보이는 영역 전체에서 drag를 시작할 수 있어야 한다.
- equal 모드의 reorder semantics는 유지되어야 한다.
- mode 버튼과 무브 버튼 외 toolbar 영역은 drag source가 아니어야 한다.

## 변경 유형

- bug fix

## 제약사항

- 기존 모드 버튼(equal/focus/auto)의 클릭 동작은 유지해야 한다.
- right-click, non-primary pointer no-op 규칙은 유지해야 한다.
- toolbar surface outside move button은 drag source가 아니어야 한다.
- 기존 UI를 임의로 재디자인하지 않는다.
- vendor patch는 `patch-package` 재현 가능 상태를 유지해야 한다.

## 핵심 해석

- 이 작업의 핵심은 “equal 모드에서 move button 영역 전체를 drag source로 다시 연결”하는 것이다.
- 해결 방향은 grip 내부의 매우 좁은 드래그 가능 영역을 넓혀서, 사용자가 기대하는 버튼 hit area와 실제 drag source를 일치시키는 것이다.
