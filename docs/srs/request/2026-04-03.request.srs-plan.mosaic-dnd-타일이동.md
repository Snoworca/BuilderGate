# 요구사항 정리 문서

- 작성일: 2026-04-03
- 문서 유형: SRS-Plan 요구사항 정리

## 원문 요구사항

> PRD FR-1.2 (P0): 드래그 앤 드롭으로 타일(세션 창) 위치 변경
> - 그리드 모드에서 세션 창을 꾹 눌러서 다른 위치로 이동
> - UI/UX 설계: 드래그 핸들, 시각적 피드백, 터미널 입력 충돌 방지
>
> 부수 간극:
> - 모바일에서 viewMode가 'grid'로 저장된 상태에서 전환 시 강제 'tab' reset 누락
> - 컨텍스트 메뉴 복사 항목의 disabled 상태 미적용 (hasSelection 미호출)

## 의도 요약

React Mosaic 그리드 레이아웃에서 MosaicWindow를 활용하여 드래그 앤 드롭 타일 재배치를 구현하고, 부수적으로 모바일 viewMode reset과 복사 메뉴 disabled 상태를 수정한다.

## 핵심 목표

1. 그리드 모드에서 세션 타일을 드래그하여 다른 위치로 이동 가능하게 한다
2. 드래그 핸들을 좌상단 툴바의 그립 아이콘으로 한정하여 터미널 입력과 충돌하지 않게 한다
3. 드래그 시 시각적 피드백(고스트, 드롭 영역 하이라이트)을 제공한다

## 제약 사항

- React Mosaic v6.1.1의 MosaicWindow 컴포넌트를 활용해야 함
- 터미널(xterm.js) 영역은 드래그 핸들이 되면 안 됨
- 모바일에서는 그리드 모드가 없으므로 DnD 불필요

## 구조화된 요구사항 목록

### 요구사항 1: 드래그 앤 드롭 타일 위치 변경
- [원문] PRD FR-1.2: 드래그 앤 드롭으로 타일(세션 창) 위치 변경
- [정리] renderTile에서 MosaicWindow로 래핑, connectDragSource로 그립 아이콘을 드래그 핸들로 등록

### 요구사항 2: 모바일 viewMode 강제 reset
- [원문] 모바일 전환 시 viewMode가 'grid'로 남아있을 수 있음
- [정리] isMobile이 true로 변경될 때 activeWorkspace.viewMode가 'grid'이면 'tab'으로 강제 변경

### 요구사항 3: 복사 메뉴 disabled 상태
- [원문] 텍스트 미선택 시 복사 메뉴 disabled 미적용
- [정리] 컨텍스트 메뉴 복사 항목에 hasSelection() 체크 추가
