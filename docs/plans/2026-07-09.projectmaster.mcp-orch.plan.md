## §1 개요

### 1.1 목표

BuilderGate에 독립 MCP control-plane을 추가해 loopback-first HTTP MCP endpoint, 외부 IP/CIDR whitelist 보안 설정, sessionId/sessionKey 조회, 세션 목록/검색/alias 변경, 안전한 세션 간 메시지 전달, MCP-launched agent, webhook assignment, Tools dialog 관리 UI, audit/status 관측을 구현한다.

### 1.2 범위

- MCP listener, config, trusted proxy, Origin, capability token, webhook key 보안 경계
- stable `sessionKey`와 runtime UUID `sessionId` generation binding
- `SessionInputGateway`를 통한 WebSocket/restore/MCP/webhook 입력 정책 통합
- MCP tool surface, control REST API, webhook route, Tools dialog UI
- agent profile, open_agent, leader/follower claim, close/self-close lifecycle
- audit, status, final wire-contract precedence, regression and manual validation

### 1.3 제외사항

- stdio MCP transport는 이번 계획의 구현 대상이 아니다.
- 일반 Settings 화면으로 MCP whitelist를 이동하지 않는다. MCP 설정은 Tools dialog의 MCP 관리 다이얼로그 보안 탭에서 관리한다.
- browser JWT를 MCP session actor 또는 webhook actor credential로 재사용하지 않는다.
- `REL-BGSTAB-005` draft warning은 MCP scope 밖의 기존 blocker다.

### 1.4 전제조건 / 가정

- 모든 MCP scope 요구사항은 `planned/evolving`이고 feasibility high로 평가되었다.
- `IR-MCP-005`가 최종 wire-contract precedence를 제공하므로 앞선 요구사항보다 우선한다.
- 로컬 dev/manual validation은 필요 시 `https://localhost:2222`를 대상으로 한다.
