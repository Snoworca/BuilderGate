# BuilderGate Wave 3 Session Lifecycle Plan

## §1 개요

### 1.1 목표

Implement `FR-BGSTAB-010` by unifying server session finalization semantics and persisting workspace tab lifecycle metadata so intentional session stops are not recovered as orphans.

### 1.2 범위

- Shared `SessionManager` finalizer for explicit delete and PTY natural exit.
- Workspace tab lifecycle metadata for active/stopped and recoverable/non-recoverable semantics.
- Direct session delete integration with workspace-owned tabs.
- Orphan recovery gate and generation updates.
- Backend and frontend optional lifecycle types with no visual UI change.
- Focused server regression tests, frontend typecheck, full server test, and review evidence.

### 1.3 제외사항

- Verified process-tree termination and OS descendant kill enforcement.
- Shutdown recoverability policy changes, which are reserved for Wave 5.
- Frontend layout, labels, icons, or visual behavior changes.
- Settings UI controls for lifecycle policy.

### 1.4 전제조건 / 가정

- `FR-BGSTAB-010` is planned/evolving.
- Wave 1 cleanup telemetry remains the source for cleanup observation evidence.
- Legacy tabs without lifecycle fields are treated as active/recoverable for startup orphan recovery.
