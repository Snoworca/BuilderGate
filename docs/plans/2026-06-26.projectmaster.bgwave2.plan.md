# BuilderGate Wave 2 Verified Process-Tree Termination Plan

## §1 Overview

### 1.1 Goal

Implement `FR-BGSTAB-011` by introducing verified process-tree termination and asynchronous session termination APIs. The implementation must terminate only processes whose ownership can be verified from Wave1 process metadata and OS start identity.

### 1.2 Scope

- Add `ProcessTreeTerminator` with inspect and terminate methods.
- Add best-effort OS process start identity capture to session metadata.
- Add default process information providers for Windows and procfs-style POSIX hosts.
- Add async `terminateSession`, `terminateMultipleSessions`, and `terminateAllSessions`.
- Route direct session delete, tab delete, workspace delete, and tab restart cleanup through async termination.
- Keep `deleteSession(id)` as the existing compatibility wrapper.
- Add focused server regression tests for PID reuse, safe Windows taskkill, POSIX process group safety, POSIX cwd verification, surviving sampled descendants, unverified skip, async termination, and durable route/workspace ordering.

### 1.3 Out Of Scope

- Shutdown policy changes, reserved for Wave5.
- Broad process-name killing such as `taskkill /IM`, `killall`, or shell-built kill commands.
- UI visual changes.
- Live marker E2E tests without deterministic local preconditions.

### 1.4 Assumptions

- `session.processCleanup.mode` remains `observe` by default.
- Enforce mode must degrade or skip rather than kill when identity evidence is missing.
- Wave3 owns recoverability/lifecycle semantics; Wave2 owns OS process-tree cleanup.
