# BuilderGate Wave5 Shutdown Evidence And Daemon Stop Plan

## §1 Overview

### 1.1 Goal

Implement `REL-BGSTAB-002` by adding shutdown session cleanup evidence to graceful shutdown and daemon stop. Wave5 must keep the existing runtime cleanup mode default, avoid broad process-name kill fallback, validate internal shutdown evidence in `stop-client.js`, and report unavailable or unrun soak scenarios honestly.

### 1.2 Scope

- Add deterministic server tests for shutdown ordering, final workspace flush, timeout degradation, and internal shutdown response evidence.
- Add daemon tests for session cleanup evidence parsing, default warning policy, strict failure policy, and failure on verified remaining descendants.
- Extend `performGracefulShutdown()` with bounded owned-session cleanup after the first CWD snapshot and workspace flush.
- Extend `/api/internal/shutdown` response through the existing graceful shutdown result.
- Extend `stop-client.js` to validate and print shutdown session cleanup evidence before health nonresponse verification.
- Keep `stop.bat` and sentinel ownership unchanged.

### 1.3 Out Of Scope

- Frontend, UI, browser worker, or split WebSocket changes.
- Changing the default `session.processCleanup.mode` to `enforce`.
- WSL process-tree implementation redesign.
- Broad `taskkill /IM`, `killall`, whole `node.exe`, or process-name based kill fallback.
- Claiming live soak, bash/WSL coverage, or long-duration browser memory/input-lag validation when not run.

### 1.4 Assumptions

- Wave2 verified process-tree cleanup is already available through `SessionManager.terminateAllSessions()`.
- Current workspace CWD snapshot must occur before session finalization can delete CWD temp files.
- `remainingVerifiedDescendants` must count only verified-owned remaining descendants.
- Server-only soak is a manual/live checklist, not a CI gate.
