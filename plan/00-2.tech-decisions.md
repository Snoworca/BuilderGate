# Technical Decisions Record (ADR)

## ADR-001: Backend Runtime - Node.js

**Status**: Accepted

**Context**:
웹 쉘 백엔드를 위한 런타임 선택 필요.

**Decision**:
Node.js 18+ 사용

**Rationale**:
- node-pty 라이브러리가 Node.js에서 가장 안정적
- TypeScript 네이티브 지원
- SSE 구현이 간단
- 프론트엔드와 언어 통일 (JavaScript/TypeScript)

**Consequences**:
- (+) 빠른 개발 속도
- (+) 풍부한 npm 생태계
- (-) CPU 집약적 작업에는 부적합 (본 프로젝트와 무관)

## ADR-002: Real-time Communication - SSE over WebSocket

**Status**: Accepted

**Context**:
서버에서 클라이언트로 실시간 데이터 전송 방식 선택 필요.

**Decision**:
SSE (Server-Sent Events) + HTTP POST 조합 사용

**Rationale**:
- 서버→클라이언트 단방향 스트리밍만 필요 (쉘 출력)
- 클라이언트→서버는 빈번하지 않음 (명령 입력)
- WebSocket 대비 구현 복잡도 낮음
- 자동 재연결 내장
- HTTP/2 호환성 우수

**Alternatives Considered**:
| Option | Pros | Cons |
|--------|------|------|
| WebSocket | 양방향, 낮은 레이턴시 | 복잡한 연결 관리 |
| Polling | 구현 단순 | 높은 레이턴시, 리소스 낭비 |
| **SSE** | 단방향 최적화, 단순 | 바이너리 미지원 (불필요) |

**Consequences**:
- (+) 구현 단순화
- (+) 방화벽 친화적
- (-) IE 미지원 (최신 브라우저만 타겟이므로 무관)

## ADR-003: PTY Library - node-pty

**Status**: Accepted

**Context**:
서버에서 쉘 프로세스를 생성하고 관리할 라이브러리 선택.

**Decision**:
node-pty 사용

**Rationale**:
- xterm.js 개발팀이 유지보수
- PTY (Pseudo Terminal) 완전 지원
- Windows, macOS, Linux 크로스 플랫폼
- ANSI escape code 처리 가능

**Alternatives Considered**:
| Option | Pros | Cons |
|--------|------|------|
| child_process | 내장 모듈 | PTY 미지원, 대화형 명령 불가 |
| **node-pty** | 완전한 PTY | 네이티브 빌드 필요 |
| pty.js | 간단 | 유지보수 중단 |

**Consequences**:
- (+) vim, nano 등 대화형 프로그램 지원
- (+) 컬러 출력 완전 지원
- (-) 플랫폼별 네이티브 빌드 필요 (node-gyp)

## ADR-004: Frontend Framework - React

**Status**: Accepted

**Context**:
프론트엔드 프레임워크 선택 (React vs Svelte 5).

**Decision**:
React 18+ 사용

**Rationale**:
- xterm.js + React 통합 예제 풍부
- 검증된 SSE + hooks 패턴
- 향후 확장 시 생태계 활용 가능

**Alternatives Considered**:
| Option | Pros | Cons |
|--------|------|------|
| **React** | 큰 생태계, 검증됨 | 보일러플레이트 많음 |
| Svelte 5 | 간결한 코드, 작은 번들 | 상대적으로 적은 예제 |

**Consequences**:
- (+) 안정적인 개발
- (-) 약간 더 많은 코드량

## ADR-005: Terminal UI - xterm.js

**Status**: Accepted

**Context**:
브라우저에서 터미널을 렌더링할 라이브러리 선택.

**Decision**:
xterm.js 사용

**Rationale**:
- VS Code에서 사용하는 검증된 라이브러리
- 완전한 터미널 에뮬레이션
- 성능 최적화 (Canvas 렌더링)
- 풍부한 애드온 (fit, webgl, search 등)

**Consequences**:
- (+) 프로덕션 레벨 터미널 UI
- (+) ANSI 컬러, 유니코드 완전 지원
- (+) 접근성 지원

## ADR-006: State Management - React Hooks

**Status**: Accepted

**Context**:
프론트엔드 상태 관리 방식 선택.

**Decision**:
React 내장 hooks (useState, useEffect, useContext) 사용

**Rationale**:
- 앱 규모가 작음 (세션 목록, 현재 세션)
- Redux/Zustand 등 외부 라이브러리 불필요
- 단순성 유지

**Consequences**:
- (+) 의존성 최소화
- (+) 학습 곡선 낮음
- (-) 앱 규모 커지면 리팩토링 필요 (파일럿 범위 외)

## Compatibility Notes

### Node.js Version
- Minimum: 18.x
- Recommended: 20.x LTS
- Reason: node-pty 호환성, ES2022 지원

### Browser Support
- Chrome 90+
- Firefox 90+
- Edge 90+
- Safari 15+
- IE: Not supported (SSE, ES6+)

### OS Support
- Windows 10/11 (cmd, PowerShell)
- macOS 12+ (zsh, bash)
- Linux (bash, sh)
