## Objective

Phase 1에서 구현한 순수 함수(`paneTree.ts`)와 IndexedDB 훅(`usePaneDB.ts`)을 토대로, 실제 Pane 분할 UI를 데스크톱 환경에서 동작하게 만든다. 이 Phase가 완료되면 사용자는 하나의 터미널 탭 내에서 최대 8개의 터미널을 수평/수직으로 분할하여 동시에 볼 수 있으며, 드래그로 크기를 조절하고, 줌/포커스/교환 기능을 사용할 수 있다.

**서버 변경 없음** -- 기존 `POST /api/sessions`, `DELETE /api/sessions/:id`, `GET /api/sessions/:id/stream` API를 그대로 활용한다.

#### 2.1.3 `splitPane(paneId, direction)` -- async

- [ ] **2.1.3a** Guard: if `!canSplit`, return early (no-op)
- [ ] **2.1.3b** Guard: if `layout.zoomedPaneId !== null`, return early (no split during zoom)
- [ ] **2.1.3c** Call `POST /api/sessions` to create a new PTY session. Use `RuntimeConfigStore` defaults for `cols`, `rows`, `shell`. Pass parent session's current CWD as the new session's CWD.
- [ ] **2.1.3d** If API call fails: display error message on StatusBar for 3 seconds, return without modifying tree (**no optimistic update**)
- [ ] **2.1.3e** On success: call `paneTree.splitPane(layout.root, paneId, direction, newSession.id)` to get new root
- [ ] **2.1.3f** Update layout state: new root, `focusedPaneId` = new pane ID
- [ ] **2.1.3g** Register new session ID in `childSessionIds` tracking
- [ ] **2.1.3h** Trigger debounced save (300ms) to IndexedDB

```typescript
async splitPane(paneId: string, direction: Direction): Promise<void> {
  if (!canSplit || layout.zoomedPaneId) return;

  // 1. Create PTY session first (no optimistic update)
  let newSession: Session;
  try {
    newSession = await createSession('Sub-Terminal', undefined, parentCwd, false);
    if (!newSession) throw new Error('Session creation returned null');
  } catch (err) {
    showStatusBarError('Failed to create session for split');
    return; // rollback = no-op, tree was never modified
  }

  // 2. Only now mutate the tree
  const newPaneId = crypto.randomUUID();
  const newRoot = paneTreeSplit(layout.root, paneId, direction, newSession.id, newPaneId);

  setLayout({
    root: newRoot,
    focusedPaneId: newPaneId,
    zoomedPaneId: null,
  });

  // 3. Track as child session
  registerChildSession(newSession.id);

  // 4. Debounced save
  debouncedSave();
}
```

#### 2.1.4 `closePane(paneId)` -- async

- [ ] **2.1.4a** Guard: if `paneCount <= 1`, return early (cannot close last pane)
- [ ] **2.1.4b** Guard: if `layout.zoomedPaneId !== null`, return early (no close during zoom)
- [ ] **2.1.4c** Find the target `PaneLeaf` via `paneTree.findPane(layout.root, paneId)`
- [ ] **2.1.4d** Close SSE connection first: the `TerminalContainer` for this pane calls `EventSource.close()` on unmount (React cleanup). The tree update triggers unmount.
- [ ] **2.1.4e** Call `DELETE /api/sessions/:id` to delete the PTY session. Fire-and-forget with error logging.
- [ ] **2.1.4f** Call `paneTree.closePane(layout.root, paneId)` to get new root
- [ ] **2.1.4g** Update focus: set `focusedPaneId` to the sibling pane (first leaf of sibling subtree)
- [ ] **2.1.4h** Remove session ID from `childSessionIds` tracking
- [ ] **2.1.4i** Trigger debounced save (300ms)

```typescript
async closePane(paneId: string): Promise<void> {
  if (paneCount <= 1 || layout.zoomedPaneId) return;

  const leaf = findPane(layout.root, paneId);
  if (!leaf) return;

  // 1. Update tree (triggers TerminalContainer unmount -> SSE close)
  const newRoot = paneTreeClose(layout.root, paneId);
  if (!newRoot) return; // should not happen since paneCount > 1

  // 2. Determine new focus target
  const leaves = flattenPaneTree(newRoot);
  const newFocus = leaves[0]?.id ?? layout.focusedPaneId;

  setLayout({
    root: newRoot,
    focusedPaneId: newFocus,
    zoomedPaneId: null,
  });

  // 3. Delete PTY session (fire-and-forget)
  sessionApi.delete(leaf.sessionId).catch(console.error);

  // 4. Untrack child session
  unregisterChildSession(leaf.sessionId);

  // 5. Debounced save
  debouncedSave();
}
```

**When** `splitPane(paneId, 'vertical')` is called
**Then**

- Focus moves to the new (second) pane
- Each terminal has an independent SSE connection
- IndexedDB save is triggered after 300ms

#### T-2.2: Single Pane Horizontal Split

**Given** a single-pane layout
**When** `splitPane(paneId, 'horizontal')` is called
**Then**
- Layout tree has `direction: 'horizontal'`

- Both terminals auto-fit to their container size via `FitAddon`

#### T-2.3: Nested Split (2 levels)

**Given** a 2-pane vertical split (A | B)
**When** `splitPane(paneB, 'horizontal')` is called on pane B
**Then**
- Tree depth becomes 2
- Pane B area splits into top/bottom (B1 / B2)
- Layout: `Split(v, [A, Split(h, [B, B2])])`
- 3 terminal containers rendered

**When** `closePane(paneB)` is called
**Then**
- `DELETE /api/sessions/{B.sessionId}` is called
- Pane A becomes root `PaneLeaf` occupying full width
- Focus moves to pane A
- `paneCount` returns 1
- SSE connection for pane B's session is terminated

#### T-2.5: Resize Drag

**Given** a 2-pane vertical split at ratio 0.5
**When** user drags the resizer rightward
**Then**
- `onResize` fires continuously with new ratio values

#### T-2.6: Focus Navigation

**When** `toggleZoom(B.id)` is called again
**Then**

- Right pane now shows SA's terminal

- Swap mode is cleared after execution

**Then**

- Layout becomes 2x2 grid
- All 4 terminals are connected and responsive
- IndexedDB is saved immediately

#### T-2.10: Layout Persistence Across Reload

## Performance Notes

- `SplitPane` re-renders are scoped: `React.memo` on leaf components, keyed by `node.id`
- `resizePane` during drag should NOT trigger IndexedDB writes (only on `pointerup`)

- `FitAddon.fit()` calls batched via `requestAnimationFrame` on window resize (NFR-6112)
