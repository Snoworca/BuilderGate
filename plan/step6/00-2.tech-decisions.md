## ADR-601: 외부 라이브러리 도입 없이 자체 구현

### 결정
Pane 분할, 스와이프 캐러셀, 드래그 리사이즈를 모두 자체 구현한다.

### 근거
- SRS 제약: "외부 라이브러리 최소화, 스와이프 엔진/분할 렌더러 자체 구현"
- 기존 프로젝트에 react-split-pane, react-swipeable 등 미도입 상태
- xterm.js와의 이벤트 충돌(포인터, 터치) 세밀 제어 필요
- 번들 크기 증가 방지

### 대안 (기각)
| 라이브러리 | 기각 사유 |
|-----------|----------|
| react-split-pane | 재귀 바이너리 트리 미지원, 커스텀 포크 필요 |
| react-swipeable | xterm.js 터치 이벤트와 충돌 가능 |
| allotment | 프레임워크 의존성, 트리 모델 불일치 |

## ADR-602: IndexedDB 직접 사용 (idb 래퍼 미사용)

### 결정
IndexedDB의 native API를 직접 사용하여 `paneDb.ts` 헬퍼를 작성한다. `idb` npm 패키지를 도입하지 않는다.

### 근거
- 사용하는 패턴이 단순 (3개 Object Store, CRUD + 인덱스 조회)
- Promise 래퍼 5~10줄로 충분
- 추가 의존성 회피 (프로젝트 방침)
- 브라우저 호환성: 모든 대상 브라우저가 IndexedDB v2 지원

### 구현 패턴
```typescript
function openDB(): Promise<IDBDatabase> {
  return new Promise((resolve, reject) => {
    const request = indexedDB.open('buildergate', 1);
    request.onupgradeneeded = (e) => { /* schema init */ };
    request.onsuccess = () => resolve(request.result);
    request.onerror = () => reject(request.error);
  });
}

function txPut<T>(storeName: string, data: T): Promise<void> {
  return openDB().then(db => {
    return new Promise((resolve, reject) => {
      const tx = db.transaction(storeName, 'readwrite');
      tx.objectStore(storeName).put(data);
      tx.oncomplete = () => resolve();
      tx.onerror = () => reject(tx.error);
    });
  });
}
```

## ADR-603: PaneNode 불변 트리 모델

### 결정
PaneNode 트리를 불변(immutable) 방식으로 조작한다. 모든 변경은 새 트리를 반환한다.

### 근거
- React 상태 관리와 자연스럽게 호환 (참조 비교로 리렌더링 감지)
- 트리 조작의 테스트 용이성 (순수 함수)
- 실행 취소(undo) 구현 가능성 확보 (향후)
- SRS 섹션 3.3에서 명시적으로 불변 업데이트 요구

### 성능 고려
- 최대 8 Pane (깊이 4)의 작은 트리이므로 전체 복사 비용 무시 가능
- `structuredClone` 또는 수동 재귀 복사 사용

## ADR-604: `usePaneManager`와 `useTabManager` 분리

### 결정
Pane 관리를 `useTabManager`에 병합하지 않고 별도 `usePaneManager` 훅으로 분리한다.

### 근거
- SRS 섹션 3.4의 책임 경계 명시:
  - `useTabManager`: 탭 목록, 탭 추가/삭제/재정렬, 활성 탭 ID
  - `usePaneManager`: 활성 터미널 탭 내의 Pane 트리
- 단일 책임 원칙 (SRP)
- `useTabManager`는 파일 탭/뷰어 탭에도 사용되므로 Pane 로직과 무관

### 연결점
- `useTabManager.activeTabId` 변경 → `usePaneManager`가 해당 탭의 PaneLayout을 IndexedDB에서 로드
- Pane의 모든 sessionId → `childSessionIds` computed value로 추출

## ADR-605: 반응형 렌더링 전략 (데스크톱/모바일)

### 결정
동일한 `PaneLayout` 데이터를 `PaneRenderer` 컴포넌트에서 분기하여:
- 데스크톱(>768px): `SplitPane` 재귀 렌더러
- 모바일(≤768px): `PaneCarousel` 횡 스와이프 캐러셀

### 근거
- "하나의 데이터, 두 개의 뷰" 원칙
- 모바일에서 분할 레이아웃은 실용적이지 않음 (최소 120px 너비 × 8 Pane → 960px 필요)
- `flattenPaneTree()`로 트리를 선형 배열로 변환하여 캐러셀에 제공
- 모바일↔데스크톱 전환 시 데이터 변환 불필요 (자동 일관성)

## ADR-606: `visibilitychange` 기반 저장 전략

### 결정
`beforeunload` 대신 `visibilitychange` 이벤트에서 IndexedDB 비동기 저장을 실행한다.

### 근거
- IndexedDB API는 비동기이므로 `beforeunload`의 동기 컨텍스트에서 저장 완료를 보장할 수 없음
- `visibilitychange`(hidden)는 탭 전환/최소화 시 발생하며, 이 시점의 비동기 작업은 브라우저에서 일반적으로 허용됨
- `beforeunload`에서는 dirty 상태 확인 후 경고 표시만 수행 (best-effort)

## ADR-607: Ctrl+B Prefix 모드 상태 머신

### 결정
`TerminalView.tsx`의 `customKeyEventHandler`에 prefix 모드 상태 머신을 구현한다.

### 근거
- 기존 `customKeyEventHandler`가 이미 Ctrl+C/V, 방향키 등을 가로채고 있음
- Prefix 모드 상태를 `usePaneManager`에서 관리하고, `TerminalView`는 콜백만 호출
- 타임아웃(1500ms)은 `useEffect` 클린업으로 관리

### 상태 전이

```
NORMAL ──(Ctrl+B)──► PREFIX ──(키입력)──► 명령실행 ──► NORMAL
                      │                                    ▲
                      ├──(1500ms 타임아웃)─────────────────┘
                      ├──(Ctrl+B)──► PTY에 \x02 전송 ──► NORMAL
                      └──(ESC)─────────────────────────► NORMAL
```

## ADR-608: 터치 이벤트 레이어링 (모바일)

### 결정
모바일 스와이프와 xterm.js 터치 이벤트를 분리하기 위해 투명 터치 레이어를 배치한다.

### 근거
- xterm.js가 자체 터치 핸들러 보유 (스크롤, 텍스트 선택)
- 50ms 방향 판정으로 스와이프(X축) vs 스크롤(Y축) 분기
- 롱프레스(500ms)는 컨텍스트 메뉴 → 50ms 판정과 충돌 없음

### 구현 방식
```
TouchLayer (투명, absolute, z-index 위)
  │
  ├─ touchstart: 좌표 기록, 50ms 타이머 시작
  ├─ touchmove:
  │   ├─ X축 delta > Y축 delta → 스와이프 모드, 이벤트 소비
  │   └─ Y축 delta > X축 delta → pointer-events: none → xterm 위임
  └─ touchend: 스와이프 완료 or 클린업
```

## ADR-609: SSE 연결 전략 (Pane당 1개)

### 결정
각 Pane(`PaneLeaf`)이 자체 SSE(`EventSource`) 연결을 유지한다. 비활성(비포커스) Pane도 연결을 유지한다.

### 근거
- 기존 `TerminalContainer` + `useSSE` 구조가 세션당 독립 SSE 설계
- 서버가 HTTPS(HTTP/2)로 동작하므로 동시 연결 제한(6) 해당 없음
- 연결 재사용이나 멀티플렉싱은 불필요한 복잡도 추가
- HTTP/1.1 폴백 환경에서 7+ Pane 시 StatusBar 경고 표시 (SRS NFR-6109)

## ADR-610: Playwright 헤드리스 E2E 테스트

### 결정
Playwright를 헤드리스 모드로 사용하여 Pane 분할 시스템의 E2E 테스트를 수행한다.

### 근거
- xterm.js 렌더링, 컨텍스트 메뉴, 드래그 리사이즈, 스와이프 등 실제 브라우저 이벤트 필요
- Playwright의 `page.mouse`, `page.touchscreen`, `page.keyboard` API로 정밀한 이벤트 시뮬레이션 가능
- headless Chromium에서 IndexedDB 동작 검증 가능
- 모바일 에뮬레이션 (`page.setViewportSize`, `devices`) 지원

### 테스트 구조
```
tests/e2e/
├── pane-split.spec.ts          # 분할/닫기/리사이즈
├── pane-context-menu.spec.ts   # 컨텍스트 메뉴
├── pane-carousel.spec.ts       # 모바일 캐러셀
├── pane-keyboard.spec.ts       # Ctrl+B prefix 모드
├── pane-persistence.spec.ts    # IndexedDB 저장/복원
└── pane-preset.spec.ts         # 프리셋 레이아웃
```
