## 5. Data Model

### 5.1 Session Entity

```typescript
interface Session {
  id: string;                    // UUID v4
  name: string;                  // Display name (e.g., "Session-1")
  status: 'running' | 'idle';    // Current execution status
  createdAt: Date;               // Creation timestamp
  lastActiveAt: Date;            // Last activity timestamp
}
```

### 5.2 API DTOs

```typescript
// Request
interface CreateSessionRequest {
  name?: string;  // Optional custom name
}

interface InputRequest {
  data: string;   // Input data to write to PTY
}

interface ResizeRequest {
  cols: number;   // Terminal columns
  rows: number;   // Terminal rows
}

// Response
interface SessionResponse {
  id: string;
  name: string;
  status: 'running' | 'idle';
  createdAt: string;  // ISO 8601
  lastActiveAt: string;
}

// SSE Events
interface OutputEvent {
  data: string;
}

interface StatusEvent {
  status: 'running' | 'idle';
}

interface ErrorEvent {
  message: string;
}
```

## 7. Technology Decisions Summary

| Area | Decision | Rationale |
|------|----------|-----------|
| Backend Runtime | Node.js 18+ | PTY support, TypeScript native |
| Backend Framework | Express.js | Simple, well-documented |
| PTY Library | node-pty | Industry standard for terminal emulation |
| Real-time (S→C) | SSE | Simpler than WebSocket for one-way streaming |
| Real-time (C→S) | HTTP POST | Sufficient for command input |
| Frontend Framework | React 18 | Ecosystem, xterm.js integration |
| Build Tool | Vite | Fast HMR, TypeScript support |
| Terminal UI | xterm.js | Feature-rich terminal emulator |
| State Management | React hooks | Simple app, no need for Redux |
